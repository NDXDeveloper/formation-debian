# 2.5.3 Clients de messagerie

## Introduction

Malgré la popularité des webmails (Gmail, Outlook.com, Yahoo Mail), un client de messagerie local reste un outil de productivité incontournable dans de nombreux contextes professionnels et personnels. Un client local offre des avantages que les webmails ne peuvent pas égaler : travail hors-ligne, gestion centralisée de multiples comptes, recherche rapide dans des archives volumineuses, chiffrement bout en bout natif (PGP/GPG, S/MIME), filtrage avancé, intégration au système de notifications du bureau et maîtrise complète des données.

Debian propose trois clients de messagerie majeurs — Thunderbird, Evolution et KMail — ainsi que plusieurs alternatives légères. Cette section présente ces clients, leur configuration, leurs fonctionnalités distinctives et les critères de choix entre eux.

---

## Protocoles de messagerie

Avant d'aborder les clients, il est utile de rappeler les protocoles sur lesquels repose la messagerie électronique, car leur choix lors de la configuration du client affecte le fonctionnement quotidien.

### Réception des e-mails

| Protocole | Fonctionnement | Synchronisation | Usage recommandé |
|-----------|---------------|-----------------|-----------------|
| **IMAP** (Internet Message Access Protocol) | Les e-mails restent sur le serveur. Le client synchronise une copie locale. Les actions (lecture, suppression, déplacement) sont répercutées sur le serveur. | Bidirectionnelle (client ↔ serveur) | Usage principal. Permet l'accès depuis plusieurs appareils avec synchronisation. |
| **POP3** (Post Office Protocol v3) | Les e-mails sont téléchargés sur le client et généralement supprimés du serveur. | Unidirectionnelle (serveur → client) | Archivage local. Déconseillé pour un usage multi-appareils. |
| **Exchange ActiveSync / EWS** | Protocole propriétaire Microsoft pour Exchange/Microsoft 365. Synchronise e-mails, calendrier, contacts et tâches. | Bidirectionnelle | Environnements d'entreprise Microsoft. |
| **JMAP** (JSON Meta Application Protocol) | Successeur moderne d'IMAP, basé sur JSON/HTTP. Plus efficace pour la synchronisation mobile et les connexions lentes. | Bidirectionnelle | Adoption croissante. Supporté par Fastmail et quelques fournisseurs. |

### Envoi des e-mails

| Protocole | Port standard | Port chiffré | Utilisation |
|-----------|---------------|-------------|-------------|
| **SMTP** (Simple Mail Transfer Protocol) | 25 (relais) | 587 (submission + STARTTLS) ou 465 (SMTPS) | Envoi des e-mails sortants. Le port 587 avec STARTTLS est la configuration recommandée. |

### Chiffrement de la connexion

Tous les protocoles modernes doivent utiliser le chiffrement TLS pour protéger les identifiants et le contenu des e-mails en transit :

- **STARTTLS** : la connexion démarre en clair puis passe en chiffré après négociation. Utilisé sur les ports standard (143 pour IMAP, 587 pour SMTP).
- **SSL/TLS implicite** : la connexion est chiffrée dès le départ. Utilisé sur les ports dédiés (993 pour IMAPS, 465 pour SMTPS).

## Thunderbird

### Présentation

**Thunderbird** est le client de messagerie de la fondation Mozilla (les mêmes développeurs que Firefox). C'est le client de messagerie libre le plus populaire et le plus complet. Thunderbird gère les e-mails, les calendriers, les contacts et les flux RSS dans une interface unifiée. La version 115 « Supernova » (2023) a modernisé l'interface ; Debian Trixie embarque actuellement Thunderbird **140 ESR** (cycle annuel ESR aligné sur Firefox), apportant les améliorations cumulées des versions 116 à 140.

```bash
# Installation
sudo apt install thunderbird

# Pack de langue français
sudo apt install thunderbird-l10n-fr

# Lancement
thunderbird &
```

### Configuration d'un compte

Thunderbird propose un assistant de configuration automatique qui détecte les paramètres du serveur à partir de l'adresse e-mail. Pour la majorité des fournisseurs (Gmail, Outlook.com, Yahoo, Free, OVH, etc.), la configuration est automatique :

```
# Au premier lancement, ou Menu → Paramètres des comptes → Actions → Ajouter un compte

# 1. Saisir le nom, l'adresse e-mail et le mot de passe
# 2. Thunderbird interroge sa base de données de configuration (autoconfig)
#    et les enregistrements DNS du domaine pour détecter les serveurs
# 3. Vérifier les paramètres proposés (IMAP/POP3, SMTP, ports, chiffrement)
# 4. Confirmer

# Configuration manuelle si l'auto-détection échoue :
# Paramètres des comptes → Serveur entrant :
#   Protocole : IMAP
#   Serveur : imap.example.com
#   Port : 993
#   Sécurité : SSL/TLS
#   Authentification : Mot de passe normal (ou OAuth2 pour Gmail/Outlook)
#
# Serveur sortant (SMTP) :
#   Serveur : smtp.example.com
#   Port : 587
#   Sécurité : STARTTLS
#   Authentification : Mot de passe normal (ou OAuth2)
```

### Configuration de Gmail avec OAuth2

Gmail et les autres services Google exigent l'authentification OAuth2 (les mots de passe d'application ne sont plus recommandés). Thunderbird supporte OAuth2 nativement :

```
# 1. Ajouter un compte avec l'adresse Gmail
# 2. Thunderbird détecte automatiquement Gmail et propose OAuth2
# 3. Une fenêtre de connexion Google s'ouvre dans le navigateur intégré
# 4. Se connecter et autoriser Thunderbird à accéder au compte
# 5. Le jeton OAuth2 est stocké localement et renouvelé automatiquement

# Pour Microsoft 365 / Outlook.com :
# Le même processus OAuth2 est utilisé
# Thunderbird détecte automatiquement les paramètres Exchange/Microsoft 365
```

### Fonctionnalités clés de Thunderbird

**Gestion multi-comptes** : Thunderbird gère un nombre illimité de comptes e-mail dans la même interface. La boîte de réception unifiée (Unified Inbox) regroupe les messages de tous les comptes dans une seule vue.

**Recherche avancée** : la recherche de Thunderbird indexe l'intégralité des messages (en-têtes et corps). La recherche rapide dans la barre d'outils offre des résultats instantanés. La recherche avancée (Ctrl+Shift+F) permet des critères combinés : expéditeur, destinataire, sujet, date, contenu, dossier, pièces jointes, étiquettes.

**Filtres de messages** : les filtres automatiques trient les messages entrants selon des critères configurables : déplacer vers un dossier, appliquer une étiquette, transférer, supprimer, marquer comme lu. Les filtres peuvent être exécutés automatiquement à la réception ou manuellement sur une sélection.

```
# Outils → Filtres de messages → Nouveau
# Exemple : déplacer les newsletters dans un dossier dédié
# Condition : « De contient newsletter@example.com »
# Action : « Déplacer vers → Newsletters »

# Exemple : étiqueter les messages urgents
# Condition : « Sujet contient [URGENT] »
# Action : « Étiqueter → Important »
```

**Calendrier intégré** : depuis Thunderbird 102, le calendrier (anciennement l'extension Lightning) est intégré nativement. Il supporte les protocoles CalDAV (Google Calendar, Nextcloud, iCloud) et ICS.

**Contacts** : le carnet d'adresses supporte CardDAV pour la synchronisation avec des services distants (Google Contacts, Nextcloud, iCloud) et le format vCard pour l'import/export.

**Flux RSS/Atom** : Thunderbird peut servir de lecteur de flux RSS, affichant les articles dans la même interface que les e-mails.

### Extensions Thunderbird

Thunderbird supporte un système d'extensions (WebExtensions depuis la version 78+) :

```
# Installation : Menu → Extensions et thèmes → Rechercher

# Extensions populaires :
# - Owl for Exchange : support complet d'Exchange/EWS (payant)
# - Enigmail / OpenPGP : chiffrement PGP (intégré depuis TB 78)
# - Provider for Google Calendar : synchronisation Google Calendar
# - CardBook : gestion avancée des contacts CardDAV
# - Mail Merge : publipostage directement depuis Thunderbird
# - Quicktext : modèles de réponses rapides
# - ImportExportTools NG : import/export de messages en masse
```

### Chiffrement des e-mails (OpenPGP)

Thunderbird intègre nativement le support OpenPGP depuis la version 78, sans nécessiter d'extension externe :

```
# Configuration du chiffrement OpenPGP
# 1. Paramètres du compte → Chiffrement de bout en bout
# 2. Générer une clé OpenPGP ou importer une clé existante
# 3. La clé publique peut être attachée automatiquement aux messages

# Importer une clé GPG existante
# Si une clé GPG est déjà dans le trousseau système (gpg --list-keys),
# elle peut être importée dans Thunderbird :
# Paramètres du compte → Chiffrement → Gestionnaire de clés OpenPGP
# → Fichier → Importer une clé secrète depuis un fichier

# Envoyer un message chiffré
# Lors de la rédaction : Menu → Sécurité → Chiffrer
# ou utiliser le bouton de chiffrement dans la barre d'outils
# Le destinataire doit avoir une clé publique connue de Thunderbird

# S/MIME est également supporté pour les environnements d'entreprise
# qui utilisent des certificats X.509
```

### Emplacement des données

```bash
# Profil Thunderbird (configuration, comptes, extensions)
~/.thunderbird/

# Le profil contient :
# - prefs.js : préférences utilisateur
# - Mail/ : dossiers de courrier locaux et cache IMAP
# - ImapMail/ : cache des dossiers IMAP
# - abook.sqlite : carnet d'adresses
# - calendar-data/ : données du calendrier
# - key4.db + logins.json : mots de passe stockés

# Sauvegarder le profil Thunderbird
tar czf thunderbird-backup.tar.gz ~/.thunderbird/

# Restaurer
tar xzf thunderbird-backup.tar.gz -C ~/
```

## Evolution

### Présentation

**Evolution** est le client de messagerie et la suite de productivité personnelle intégrée à GNOME. Il offre une interface unifiée pour la messagerie, le calendrier, les contacts, les tâches et les mémos. Son intégration avec GNOME Online Accounts en fait le choix naturel pour les utilisateurs GNOME qui ont configuré des comptes Google, Microsoft ou Nextcloud dans les Paramètres système.

```bash
# Installation
sudo apt install evolution

# Support Exchange/Microsoft 365 (EWS)
sudo apt install evolution-ews

# Support des plugins supplémentaires
sudo apt install evolution-plugins
```

### Caractéristiques distinctives

**Intégration GNOME Online Accounts** : lorsqu'un compte Google ou Microsoft est ajouté dans les Paramètres GNOME (Paramètres → Comptes en ligne), Evolution récupère automatiquement la configuration e-mail, calendrier et contacts sans configuration manuelle supplémentaire. C'est l'intégration la plus transparente disponible sous GNOME.

**Support Exchange natif** : via le plugin `evolution-ews`, Evolution offre le meilleur support d'Exchange/Microsoft 365 parmi les clients Linux libres. Il synchronise les e-mails, le calendrier (incluant la planification de réunions et la consultation de disponibilité), les contacts et les listes de tâches directement depuis un serveur Exchange.

```bash
# Ajouter un compte Exchange/Microsoft 365
# 1. Édition → Préférences → Comptes de messagerie → Ajouter
# 2. Type de serveur : Exchange Web Services
# 3. Saisir l'adresse e-mail et le mot de passe (ou OAuth2)
# 4. Evolution détecte automatiquement le serveur EWS

# Alternative : ajouter un compte Microsoft dans GNOME Online Accounts
# Paramètres → Comptes en ligne → Microsoft 365
# Evolution récupérera automatiquement la configuration
```

**Suite PIM complète** : Evolution n'est pas seulement un client e-mail. C'est une suite PIM (Personal Information Manager) qui intègre un calendrier complet (vues jour, semaine, mois, liste), un gestionnaire de tâches avec priorités et dates d'échéance, un carnet d'adresses avec groupes et listes de distribution, et un module de mémos (notes).

**Recherche structurée** : Evolution utilise des « dossiers de recherche » (vSearch Folders) qui sont des vues dynamiques basées sur des critères de recherche. Par exemple, un dossier de recherche « Messages non lus cette semaine de mon équipe » affiche dynamiquement les messages correspondants, quelle que soit leur localisation réelle dans la structure de dossiers.

### Emplacement des données

```bash
# Configuration Evolution
~/.config/evolution/

# Données locales (cache mail, calendrier, contacts)
~/.local/share/evolution/

# Le backend de données est evolution-data-server, un service partagé
# qui fournit les données de calendrier et de contacts aux autres
# applications GNOME (GNOME Calendar, GNOME Contacts)
# Cela signifie que les événements ajoutés dans GNOME Calendar
# apparaissent dans Evolution et inversement
```

## KMail

### Présentation

**KMail** est le client de messagerie de l'écosystème KDE, intégré à la suite PIM **Kontact** (qui regroupe KMail, KOrganizer, KAddressBook, KNotes et Akregator). KMail est puissant et hautement configurable, mais son architecture basée sur Akonadi (un framework de cache et de synchronisation de données PIM) le rend plus complexe que Thunderbird.

```bash
# Installation de KMail seul
sudo apt install kmail

# Installation de la suite complète Kontact (recommandé)
sudo apt install kontact
# Installe : kmail, korganizer, kaddressbook, knotes, akregator

# Kontact regroupe tous les composants dans une seule fenêtre
kontact &
```

### Caractéristiques distinctives

**Intégration KDE** : KMail s'intègre parfaitement à l'environnement Plasma. Les notifications de nouveaux messages apparaissent dans la zone de notification, les contacts sont accessibles via KRunner, et le calendrier se synchronise avec l'applet horloge du panneau Plasma.

**Akonadi** : le framework Akonadi gère le stockage et la synchronisation de toutes les données PIM (e-mails, contacts, calendriers, tâches). Il fonctionne comme un cache local qui synchronise les données avec les serveurs distants en arrière-plan. L'avantage est une grande réactivité de l'interface même avec des boîtes aux lettres volumineuses. L'inconvénient est une complexité accrue et une consommation mémoire plus élevée.

```bash
# Vérifier l'état d'Akonadi
akonadictl status

# Redémarrer Akonadi en cas de problème de synchronisation
akonadictl restart

# Interface de gestion d'Akonadi
akonadiconsole &
# Permet de diagnostiquer les problèmes de synchronisation,
# de visualiser les ressources configurées et de purger le cache
```

**Chiffrement** : KMail offre un support complet de GnuPG (OpenPGP) et S/MIME via Kleopatra, le gestionnaire de certificats de KDE. L'intégration est parmi les plus abouties des clients Linux.

```bash
# Kleopatra : gestion des clés OpenPGP et des certificats S/MIME
sudo apt install kleopatra

# Générer une paire de clés
# Kleopatra → Fichier → Nouveau certificat OpenPGP

# Configurer KMail pour utiliser la clé
# Configuration → Configurer KMail → Identités → sélectionner l'identité
# → Onglet Chiffrement → Clé OpenPGP → Sélectionner la clé
```

### Emplacement des données

```bash
# Configuration KMail
~/.config/kmail2rc
~/.config/kmailrc

# Données Akonadi (cache local de tous les PIM)
~/.local/share/akonadi/

# Base de données Akonadi
~/.local/share/akonadi/db_data/

# Purger le cache Akonadi (résout de nombreux problèmes)
akonadictl stop  
rm -rf ~/.local/share/akonadi/db_data/  
akonadictl start  
# Akonadi reconstruit le cache en resynchronisant avec les serveurs
```

## Alternatives légères

### Geary

**Geary** est un client de messagerie minimaliste conçu pour GNOME. Son interface est épurée et moderne (libadwaita), axée sur la conversation (les échanges sont regroupés en fils de discussion) :

```bash
sudo apt install geary
```

Geary est idéal pour les utilisateurs qui souhaitent un client simple pour un ou deux comptes personnels. Il supporte IMAP/SMTP avec auto-configuration, les fils de conversation, la recherche et les notifications. Il ne supporte pas les calendriers, les contacts, les filtres avancés, le chiffrement PGP ni les comptes Exchange.

### Claws Mail

**Claws Mail** est un client léger et rapide, orienté vers les utilisateurs avancés qui apprécient la configurabilité et la vitesse :

```bash
# Le client + le méta-paquet des plugins « core »
sudo apt install claws-mail claws-mail-plugins

# Plugins additionnels (PGP, ManageSieve, fancy HTML, PDF viewer, mailmbox, etc.)
sudo apt install claws-mail-extra-plugins

# Plugins ciblés (à la carte si on ne veut pas tout)
sudo apt install claws-mail-fancy-plugin       # Rendu HTML enrichi (WebKit)  
sudo apt install claws-mail-pgpinline claws-mail-pgpmime  # PGP (pas de suffixe `-plugin`)  
sudo apt install claws-mail-managesieve        # Gestion des filtres Sieve serveur  
sudo apt install claws-mail-pdf-viewer  
```

Claws Mail est basé sur GTK et consomme très peu de ressources. Il est extensible via un système de plugins (PGP, anti-spam, notification, lecteur RSS, vCalendar). Son interface est plus technique et moins visuelle que celle de Thunderbird ou Evolution, mais elle est extrêmement rapide même avec des boîtes aux lettres de plusieurs dizaines de milliers de messages.

### Mutt / NeoMutt (terminal)

Pour les utilisateurs du terminal, **Mutt** (et son fork **NeoMutt**) est un client de messagerie en mode texte réputé pour sa puissance et sa configurabilité :

```bash
sudo apt install neomutt

# NeoMutt est un fork actif de Mutt avec des fonctionnalités supplémentaires
# Configuration dans ~/.config/neomutt/neomuttrc ou ~/.muttrc
```

Mutt/NeoMutt est le choix des administrateurs système et des développeurs qui vivent dans le terminal. Il est extrêmement rapide, entièrement contrôlable au clavier, et peut être configuré pour fonctionner avec n'importe quel serveur IMAP/POP3/SMTP. Sa courbe d'apprentissage est raide mais ses utilisateurs sont parmi les plus productifs dans la gestion de gros volumes d'e-mails.

## Critères de choix

**Choisir Thunderbird si** : polyvalence maximale est nécessaire, plusieurs comptes de fournisseurs différents doivent être gérés, l'indépendance vis-à-vis de l'environnement de bureau est souhaitée, le chiffrement OpenPGP intégré est nécessaire, ou si l'écosystème d'extensions doit être riche. Thunderbird est le choix le plus polyvalent et le plus recommandé en général.

**Choisir Evolution si** : l'environnement est GNOME, un compte Exchange/Microsoft 365 doit être synchronisé (e-mails, calendrier, contacts), ou si l'intégration avec GNOME Online Accounts est une priorité. Evolution est le meilleur choix pour les environnements d'entreprise utilisant Microsoft Exchange avec des postes GNOME.

**Choisir KMail/Kontact si** : l'environnement est KDE Plasma et une suite PIM complète intégrée à l'écosystème KDE est souhaitée. Kontact offre la meilleure intégration avec le bureau Plasma (notifications, KRunner, calendrier du panneau).

**Choisir Geary si** : un client simple et élégant suffit pour un ou deux comptes personnels sous GNOME, sans besoin de fonctionnalités avancées.

**Choisir Claws Mail si** : la vitesse et la légèreté sont prioritaires, la machine a des ressources limitées, ou si une interface hautement configurable est préférée à une interface moderne.

## Bonnes pratiques de configuration

### Sauvegardes

Les données de messagerie locale représentent souvent des années de correspondance irremplaçable. Une stratégie de sauvegarde est essentielle :

```bash
# Thunderbird : sauvegarder le profil complet
tar czf thunderbird-$(date +%Y%m%d).tar.gz ~/.thunderbird/

# Evolution : sauvegarder via l'outil intégré
# Fichier → Sauvegarder les données d'Evolution
# Génère une archive .tar.gz contenant toute la configuration et les données

# Restaurer : Fichier → Restaurer les données d'Evolution

# KMail : sauvegarder les données Akonadi et la configuration
tar czf kmail-$(date +%Y%m%d).tar.gz \
  ~/.config/kmail* ~/.local/share/akonadi/ ~/.local/share/kmail2/

# Automatiser les sauvegardes avec un timer systemd ou cron
# Voir Module 5 pour les détails sur la planification
```

### Archivage des e-mails

Pour les boîtes aux lettres volumineuses, l'archivage des anciens messages libère de l'espace sur le serveur et améliore les performances :

```
# Thunderbird : archivage automatique
# Clic droit sur un dossier → Propriétés → Politique de rétention
# Définir un âge maximum des messages (ex: archiver après 12 mois)
# Les messages sont déplacés vers un dossier d'archives local

# Archivage manuel : sélectionner les messages → touche A
# Déplace vers le dossier Archives/ (classé par année)

# Evolution : archivage via les dossiers de recherche
# Créer un dossier de recherche pour les messages anciens
# Sélectionner tout → déplacer vers un dossier local
```

### Sécurité

```bash
# Désactiver le chargement automatique des images distantes
# (prévient le pistage par pixel de suivi)
# Thunderbird : Paramètres → Vie privée et sécurité
#   → Décocher « Autoriser le contenu distant dans les messages »
# Evolution : Préférences → Préférences du courriel → HTML
#   → Décocher « Charger les images uniquement dans les courriels des contacts »
# KMail : Configuration → Sécurité → Lecture
#   → Décocher « Charger automatiquement les images externes »

# Afficher les messages en texte brut par défaut (sécurité maximale)
# Thunderbird : Affichage → Corps du message en → Texte brut
# Réduit la surface d'attaque des e-mails HTML malveillants

# Vérifier les en-têtes des e-mails suspects
# Thunderbird : Affichage → En-têtes → Tous
# Permet d'examiner le parcours du message et de détecter le phishing
```

## Résumé

Le choix d'un client de messagerie sous Debian dépend de l'environnement de bureau, des besoins en fonctionnalités PIM (calendrier, contacts, tâches) et de l'écosystème de serveurs de messagerie utilisé. Thunderbird est le choix polyvalent par défaut, adapté à tous les environnements et tous les fournisseurs. Evolution excelle dans l'intégration GNOME et le support Exchange. KMail/Kontact offre la meilleure intégration avec l'écosystème KDE.

Quel que soit le client choisi, les bonnes pratiques de sécurité (désactivation du contenu distant, chiffrement OpenPGP) et de sauvegarde (archivage régulier du profil) sont essentielles pour une utilisation sereine de la messagerie.
