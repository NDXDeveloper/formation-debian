# 2.1.1 GNOME (environnement par défaut)

## Introduction

GNOME (historiquement *GNU Network Object Model Environment*, mais cet acronyme n'est plus utilisé officiellement par le projet ; depuis 2021, les références à GNU ont même été retirées du code et de la documentation) est l'environnement de bureau par défaut de Debian depuis Debian 8 « Jessie » (2015). Ce choix n'est pas anodin : GNOME est l'un des projets de bureau libre les plus aboutis, soutenu par la fondation GNOME et un large écosystème de contributeurs. Il incarne une vision opinionée de l'interface utilisateur Linux, privilégiant la simplicité, la cohérence visuelle et un flux de travail centré sur les activités plutôt que sur la métaphore de bureau traditionnelle.

Debian 12 « Bookworm » intégrait GNOME 43, tandis que Debian 13 « Trixie » propose **GNOME 48** avec des améliorations significatives en matière de performances, de support Wayland et de personnalisation (couleur d'accentuation, notifications repensées, améliorations de Nautilus).

---

## Philosophie et principes de conception

GNOME repose sur un ensemble de principes de conception clairement définis, qui le distinguent des autres environnements de bureau :

### Simplicité par soustraction

GNOME fait le choix délibéré de limiter le nombre d'options exposées à l'utilisateur. Là où KDE Plasma propose des dizaines de réglages pour chaque composant, GNOME sélectionne des comportements par défaut soigneusement étudiés et masque la complexité. Cette philosophie, parfois qualifiée d'« opinionated design », divise les utilisateurs : certains apprécient l'interface épurée, d'autres regrettent le manque de flexibilité native (que l'on peut retrouver via les extensions).

### Focus et concentration

L'interface de GNOME est conçue pour minimiser les distractions. Il n'y a pas de barre des tâches permanente en bas de l'écran, pas d'icônes sur le bureau par défaut, pas de menus contextuels complexes. L'espace de travail est dédié à l'application en cours. Ce paradigme s'inspire des recherches en ergonomie sur la surcharge cognitive et favorise la productivité pour les utilisateurs qui travaillent avec un nombre réduit d'applications simultanées.

### Cohérence visuelle

GNOME impose des guidelines strictes aux développeurs d'applications via sa bibliothèque **libadwaita** (successeur de libhandy). Toutes les applications GNOME natives partagent le même langage visuel : barre d'en-tête intégrée (header bars), typographie cohérente, palette de couleurs unifiée, animations fluides. Cette cohérence est l'un des points forts de GNOME par rapport à d'autres environnements où chaque application peut avoir un style radicalement différent.

### Accessibilité comme priorité

GNOME est l'environnement de bureau Linux qui investit le plus dans l'accessibilité. Le lecteur d'écran **Orca**, le zoom d'écran, le clavier virtuel, le mode contraste élevé et les options de taille de police sont intégrés nativement. GNOME est souvent le seul choix viable pour les utilisateurs malvoyants ou à mobilité réduite sous Linux.

## Architecture de GNOME

Comprendre l'architecture de GNOME permet de mieux appréhender son fonctionnement, sa configuration et le diagnostic de problèmes éventuels.

### Les composants principaux

```
┌─────────────────────────────────────────────────────┐
│                   GNOME Shell                       │
│  (barre supérieure, vue activités, notifications)   │
├─────────────────────────────────────────────────────┤
│                    Mutter                           │
│  (gestionnaire de fenêtres + compositeur Wayland)   │
├─────────────────────────────────────────────────────┤
│              GTK4 / libadwaita                      │
│  (toolkit graphique et bibliothèque de widgets)     │
├─────────────────────────────────────────────────────┤
│         Serveur d'affichage (Wayland ou Xorg)       │
├─────────────────────────────────────────────────────┤
│               Noyau Linux + DRM/KMS                 │
└─────────────────────────────────────────────────────┘
```

**Mutter** est le gestionnaire de fenêtres et compositeur de GNOME. Sous Wayland (configuration par défaut), Mutter joue un double rôle : il est à la fois le compositeur Wayland (équivalent du serveur d'affichage) et le gestionnaire de fenêtres. Sous Xorg, Mutter fonctionne comme un gestionnaire de fenêtres compositeur classique au-dessus du serveur X.

**GNOME Shell** est le shell graphique qui fournit l'ensemble de l'interface utilisateur : la barre supérieure (top bar), la vue des activités (Activities Overview), le sélecteur d'espaces de travail, le système de notifications, les menus système et le lanceur d'applications. GNOME Shell est implémenté comme un **plugin de Mutter**, en **C et JavaScript** (GJS, GNOME JavaScript). Le rendu graphique passe par **Clutter** et **Cogl**, qui depuis GNOME 42 (mars 2022) ne sont plus des bibliothèques externes distinctes mais sont **intégrées au sein de Mutter** lui-même (les anciens dépôts Clutter/Cogl ont été archivés). C'est cette architecture C + JavaScript qui permet à GNOME Shell d'être étendu de manière dynamique via le système d'extensions.

**GTK4** (GIMP Toolkit, version 4) est le toolkit graphique utilisé par les applications GNOME. Couplé à **libadwaita**, il fournit les widgets, les thèmes et les motifs d'interface communs à toutes les applications de l'écosystème.

### Services d'arrière-plan (daemons)

GNOME repose sur plusieurs services qui fonctionnent en arrière-plan :

| Service | Rôle |
|---------|------|
| `gnome-session` | Gestion du cycle de vie de la session (démarrage, sauvegarde, fermeture) |
| `gnome-settings-daemon` (gsd) | Application des paramètres système (clavier, souris, écrans, alimentation, son, couleurs, etc.) |
| `gnome-shell` | Interface utilisateur principale |
| `tracker-miner-fs` | Indexation des fichiers pour la recherche rapide |
| `gvfsd` | Système de fichiers virtuel GNOME (accès réseau, clés USB, etc.) |
| `gnome-keyring-daemon` | Stockage sécurisé des mots de passe, clés SSH et certificats |
| `xdg-desktop-portal-gnome` | Portails pour les applications sandboxées (Flatpak) : sélecteur de fichiers, partage d'écran, etc. |
| `evolution-data-server` | Gestion centralisée des contacts, calendriers et comptes en ligne |

Ces services sont gérés par la session D-Bus utilisateur et démarrés automatiquement par `gnome-session`. Ils peuvent être inspectés avec :

```bash
# Lister les processus GNOME en cours
ps aux | grep gnome

# Voir les services de la session utilisateur via systemd
systemctl --user list-units | grep gnome

# Consulter les logs de la session GNOME
journalctl --user -b | grep gnome
```

## L'interface GNOME en détail

### La barre supérieure (Top Bar)

La barre supérieure est l'élément permanent de l'interface GNOME. Elle occupe le haut de l'écran et se compose de trois zones :

- **À gauche** : le bouton « Activités » (ou le coin actif en haut à gauche), qui ouvre la vue d'ensemble.
- **Au centre** : l'horloge et le calendrier. Un clic affiche un panneau avec les notifications et les événements de l'agenda.
- **À droite** : le menu système, qui regroupe les contrôles rapides : réseau, Bluetooth, volume sonore, luminosité, batterie (sur portable), paramètres d'accessibilité, verrouillage de session et extinction.

### La vue Activités (Activities Overview)

La vue Activités est le cœur du flux de travail GNOME. Elle s'ouvre en cliquant sur « Activités », en appuyant sur la touche `Super` (touche Windows), ou en poussant le curseur dans le coin supérieur gauche. Elle offre :

- **La vue des fenêtres ouvertes** : toutes les fenêtres de l'espace de travail actif sont affichées en mosaïque, permettant une sélection visuelle rapide.
- **Le sélecteur d'espaces de travail** : visible sur le côté, il permet de naviguer entre les espaces de travail virtuels. GNOME utilise des espaces de travail dynamiques par défaut : un nouvel espace est créé automatiquement lorsque le dernier est occupé.
- **La barre de recherche** : en tapant directement dans la vue Activités, une recherche universelle se lance. Elle couvre les applications installées, les fichiers récents (via Tracker), les contacts, les paramètres système, les caractères spéciaux et les résultats de calculatrice. Cette recherche est l'un des moyens les plus rapides de naviguer dans GNOME.

### Le lanceur d'applications (App Grid)

Accessible depuis la vue Activités via l'icône en grille (ou un balayage vers le haut sur écran tactile), le lanceur affiche toutes les applications installées sous forme d'une grille paginée. Les applications peuvent être organisées en dossiers par glisser-déposer.

### Le Dash

Le Dash est la barre verticale de favoris visible dans la vue Activités. Elle contient les applications épinglées par l'utilisateur ainsi que les applications en cours d'exécution (identifiées par un indicateur lumineux). Le Dash se comporte comme un dock contextuel, uniquement visible lorsque la vue Activités est active — contrairement aux docks permanents que l'on retrouve dans d'autres environnements.

### Les notifications

GNOME gère les notifications via un système centralisé accessible depuis l'horloge en barre supérieure. Les notifications sont non intrusives : elles apparaissent brièvement en haut de l'écran puis sont archivées dans le panneau de notifications. Le mode « Ne pas déranger » permet de les masquer temporairement.

## Configuration de GNOME

### Paramètres système (gnome-control-center)

L'application **Paramètres** (`gnome-control-center`) est le point central de configuration de GNOME. Elle couvre :

- Réseau et Wi-Fi
- Bluetooth
- Affichage (résolution, taux de rafraîchissement, disposition multi-écrans, mode nuit)
- Son (entrées/sorties audio, volume par application)
- Alimentation (comportement du bouton d'alimentation, mise en veille, économie d'énergie)
- Apparence (thème clair/sombre, image de fond, couleur d'accentuation depuis GNOME 47)
- Comptes en ligne (Google, Microsoft, Nextcloud — intégration calendrier, contacts, fichiers)
- Clavier (raccourcis, méthodes de saisie, disposition)
- Accessibilité
- Utilisateurs et partage
- Date et heure, région et langue

### dconf et gsettings : la configuration en profondeur

Sous la surface des Paramètres, GNOME stocke l'ensemble de sa configuration dans une base de données binaire appelée **dconf**. L'outil en ligne de commande `gsettings` permet d'interagir avec cette base :

```bash
# Lire un paramètre
gsettings get org.gnome.desktop.interface color-scheme

# Modifier un paramètre (activer le thème sombre)
gsettings set org.gnome.desktop.interface color-scheme 'prefer-dark'

# Lister tous les schémas disponibles
gsettings list-schemas

# Lister les clés d'un schéma
gsettings list-keys org.gnome.desktop.interface

# Afficher la description d'une clé
gsettings describe org.gnome.desktop.interface font-name

# Réinitialiser un paramètre à sa valeur par défaut
gsettings reset org.gnome.desktop.interface font-name
```

L'outil graphique **dconf-editor** offre une interface de navigation complète dans l'arbre de configuration :

```bash
sudo apt install dconf-editor
```

`dconf-editor` est particulièrement utile pour explorer les paramètres cachés qui ne sont pas exposés dans l'application Paramètres. Il permet de naviguer dans l'arborescence, de lire les descriptions et de modifier les valeurs avec une validation de type.

### Exemples de réglages courants via gsettings

```bash
# Afficher les pourcentages de batterie dans la barre supérieure
gsettings set org.gnome.desktop.interface show-battery-percentage true

# Configurer l'action du bouton de fermeture de fenêtre
# (par défaut, seul le bouton fermer est affiché)
gsettings set org.gnome.desktop.wm.preferences button-layout 'appmenu:minimize,maximize,close'

# Définir le comportement du pavé tactile (tapotement pour clic)
gsettings set org.gnome.desktop.peripherals.touchpad tap-to-click true

# Configurer les espaces de travail statiques (au lieu de dynamiques)
gsettings set org.gnome.mutter dynamic-workspaces false  
gsettings set org.gnome.desktop.wm.preferences num-workspaces 4  

# Activer le mode « Ne pas déranger »
gsettings set org.gnome.desktop.notifications show-banners false
```

## GNOME Tweaks : personnalisation avancée

L'application **GNOME Tweaks** (anciennement Tweak Tool) comble le fossé entre les Paramètres officiels et dconf-editor. Elle expose des réglages que GNOME considère comme secondaires mais qui sont essentiels pour de nombreux utilisateurs :

```bash
sudo apt install gnome-tweaks
```

Les réglages accessibles via GNOME Tweaks incluent :

- **Apparence** : thème GTK, thème d'icônes, thème du curseur, thème du shell GNOME.
- **Barre de titre** : ajout des boutons minimiser et maximiser (absents par défaut), actions au double-clic et clic du milieu.
- **Polices** : police de l'interface, police des documents, police à chasse fixe, facteur de mise à l'échelle des polices, anti-aliasing et hinting.
- **Fenêtres** : comportement du focus, action du clic sur la barre de titre, rattachement aux bords (edge tiling).
- **Démarrage** : applications lancées automatiquement à l'ouverture de session.
- **Clavier et souris** : profil d'accélération, émulation du clic du milieu, touches supplémentaires (Compose, Caps Lock comme Ctrl, etc.).
- **Extensions** : activation et désactivation des extensions GNOME Shell (remplacé par l'application Extensions dédiée dans les versions récentes).

## Les extensions GNOME Shell

Les extensions sont l'un des mécanismes les plus puissants de GNOME. Elles permettent de modifier profondément le comportement et l'apparence de GNOME Shell sans toucher au code source du projet. Une extension est essentiellement un module JavaScript qui s'injecte dans le processus GNOME Shell.

### Installation et gestion

```bash
# Installer le support des extensions et le navigateur d'intégration
sudo apt install gnome-shell-extensions gnome-shell-extension-prefs

# Installer le connecteur pour le site extensions.gnome.org (optionnel)
sudo apt install gnome-browser-connector
```

Les extensions peuvent être installées de trois manières :

1. **Via le site web** [extensions.gnome.org](https://extensions.gnome.org) : un catalogue en ligne avec installation en un clic (nécessite `gnome-browser-connector` et l'extension de navigateur correspondante).
2. **Via les dépôts Debian** : certaines extensions populaires sont empaquetées dans les dépôts (`apt search gnome-shell-extension`).
3. **Manuellement** : en plaçant les fichiers de l'extension dans `~/.local/share/gnome-shell/extensions/`.

La gestion des extensions s'effectue via l'application **Extensions** :

```bash
sudo apt install gnome-shell-extension-manager
```

Ou en ligne de commande avec `gnome-extensions` :

```bash
# Lister les extensions installées
gnome-extensions list

# Activer une extension
gnome-extensions enable nom-extension@auteur

# Désactiver une extension
gnome-extensions disable nom-extension@auteur

# Afficher les informations d'une extension
gnome-extensions info nom-extension@auteur
```

### Extensions populaires dans l'écosystème Debian

Quelques extensions fréquemment utilisées pour adapter GNOME à différents flux de travail :

| Extension | Fonction |
|-----------|----------|
| **Dash to Dock** | Transforme le Dash en dock permanent visible sur le bureau (comportement similaire à macOS ou Ubuntu) |
| **Dash to Panel** | Transforme la barre supérieure en barre des tâches classique intégrant le Dash (comportement similaire à Windows) |
| **AppIndicator/KStatusNotifierItem** | Affiche les icônes de la zone de notification (tray icons) des applications qui en dépendent |
| **GSConnect** | Intégration avec les téléphones Android via le protocole KDE Connect (partage de fichiers, notifications, presse-papiers) |
| **Clipboard Indicator** | Gestionnaire de presse-papiers avec historique |
| **Caffeine** | Empêche la mise en veille et l'extinction de l'écran (utile lors de présentations ou de téléchargements longs) |
| **Blur my Shell** | Ajoute un effet de flou à la vue Activités et à la barre supérieure |

### Précautions avec les extensions

Les extensions sont soumises à des considérations importantes :

- **Compatibilité** : les extensions sont liées à une version spécifique de GNOME Shell. Lors d'une mise à jour majeure de Debian (et donc de GNOME), certaines extensions peuvent cesser de fonctionner. C'est la cause la plus fréquente de problèmes après une montée de version.
- **Stabilité** : une extension mal écrite peut provoquer des crashs de GNOME Shell. Si GNOME Shell refuse de démarrer, il est possible de le lancer en mode sans extensions pour diagnostiquer le problème :

```bash
# Désactiver toutes les extensions (depuis un TTY ou SSH)
gsettings set org.gnome.shell disable-user-extensions true

# Redémarrer GNOME Shell (sous Xorg uniquement)
# Alt+F2, puis taper 'r' et Entrée

# Sous Wayland, il faut se déconnecter et se reconnecter
```

- **Sécurité** : les extensions ont accès à l'ensemble du processus GNOME Shell. Il est recommandé de n'installer que des extensions provenant de sources fiables et de vérifier régulièrement les mises à jour.

## Raccourcis clavier essentiels

La maîtrise des raccourcis clavier est fondamentale pour exploiter efficacement GNOME. Voici les raccourcis les plus importants :

### Navigation générale

| Raccourci | Action |
|-----------|--------|
| `Super` | Ouvrir/fermer la vue Activités |
| `Super` + saisie au clavier | Recherche universelle (applications, fichiers, paramètres) |
| `Super + A` | Ouvrir la grille d'applications |
| `Super + L` | Verrouiller la session |
| `Alt + F2` | Boîte de commande (Xorg uniquement) |

### Gestion des fenêtres

| Raccourci | Action |
|-----------|--------|
| `Super + ↑` | Maximiser la fenêtre active |
| `Super + ↓` | Restaurer / dé-maximiser la fenêtre |
| `Super + ←` / `Super + →` | Ancrer la fenêtre à gauche / droite (tiling demi-écran) |
| `Alt + Tab` | Basculer entre les applications |
| `Alt + ²` (touche au-dessus de Tab) | Basculer entre les fenêtres d'une même application |
| `Super + H` | Minimiser la fenêtre active |
| `Alt + F4` | Fermer la fenêtre active |

### Espaces de travail

| Raccourci | Action |
|-----------|--------|
| `Super + Page Haut` / `Page Bas` | Naviguer entre les espaces de travail |
| `Super + Shift + Page Haut` / `Page Bas` | Déplacer la fenêtre active vers un autre espace de travail |

### Système

| Raccourci | Action |
|-----------|--------|
| `Ctrl + Alt + Suppr` | Menu d'extinction (configurable) |
| `Ctrl + Alt + T` | Ouvrir un terminal (si configuré, non actif par défaut dans GNOME pur) |
| `Impr. écran` | Capture d'écran (outil intégré GNOME Screenshot) |
| `Shift + Impr. écran` | Capture d'une zone sélectionnée |

Les raccourcis sont entièrement personnalisables dans **Paramètres → Clavier → Raccourcis clavier**.

## GNOME sur Wayland vs Xorg sous Debian

Depuis **Debian 10 « Buster » (juillet 2019)**, GNOME utilise **Wayland par défaut** comme protocole d'affichage — Debian a été parmi les premières grandes distributions à adopter ce changement. Une session Xorg reste disponible comme option de secours sur l'écran de connexion GDM (cliquer sur l'icône d'engrenage avant de se connecter).

### Avantages de GNOME sous Wayland

- **Sécurité** : les applications sont isolées les unes des autres. Une application ne peut pas capturer les frappes clavier ou le contenu d'une autre fenêtre (contrairement à Xorg où toute application a accès à l'intégralité du serveur X).
- **Performances** : le rendu est plus direct, avec moins de copies de tampons graphiques. Le tearing (déchirement d'image) est éliminé par conception.
- **HiDPI et mise à l'échelle** : Wayland gère la mise à l'échelle fractionnaire et par écran, ce qui est particulièrement utile sur les configurations multi-écrans avec des résolutions différentes.
- **Gestes tactiles** : les gestes multi-doigts (balayage à trois doigts pour les espaces de travail, pincement pour zoomer) sont intégrés nativement.

### Situations nécessitant Xorg

Certains cas de figure peuvent justifier l'utilisation de la session Xorg :

- **Pilotes NVIDIA propriétaires** : bien que le support Wayland pour NVIDIA se soit amélioré, certaines configurations anciennes ou certains cas d'usage (CUDA, multi-GPU) peuvent encore poser problème.
- **Applications utilisant le protocole X11** : certaines applications anciennes ou certains logiciels propriétaires ne fonctionnent pas correctement sous Wayland. Le mécanisme **XWayland** assure une couche de compatibilité, mais elle n'est pas parfaite dans tous les cas.
- **Partage d'écran et capture vidéo** : les applications doivent supporter le portail `xdg-desktop-portal` pour fonctionner correctement sous Wayland. Les outils de capture basés sur X11 (comme `xdotool`, `xclip` en mode X11, `xrandr`) ne fonctionnent pas.
- **Outils d'automatisation** : `xdotool`, `xte` et les outils de simulation de clavier/souris basés sur X11 n'ont pas d'équivalent direct sous Wayland (pour des raisons de sécurité par conception).

### Vérifier la session en cours

```bash
# Vérifier si la session utilise Wayland ou Xorg
echo $XDG_SESSION_TYPE
# Résultat : "wayland" ou "x11"

# Vérifier via loginctl
loginctl show-session $(loginctl | grep $(whoami) | awk '{print $1}') -p Type
```

## Performances et optimisation

### Consommation de ressources

GNOME est l'un des environnements de bureau les plus gourmands en ressources. Une session GNOME typique consomme entre 800 Mo et 1,5 Go de RAM au repos, selon les extensions et services activés. Cette consommation s'explique par la richesse des fonctionnalités, l'indexation des fichiers (Tracker) et le compositeur Mutter.

### Pistes d'optimisation

Pour réduire la consommation de ressources sur les machines moins puissantes :

```bash
# Désactiver l'indexation de fichiers (Tracker)
systemctl --user mask tracker-miner-fs-3.service  
systemctl --user mask tracker-miner-fs-control-3.service  

# Réduire les animations
gsettings set org.gnome.desktop.interface enable-animations false

# Désactiver les extensions non essentielles
gnome-extensions disable nom-extension@auteur

# Désactiver les services de comptes en ligne si non utilisés
# (via Paramètres → Comptes en ligne → supprimer les comptes)
```

### Diagnostic des problèmes de performance

```bash
# Vérifier la consommation mémoire de GNOME Shell
ps aux | grep gnome-shell | grep -v grep

# Vérifier les fuites mémoire (GNOME Shell peut en avoir sur les sessions longues)
# La consommation de gnome-shell ne devrait pas dépasser 500-800 Mo
# Un redémarrage de session résout généralement les fuites

# Vérifier les logs pour des erreurs de rendu
journalctl --user -b | grep -i "mutter\|gnome-shell\|error"

# Sous Wayland, vérifier les performances du compositeur
# L'outil Looking Glass (Alt+F2 puis 'lg' sous Xorg) permet d'inspecter GNOME Shell
```

## Résumé

GNOME est un environnement de bureau mature, cohérent et bien intégré à Debian. Son approche minimaliste et son flux de travail centré sur les activités en font un choix productif pour les utilisateurs qui adoptent sa philosophie. Le système d'extensions permet de pallier les limitations perçues et d'adapter GNOME à des flux de travail très variés. Son support Wayland avancé et son investissement dans l'accessibilité en font l'environnement de référence pour les installations Debian modernes.

Les sections suivantes présentent les alternatives à GNOME : KDE Plasma pour les utilisateurs recherchant une personnalisation maximale, puis XFCE et LXDE pour ceux qui privilégient la légèreté.
