# 2.2.3 Gestion des formats de fichiers

## Introduction

Un poste de travail Debian est amené à manipuler une grande variété de formats de fichiers au quotidien : documents bureautiques reçus par e-mail, images dans divers formats, archives compressées, fichiers audio et vidéo, documents PDF, polices de caractères, fichiers de développement, etc. La capacité du système à ouvrir, afficher, convertir et manipuler ces formats de manière transparente est un facteur déterminant de l'expérience utilisateur.

Debian, en tant que distribution engagée dans le logiciel libre, ne peut pas inclure par défaut certains codecs et formats propriétaires pour des raisons de licence ou de brevets. Cette section explique comment le système identifie et gère les types de fichiers, comment installer le support des formats courants (y compris propriétaires), et comment résoudre les problèmes d'ouverture de fichiers les plus fréquents.

---

## Identification des types de fichiers

### Types MIME

Le système d'identification des fichiers sous Linux repose sur les **types MIME** (Multipurpose Internet Mail Extensions), un standard qui attribue un identifiant unique à chaque format. Le type MIME est composé d'un type principal et d'un sous-type séparés par une barre oblique : `application/pdf`, `image/jpeg`, `text/plain`, `audio/mpeg`, etc.

L'identification du type MIME se fait par deux mécanismes complémentaires :

**L'analyse du contenu (magic bytes)** : les premiers octets du fichier contiennent souvent une signature caractéristique du format. Un fichier PDF commence par `%PDF`, un fichier PNG par la séquence `\x89PNG`, un fichier ZIP par `PK`. Cette méthode est fiable mais nécessite de lire le fichier.

**L'extension du nom de fichier** : l'extension (`.pdf`, `.jpg`, `.docx`) donne une indication rapide du format. Cette méthode est moins fiable car l'extension peut être absente, incorrecte ou trompeuse.

```bash
# Identifier le type MIME d'un fichier (analyse du contenu)
file --mime-type document.pdf
# Résultat : document.pdf: application/pdf

# Identifier le type MIME via la base de données freedesktop (extension + contenu)
xdg-mime query filetype rapport.docx
# Résultat : application/vnd.openxmlformats-officedocument.wordprocessingml.document

# La commande file donne des informations plus détaillées
file rapport.docx
# Résultat : rapport.docx: Microsoft Word 2007+

file image.png
# Résultat : image.png: PNG image data, 1920 x 1080, 8-bit/color RGBA, non-interlaced

# Identifier le type même sans extension correcte
cp photo.jpg mystere  
file mystere  
# Résultat : mystere: JPEG image data, JFIF standard 1.01
```

### Base de données des types MIME

Le système maintient une base de données qui associe les types MIME aux extensions de fichiers, aux magic bytes, aux icônes et aux descriptions. Cette base est gérée par le paquet `shared-mime-info` :

```bash
# Le paquet fondamental pour la reconnaissance des types de fichiers
sudo apt install shared-mime-info

# Emplacement de la base de données
/usr/share/mime/

# Mettre à jour la base de données MIME après modification manuelle
sudo update-mime-database /usr/share/mime

# Consulter la définition d'un type MIME
cat /usr/share/mime/application/pdf.xml
```

### Associations fichier-application

Le lien entre un type de fichier et l'application qui l'ouvre est géré par le système d'associations MIME, présenté dans la section 2.2. Pour mémoire, les commandes essentielles :

```bash
# Connaître l'application par défaut pour un type de fichier
xdg-mime query default application/pdf

# Modifier l'application par défaut
xdg-mime default org.gnome.Evince.desktop application/pdf

# Ouvrir un fichier avec l'application par défaut
xdg-open fichier.pdf

# Le fichier de configuration des associations utilisateur
cat ~/.config/mimeapps.list
```

Les gestionnaires de fichiers (Nautilus, Dolphin, Thunar) permettent également de modifier l'application par défaut via le clic droit → Propriétés → Ouvrir avec, ou via le clic droit → Ouvrir avec une autre application.

## Documents bureautiques

### Formats Microsoft Office

La compatibilité avec les formats Microsoft Office est l'une des préoccupations les plus courantes des utilisateurs Linux. Debian gère ces formats principalement via LibreOffice.

| Format | Extension | Type MIME | Application par défaut |
|--------|-----------|-----------|----------------------|
| Word (ancien) | `.doc` | `application/msword` | LibreOffice Writer |
| Word (moderne) | `.docx` | `application/vnd.openxmlformats-officedocument.wordprocessingml.document` | LibreOffice Writer |
| Excel (ancien) | `.xls` | `application/vnd.ms-excel` | LibreOffice Calc |
| Excel (moderne) | `.xlsx` | `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` | LibreOffice Calc |
| PowerPoint (ancien) | `.ppt` | `application/vnd.ms-powerpoint` | LibreOffice Impress |
| PowerPoint (moderne) | `.pptx` | `application/vnd.openxmlformats-officedocument.presentationml.presentation` | LibreOffice Impress |

Les formats modernes (`.docx`, `.xlsx`, `.pptx`) sont basés sur le standard Office Open XML (OOXML). LibreOffice les gère correctement dans la grande majorité des cas. Les problèmes de compatibilité les plus fréquents concernent les mises en page complexes, les macros VBA, les polices de caractères Windows absentes sous Linux, et certains graphiques SmartArt.

#### Installation des polices Microsoft

L'absence des polices Microsoft (Arial, Times New Roman, Calibri, Cambria) est la cause la plus visible de différences de rendu entre un document ouvert sous Windows et sous Linux. L'installation de ces polices améliore significativement la fidélité d'affichage :

```bash
# Polices Microsoft de base (Arial, Times New Roman, Courier New, etc.)
# Le paquet nécessite l'acceptation de la licence Microsoft
sudo apt install ttf-mscorefonts-installer

# Vérifier l'installation
fc-list | grep -i arial

# Rafraîchir le cache des polices
fc-cache -fv
```

Les polices Calibri et Cambria (utilisées par défaut dans Microsoft Office depuis 2007) ne sont pas incluses dans le paquet `ttf-mscorefonts-installer`. Elles peuvent être copiées depuis une installation Windows ou remplacées par les polices métriquement compatibles du projet **Croscore** :

```bash
# Polices Croscore (compatibles métriquement avec Calibri, Cambria, etc.)
# Carlito → compatible avec Calibri
# Caladea → compatible avec Cambria
sudo apt install fonts-crosextra-carlito fonts-crosextra-caladea

# Polices Liberation (compatibles métriquement avec Arial, Times, Courier)
# Déjà installées par défaut dans la plupart des installations Debian
sudo apt install fonts-liberation2
```

### Formats OpenDocument (ODF)

Les formats OpenDocument (`.odt`, `.ods`, `.odp`) sont les formats natifs de LibreOffice et le standard ISO pour les documents bureautiques. Ils sont intégralement supportés par Debian sans aucune configuration supplémentaire.

| Format | Extension | Application |
|--------|-----------|-------------|
| Texte | `.odt` | LibreOffice Writer |
| Tableur | `.ods` | LibreOffice Calc |
| Présentation | `.odp` | LibreOffice Impress |
| Dessin | `.odg` | LibreOffice Draw |
| Formule | `.odf` | LibreOffice Math |

### Conversion entre formats

```bash
# Conversion en ligne de commande via LibreOffice (mode headless)
# Convertir un fichier DOCX en PDF
libreoffice --headless --convert-to pdf document.docx

# Convertir un fichier DOCX en ODT
libreoffice --headless --convert-to odt document.docx

# Convertir un fichier ODP en PDF
libreoffice --headless --convert-to pdf presentation.odp

# Conversion par lot (tous les fichiers DOCX d'un répertoire)
libreoffice --headless --convert-to pdf *.docx

# Convertir en spécifiant le répertoire de sortie
libreoffice --headless --convert-to pdf --outdir ~/conversions/ document.docx
```

## Documents PDF

### Lecteurs PDF

```bash
# Evince (lecteur GNOME, installé par défaut)
sudo apt install evince

# Okular (lecteur KDE, annotations avancées, support multi-formats)
sudo apt install okular

# Atril (lecteur MATE, léger)
sudo apt install atril

# Zathura (lecteur minimaliste, navigation clavier style Vim)
sudo apt install zathura zathura-pdf-poppler
```

### Manipulation de fichiers PDF

```bash
# Outils en ligne de commande pour manipuler les PDF
sudo apt install poppler-utils

# Extraire le texte d'un PDF
pdftotext document.pdf document.txt

# Extraire les images d'un PDF
pdfimages document.pdf images/

# Obtenir les informations d'un PDF (métadonnées, nombre de pages, version)
pdfinfo document.pdf

# Fusionner des PDF
# Via pdfunite (poppler-utils)
pdfunite fichier1.pdf fichier2.pdf fichier3.pdf resultat.pdf

# Séparer un PDF (extraire des pages)
# Via pdfseparate (poppler-utils)
pdfseparate -f 1 -l 5 document.pdf page-%d.pdf   # Pages 1 à 5

# Manipulation avancée avec pdftk
sudo apt install pdftk-java

# Fusionner
pdftk fichier1.pdf fichier2.pdf cat output resultat.pdf

# Extraire les pages 3 à 7
pdftk document.pdf cat 3-7 output extrait.pdf

# Rotation des pages
pdftk document.pdf cat 1-endeast output pivote.pdf

# Protéger par mot de passe
pdftk document.pdf output protege.pdf user_pw motdepasse

# Outil graphique pour réorganiser des pages PDF
sudo apt install pdfarranger
```

### Création et édition de PDF

La création de PDF est native dans toutes les applications Linux grâce au système d'impression CUPS, qui intègre un pilote d'imprimante virtuelle PDF. Toute application disposant d'une fonction d'impression peut générer un PDF via « Imprimer vers fichier ».

Pour l'édition de formulaires PDF et les annotations :

```bash
# Okular permet les annotations (surlignage, notes, tampons)
# Les annotations sont sauvegardées dans un fichier séparé par défaut
# Pour les intégrer au PDF : Fichier → Enregistrer sous

# Xournal++ (annotation, écriture manuscrite, signature de PDF)
sudo apt install xournalpp

# LibreOffice Draw peut ouvrir et éditer le contenu de fichiers PDF
# (conversion en objets éditables, avec perte possible de mise en page)
libreoffice --draw document.pdf
```

## Images

### Formats courants

Debian supporte nativement tous les formats d'image courants grâce aux bibliothèques partagées installées par défaut :

| Format | Extension | Type | Support Debian |
|--------|-----------|------|---------------|
| **JPEG** | `.jpg`, `.jpeg` | Bitmap, compression avec perte | Natif (`libjpeg`) |
| **PNG** | `.png` | Bitmap, compression sans perte, transparence | Natif (`libpng`) |
| **GIF** | `.gif` | Bitmap, animation, palette limitée | Natif |
| **WebP** | `.webp` | Bitmap, compression moderne (Google) | Natif (`libwebp`) |
| **AVIF** | `.avif` | Bitmap, compression nouvelle génération | Supporté (`libavif`) |
| **SVG** | `.svg` | Vectoriel | Natif (`librsvg`) |
| **TIFF** | `.tif`, `.tiff` | Bitmap, haute qualité, multi-pages | Natif (`libtiff`) |
| **BMP** | `.bmp` | Bitmap, non compressé (héritage Windows) | Natif |
| **HEIF/HEIC** | `.heif`, `.heic` | Bitmap, format Apple/Android moderne | À installer |
| **RAW** | `.cr2`, `.nef`, `.arw`, etc. | Données brutes du capteur photo | Via `dcraw` ou `darktable` |

```bash
# Support HEIF/HEIC (photos iPhone et certains Android)
sudo apt install heif-gdk-pixbuf heif-thumbnailer libheif-examples

# Après installation, les visionneuses d'images qui s'appuient sur GdkPixbuf
# (Loupe et son prédécesseur Eye of GNOME, Gwenview, Ristretto, etc.) peuvent
# afficher les fichiers HEIC. L'outil heif-convert permet la conversion en
# ligne de commande :
heif-convert photo.heic photo.jpg

# Outils de conversion d'images en ligne de commande
sudo apt install imagemagick

# Convertir un format vers un autre
convert image.png image.jpg  
convert image.webp image.png  

# Redimensionner une image
convert image.jpg -resize 1920x1080 image_reduite.jpg

# Conversion par lot (tous les PNG en JPEG)
mogrify -format jpg *.png

# Optimiser des JPEG (réduire la taille sans perte visible)
sudo apt install jpegoptim  
jpegoptim --max=85 photo.jpg  

# Optimiser des PNG (compression sans perte)
sudo apt install optipng  
optipng image.png  
```

### Vignettes dans les gestionnaires de fichiers

Les gestionnaires de fichiers génèrent des vignettes (thumbnails) pour prévisualiser les fichiers. Ce mécanisme repose sur des « tumbler » ou « thumbnailer » spécialisés par type de fichier :

```bash
# Vignettes vidéo dans les gestionnaires de fichiers
sudo apt install ffmpegthumbnailer

# Vignettes PDF
# Généralement incluses via poppler-glib (déjà installé avec Evince)

# Vignettes pour les fichiers LibreOffice/Microsoft Office
sudo apt install gnome-sushi     # Prévisualisation rapide dans Nautilus (barre espace)
# Ou pour Thunar (XFCE)
sudo apt install tumbler tumbler-plugins-extra

# Régénérer les vignettes en cas de problème
rm -rf ~/.cache/thumbnails/*
# Les vignettes seront recréées automatiquement à la prochaine ouverture du dossier
```

## Archives et compression

### Formats d'archives supportés

| Format | Extension | Outil CLI | Paquet requis | Libre |
|--------|-----------|-----------|---------------|-------|
| **tar** | `.tar` | `tar` | `tar` (installé) | Oui |
| **gzip** | `.gz`, `.tar.gz`, `.tgz` | `gzip`, `tar` | `gzip` (installé) | Oui |
| **bzip2** | `.bz2`, `.tar.bz2` | `bzip2`, `tar` | `bzip2` (installé) | Oui |
| **xz** | `.xz`, `.tar.xz` | `xz`, `tar` | `xz-utils` (installé) | Oui |
| **zstd** | `.zst`, `.tar.zst` | `zstd`, `tar` | `zstd` (installé) | Oui |
| **ZIP** | `.zip` | `zip`, `unzip` | `zip`, `unzip` | Oui |
| **7-Zip** | `.7z` | `7z` | `p7zip-full` | Oui |
| **RAR** | `.rar` | `unrar` | `unrar-free` ou `unrar` | Libre / Non-free |
| **CAB** | `.cab` | `cabextract` | `cabextract` | Oui |
| **ISO** | `.iso` | `mount`, `xorriso` | Outils système | Oui |

```bash
# Installer le support complet des archives
sudo apt install zip unzip p7zip-full unrar-free cabextract xz-utils zstd

# Pour le support RAR complet (y compris les archives RAR5)
# Le paquet unrar (non-free) offre une meilleure compatibilité que unrar-free
sudo apt install unrar

# Opérations courantes en ligne de commande

# Extraire une archive (tar détecte automatiquement la compression)
tar xf archive.tar.gz  
tar xf archive.tar.xz  
tar xf archive.tar.zst  

# Créer une archive
tar czf archive.tar.gz dossier/        # gzip  
tar cJf archive.tar.xz dossier/        # xz (meilleure compression)  
tar --zstd -cf archive.tar.zst dossier/ # zstd (rapide et efficace)  

# ZIP
zip -r archive.zip dossier/  
unzip archive.zip  

# 7-Zip
7z a archive.7z dossier/
7z x archive.7z

# RAR (extraction uniquement avec unrar)
unrar x archive.rar
```

### Intégration graphique

Les gestionnaires d'archives graphiques (File Roller pour GNOME, Ark pour KDE, Xarchiver pour XFCE) prennent en charge automatiquement tous les formats dont les outils en ligne de commande sont installés. Il suffit d'installer les paquets CLI listés ci-dessus pour que le gestionnaire d'archives graphique puisse ouvrir le format correspondant.

```bash
# Gestion des archives par clic droit dans le gestionnaire de fichiers
# Nautilus : clic droit → Extraire ici / Compresser
# Dolphin : clic droit → Extraire → Extraire l'archive ici
# Thunar : clic droit → Extraire ici (nécessite thunar-archive-plugin)
sudo apt install thunar-archive-plugin
```

## Polices de caractères

### Formats de polices

Linux supporte les trois principaux formats de polices : TrueType (`.ttf`), OpenType (`.otf`) et Web Open Font Format (`.woff`, `.woff2`). Le système de gestion des polices repose sur **Fontconfig**, qui maintient un index de toutes les polices disponibles et les rend accessibles à toutes les applications.

### Installation de polices

```bash
# Polices disponibles dans les dépôts Debian
sudo apt install fonts-noto              # Noto (Google) — couverture Unicode massive  
sudo apt install fonts-noto-cjk          # Noto CJK (chinois, japonais, coréen)  
sudo apt install fonts-noto-color-emoji  # Emojis couleur  
sudo apt install fonts-liberation2       # Liberation (compatibles métriquement avec Arial/Times/Courier)  
sudo apt install fonts-firacode          # Fira Code (police monospace avec ligatures pour le développement)  
sudo apt install fonts-jetbrains-mono    # JetBrains Mono (police monospace pour IDE)  
sudo apt install fonts-hack              # Hack (police monospace)  
sudo apt install fonts-roboto            # Roboto (Google, utilisée dans Android/Material Design)  
sudo apt install fonts-open-sans         # Open Sans  
sudo apt install fonts-lato              # Lato  

# Installation manuelle de polices (fichiers .ttf ou .otf)
# Pour l'utilisateur courant uniquement
mkdir -p ~/.local/share/fonts  
cp police.ttf ~/.local/share/fonts/  
fc-cache -fv  

# Pour tous les utilisateurs du système
sudo cp police.ttf /usr/local/share/fonts/  
sudo fc-cache -fv  

# Lister les polices installées
fc-list

# Rechercher une police par nom
fc-list | grep -i "fira"

# Vérifier quelle police sera utilisée pour un texte donné
# (utile pour diagnostiquer les problèmes de substitution)
fc-match "Arial"
# Résultat : Liberation Sans si Arial n'est pas installé

fc-match "Calibri"
# Résultat : Carlito si Calibri n'est pas installé et Carlito est installé
```

### Gestionnaires de polices graphiques

```bash
# GNOME Font Viewer (prévisualisation et installation en un clic)
sudo apt install gnome-font-viewer
# Double-cliquer sur un fichier .ttf/.otf ouvre le visualiseur avec un bouton Installer

# Pour une gestion plus avancée des collections de polices
# Font Manager
sudo apt install font-manager
```

## Résolution des problèmes courants

### « Aucune application pour ouvrir ce fichier »

Ce message indique qu'aucune application installée n'est associée au type MIME du fichier. Les étapes de diagnostic :

```bash
# 1. Identifier le type du fichier
file fichier_inconnu  
xdg-mime query filetype fichier_inconnu  

# 2. Chercher une application capable de l'ouvrir
# Rechercher dans les dépôts par type MIME
appstreamcli what-provides mediatype "le/type-mime"

# 3. Installer l'application appropriée
sudo apt install application_trouvee

# 4. Définir l'association
xdg-mime default application.desktop le/type-mime
```

### Fichier ouvert par la mauvaise application

```bash
# Identifier l'application actuellement associée
xdg-mime query default $(xdg-mime query filetype fichier)

# Changer l'application par défaut
# Méthode graphique : clic droit sur le fichier → Propriétés → Ouvrir avec
# Méthode CLI :
xdg-mime default nouvelle-application.desktop $(xdg-mime query filetype fichier)

# Vérifier le fichier de configuration
cat ~/.config/mimeapps.list
```

### Caractères mal affichés dans un document

Les problèmes d'encodage de caractères sont fréquents avec les fichiers provenant de systèmes Windows ou de systèmes utilisant des encodages régionaux (ISO-8859-1, Windows-1252, Shift-JIS).

```bash
# Détecter l'encodage d'un fichier texte
file -i fichier.txt
# Résultat possible : text/plain; charset=iso-8859-1

# Convertir l'encodage
iconv -f ISO-8859-1 -t UTF-8 fichier.txt > fichier_utf8.txt

# Convertir en place avec recode
sudo apt install recode  
recode ISO-8859-1..UTF-8 fichier.txt  

# Problème courant : les noms de fichiers dans les archives ZIP
# créées sous Windows peuvent avoir des caractères mal encodés
# Utiliser convmv pour corriger les noms de fichiers
sudo apt install convmv  
convmv -f cp437 -t utf-8 --notest -r dossier_extrait/  
```

### Vignettes manquantes dans le gestionnaire de fichiers

```bash
# Vérifier que les thumbnailers nécessaires sont installés
sudo apt install ffmpegthumbnailer   # Vidéos  
sudo apt install tumbler             # Framework de vignettes (XFCE)  
sudo apt install gnome-sushi         # Prévisualisation rapide GNOME  

# Purger le cache de vignettes et le reconstruire
rm -rf ~/.cache/thumbnails/*

# Vérifier que la génération de vignettes n'est pas désactivée
# GNOME :
gsettings get org.gnome.desktop.thumbnailers disable-all
# Doit retourner false

# Si la vignette ne se génère pas pour un type de fichier spécifique,
# vérifier qu'un thumbnailer est enregistré pour ce type MIME
ls /usr/share/thumbnailers/
```

## Résumé

La gestion des formats de fichiers sous Debian est largement transparente pour les formats ouverts et les standards courants. Les formats propriétaires ou protégés par des brevets (HEIC, certains codecs, polices Microsoft) nécessitent l'installation de paquets supplémentaires, généralement disponibles dans les dépôts `non-free` ou `non-free-firmware` de Debian.

Les points essentiels à retenir sont l'installation des polices de compatibilité Microsoft pour les documents bureautiques, le support du format HEIC pour les photos provenant d'appareils Apple, et l'installation des outils d'archives (`p7zip-full`, `unrar`) pour une prise en charge complète des formats compressés. La section suivante (2.2.4) aborde la configuration multimédia en détail : codecs audio et vidéo et PipeWire.
