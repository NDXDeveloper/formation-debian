# 2.1.3 XFCE et LXDE

## Introduction

Après GNOME et KDE Plasma, les deux environnements de bureau majeurs de l'écosystème Linux, Debian propose également des alternatives légères destinées aux machines disposant de ressources limitées ou aux utilisateurs qui privilégient la rapidité et la sobriété. Parmi ces alternatives, **XFCE** et **LXDE** (ainsi que son successeur **LXQt**) occupent une place historique et restent largement utilisés.

Ces environnements partagent un objectif commun : offrir un bureau fonctionnel et agréable sans imposer une charge significative sur le processeur, la mémoire vive ou le processeur graphique. Ils sont le choix naturel pour les machines anciennes, les ordinateurs mono-cartes (Raspberry Pi), les environnements virtualisés, les postes de travail d'entreprise standardisés, ou tout simplement pour les utilisateurs qui considèrent que les ressources système doivent être allouées aux applications plutôt qu'au bureau lui-même.

Cette section présente XFCE, LXDE et LXQt en détail, avec leurs forces respectives, leurs différences architecturales et les critères de choix entre eux.

# XFCE

## Présentation

XFCE (prononcé « ex-ef-si-i » ou simplement « xfce ») est un environnement de bureau libre créé en 1996 par Olivier Fourdan. Son nom était à l'origine l'acronyme de « XForms Common Environment », bien que XFCE ait abandonné le toolkit XForms depuis longtemps au profit de GTK. Aujourd'hui, XFCE est développé par une petite équipe de contributeurs qui maintiennent un rythme de publication volontairement lent, privilégiant la stabilité absolue à la nouveauté.

XFCE est souvent décrit comme le « juste milieu » de l'écosystème Linux : plus léger que GNOME et KDE Plasma, plus complet et plus poli que LXDE, et suffisamment modulaire pour s'adapter à de très nombreux cas d'usage. C'est l'environnement de bureau léger le plus populaire, utilisé par défaut dans plusieurs distributions (Xubuntu, Manjaro XFCE, Linux Mint XFCE).

Debian 12 « Bookworm » intégrait XFCE 4.18, tandis que Debian 13 « Trixie » propose **XFCE 4.20**, une version majeure qui introduit un support expérimental de Wayland via la nouvelle bibliothèque `libxfce4windowing`, ainsi que des améliorations significatives de Thunar (barre d'outils, décorations côté client, glisser-déposer amélioré).

## Philosophie

XFCE adhère à trois principes fondamentaux :

**Légèreté** : l'environnement doit fonctionner de manière fluide sur du matériel modeste. XFCE évite les effets graphiques gourmands, les animations superflues et les services d'arrière-plan non essentiels. Au repos, une session XFCE consomme typiquement entre 300 et 500 Mo de RAM.

**Stabilité** : le cycle de développement de XFCE est intentionnellement long (deux à trois ans entre les versions majeures). Les nouvelles fonctionnalités sont introduites avec prudence et les changements radicaux sont rares. Un utilisateur XFCE retrouve ses repères d'une version à l'autre sans réapprendre l'interface.

**Modularité** : chaque composant de XFCE est un logiciel indépendant qui peut être utilisé séparément ou remplacé par une alternative. Il est courant de voir des utilisateurs utiliser le gestionnaire de fichiers de XFCE (Thunar) dans un autre environnement, ou remplacer le gestionnaire de fenêtres de XFCE (Xfwm4) par un gestionnaire en mosaïque comme i3.

## Architecture

```
┌───────────────────────────────────────────────────────┐
│                    xfce4-panel                        │
│  (panneaux, lanceurs, plugins, zone de notification)  │
├───────────────────────────────────────────────────────┤
│                    xfdesktop                          │
│  (fond d'écran, icônes du bureau, menu clic droit)    │
├───────────────────────────────────────────────────────┤
│                     Xfwm4                             │
│  (gestionnaire de fenêtres + compositeur)             │
├───────────────────────────────────────────────────────┤
│                  xfce4-session                        │
│  (gestion de la session, démarrage, sauvegarde)       │
├───────────────────────────────────────────────────────┤
│                   xfconf                              │
│  (système de configuration centralisé)                │
├───────────────────────────────────────────────────────┤
│                    GTK 3                              │
├───────────────────────────────────────────────────────┤
│                  Xorg (X11)                           │
├───────────────────────────────────────────────────────┤
│               Noyau Linux + DRM/KMS                   │
└───────────────────────────────────────────────────────┘
```

### Composants principaux

| Composant | Paquet | Rôle |
|-----------|--------|------|
| **Xfwm4** | `xfwm4` | Gestionnaire de fenêtres avec compositeur intégré. Gère le placement, le dimensionnement, le focus, les espaces de travail et un compositeur léger pour la transparence et les ombres portées |
| **xfce4-panel** | `xfce4-panel` | Système de panneaux modulaire. Chaque panneau est composé de plugins enfichables (lanceur, gestionnaire de tâches, horloge, zone de notification, etc.) |
| **xfdesktop** | `xfdesktop4` | Gestion du bureau : fond d'écran, icônes sur le bureau, menu clic droit avec lanceur d'applications et liste des fenêtres |
| **Thunar** | `thunar` | Gestionnaire de fichiers léger et rapide avec support de plugins |
| **xfce4-session** | `xfce4-session` | Gestionnaire de sessions (login, sauvegarde et restauration de session, démarrage automatique des applications) |
| **xfconf** | `xfconf` | Système de configuration centralisé, stocke les paramètres de tous les composants XFCE dans des fichiers XML |
| **xfce4-settings** | `xfce4-settings` | Collection de dialogues de configuration (écran, clavier, souris, apparence, gestionnaire de fenêtres, etc.) |
| **xfce4-power-manager** | `xfce4-power-manager` | Gestion de l'alimentation et de la luminosité |
| **xfce4-notifyd** | `xfce4-notifyd` | Démon de notifications |
| **xfce4-appfinder** | `xfce4-appfinder` | Lanceur d'applications avec recherche et historique |

### Système de configuration (xfconf)

XFCE stocke toute sa configuration dans des fichiers XML situés dans `~/.config/xfce4/xfconf/xfce-perchannel-xml/`. Chaque composant a son propre fichier, appelé « canal » (channel). L'outil en ligne de commande `xfconf-query` permet d'interroger et de modifier ces paramètres :

```bash
# Lister tous les canaux de configuration
xfconf-query -l

# Lister les propriétés d'un canal
xfconf-query -c xfce4-desktop -l

# Lire une propriété
xfconf-query -c xfce4-desktop -p /backdrop/screen0/monitorVGA-1/workspace0/last-image

# Modifier une propriété (exemple : changer le fond d'écran)
xfconf-query -c xfce4-desktop -p /backdrop/screen0/monitorVGA-1/workspace0/last-image \
  -s /usr/share/backgrounds/xfce/xfce-shapes.svg

# Créer une propriété qui n'existe pas encore
xfconf-query -c xfce4-desktop -p /desktop-icons/show-thumbnails -n -t bool -s true

# Surveiller les changements en temps réel (utile pour le débogage)
xfconf-query -c xfce4-panel -m
```

Les fichiers de configuration principaux :

```
~/.config/xfce4/xfconf/xfce-perchannel-xml/
├── thunar.xml                # Gestionnaire de fichiers
├── xfce4-desktop.xml         # Bureau (fond d'écran, icônes)
├── xfce4-keyboard-shortcuts.xml  # Raccourcis clavier
├── xfce4-panel.xml           # Configuration des panneaux
├── xfce4-power-manager.xml   # Gestion de l'alimentation
├── xfce4-session.xml         # Session
├── xfwm4.xml                 # Gestionnaire de fenêtres
└── xsettings.xml             # Paramètres d'apparence (thème, polices, DPI)
```

## L'interface XFCE

### Les panneaux

XFCE propose par défaut deux panneaux :

**Le panneau supérieur** contient le menu « Applications » (Whisker Menu ou menu classique), les lanceurs rapides, le gestionnaire de tâches (liste des fenêtres ouvertes) et la zone de notification avec l'horloge.

**Le panneau inférieur** (dock) affiche des lanceurs rapides pour les applications fréquemment utilisées et le bouton « Afficher le bureau ».

Les panneaux XFCE sont entièrement modulaires. Chaque élément est un **plugin de panneau** qui peut être ajouté, supprimé, déplacé ou configuré indépendamment. Parmi les plugins disponibles :

| Plugin | Fonction |
|--------|----------|
| **Whisker Menu** | Menu d'applications moderne avec recherche, favoris et catégories (installé par défaut dans la plupart des configurations) |
| **Application Menu** | Menu hiérarchique classique |
| **Gestionnaire de tâches** | Liste des fenêtres avec vignettes de prévisualisation |
| **Zone de notification** | Icônes des applications en arrière-plan |
| **Indicateur d'espaces de travail** | Miniatures ou boutons des espaces de travail |
| **Lanceurs** | Raccourcis vers les applications ou les commandes |
| **Horloge** | Affichage de l'heure (numérique, analogique, binaire, floue) |
| **PulseAudio** / **PipeWire** | Contrôle du volume audio |
| **Moniteur système** | Graphiques CPU, mémoire, réseau, swap |
| **Séparateur** / **Espacement** | Organisation visuelle du panneau |
| **Dictionnaire** | Recherche de définitions |
| **Météo** | Affichage des prévisions météorologiques |
| **Notes** | Post-it accessibles depuis le panneau |
| **Minuterie** | Compte à rebours et alarme |
| **Actions de session** | Boutons verrouiller, déconnecter, éteindre |

Un ou plusieurs panneaux supplémentaires peuvent être créés à volonté. Chaque panneau peut être positionné sur n'importe quel bord de l'écran, configuré en taille fixe ou en mode « intelligent » (masquage automatique), et peut adopter un comportement opaque ou semi-transparent.

### Le bureau

Le bureau XFCE (géré par `xfdesktop`) offre une expérience traditionnelle :

- **Icônes sur le bureau** : activées par défaut, elles affichent les fichiers et dossiers du répertoire `~/Bureau` ainsi que les icônes spéciales (Corbeille, Système de fichiers, Dossier personnel).
- **Menu clic droit** : un clic droit sur le bureau affiche un menu contextuel donnant accès au lanceur d'applications, aux fenêtres ouvertes et aux espaces de travail. Ce menu est configurable et peut être personnalisé.
- **Fond d'écran** : configurable par espace de travail, avec prise en charge des diaporamas et des dossiers d'images.

### Le gestionnaire de fenêtres (Xfwm4)

Xfwm4 est un gestionnaire de fenêtres classique qui offre un comportement prévisible et configurable :

- Décoration des fenêtres avec boutons minimiser, maximiser, fermer, ombrer (shade), coller (sticky) et menu. L'ordre et la présence des boutons sont personnalisables.
- Ancrage des fenêtres aux bords et aux coins de l'écran (tiling basique).
- Espaces de travail multiples (4 par défaut, configurable) avec possibilité d'envelopper la navigation (passer du dernier au premier espace).
- Compositeur intégré léger : ombres portées, transparence des fenêtres, transparence au déplacement. Le compositeur peut être désactivé pour les machines les plus anciennes.
- Thèmes de décoration de fenêtres indépendants du thème GTK.

```bash
# Configuration du compositeur Xfwm4
# Paramètres → Peaufinage des fenêtres → Compositeur

# Activer/désactiver le compositeur en ligne de commande
xfconf-query -c xfwm4 -p /general/use_compositing -s true  
xfconf-query -c xfwm4 -p /general/use_compositing -s false  
```

### Thunar : le gestionnaire de fichiers

Thunar est conçu pour être rapide et simple tout en restant fonctionnel. Ses caractéristiques principales :

- **Démarrage rapide** : Thunar se lance quasi instantanément, même sur du matériel ancien. C'est l'un des gestionnaires de fichiers les plus réactifs de l'écosystème Linux.
- **Navigation par onglets** : gestion de plusieurs emplacements dans une même fenêtre.
- **Panneau latéral** : raccourcis vers les emplacements fréquents, signets personnalisés et arborescence de répertoires.
- **Actions personnalisées** : système puissant permettant d'ajouter des entrées au menu contextuel selon le type de fichier. Par exemple, on peut créer une action « Ouvrir un terminal ici », « Redimensionner l'image » ou « Convertir en PDF » qui apparaît uniquement pour les types de fichiers pertinents.
- **Vignettes** : prévisualisation des images, vidéos et fichiers PDF via le plugin `tumbler`.
- **Plugins** : extensible via des plugins comme `thunar-archive-plugin` (intégration d'archives), `thunar-media-tags-plugin` (métadonnées audio) et `thunar-volman` (gestion automatique des volumes amovibles).

```bash
# Installer les plugins Thunar utiles
sudo apt install thunar-archive-plugin thunar-media-tags-plugin thunar-volman

# Les actions personnalisées se configurent dans
# Thunar → Édition → Configurer les actions personnalisées
```

## Personnalisation de XFCE

### Thèmes et apparence

La personnalisation de l'apparence se fait via **Paramètres → Apparence** et **Paramètres → Gestionnaire de fenêtres** :

- **Thème GTK** : contrôle l'apparence des applications (boutons, menus, barres de défilement). XFCE est livré avec plusieurs thèmes dont Greybird (utilisé par défaut dans de nombreuses configurations), Adwaita et les thèmes Xfce classiques.
- **Thème de fenêtres** : contrôle la barre de titre et les boutons de fenêtre (indépendant du thème GTK).
- **Thème d'icônes** : jeu d'icônes pour les applications et les fichiers.
- **Polices** : police de l'interface, rendu (anti-aliasing, hinting), facteur DPI.

```bash
# Installer des thèmes supplémentaires depuis les dépôts
sudo apt install arc-theme numix-gtk-theme papirus-icon-theme numix-icon-theme

# Appliquer un thème GTK en ligne de commande
xfconf-query -c xsettings -p /Net/ThemeName -s "Arc-Dark"

# Appliquer un thème de fenêtres
xfconf-query -c xfwm4 -p /general/theme -s "Arc-Dark"

# Appliquer un thème d'icônes
xfconf-query -c xsettings -p /Net/IconThemeName -s "Papirus-Dark"
```

### Raccourcis clavier

Les raccourcis clavier se configurent dans **Paramètres → Clavier → Raccourcis d'applications** et **Paramètres → Gestionnaire de fenêtres → Clavier**. Les raccourcis par défaut les plus utiles :

| Raccourci | Action |
|-----------|--------|
| `Alt + F2` | Lanceur de commandes (appfinder) |
| `Super + L` | Verrouiller l'écran (si configuré) |
| `Alt + Tab` | Basculer entre les fenêtres |
| `Alt + F4` | Fermer la fenêtre |
| `Alt + F9` | Minimiser la fenêtre |
| `Alt + F10` | Maximiser / restaurer la fenêtre |
| `Ctrl + Alt + D` | Afficher le bureau |
| `Ctrl + Alt + ←` / `→` | Naviguer entre les espaces de travail |
| `Ctrl + Alt + Suppr` | Verrouiller la session |
| `Impr. écran` | Capture d'écran |

## XFCE et Wayland

XFCE 4.20, inclus dans Debian 13 Trixie, introduit un **support expérimental de Wayland** via la bibliothèque `libxfce4windowing`. Plusieurs composants ont été portés (Thunar, xfce4-panel, xfdesktop), mais le portage n'est pas encore complet : Xfwm4 ne fonctionne pas sous Wayland, et l'utilisation d'un compositeur Wayland externe (comme Labwc ou Wayfire) est nécessaire. Le support Wayland de XFCE est donc considéré comme un **travail en cours**, et Xorg reste le mode de fonctionnement recommandé et par défaut pour XFCE dans Trixie.

Pour la très grande majorité des cas d'usage, Xorg reste parfaitement fonctionnel et fiable avec XFCE. Si un support Wayland complet et mature est essentiel, GNOME ou KDE Plasma sont les alternatives recommandées.

# LXDE

## Présentation

LXDE (Lightweight X11 Desktop Environment) est un environnement de bureau ultra-léger créé en 2006 par Hong Jen Yee (PCMan) à Taïwan. Son objectif initial était de fournir un bureau fonctionnel pour les machines très anciennes ou à très faibles ressources, comme les premiers netbooks et les ordinateurs recyclés pour les pays en développement.

LXDE est construit sur **GTK 2** (une version obsolète du toolkit), ce qui explique à la fois sa légèreté exceptionnelle et ses limitations en termes de modernité visuelle. Au repos, une session LXDE consomme typiquement entre 150 et 300 Mo de RAM, ce qui en fait l'un des environnements de bureau les plus économes en ressources.

**Remarque importante sur l'état du projet** : le développement de LXDE est aujourd'hui au point mort. Le créateur du projet a redirigé ses efforts vers **LXQt**, un successeur qui utilise le toolkit Qt au lieu de GTK 2. LXDE reste disponible dans les dépôts Debian et fonctionne de manière stable, mais il ne reçoit plus de nouvelles fonctionnalités. Pour les nouvelles installations, LXQt est le choix recommandé dans cette famille d'environnements.

## Architecture

LXDE est un assemblage de composants indépendants, plus faiblement couplés que dans les autres environnements :

| Composant | Rôle |
|-----------|------|
| **Openbox** | Gestionnaire de fenêtres (projet externe, utilisé par défaut) |
| **LXPanel** | Barre des tâches avec menu, gestionnaire de tâches et zone de notification |
| **PCManFM** | Gestionnaire de fichiers (gère également le bureau et le fond d'écran) |
| **LXSession** | Gestionnaire de sessions |
| **LXAppearance** | Configuration de l'apparence (thèmes GTK, icônes, polices) |
| **LXTerminal** | Émulateur de terminal |
| **LXTask** | Gestionnaire de tâches (processus) |
| **GPicView** | Visionneuse d'images |
| **L3afpad** | Éditeur de texte minimaliste — fork actif de l'historique Leafpad. Le paquet `leafpad` n'existe plus dans Trixie ; l'installation se fait via `apt install l3afpad`. |

### Openbox : le gestionnaire de fenêtres

Contrairement à GNOME (Mutter), KDE (KWin) ou XFCE (Xfwm4) qui utilisent un gestionnaire de fenêtres développé par le même projet, LXDE s'appuie sur **Openbox**, un gestionnaire de fenêtres indépendant réputé pour sa légèreté et sa configurabilité. Openbox peut d'ailleurs être utilisé seul, sans LXDE, comme base pour construire un environnement personnalisé.

La configuration d'Openbox se fait via des fichiers XML :

```bash
# Fichier de configuration principal d'Openbox dans le contexte LXDE
~/.config/openbox/lxde-rc.xml

# Ce fichier contrôle :
# - Le comportement des fenêtres (focus, placement, résistance des bords)
# - Les raccourcis clavier
# - Les actions liées à la souris
# - Les thèmes de décoration
# - Les menus (clic droit sur le bureau)
# - Les règles par application (taille, position, espace de travail par défaut)

# L'outil graphique de configuration
sudo apt install obconf  
obconf   # Lance l'interface de configuration d'Openbox  
```

### PCManFM : le gestionnaire de fichiers et de bureau

PCManFM (PCMan File Manager) est un gestionnaire de fichiers léger qui assure également la gestion du bureau (fond d'écran et icônes). Cette double responsabilité est une particularité de LXDE. Ses caractéristiques :

- Démarrage quasi instantané
- Navigation par onglets
- Panneau latéral avec signets
- Support des volumes amovibles
- Gestion du bureau (fond d'écran, icônes)
- Accès réseau basique via GVFS

PCManFM est fonctionnel mais limité par rapport à Thunar ou Dolphin : pas de vue partagée, pas de terminal intégré, système de plugins rudimentaire.

## Interface LXDE

L'interface par défaut de LXDE est minimaliste et immédiatement familière :

- Un **panneau inférieur** avec un menu d'applications à gauche, le gestionnaire de tâches au centre et la zone de notification (horloge, volume, réseau) à droite.
- Un **bureau** avec icônes (Corbeille, dossier personnel) et un fond d'écran configurable.
- Un **menu clic droit** sur le bureau donnant accès aux applications et aux paramètres.

L'ensemble est sobre, fonctionnel et extrêmement réactif, même sur du matériel vieux de 15 ans.

## Personnalisation de LXDE

```bash
# Apparence GTK et icônes
lxappearance

# Configuration d'Openbox (gestionnaire de fenêtres)
obconf

# Configuration du panneau
# Clic droit sur le panneau → Paramètres du tableau de bord

# Fond d'écran et bureau
pcmanfm --desktop-pref

# Configuration des applications de démarrage
# Éditer ~/.config/lxsession/LXDE/autostart
```

## LXDE et Wayland

Comme XFCE, LXDE est exclusivement basé sur X11 et n'a aucune perspective de portage vers Wayland. Le projet étant effectivement en fin de vie, cette migration ne sera jamais réalisée pour LXDE lui-même. Les utilisateurs souhaitant un environnement ultra-léger avec un support futur de Wayland doivent se tourner vers LXQt.

# LXQt : le successeur de LXDE

## Présentation

**LXQt** est né en 2013 de la fusion entre LXDE-Qt (une réécriture de LXDE en Qt initiée par le créateur de LXDE) et le projet Razor-Qt. Il représente la continuité naturelle de LXDE avec une base technologique modernisée.

LXQt utilise le toolkit **Qt** (le même que KDE Plasma) au lieu de GTK 2. Ce choix apporte plusieurs avantages : un meilleur rendu sur les écrans HiDPI, un support futur de Wayland plus réaliste, et un toolkit activement maintenu et en pleine évolution. Malgré l'utilisation de Qt, LXQt ne dépend pas des KDE Frameworks et reste un projet indépendant avec sa propre architecture légère.

Debian propose LXQt dans ses dépôts officiels comme une alternative distincte de LXDE.

## Architecture

```
┌───────────────────────────────────────────────────────┐
│                    lxqt-panel                         │
│  (panneaux, plugins, lanceurs, zone de notification)  │
├───────────────────────────────────────────────────────┤
│                   lxqt-desktop                        │
│  (fond d'écran, icônes)                               │
├───────────────────────────────────────────────────────┤
│              Openbox ou KWin                          │
│  (gestionnaire de fenêtres, au choix)                 │
├───────────────────────────────────────────────────────┤
│                  lxqt-session                         │
│  (gestion de la session)                              │
├───────────────────────────────────────────────────────┤
│                     Qt 6                              │
├───────────────────────────────────────────────────────┤
│                  Xorg (X11)                           │
│  (Wayland prévu pour les versions futures)            │
├───────────────────────────────────────────────────────┤
│               Noyau Linux + DRM/KMS                   │
└───────────────────────────────────────────────────────┘
```

### Composants principaux

| Composant | Paquet | Rôle |
|-----------|--------|------|
| **Openbox** (défaut) ou **KWin** | `openbox` / `kwin-x11` | Gestionnaire de fenêtres. LXQt utilise Openbox par défaut mais peut utiliser KWin pour bénéficier des effets de bureau et du support Wayland futur |
| **lxqt-panel** | `lxqt-panel` | Panneau modulaire avec plugins |
| **PCManFM-Qt** | `pcmanfm-qt` | Gestionnaire de fichiers (réécriture Qt de PCManFM) et gestion du bureau |
| **lxqt-session** | `lxqt-session` | Gestionnaire de sessions |
| **lxqt-config** | `lxqt-config` | Centre de configuration |
| **lxqt-globalkeys** | `lxqt-globalkeys` | Gestion des raccourcis clavier globaux |
| **lxqt-notificationd** | `lxqt-notificationd` | Démon de notifications |
| **lxqt-powermanagement** | `lxqt-powermanagement` | Gestion de l'alimentation |
| **lxqt-runner** | `lxqt-runner` | Lanceur d'applications rapide (équivalent de Krunner en plus simple) |
| **QTerminal** | `qterminal` | Émulateur de terminal |
| **FeatherPad** | `featherpad` | Éditeur de texte avec coloration syntaxique, onglets et numérotation des lignes |
| **LXImage-Qt** | `lximage-qt` | Visionneuse d'images et outil de capture d'écran |

### Avantages de LXQt par rapport à LXDE

- **Toolkit moderne** : Qt 6 est activement développé, contrairement à GTK 2 qui ne reçoit plus de mises à jour.
- **Meilleur rendu HiDPI** : la mise à l'échelle est nettement meilleure qu'avec GTK 2.
- **Support Wayland en cours d'intégration** : LXQt 2.1.0 (version livrée dans Trixie) introduit *en amont* un paquet `lxqt-wayland-session` permettant de lancer une session LXQt Wayland multi-compositeur (Labwc, KWin, Wayfire, Sway, River, Niri, Hyprland). **Ce paquet n'est cependant pas encore disponible dans les dépôts Debian Trixie** au moment de la sortie de cette stable (`apt-cache search lxqt-wayland-session` retourne vide ; il faudra l'attendre via `trixie-backports` ou la prochaine stable). Sur Trixie, LXQt fonctionne donc en pratique sous Xorg avec Openbox.
- **Développement actif** : contrairement à LXDE, LXQt reçoit des mises à jour régulières avec de nouvelles fonctionnalités.
- **Consommation de ressources comparable** : malgré l'utilisation de Qt, LXQt reste très léger (250 à 450 Mo de RAM au repos).
- **PCManFM-Qt** : le gestionnaire de fichiers a été significativement amélioré par rapport à la version GTK.

### Configuration de LXQt

LXQt utilise des fichiers de configuration au format INI dans `~/.config/lxqt/` :

```bash
# Centre de configuration graphique
lxqt-config

# Fichiers de configuration principaux
~/.config/lxqt/
├── lxqt.conf              # Configuration globale (thème, polices)
├── panel.conf             # Configuration du panneau
├── session.conf           # Session (gestionnaire de fenêtres, modules)
├── globalkeyshortcuts.conf # Raccourcis clavier
├── powermanagement.conf   # Alimentation
└── notifications.conf     # Notifications

# Choix du gestionnaire de fenêtres
# Éditer ~/.config/lxqt/session.conf → window_manager=openbox
# ou window_manager=kwin_x11
```

| **Notifications** | Système intégré (xfce4-notifyd) | Basique | Système intégré |

## Critères de choix

**Choisir XFCE si** :
- La machine dispose d'au moins 1 Go de RAM et d'un processeur raisonnable.
- Un environnement léger mais complet est souhaité, sans compromettre les fonctionnalités.
- La stabilité et la prévisibilité à long terme sont prioritaires.
- L'absence de Wayland n'est pas un problème à court terme.
- Le profil d'utilisation est un poste de travail, une station de développement ou un poste d'entreprise standardisé.

**Choisir LXQt si** :
- La machine est très limitée en ressources (512 Mo – 1 Go de RAM).
- Un environnement ultra-léger mais activement maintenu est nécessaire.
- La perspective d'un support Wayland futur est souhaitée.
- L'environnement doit fonctionner sur un Raspberry Pi ou un appareil embarqué avec une interface graphique.

**Choisir LXDE si** :
- La machine est extrêmement ancienne (moins de 512 Mo de RAM).
- L'objectif est d'obtenir le bureau le plus léger possible avec un assemblage de composants existants.
- L'absence de mises à jour et de nouvelles fonctionnalités n'est pas un problème (environnement figé et stable).

## Installation sur Debian

```bash
# Installation complète de XFCE
sudo tasksel install xfce-desktop
# ou
sudo apt install task-xfce-desktop

# Installation minimale de XFCE (environnement seul)
sudo apt install xfce4

# Installation avec extras (plugins de panneau, utilitaires)
sudo apt install xfce4 xfce4-goodies

# Installation complète de LXDE
sudo apt install task-lxde-desktop

# Installation minimale de LXDE
sudo apt install lxde-core

# Installation complète de LXQt
sudo apt install task-lxqt-desktop

# Installation minimale de LXQt
sudo apt install lxqt
```

Le méta-paquet `xfce4-goodies` est particulièrement recommandé pour XFCE : il installe un ensemble de plugins de panneau supplémentaires, le Whisker Menu, des utilitaires (xfce4-screenshooter, xfce4-taskmanager, Catfish) et des thèmes supplémentaires qui enrichissent significativement l'expérience de base.

## Résumé

XFCE, LXDE et LXQt occupent le créneau des environnements de bureau légers dans Debian, chacun avec un positionnement distinct. XFCE est le plus complet et le plus abouti des trois, offrant un équilibre remarquable entre légèreté et fonctionnalité. LXQt reprend le flambeau de LXDE avec une base technologique moderne et un développement actif. LXDE reste une option viable pour le matériel le plus contraint, mais son absence de développement le destine principalement aux installations existantes plutôt qu'aux nouveaux déploiements.

Pour la majorité des utilisateurs cherchant une alternative légère à GNOME ou KDE Plasma, XFCE est la recommandation par défaut. Pour les cas extrêmes de contrainte matérielle ou les usages embarqués nécessitant un bureau graphique, LXQt prend le relais avec une perspective d'avenir plus solide que LXDE.
