# 2.4.1 Thèmes et apparence

## Introduction

L'apparence visuelle d'un environnement de bureau influence directement le confort, la lisibilité et le plaisir d'utilisation au quotidien. Sous Linux, la personnalisation visuelle va bien au-delà du simple changement de fond d'écran : chaque couche de l'interface — les contrôles des applications, les décorations de fenêtres, les icônes, les polices, les curseurs, les couleurs — peut être thémée indépendamment.

Cette section présente les différentes couches de thèmes, les méthodes d'installation et de configuration, les thèmes populaires dans l'écosystème Debian, et les bonnes pratiques pour obtenir une interface cohérente, y compris lorsque des applications GTK et Qt cohabitent.

---

## Les couches de thèmes

L'apparence d'un bureau Linux est composée de plusieurs couches distinctes, chacune gérée par un mécanisme de thème spécifique. Comprendre cette architecture est essentiel pour personnaliser l'interface de manière cohérente.

### Thème d'applications (widget theme)

Le thème d'applications contrôle l'apparence de tous les éléments d'interface à l'intérieur des fenêtres : boutons, menus, barres de défilement, champs de saisie, onglets, listes déroulantes, cases à cocher, barres de progression, etc. Ce thème dépend du toolkit graphique utilisé par l'application :

- **GTK 3/4** : utilisé par GNOME, XFCE, MATE, Cinnamon et les applications de ces écosystèmes (Nautilus, Firefox, GIMP, LibreOffice avec intégration GTK).
- **Qt 5/6** : utilisé par KDE Plasma et les applications KDE (Dolphin, Kate, Kdenlive, VLC avec interface Qt).

Une application GTK et une application Qt sur le même bureau utilisent des thèmes différents. Sans configuration spécifique, elles ont une apparence incohérente. La section « Cohérence GTK/Qt » plus bas dans ce chapitre traite ce problème.

### Thème de fenêtres (window decoration)

Le thème de fenêtres contrôle l'apparence de la barre de titre et des boutons de fenêtre (fermer, maximiser, minimiser). Sous GNOME avec libadwaita, la barre de titre est intégrée à l'application (header bar) et suit le thème GTK. Sous KDE et XFCE, la décoration de fenêtres est un thème séparé géré par le gestionnaire de fenêtres (KWin, Xfwm4).

### Thème d'icônes

Le thème d'icônes détermine l'apparence de toutes les icônes du système : icônes d'applications dans les menus et sur le bureau, icônes de dossiers et de fichiers dans le gestionnaire de fichiers, icônes d'actions dans les barres d'outils (copier, coller, enregistrer), icônes de la zone de notification (réseau, batterie, volume). Un thème d'icônes cohérent unifie visuellement l'ensemble du bureau.

### Thème de curseur

Le thème de curseur définit l'apparence du pointeur de souris dans ses différents états : flèche standard, main (lien cliquable), curseur de texte, chargement, redimensionnement, etc.

### Schéma de couleurs

Le schéma de couleurs définit la palette utilisée par l'interface : couleur de fond, couleur de texte, couleur d'accentuation, couleur des sélections, couleur des éléments actifs et inactifs. Sous GNOME, le choix se résume essentiellement à clair ou sombre (avec une couleur d'accentuation depuis GNOME 47). Sous KDE, chaque composant de couleur peut être ajusté individuellement.

### Fond d'écran

Le fond d'écran est l'élément le plus personnel de l'interface. Il peut être une image statique, un diaporama rotatif ou, selon l'environnement, un fond dynamique qui change au fil de la journée.

## Personnalisation sous GNOME

### Le paradigme libadwaita

GNOME a fait un choix architectural fort avec l'introduction de **libadwaita** (GNOME 42+) : les applications GNOME natives utilisent leur propre bibliothèque de widgets qui impose un style visuel spécifique. Ce style n'est pas modifiable par les thèmes GTK traditionnels, ce qui garantit une cohérence visuelle mais limite la personnalisation des applications récentes.

En pratique, cela signifie que les thèmes GTK tiers ne s'appliquent qu'aux applications GTK 3 qui n'utilisent pas libadwaita. Les applications libadwaita (Nautilus, GNOME Text Editor, GNOME Console, etc.) suivent uniquement le choix clair/sombre et la couleur d'accentuation.

### Mode clair et sombre

```bash
# Activer le thème sombre
gsettings set org.gnome.desktop.interface color-scheme 'prefer-dark'

# Revenir au thème clair
gsettings set org.gnome.desktop.interface color-scheme 'default'

# Via les Paramètres : Apparence → Style
```

Les applications libadwaita réagissent automatiquement à ce changement. Les applications GTK 3 plus anciennes nécessitent parfois un thème GTK explicitement sombre.

### Couleur d'accentuation (GNOME 47+)

À partir de GNOME 47, une couleur d'accentuation personnalisable est disponible. Elle affecte les éléments interactifs (boutons, liens, sélections, indicateurs) :

```bash
# Paramètres → Apparence → Couleur
# Choix parmi plusieurs couleurs prédéfinies

# Ou en ligne de commande
gsettings set org.gnome.desktop.interface accent-color 'blue'
# Valeurs : blue, teal, green, yellow, orange, red, pink, purple, slate
```

### Thèmes GTK (applications non-libadwaita)

Pour les applications GTK 3 qui ne sont pas passées à libadwaita, un thème GTK peut être appliqué :

```bash
# Installer des thèmes depuis les dépôts
sudo apt install arc-theme          # Arc (clair, sombre, mixte)  
sudo apt install numix-gtk-theme    # Numix  
# Note : `materia-gtk-theme` a été retiré de Debian Trixie. Pour un look
# « Material Design », on peut installer manuellement le thème depuis le
# dépôt upstream (https://github.com/nana-4/materia-theme) ou utiliser
# une alternative empaquetée comme `arc-theme` ou `breeze-gtk-theme`.
sudo apt install greybird-gtk-theme # Greybird (thème par défaut de Xubuntu)

# Appliquer un thème via gsettings
gsettings set org.gnome.desktop.interface gtk-theme 'Arc-Dark'

# Vérifier le thème actuel
gsettings get org.gnome.desktop.interface gtk-theme

# Via GNOME Tweaks (recommandé pour une vue d'ensemble)
sudo apt install gnome-tweaks  
gnome-tweaks &  
# Apparence → Thème des applications Legacy
```

### Thèmes d'icônes

```bash
# Thèmes d'icônes populaires dans les dépôts Debian
sudo apt install papirus-icon-theme       # Papirus (le plus populaire, riche et complet)  
sudo apt install numix-icon-theme         # Numix  
sudo apt install numix-icon-theme-circle  # Numix Circle (icônes rondes)  
sudo apt install breeze-icon-theme        # Breeze (icônes KDE, cohérent avec Qt)  
sudo apt install adwaita-icon-theme       # Adwaita (GNOME, installé par défaut)  
sudo apt install tango-icon-theme         # Tango (classique)  

# Appliquer
gsettings set org.gnome.desktop.interface icon-theme 'Papirus-Dark'

# Via GNOME Tweaks → Apparence → Icônes
```

### Thème de curseur

```bash
# Thèmes de curseur disponibles
sudo apt install breeze-cursor-theme         # Breeze (style KDE)  
sudo apt install dmz-cursor-theme            # DMZ (blanc ou noir, sobre)  
sudo apt install xcursor-themes              # Collection de curseurs classiques  

# Appliquer
gsettings set org.gnome.desktop.interface cursor-theme 'Breeze_Snow'  
gsettings set org.gnome.desktop.interface cursor-size 24  
# Tailles courantes : 24 (défaut), 32, 48 (grand)

# Via GNOME Tweaks → Apparence → Curseur
```

### Polices

```bash
# Configurer les polices de l'interface
gsettings set org.gnome.desktop.interface font-name 'Cantarell 11'  
gsettings set org.gnome.desktop.interface document-font-name 'Cantarell 11'  
gsettings set org.gnome.desktop.interface monospace-font-name 'JetBrains Mono 10'  

# Facteur d'échelle des polices (pour agrandir le texte sans changer l'échelle globale)
gsettings set org.gnome.desktop.interface text-scaling-factor 1.0
# 1.0 = normal, 1.25 = 25% plus grand, etc.

# Paramètres de rendu des polices
gsettings set org.gnome.desktop.interface font-antialiasing 'rgba'
# Valeurs : none, grayscale, rgba

gsettings set org.gnome.desktop.interface font-hinting 'slight'
# Valeurs : none, slight, medium, full

# Via GNOME Tweaks → Polices
```

### Fond d'écran

```bash
# Changer le fond d'écran
gsettings set org.gnome.desktop.background picture-uri \
  'file:///usr/share/backgrounds/gnome/blobs-l.svg'
gsettings set org.gnome.desktop.background picture-uri-dark \
  'file:///usr/share/backgrounds/gnome/blobs-d.svg'

# Mode d'affichage (zoom, centered, stretched, wallpaper, spanned)
gsettings set org.gnome.desktop.background picture-options 'zoom'

# Couleur de fond (si pas d'image ou image transparente)
gsettings set org.gnome.desktop.background primary-color '#1a1a2e'

# Fond d'écran de l'écran de verrouillage
gsettings set org.gnome.desktop.screensaver picture-uri \
  'file:///usr/share/backgrounds/gnome/blobs-l.svg'

# Installer des fonds d'écran supplémentaires
sudo apt install gnome-backgrounds
# Les fonds d'écran Debian sont dans
ls /usr/share/backgrounds/
```

### Extensions GNOME pour la personnalisation

Certaines extensions GNOME Shell sont particulièrement utiles pour la personnalisation visuelle :

```bash
# User Themes — permet d'appliquer des thèmes au shell GNOME lui-même
# (barre supérieure, vue d'ensemble, notifications)
sudo apt install gnome-shell-extension-user-theme
# Nécessaire pour thèmer la barre supérieure et la vue Activités

# Une fois activée, GNOME Tweaks → Apparence → Shell
# permet de sélectionner un thème Shell

# Blur my Shell — ajoute un effet de flou esthétique
# Installable via extensions.gnome.org ou Extension Manager
sudo apt install gnome-shell-extension-manager
```

## Personnalisation sous KDE Plasma

KDE Plasma est l'environnement le plus personnalisable visuellement. Chaque couche de l'apparence est indépendante et peut être modifiée séparément via **Paramètres système → Apparence**.

### Thèmes globaux (Global Themes)

Un thème global applique un ensemble cohérent de thèmes sur toutes les couches en un seul clic : thème Plasma, couleurs, décoration de fenêtres, icônes, curseur et parfois le fond d'écran.

```bash
# Paramètres système → Apparence → Thème global

# Installer des thèmes globaux depuis le KDE Store
# Cliquer sur « Télécharger de nouveaux thèmes globaux » en bas du panneau

# Les thèmes sont installés dans
~/.local/share/plasma/look-and-feel/
```

### Style d'application (Qt)

Le style d'application contrôle le rendu des widgets Qt. Breeze est le style par défaut :

```bash
# Paramètres système → Apparence → Style de l'application
# Styles disponibles : Breeze, Oxygen, Fusion, Windows (nostalgie)

# Sous-section « Style des applications GNOME/GTK »
# Permet de configurer le thème GTK pour les applications GNOME
# Breeze GTK est recommandé pour la cohérence
sudo apt install breeze-gtk-theme
```

### Thème Plasma

Le thème Plasma contrôle l'apparence des éléments spécifiques à Plasma : les panneaux, les widgets, les menus contextuels, la vue d'ensemble. Il est basé sur des fichiers SVG, ce qui permet un rendu net à toutes les résolutions.

```bash
# Paramètres système → Apparence → Thème de Plasma

# Thèmes populaires installables depuis le KDE Store via le même panneau :
# Layan, Sweet, Nordian, WhiteSur, Orchis, etc.

# Les thèmes sont stockés dans
~/.local/share/plasma/desktoptheme/
```

### Schéma de couleurs

C'est l'une des fonctionnalités les plus puissantes de KDE. Le schéma de couleurs définit chaque composant individuel de la palette : fond des fenêtres, fond des vues, texte actif, texte inactif, surbrillance, lien, boutons, barre de titre, etc.

```bash
# Paramètres système → Apparence → Couleurs

# Schémas fournis : Breeze Light, Breeze Dark, Breeze Classic
# Des dizaines de schémas téléchargeables depuis le KDE Store

# Créer un schéma personnalisé :
# Sélectionner un schéma de base, cliquer sur « Modifier les couleurs »
# Ajuster chaque groupe de couleurs individuellement

# Les schémas sont stockés dans
~/.local/share/color-schemes/

# Appliquer en ligne de commande
kwriteconfig6 --file kdeglobals --group General --key ColorScheme "BreezeDark"
```

### Décorations de fenêtres

```bash
# Paramètres système → Apparence → Décorations de fenêtres

# Breeze est la décoration par défaut
# Personnalisation disponible :
# - Taille des boutons
# - Présence et ordre des boutons (fermer, maximiser, minimiser, coller, ombrer)
# - Épaisseur de la bordure
# - Ombrage
# - Opacité de la barre de titre

# Format Aurorae : thèmes basés sur des images SVG/PNG
# Installables depuis le KDE Store
# Stockés dans ~/.local/share/aurorae/themes/

# Boutons de fenêtres : l'ordre est configurable
# Style macOS (fermer à gauche) :
# Paramètres → Décorations de fenêtres → Boutons de barre de titre
# Glisser-déposer les boutons entre gauche et droite
```

### Icônes, curseur et polices

```bash
# Icônes
# Paramètres système → Apparence → Icônes
# Breeze, Breeze Dark, Oxygen, et téléchargements depuis le KDE Store

# Curseur
# Paramètres système → Apparence → Curseurs
# Taille configurable

# Polices
# Paramètres système → Apparence → Polices
# Configuration séparée pour : général, taille fixe (terminal),
# petite, barre d'outils, menu, titre de fenêtre
# Option « Forcer le DPI des polices » pour un rendu cohérent

# Anti-aliasing et hinting configurables par police
```

### Effets visuels et animations

```bash
# Paramètres système → Comportement de l'espace de travail → Effets de bureau
```

## Personnalisation sous XFCE

XFCE offre une personnalisation solide avec une approche plus simple que KDE, via **Paramètres → Apparence** et **Paramètres → Gestionnaire de fenêtres**.

### Thème GTK

```bash
# Paramètres → Apparence → Style
# ou en ligne de commande
xfconf-query -c xsettings -p /Net/ThemeName -s "Arc-Dark"

# Thèmes recommandés pour XFCE
# Note : `materia-gtk-theme` n'est plus dans Debian Trixie (cf. la note de
# la section GNOME plus haut). Si vous voulez Materia, installez-le depuis
# https://github.com/nana-4/materia-theme dans `~/.local/share/themes/`.
sudo apt install arc-theme greybird-gtk-theme numix-gtk-theme
```

### Thème de fenêtres (Xfwm4)

```bash
# Paramètres → Gestionnaire de fenêtres → Style
# Thèmes indépendants du thème GTK

xfconf-query -c xfwm4 -p /general/theme -s "Arc-Dark"

# Configurer les boutons de fenêtre (ordre et présence)
# Paramètres → Gestionnaire de fenêtres → Style → Disposition des boutons
```

### Icônes, polices et curseur

```bash
# Icônes
xfconf-query -c xsettings -p /Net/IconThemeName -s "Papirus-Dark"

# Polices
xfconf-query -c xsettings -p /Gtk/FontName -s "Noto Sans 10"

# DPI (pour écrans HiDPI)
xfconf-query -c xsettings -p /Xft/DPI -s 144

# Anti-aliasing
xfconf-query -c xsettings -p /Xft/Antialias -s 1  
xfconf-query -c xsettings -p /Xft/HintStyle -s "hintslight"  
xfconf-query -c xsettings -p /Xft/RGBA -s "rgb"  

# Curseur
xfconf-query -c xsettings -p /Gtk/CursorThemeName -s "Breeze_Snow"  
xfconf-query -c xsettings -p /Gtk/CursorThemeSize -s 24  
```

### Panneau XFCE

L'apparence des panneaux XFCE est contrôlée séparément et peut être personnalisée :

```bash
# Clic droit sur le panneau → Paramètres du tableau de bord

# Options visuelles :
# - Mode (horizontal, vertical, bureau)
# - Taille (hauteur en pixels)
# - Longueur (en pourcentage de l'écran)
# - Transparence et opacité
# - Style (couleur de fond, image de fond)
# - Masquage automatique

# Ajout de plugins visuels
# Séparateur avec style (transparent, pointillés, ligne)
# Espacement pour centrer les éléments
```

## Installation de thèmes tiers

### Depuis les dépôts Debian

```bash
# Rechercher des thèmes dans les dépôts
apt search gtk-theme  
apt search icon-theme  
apt search cursor-theme  
apt search plasma-theme  

# Thèmes GTK populaires disponibles dans les dépôts
sudo apt install arc-theme                # Arc (clair, sombre, darker)
# materia-gtk-theme n'est plus packagé dans Trixie — installer manuellement
# depuis https://github.com/nana-4/materia-theme si besoin du look Material.
sudo apt install numix-gtk-theme          # Numix  
sudo apt install greybird-gtk-theme       # Greybird  

# Thèmes d'icônes populaires
sudo apt install papirus-icon-theme       # Papirus (5000+ icônes)  
sudo apt install numix-icon-theme-circle  # Numix Circle  
sudo apt install breeze-icon-theme        # Breeze  

# Thèmes de curseur
sudo apt install breeze-cursor-theme  
sudo apt install dmz-cursor-theme  
```

### Installation manuelle

Les thèmes qui ne sont pas dans les dépôts peuvent être installés manuellement depuis des sources comme [gnome-look.org](https://www.gnome-look.org), [store.kde.org](https://store.kde.org) ou des dépôts GitHub :

```bash
# Thèmes GTK — installer pour l'utilisateur courant
mkdir -p ~/.local/share/themes/
# Extraire l'archive du thème dans ce répertoire
tar xf theme-telecharge.tar.gz -C ~/.local/share/themes/
# Le thème apparaît dans les outils de sélection (GNOME Tweaks, Paramètres XFCE)

# Thèmes GTK — installer pour tous les utilisateurs
sudo tar xf theme-telecharge.tar.gz -C /usr/share/themes/

# Thèmes d'icônes
mkdir -p ~/.local/share/icons/  
tar xf icon-theme.tar.gz -C ~/.local/share/icons/  
# Rafraîchir le cache des icônes
gtk-update-icon-cache ~/.local/share/icons/NomDuTheme/

# Thèmes de curseur
mkdir -p ~/.local/share/icons/  
tar xf cursor-theme.tar.gz -C ~/.local/share/icons/  
# Les thèmes de curseur utilisent le même répertoire que les icônes

# Thèmes Plasma (KDE)
# L'installation via Paramètres système → « Télécharger » est recommandée
# Installation manuelle :
mkdir -p ~/.local/share/plasma/desktoptheme/
# Copier le thème dans ce répertoire
```

### Structure des répertoires de thèmes

```bash
# Résumé des emplacements d'installation

# Thèmes GTK (applications)
/usr/share/themes/             # Système (tous les utilisateurs)
~/.local/share/themes/         # Utilisateur
~/.themes/                     # Ancien emplacement (encore supporté)

# Icônes et curseurs
/usr/share/icons/              # Système
~/.local/share/icons/          # Utilisateur
~/.icons/                      # Ancien emplacement (encore supporté)

# Thèmes Plasma
/usr/share/plasma/desktoptheme/       # Système
~/.local/share/plasma/desktoptheme/   # Utilisateur

# Thèmes globaux KDE (look and feel)
~/.local/share/plasma/look-and-feel/

# Décorations de fenêtres KDE (Aurorae)
~/.local/share/aurorae/themes/

# Schémas de couleurs KDE
~/.local/share/color-schemes/

# Polices
/usr/share/fonts/              # Système
/usr/local/share/fonts/        # Système (ajouts locaux)
~/.local/share/fonts/          # Utilisateur
~/.fonts/                      # Ancien emplacement (encore supporté)
```

## Cohérence visuelle GTK/Qt

Sur un bureau qui mélange des applications GTK (Firefox, GIMP, LibreOffice) et des applications Qt (VLC, Kdenlive, KeePassXC), l'apparence peut être visuellement incohérente si rien n'est fait pour harmoniser les deux toolkits.

### Sous KDE Plasma (harmoniser GTK)

KDE Plasma configure automatiquement les applications GTK pour utiliser le thème Breeze GTK, ce qui assure une cohérence visuelle raisonnable. Si l'harmonisation n'est pas satisfaisante :

```bash
# Installer le thème GTK Breeze
sudo apt install breeze-gtk-theme

# Configurer dans Paramètres système → Apparence → Style de l'application
# → Style des applications GNOME/GTK
# Sélectionner "Breeze" comme thème GTK

# Le schéma de couleurs GTK suit automatiquement les couleurs de Plasma
```

### Sous GNOME (harmoniser Qt)

Les applications Qt dans un environnement GNOME nécessitent une configuration explicite :

```bash
# Méthode 1 : Adwaita-qt (thème Qt qui imite Adwaita/libadwaita)
sudo apt install adwaita-qt6 adwaita-qt

# Configurer via la variable d'environnement
echo 'export QT_STYLE_OVERRIDE=Adwaita-Dark' >> ~/.profile
# ou Adwaita pour le mode clair

# Méthode 2 : qt5ct/qt6ct pour un contrôle fin
sudo apt install qt5ct qt6ct  
echo 'export QT_QPA_PLATFORMTHEME=qt6ct' >> ~/.profile  
# Se déconnecter/reconnecter, puis lancer qt5ct et qt6ct pour configurer
# le thème, les polices et les icônes des applications Qt 5 et Qt 6.
# Avec qt6ct comme valeur, les applis Qt 5 utilisent automatiquement
# qt5ct si celui-ci est installé (mécanisme de fallback intégré).

# Méthode 3 : kvantum (moteur de thèmes Qt avancé)
# Sous Trixie, le paquet `qt6-style-kvantum` est transitionnel ;
# le paquet réel se nomme `qt-style-kvantum`.
sudo apt install qt-style-kvantum
# Pour le support Qt 5 (à installer en plus si besoin)
sudo apt install qt5-style-kvantum
# Kvantum permet d'appliquer des thèmes SVG aux applications Qt
# Installer un thème Kvantum (ex: Kvantum-Arc, Kvantum-Materia)
# Configurer via kvantummanager
kvantummanager &
```

### Sous XFCE (harmoniser Qt)

```bash
# XFCE utilise GTK, donc les applications Qt nécessitent la même approche
# que sous GNOME

sudo apt install qt6ct  
echo 'export QT_QPA_PLATFORMTHEME=qt6ct' >> ~/.profile  

# Dans qt6ct, sélectionner le thème GTK+ ou Kvantum
# et configurer les polices pour correspondre à celles de XFCE
```

## Mode sombre système et applications

Le mode sombre est devenu un standard dans les systèmes modernes. Sous Linux, son implémentation repose sur le **portail XDG Settings** (`org.freedesktop.portal.Settings`), qui expose la préférence dans le namespace `org.freedesktop.appearance` avec la clé `color-scheme`. Les valeurs possibles sont `default` (0), `prefer-dark` (1), `prefer-light` (2). Ce mécanisme est indépendant du toolkit : Firefox, Chromium, GTK 4 (libadwaita), Qt 6, Electron récent et Flatpak interrogent tous cette même source de vérité via D-Bus.

```bash
# Lire la préférence courante via dbus-send
dbus-send --session --print-reply=literal \
  --dest=org.freedesktop.portal.Desktop \
  /org/freedesktop/portal/desktop \
  org.freedesktop.portal.Settings.Read \
  string:org.freedesktop.appearance string:color-scheme
```

### Comportement par toolkit

| Toolkit/Framework | Réaction au mode sombre système |
|-------------------|-------------------------------|
| **libadwaita (GTK 4)** | Automatique — suit `color-scheme` |
| **GTK 3** | Suit le thème GTK configuré. Un thème explicitement sombre (ex: `Adwaita-dark`) est nécessaire |
| **Qt / KDE** | Suit le schéma de couleurs Plasma. Le mode sombre est un schéma de couleurs |
| **Electron** | Variable. Certaines applications (VS Code, Slack) détectent la préférence système, d'autres non |
| **Firefox** | Détecte automatiquement la préférence système et adapte les pages web qui supportent `prefers-color-scheme` |
| **Flatpak** | Les applications Flatpak accèdent à la préférence via le portail XDG `Settings` |

### Applications récalcitrantes

Certaines applications ne suivent pas la préférence système et doivent être configurées individuellement :

```bash
# Firefox : forcer le thème sombre pour l'interface
```

## Polices : configuration avancée

### Fontconfig : le système de polices de Linux

**Fontconfig** est la bibliothèque qui gère la sélection, la substitution et le rendu des polices sur l'ensemble du système. Sa configuration permet de contrôler finement quel police est utilisée, comment elle est rendue et quelles substitutions sont appliquées lorsqu'une police demandée n'est pas disponible.

```bash
# Configuration système de fontconfig
/etc/fonts/fonts.conf            # Configuration principale (ne pas modifier)
/etc/fonts/conf.d/               # Liens symboliques vers les fichiers de configuration
/etc/fonts/conf.avail/           # Configurations disponibles

# Configuration utilisateur
~/.config/fontconfig/fonts.conf  # Surcharges personnelles
```

### Substitution de polices

Lorsqu'un document demande une police qui n'est pas installée (par exemple Calibri dans un document Word), fontconfig peut être configuré pour substituer automatiquement une police compatible :

```bash
# Créer ou éditer ~/.config/fontconfig/fonts.conf
mkdir -p ~/.config/fontconfig/  
nano ~/.config/fontconfig/fonts.conf  
```

```xml
<?xml version="1.0"?>
<!DOCTYPE fontconfig SYSTEM "urn:fontconfig:fonts.dtd">
<fontconfig>
  <!-- Substituer Calibri par Carlito -->
  <alias>
    <family>Calibri</family>
    <prefer>
      <family>Carlito</family>
    </prefer>
  </alias>

  <!-- Substituer Cambria par Caladea -->
  <alias>
    <family>Cambria</family>
    <prefer>
      <family>Caladea</family>
    </prefer>
  </alias>

  <!-- Police par défaut pour les sans-serif -->
  <alias>
    <family>sans-serif</family>
    <prefer>
      <family>Noto Sans</family>
    </prefer>
  </alias>

  <!-- Police par défaut pour les monospace -->
  <alias>
    <family>monospace</family>
    <prefer>
      <family>JetBrains Mono</family>
    </prefer>
  </alias>
</fontconfig>
```

```bash
# Recharger la configuration
fc-cache -fv

# Vérifier la substitution
fc-match Calibri
# Résultat attendu : Carlito-Regular.ttf: "Carlito" "Regular"
```

### Rendu des polices

La qualité du rendu des polices dépend de trois paramètres : l'anti-aliasing (lissage des bords), le hinting (ajustement aux pixels) et le sous-pixel rendering (utilisation des sous-pixels de l'écran LCD) :

```bash
# Vérifier la configuration actuelle
# GNOME
gsettings get org.gnome.desktop.interface font-antialiasing  
gsettings get org.gnome.desktop.interface font-hinting  

# Recommandations par type d'écran :
# Écrans LCD/IPS : antialiasing=rgba, hinting=slight
# Écrans OLED : antialiasing=grayscale, hinting=slight
# Écrans HiDPI (>150 DPI) : antialiasing=grayscale, hinting=none
#   (les pixels sont assez petits pour que le sous-pixel rendering soit inutile)
```

## Résumé

La personnalisation visuelle sous Debian s'appuie sur un système de couches (thème d'applications, décorations de fenêtres, icônes, curseurs, couleurs, polices) dont la granularité varie selon l'environnement de bureau. KDE Plasma offre la personnalisation la plus fine avec un contrôle indépendant de chaque couche. GNOME privilégie la cohérence via libadwaita au détriment de la liberté de thème. XFCE propose un équilibre pragmatique entre les deux.

La cohérence visuelle entre applications GTK et Qt est un sujet transversal que chaque environnement résout différemment : thème Breeze GTK sous KDE, Adwaita-Qt ou qt6ct sous GNOME et XFCE. Le mode sombre système est désormais bien supporté par les applications modernes via le portail XDG Settings.

La section suivante (2.4.2) aborde les raccourcis clavier, un levier de productivité souvent sous-exploité.
