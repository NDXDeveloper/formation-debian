# 2.3.3 Configuration multi-écrans

## Introduction

Les configurations multi-écrans sont devenues la norme dans de nombreux contextes : postes de développement avec deux ou trois moniteurs, ordinateurs portables connectés à un écran externe au bureau, stations de travail pour la création de contenu, ou encore configurations de présentation avec vidéoprojecteur. Sous Linux, la gestion multi-écrans a considérablement évolué au fil des années, et Wayland a apporté des améliorations majeures par rapport à Xorg, notamment pour les configurations mélangeant des écrans de résolutions et de facteurs d'échelle différents.

Cette section couvre la configuration multi-écrans sous Debian : la détection des sorties vidéo, la disposition des écrans, la gestion de la résolution et du taux de rafraîchissement, la mise à l'échelle (scaling) pour les écrans HiDPI, et les spécificités de chaque environnement de bureau.

## Types de connecteurs et sorties vidéo

### Connecteurs courants

| Connecteur | Signal | Résolution maximale typique | Audio | Remarques |
|------------|--------|----------------------------|-------|-----------|
| **HDMI 2.0** | Numérique | 4K @ 60 Hz | Oui | Standard le plus courant, présent sur la majorité des écrans et portables |
| **HDMI 2.1** | Numérique | 8K @ 60 Hz / 4K @ 120 Hz | Oui | Support VRR, eARC |
| **DisplayPort 1.4** | Numérique | 8K @ 60 Hz / 4K @ 120 Hz | Oui | Supporte le daisy-chaining (Multi-Stream Transport) |
| **DisplayPort 2.0** | Numérique | 16K @ 60 Hz / 4K @ 240 Hz | Oui | Dernière génération |
| **USB-C / Thunderbolt** | Numérique (Alt Mode DP) | Dépend de la version | Oui | Transport DisplayPort via USB-C, charge simultanée possible |
| **VGA (D-Sub)** | Analogique | 2048 × 1536 (théorique) | Non | Connecteur historique, en voie de disparition |
| **DVI** | Numérique/Analogique | 2560 × 1600 (dual-link) | Non | Remplacé par HDMI et DisplayPort |

### Détection des sorties vidéo

```bash
# Sous Xorg : lister les sorties et leur état
xrandr
# Résultat typique :
# Screen 0: minimum 320 x 200, current 3840 x 1080, maximum 16384 x 16384
# eDP-1 connected primary 1920x1080+0+0 (normal left inverted right) 309mm x 174mm
#    1920x1080     60.01*+  60.01    59.97    59.96
#    1680x1050     59.95    59.88
#    ...
# HDMI-1 connected 1920x1080+1920+0 (normal left inverted right) 527mm x 296mm
#    1920x1080     60.00*+  50.00    59.94
#    ...
# DP-1 disconnected (normal left inverted right)

# eDP-1     = écran intégré (embedded DisplayPort, portable)
# HDMI-1    = sortie HDMI
# DP-1      = sortie DisplayPort
# VGA-1     = sortie VGA
# DP-1-1    = sortie DP via MST (Multi-Stream Transport, daisy-chain)
# HDMI-1-1  = sortie HDMI via hub/dock

# Sous Wayland : les commandes varient selon le compositeur
# GNOME (Mutter)
# Pas d'outil CLI natif, utiliser les Paramètres système
# Ou installer gnome-randr :
# pip install gnome-randr
# gnome-randr

# KDE (KWin)
kscreen-doctor --outputs
# Affiche les écrans détectés avec leurs modes et leur disposition

# Informations détaillées via le noyau
ls /sys/class/drm/
# card0-eDP-1, card0-HDMI-A-1, card0-DP-1, etc.

# Informations EDID de l'écran (modèle, résolutions supportées, etc.)
sudo apt install edid-decode  
cat /sys/class/drm/card0-HDMI-A-1/edid | edid-decode  
```

## Configuration sous Xorg

### xrandr : l'outil de référence

`xrandr` (X Resize and Rotate) est l'outil en ligne de commande standard pour la configuration des écrans sous Xorg. Il permet de modifier la résolution, le taux de rafraîchissement, la disposition, la rotation et la mise à l'échelle des écrans.

#### Opérations courantes

```bash
# Afficher la configuration actuelle
xrandr

# Activer un écran externe en extension (à droite de l'écran principal)
xrandr --output HDMI-1 --auto --right-of eDP-1

# Activer un écran externe à gauche
xrandr --output HDMI-1 --auto --left-of eDP-1

# Activer un écran externe au-dessus
xrandr --output HDMI-1 --auto --above eDP-1

# Mode miroir (duplication)
xrandr --output HDMI-1 --auto --same-as eDP-1

# Définir une résolution et un taux de rafraîchissement spécifiques
xrandr --output HDMI-1 --mode 2560x1440 --rate 75

# Définir un écran comme principal
# (la barre des tâches et les notifications apparaissent sur l'écran principal)
xrandr --output HDMI-1 --primary

# Désactiver un écran
xrandr --output HDMI-1 --off

# Définir la position exacte (en pixels, coin supérieur gauche)
xrandr --output HDMI-1 --pos 1920x0  
xrandr --output eDP-1 --pos 0x0  

# Rotation
xrandr --output HDMI-1 --rotate left      # 90° antihoraire (portrait)  
xrandr --output HDMI-1 --rotate right     # 90° horaire  
xrandr --output HDMI-1 --rotate inverted  # 180°  
xrandr --output HDMI-1 --rotate normal    # Orientation normale  

# Configuration complète en une seule commande
xrandr \
  --output eDP-1 --mode 1920x1080 --pos 0x0 --primary \
  --output HDMI-1 --mode 2560x1440 --pos 1920x0 --rotate normal
```

#### Ajout d'un mode personnalisé

Si la résolution souhaitée n'apparaît pas dans la liste des modes détectés :

```bash
# Générer les paramètres du mode avec cvt
cvt 1920 1200 60
# Résultat : Modeline "1920x1200_60.00" 193.25 1920 2056 2256 2592 1200 ...

# Créer le nouveau mode
xrandr --newmode "1920x1200_60.00" 193.25 1920 2056 2256 2592 1200 1203 1209 1245 -hsync +vsync

# Ajouter le mode à la sortie
xrandr --addmode HDMI-1 "1920x1200_60.00"

# Activer le mode
xrandr --output HDMI-1 --mode "1920x1200_60.00"
```

#### Rendre la configuration persistante

Les changements effectués avec `xrandr` sont temporaires et perdus au redémarrage ou à la déconnexion/reconnexion d'un écran. Plusieurs méthodes permettent de rendre la configuration persistante :

```bash
# Méthode 1 : script dans le profil de session
# Créer ~/.xprofile (exécuté au démarrage de la session X)
nano ~/.xprofile

#!/bin/bash
xrandr \
  --output eDP-1 --mode 1920x1080 --pos 0x0 --primary \
  --output HDMI-1 --mode 2560x1440 --pos 1920x0

chmod +x ~/.xprofile

# Méthode 2 : autorandr (gestion automatique de profils multi-écrans)
sudo apt install autorandr

# Configurer les écrans comme souhaité avec xrandr, puis sauvegarder le profil
autorandr --save bureau          # Profil "bureau" (avec écran externe)  
autorandr --save mobile          # Profil "mobile" (écran interne seul)  
autorandr --save presentation    # Profil "présentation" (miroir)  

# autorandr détecte automatiquement les écrans connectés et applique
# le profil correspondant grâce à un hook udev
# Lister les profils
autorandr --list

# Appliquer un profil manuellement
autorandr bureau

# Méthode 3 : fichier de configuration Xorg
# Créer /etc/X11/xorg.conf.d/10-monitors.conf
# (voir section Configuration Xorg statique ci-dessous)
```

#### Configuration Xorg statique

Pour les installations nécessitant une configuration fixe (serveur avec affichage, borne interactive) :

```bash
sudo nano /etc/X11/xorg.conf.d/10-monitors.conf
```

```
Section "Monitor"
    Identifier  "eDP-1"
    Option      "Primary" "true"
EndSection

Section "Monitor"
    Identifier  "HDMI-1"
    Option      "RightOf" "eDP-1"
    Option      "PreferredMode" "2560x1440"
EndSection
```

## Configuration sous Wayland

Sous Wayland, il n'existe pas d'outil universel comme `xrandr`. La configuration des écrans est gérée par chaque compositeur (Mutter pour GNOME, KWin pour KDE) via ses propres interfaces. Cette approche est intentionnelle : chaque compositeur peut exploiter au mieux les capacités de Wayland (mise à l'échelle fractionnaire par écran, taux de rafraîchissement différents par écran).

### Avantages de Wayland pour le multi-écrans

| Fonctionnalité | Xorg | Wayland |
|----------------|------|---------|
| **Mise à l'échelle par écran** | Non (un seul facteur DPI global) | Oui (chaque écran a son propre facteur) |
| **Mise à l'échelle fractionnaire** | Approximative et floue | Native et nette |
| **Taux de rafraîchissement par écran** | Tous les écrans au même taux | Chaque écran indépendant |
| **VRR (Variable Refresh Rate)** | Non | Oui (GNOME 46+, KDE Plasma 6+) |
| **Hotplug (branchement à chaud)** | Supporté mais parfois instable | Natif et transparent |
| **Mémorisation de la disposition** | Non natif (autorandr nécessaire) | Natif dans GNOME et KDE |

## Configuration par environnement de bureau

### GNOME

GNOME gère les écrans via **Paramètres → Écrans** (`gnome-control-center display`). L'interface est visuelle : les écrans sont représentés par des rectangles que l'on peut glisser-déposer pour ajuster leur position relative.

Les options disponibles :

- **Disposition** : glisser les écrans pour définir leur position relative (gauche, droite, dessus, dessous).
- **Écran principal** : sélectionner l'écran qui affiche la barre supérieure et les notifications.
- **Résolution** : sélection parmi les modes supportés par l'écran.
- **Taux de rafraîchissement** : choix du taux parmi ceux supportés.
- **Échelle** : mise à l'échelle (100%, 125%, 150%, 175%, 200%, etc.). Sous Wayland, cette mise à l'échelle est fractionnaire et par écran.
- **Rotation** : portrait, paysage, inversé.
- **Joindre les affichages** / **Miroir** : mode extension ou mode duplication.

```bash
# Sous Wayland, GNOME mémorise automatiquement la disposition des écrans
# La configuration est sauvegardée dans :
~/.config/monitors.xml

# Ce fichier XML contient la configuration pour chaque combinaison d'écrans
# identifiés par leur EDID (modèle, numéro de série)
# GNOME applique automatiquement la bonne configuration lorsque les mêmes
# écrans sont reconnectés

# Appliquer la mise à l'échelle fractionnaire sous Wayland
# Activer le support (si les options 125%, 150% n'apparaissent pas)
gsettings set org.gnome.mutter experimental-features "['scale-monitor-framebuffer']"
# Note : sur les versions récentes de GNOME (45+), cette option est activée par défaut

# Sous Xorg, la mise à l'échelle fractionnaire GNOME est différente :
gsettings set org.gnome.mutter experimental-features "['x11-randr-fractional-scaling']"
```

#### Gestion des espaces de travail en multi-écrans

GNOME propose deux modes pour les espaces de travail en configuration multi-écrans :

```bash
# Mode 1 : espaces de travail sur tous les écrans (défaut)
# Changer d'espace de travail change les fenêtres sur TOUS les écrans
gsettings set org.gnome.mutter workspaces-only-on-primary false

# Mode 2 : espaces de travail uniquement sur l'écran principal
# L'écran secondaire conserve ses fenêtres quel que soit l'espace de travail
# Utile pour garder une référence (documentation, chat) toujours visible
gsettings set org.gnome.mutter workspaces-only-on-primary true
```

### KDE Plasma

KDE Plasma offre la configuration multi-écrans la plus détaillée parmi les environnements de bureau, via **Paramètres système → Affichage et écran → Configuration d'affichage**.

Les options disponibles :

- **Disposition** : glisser-déposer des écrans avec alignement (bords, coins, centré).
- **Résolution et taux de rafraîchissement** : sélection indépendante par écran.
- **Mise à l'échelle** : curseur de 100% à 300% par incréments fins par écran (Wayland).
- **Rotation** : quatre orientations.
- **Réplication** : mode miroir.
- **Priorité** : écran principal pour la barre des tâches et les notifications.
- **Profils** : KDE mémorise les configurations pour chaque combinaison d'écrans.

```bash
# Outil en ligne de commande : kscreen-doctor
kscreen-doctor --outputs
# Affiche les écrans détectés, leurs modes et la configuration actuelle

# Modifier la configuration
kscreen-doctor output.HDMI-1.enable  
kscreen-doctor output.HDMI-1.mode.2560x1440@75  
kscreen-doctor output.HDMI-1.position.1920,0  
kscreen-doctor output.HDMI-1.scale.1.25  

# Désactiver un écran
kscreen-doctor output.HDMI-1.disable

# La configuration est sauvegardée dans
~/.local/share/kscreen/
# KDE mémorise la disposition pour chaque combinaison d'écrans
# et l'applique automatiquement à la reconnexion
```

#### Barre des tâches et multi-écrans dans Plasma

KDE Plasma permet de configurer indépendamment les panneaux sur chaque écran :

- Un panneau unique sur l'écran principal (défaut).
- Un panneau sur chaque écran (clic droit sur le panneau → « Afficher un panneau sur chaque écran »).
- Des panneaux différents sur chaque écran (ajouter un panneau manuellement sur l'écran secondaire).

Le gestionnaire de tâches peut être configuré pour afficher uniquement les fenêtres de l'écran courant ou toutes les fenêtres sur tous les panneaux.

### XFCE

XFCE gère les écrans via **Paramètres → Affichage** (`xfce4-display-settings`).

```bash
# Ouvrir les paramètres d'affichage
xfce4-display-settings

# L'interface permet :
# - Activer/désactiver les écrans
# - Choisir la résolution et le taux de rafraîchissement
# - Définir la disposition (position relative)
# - Sélectionner l'écran principal
# - Activer le mode miroir
# - Rotation

# XFCE peut sauvegarder des profils d'affichage
# Dialogue affiché automatiquement à la connexion/déconnexion d'un écran

# Configuration en ligne de commande (via xrandr, puisque XFCE fonctionne sous Xorg)
# Voir la section xrandr ci-dessus

# Configuration via xfconf
xfconf-query -c displays -l
```

### LXQt

LXQt utilise `lxqt-config-monitor` pour la configuration des écrans. L'interface est minimaliste mais fonctionnelle. Si KWin est utilisé comme gestionnaire de fenêtres, `kscreen-doctor` peut également être utilisé.

```bash
# Configuration des écrans
lxqt-config-monitor

# En complément, xrandr reste l'outil principal sous LXQt (session X11)
```

## Écrans HiDPI (haute résolution)

### Comprendre le scaling

Les écrans HiDPI (High Dots Per Inch) — écrans 4K (3840×2160) sur des diagonales de 13" à 27", écrans Retina, etc. — nécessitent une mise à l'échelle (scaling) pour que les éléments de l'interface restent lisibles. Sans mise à l'échelle, le texte et les icônes sont minuscules sur un écran 4K 15".

Le facteur d'échelle représente le multiplicateur appliqué à la taille de tous les éléments de l'interface :

| Facteur | Effet | Cas d'usage typique |
|---------|-------|---------------------|
| **100%** | Taille native (1 pixel logique = 1 pixel physique) | Écrans Full HD (1920×1080) |
| **125%** | Légèrement agrandi | Écrans QHD (2560×1440) 27" |
| **150%** | Agrandi de 50% | Écrans QHD 14-15", écrans 4K 27-32" |
| **175%** | Agrandi de 75% | Écrans 4K 24" |
| **200%** | Doublé (1 pixel logique = 4 pixels physiques) | Écrans 4K 13-15" |

### Mise à l'échelle sous Wayland

Wayland gère la mise à l'échelle de manière native et par écran. Chaque écran peut avoir son propre facteur d'échelle, et le rendu est net à tous les facteurs, y compris fractionnaires.

```bash
# GNOME Wayland
# Paramètres → Écrans → Échelle
# Sélectionner le facteur souhaité pour chaque écran indépendamment

# KDE Plasma Wayland
# Paramètres système → Affichage et écran → Configuration d'affichage
# Curseur d'échelle pour chaque écran

# Vérifier le facteur d'échelle actif
# GNOME
gsettings get org.gnome.desktop.interface scaling-factor
# Note : ce paramètre ne reflète que l'échelle entière sous Xorg
# Sous Wayland, l'échelle fractionnaire est gérée par Mutter directement
```

### Mise à l'échelle sous Xorg

Sous Xorg, la mise à l'échelle est plus complexe et moins satisfaisante :

```bash
# Méthode 1 : mise à l'échelle globale via le DPI Xorg
# Applique un facteur à l'ensemble du serveur X (tous les écrans)
xrandr --dpi 192    # 192 DPI = facteur 2x (96 DPI étant la référence)  
xrandr --dpi 144    # 144 DPI = facteur 1.5x  

# Méthode 2 : mise à l'échelle par écran via xrandr --scale
# Cette méthode effectue un rendu à une résolution supérieure puis réduit
# Le résultat est souvent flou
xrandr --output eDP-1 --scale 1x1 \
       --output HDMI-1 --scale 2x2
# L'écran HDMI est rendu en résolution doublée puis réduit

# Méthode 3 : mise à l'échelle fractionnaire GNOME Xorg
gsettings set org.gnome.mutter experimental-features "['x11-randr-fractional-scaling']"
# Permet de sélectionner 125%, 150%, etc. dans les Paramètres GNOME sous Xorg
# Le rendu utilise une technique de surrendu et réduction, parfois flou

# Ajuster la taille des polices (complément à la mise à l'échelle)
# GNOME
gsettings set org.gnome.desktop.interface text-scaling-factor 1.25

# XFCE
xfconf-query -c xsettings -p /Xft/DPI -s 144

# KDE
# Paramètres système → Polices → Forcer le DPI des polices
```

### Configuration mixte : écrans HiDPI et non-HiDPI

La configuration la plus délicate est celle qui mélange un écran HiDPI (portable 4K) et un écran standard (moniteur externe Full HD). Sous Xorg, cette configuration est problématique car il n'existe qu'un seul facteur d'échelle global. Sous Wayland, chaque écran a son propre facteur.

```bash
# Sous Wayland (GNOME ou KDE) :
# Chaque écran est configuré indépendamment dans les Paramètres
# Exemple : écran portable 4K à 200%, écran externe FHD à 100%
# Le rendu est net sur les deux écrans

# Sous Xorg (solution de contournement) :
# Configurer le DPI global pour l'écran HiDPI
xrandr --dpi 192

# Puis appliquer un facteur de réduction sur l'écran non-HiDPI
xrandr --output HDMI-1 --scale 2x2 --filter bilinear
# L'écran FHD est rendu en 3840x2160 (virtuel) puis réduit à 1920x1080
# Le résultat est fonctionnel mais le texte peut paraître légèrement flou
# sur l'écran non-HiDPI

# La meilleure solution sous Xorg pour les configurations mixtes reste
# de passer à Wayland si l'environnement de bureau le supporte
```

## Cas d'usage spécifiques

### Ordinateur portable avec station d'accueil (dock)

Les stations d'accueil USB-C/Thunderbolt permettent de connecter plusieurs écrans via un seul câble. La prise en charge dépend du dock, du GPU et du pilote :

```bash
# Vérifier les périphériques Thunderbolt
# Installer l'outil de gestion Thunderbolt
sudo apt install bolt

# Lister les périphériques Thunderbolt
boltctl list  
boltctl info <uuid>  

# Autoriser un périphérique Thunderbolt (si la sécurité Thunderbolt est active)
boltctl authorize <uuid>
# Ou autoriser de manière permanente
boltctl enroll <uuid>

# Vérifier les écrans connectés via le dock
xrandr    # sous Xorg  
kscreen-doctor --outputs    # sous KDE Wayland  

# Les écrans via dock apparaissent avec des noms comme
# DP-3, DP-4 (DisplayPort via USB-C/Thunderbolt)
```

Les environnements de bureau mémorisent la disposition des écrans associée à chaque dock. Lorsque le portable est connecté au dock, la configuration multi-écrans s'applique automatiquement. Lorsqu'il est déconnecté, le système revient à l'écran interne seul.

### Mode présentation (vidéoprojecteur)

```bash
# Mode miroir (duplication)
# Xorg
xrandr --output HDMI-1 --auto --same-as eDP-1

# Si les résolutions diffèrent, forcer une résolution commune
xrandr --output eDP-1 --mode 1920x1080 --output HDMI-1 --mode 1920x1080 --same-as eDP-1

# Les environnements de bureau proposent un dialogue automatique
# lors de la connexion d'un écran :
# - Miroir (présentation)
# - Extension (bureau étendu)
# - Écran externe uniquement

# Raccourcis clavier courants pour le multi-écrans :
# GNOME : Super + P (cycle entre les modes d'affichage)
# KDE :   Meta + P (similaire)
# XFCE :  le dialogue xfce4-display-settings apparaît automatiquement
```

### Écran en mode portrait (vertical)

Les écrans en mode portrait sont courants pour le développement (lecture de code), la lecture de documents et les affichages d'information :

```bash
# Xorg : rotation d'un écran
xrandr --output HDMI-1 --rotate left     # 90° antihoraire  
xrandr --output HDMI-1 --rotate right    # 90° horaire  

# Wayland : via les Paramètres d'affichage de l'environnement de bureau
# L'option de rotation est disponible dans les paramètres de chaque écran

# Ajuster la position après rotation
# Un écran 1920x1080 en portrait devient 1080x1920
# Il faut repositionner les écrans dans les Paramètres ou via xrandr/kscreen-doctor
xrandr --output eDP-1 --mode 1920x1080 --pos 0x0 \
       --output HDMI-1 --mode 1920x1080 --rotate left --pos 1920x0
```

## Dépannage

### L'écran externe n'est pas détecté

```bash
# 1. Vérifier la connexion physique
# Tester avec un autre câble si possible
# Vérifier que le câble supporte la résolution souhaitée
# (certains câbles HDMI anciens ne supportent pas le 4K)

# 2. Vérifier la détection côté noyau
dmesg | tail -20    # Après avoir branché l'écran
# Chercher des messages "drm", "connector", "hotplug"

# 3. Vérifier les sorties détectées
xrandr              # Xorg  
cat /sys/class/drm/card0-*/status  
# "connected" ou "disconnected" pour chaque sortie

# 4. Forcer la détection
xrandr --output HDMI-1 --auto

# 5. Vérifier le pilote graphique
lspci -k | grep -A 3 -i vga
# Si le pilote "nouveau" est utilisé avec une carte NVIDIA,
# certaines sorties peuvent ne pas fonctionner
# → Installer le pilote NVIDIA propriétaire (voir section 2.3.1)

# 6. Pour les docks USB-C/Thunderbolt
# Vérifier l'autorisation Thunderbolt
boltctl list
# Vérifier le support DisplayPort Alt Mode
dmesg | grep -i "typec\|ucsi\|alt mode"
```

### Résolution incorrecte ou non disponible

```bash
# 1. Vérifier les modes supportés par l'écran
xrandr --verbose | grep -A 20 "HDMI-1"

# 2. Vérifier les informations EDID
sudo apt install edid-decode read-edid  
sudo get-edid -b 0 | edid-decode  
# Si l'EDID est corrompu ou non lu, le système ne connaît pas
# les résolutions supportées par l'écran

# 3. Forcer une résolution via un mode personnalisé
# (voir section xrandr / Ajout d'un mode personnalisé)

# 4. Vérifier les limitations du GPU
# Les GPU intégrés ont des limites de résolution et de nombre d'écrans
# Intel UHD : généralement 3 écrans max, 4K@60Hz via DP, 4K@30Hz via HDMI 1.4
# Vérifier dans la documentation du GPU
```

### Décalage ou espacement entre les écrans

Lorsque le curseur de la souris « saute » ou qu'il existe un espace mort entre deux écrans, c'est un problème d'alignement :

```bash
# Sous Xorg : ajuster précisément la position
# Si eDP-1 est en 1920x1080 et HDMI-1 en 2560x1440 à droite,
# les bords supérieurs sont alignés par défaut
# Pour aligner les bords inférieurs :
xrandr --output eDP-1 --pos 0x360 --output HDMI-1 --pos 1920x0
# (360 = 1440 - 1080, décalage vertical pour aligner le bas)

# Sous Wayland : glisser-déposer dans les Paramètres d'affichage
# pour ajuster l'alignement visuellement
```

### Écran externe qui clignote ou artefacts visuels

```bash
# 1. Essayer un taux de rafraîchissement inférieur
xrandr --output HDMI-1 --mode 2560x1440 --rate 60
# Au lieu de 75 Hz ou 144 Hz

# 2. Vérifier la qualité du câble
# Les câbles HDMI et DP de mauvaise qualité peuvent causer des artefacts
# à haute résolution ou haut taux de rafraîchissement

# 3. Sous NVIDIA, vérifier le mode de composition
```

## Résumé

La configuration multi-écrans sous Debian est une tâche bien maîtrisée par les environnements de bureau modernes, avec un avantage marqué pour Wayland qui apporte la mise à l'échelle par écran, les taux de rafraîchissement indépendants et le VRR. Sous Xorg, `xrandr` reste l'outil de référence, complété par `autorandr` pour la gestion automatique des profils.

GNOME et KDE Plasma offrent tous deux une interface graphique intuitive de glisser-déposer pour organiser les écrans, avec une mémorisation automatique des configurations. Pour les écrans HiDPI, Wayland est fortement recommandé, particulièrement dans les configurations mixtes (HiDPI + standard).

La section suivante (2.3.4) couvre la gestion du Bluetooth et des périphériques : appairage, gestion des profils, claviers, souris, imprimantes et autres périphériques connectés.
