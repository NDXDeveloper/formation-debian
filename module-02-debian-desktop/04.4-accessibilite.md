# 2.4.4 Accessibilité

## Introduction

L'accessibilité numérique désigne l'ensemble des pratiques et des technologies qui permettent à toutes les personnes, quelles que soient leurs capacités physiques, sensorielles ou cognitives, d'utiliser un système informatique de manière autonome. Sous Linux, l'accessibilité est un domaine où GNOME se distingue particulièrement, ayant investi depuis plus de 20 ans dans le framework AT-SPI (Assistive Technology Service Provider Interface), le lecteur d'écran Orca et l'intégration d'options d'accessibilité à tous les niveaux de l'interface.

L'accessibilité ne concerne pas uniquement les personnes en situation de handicap permanent. Des situations temporaires (bras dans le plâtre, lunettes cassées, environnement bruyant) ou liées à l'âge (baisse de la vue, motricité réduite) rendent les fonctions d'accessibilité utiles à un public bien plus large. Certaines fonctionnalités d'accessibilité — le zoom d'écran, l'agrandissement du curseur, le contraste élevé, les raccourcis clavier — améliorent le confort de tous les utilisateurs.

Cette section présente les fonctionnalités d'accessibilité disponibles sous Debian, leur configuration et les outils spécialisés pour chaque type de besoin.

## Le framework d'accessibilité sous Linux

### AT-SPI2 : l'infrastructure d'accessibilité

**AT-SPI2** (Assistive Technology Service Provider Interface, version 2) est le framework qui permet aux technologies d'assistance (lecteurs d'écran, loupes, claviers virtuels) de communiquer avec les applications graphiques. AT-SPI2 fonctionne via D-Bus et fournit une interface standardisée pour que les outils d'accessibilité puissent interroger l'état de l'interface utilisateur : quel élément a le focus, quel texte est affiché, quels boutons sont disponibles, quelle est la structure de la page, etc.

```bash
# AT-SPI2 est installé par défaut avec les environnements de bureau
sudo apt install at-spi2-core libatk-adaptor

# Vérifier que le bus AT-SPI est actif
busctl --user list | grep accessibility
# ou
ps aux | grep at-spi

# Le registre AT-SPI doit être démarré pour que les technologies
# d'assistance fonctionnent
# Il est normalement lancé automatiquement avec la session graphique
```

### Compatibilité des toolkits

Le fonctionnement des technologies d'assistance dépend de la capacité des applications à exposer leurs éléments d'interface via AT-SPI2. Cette capacité varie selon le toolkit :

| Toolkit | Support AT-SPI2 | Qualité |
|---------|----------------|---------|
| **GTK 3/4 (libadwaita)** | Natif et complet | Excellente — les applications GNOME sont les mieux supportées |
| **Qt 5/6** | Natif via le plugin `qt-at-spi` | Bonne — les applications KDE sont accessibles |
| **Electron** | Partiel (basé sur Chromium accessibility) | Variable selon l'application |
| **Java Swing/AWT** | Via Java Access Bridge | Fonctionnelle |
| **Firefox** | Natif | Excellente |
| **LibreOffice** | Natif (ATK) | Bonne |

```bash
# Activer le support AT-SPI pour les applications Qt
# Normalement automatique, mais vérifier :
export QT_ACCESSIBILITY=1
# Ajouter à ~/.profile si ce n'est pas défini par défaut
```

## Accessibilité visuelle

### Lecteur d'écran : Orca

**Orca** est le lecteur d'écran de référence sous Linux. Il convertit le contenu de l'écran en synthèse vocale et/ou en braille, permettant aux personnes aveugles ou malvoyantes d'utiliser l'ordinateur. Orca est intégré à GNOME et fonctionne également avec les applications Qt, Firefox, LibreOffice et de nombreuses autres applications accessibles.

```bash
# Orca est normalement installé avec GNOME
sudo apt install orca

# Activer/désactiver Orca avec le raccourci clavier
# Super + Alt + S  (sous GNOME)

# Lancer Orca manuellement
orca &

# Lancer la configuration d'Orca
orca --setup
```

#### Configuration d'Orca

La boîte de dialogue de configuration d'Orca (`orca --setup` ou `Orca Preferences` depuis le menu) permet d'ajuster :

**Voix** :
- Synthétiseur vocal (eSpeak NG par défaut, d'autres sont installables)
- Voix et langue
- Débit de parole (lent à très rapide)
- Hauteur et volume
- Verbosité (combien de détails sont annoncés)

**Braille** :
- Activation de la sortie braille
- Table braille (français, anglais, abrégé, intégral)
- Configuration de l'afficheur braille connecté

**Raccourcis** :
- Orca utilise une touche modificatrice dédiée, appelée « touche Orca »
- Par défaut, la touche Orca est `Insert` (ou `Caps Lock` selon la configuration)
- Les raccourcis Orca sont distincts des raccourcis du bureau

```bash
# Installer des synthétiseurs vocaux supplémentaires
sudo apt install espeak-ng                # eSpeak NG (par défaut, léger, multilingue)  
sudo apt install speech-dispatcher        # Dispatcher de synthèse vocale  
sudo apt install mbrola mbrola-fr1        # Voix MBROLA (qualité supérieure)  

# Configurer le système de synthèse vocale
sudo apt install speech-dispatcher-espeak-ng
# Le fichier de configuration est
/etc/speech-dispatcher/speechd.conf
# La configuration utilisateur est dans
~/.config/speech-dispatcher/

# Tester la synthèse vocale
spd-say "Bonjour, ceci est un test de synthèse vocale"  
spd-say -l fr "Ceci est un test en français"  

# Lister les voix disponibles
spd-list-modules  
spd-list-voices  
```

#### Navigation avec Orca

Orca propose plusieurs modes de navigation :

**Mode focus** : Orca suit le focus de l'interface. L'utilisateur navigue avec Tab entre les éléments interactifs (boutons, champs, liens) et Orca annonce chaque élément au fur et à mesure.

**Mode navigation** (browse mode) : dans les navigateurs web et les documents, Orca permet de naviguer par structure — en-têtes, paragraphes, liens, listes, tableaux, repères (landmarks) — de manière similaire aux lecteurs d'écran sous Windows (NVDA, JAWS).

**Raccourcis Orca essentiels** (touche Orca = Insert par défaut) :

| Raccourci | Action |
|-----------|--------|
| `Orca + Espace` | Basculer entre mode focus et mode navigation |
| `Orca + S` | Lire la ligne courante |
| `Orca + W` | Lire le mot courant |
| `Orca + L` | Lire le caractère courant |
| `H` / `Shift+H` | En-tête suivant / précédent (mode navigation) |
| `K` / `Shift+K` | Lien suivant / précédent |
| `T` / `Shift+T` | Tableau suivant / précédent |
| `Orca + ;` | Lire la barre de titre |
| `Orca + Tab` | Connaître l'élément ayant le focus |
| `Orca + Q` | Quitter Orca |

### Zoom d'écran (loupe)

Le zoom d'écran agrandit une portion de l'écran pour les personnes malvoyantes qui n'ont pas besoin d'un lecteur d'écran complet.

#### GNOME : zoom intégré

```bash
# Activer le zoom via le raccourci clavier
# Super + Alt + 8   → Activer/désactiver le zoom

# Zoomer / dézoomer (quand le zoom est actif)
# Super + Alt + =   → Zoomer
# Super + Alt + -   → Dézoomer

# Via les Paramètres
# Paramètres → Accessibilité → Zoom

# Configuration en ligne de commande
# Activer le zoom
gsettings set org.gnome.desktop.a11y.magnifier mag-factor 2.0

# Mode de suivi du curseur
gsettings set org.gnome.desktop.a11y.magnifier mouse-tracking 'proportional'
# Valeurs : centered (le curseur reste au centre de la zone zoomée),
#           proportional (le zoom suit le curseur proportionnellement),
#           push (le zoom se déplace quand le curseur atteint le bord)

# Mode de suivi du focus
gsettings set org.gnome.desktop.a11y.magnifier focus-tracking 'proportional'

# Lentille (loupe ronde qui suit le curseur, au lieu du zoom plein écran)
gsettings set org.gnome.desktop.a11y.magnifier lens-mode true

# Couleurs inversées dans la zone zoomée
gsettings set org.gnome.desktop.a11y.magnifier invert-lightness true

# Filtres daltonisme dans la zone zoomée
gsettings set org.gnome.desktop.a11y.magnifier color-saturation 0.5
```

#### KDE Plasma : effet de zoom KWin

```bash
# Paramètres système → Comportement de l'espace de travail → Effets de bureau
# → Accessibilité → Zoom

# Raccourcis par défaut :
# Meta + =     → Zoomer
# Meta + -     → Dézoomer
# Meta + 0     → Réinitialiser le zoom

# Le zoom KDE est un effet KWin, plus simple que le zoom GNOME
# mais fonctionnel pour un usage ponctuel

# Alternative : utiliser Meta + molette de la souris (si l'effet est activé)
```

### Contraste élevé et thèmes à fort contraste

Le contraste élevé améliore la lisibilité pour les personnes ayant une vision réduite, mais aussi dans des conditions d'éclairage difficiles (soleil direct sur l'écran) :

```bash
# GNOME : activer le contraste élevé
gsettings set org.gnome.desktop.a11y.interface high-contrast true

# Ou via Paramètres → Accessibilité → Contraste élevé
# Sous GNOME 48 / libadwaita, ce paramètre est natif : les applications
# adaptatives basculent automatiquement vers leur palette haut contraste,
# sans avoir besoin d'un thème externe.

# Thème GTK 3 à fort contraste (utile pour les vieilles applications GTK 3
# qui n'ont pas migré vers libadwaita)
gsettings set org.gnome.desktop.interface gtk-theme 'HighContrast'

# KDE : utiliser un schéma de couleurs à fort contraste
# Paramètres système → Apparence → Couleurs → Breeze High Contrast

# Paquet historique pour le thème HighContrast GTK 2
# (utile uniquement si des applications GTK 2 sont encore installées)
sudo apt install gnome-accessibility-themes
```

### Taille du texte et du curseur

```bash
# Agrandir le texte de l'interface (sans changer l'échelle globale)
# GNOME
gsettings set org.gnome.desktop.interface text-scaling-factor 1.5
# 1.0 = normal, 1.5 = 50% plus grand, 2.0 = doublé

# Agrandir le curseur de souris
gsettings set org.gnome.desktop.interface cursor-size 48
# Tailles : 24 (défaut), 32, 48, 64, 96

# KDE
# Paramètres système → Apparence → Polices → Forcer le DPI
# Paramètres système → Apparence → Curseurs → Taille

# XFCE
xfconf-query -c xsettings -p /Xft/DPI -s 144  
xfconf-query -c xsettings -p /Gtk/CursorThemeSize -s 48  
```

### Filtres pour le daltonisme

GNOME propose des filtres de correction pour les différentes formes de daltonisme :

```bash
# GNOME (via les paramètres de la loupe d'écran)
# La loupe peut appliquer des filtres de couleur à l'écran entier

# Alternatives logicielles
# Compiz / picom : certains compositeurs proposent des shaders daltonisme

# Le mode nuit (filtre de lumière bleue) est aussi utile
# pour réduire la fatigue visuelle
# Paramètres → Écrans → Mode nuit (GNOME)
# Paramètres système → Affichage → Couleur nocturne (KDE)
```

## Accessibilité auditive

### Alertes visuelles

Pour les personnes sourdes ou malentendantes, les alertes sonores du système peuvent être remplacées ou complétées par des alertes visuelles :

```bash
# GNOME : activer les alertes visuelles
gsettings set org.gnome.desktop.wm.preferences visual-bell true  
gsettings set org.gnome.desktop.wm.preferences visual-bell-type 'frame-flash'  
# Types : frame-flash (la barre de titre clignote) ou fullscreen-flash

# Paramètres → Accessibilité → Audition → Alertes visuelles

# KDE
# Paramètres système → Accessibilité → Cloche
# Utiliser une notification visuelle au lieu d'une cloche sonore
```

### Sous-titres et transcription

```bash
# Les sous-titres dans les lecteurs vidéo sont gérés par chaque application
# VLC : Sous-titres → Piste de sous-titres
# mpv : touche 'v' pour activer/désactiver les sous-titres

# Les visioconférences modernes (Zoom, Teams, Meet) proposent
# des sous-titres automatiques dans le navigateur

# Transcription en temps réel
# Plusieurs solutions tierces existent (aucune n'est intégrée nativement à GNOME) :
#  - LiveCaptions (https://github.com/abb128/LiveCaptions) : application
#    Linux qui sous-titre tout flux audio, fonctionne hors-ligne.
#  - Extensions GNOME Shell « Blurt » et « Speech2Text » : transcription locale
#    via whisper.cpp, accessible depuis n'importe quel champ texte.
#  - GNOME 46 a néanmoins modernisé Orca (perfs, compatibilité Wayland et apps
#    sandboxées) — sans pour autant ajouter la transcription en temps réel.
```

### Contrôle du volume par application

PipeWire et `pavucontrol` permettent de contrôler le volume de chaque application indépendamment, ce qui est utile pour les personnes qui doivent amplifier certaines sources sonores :

```bash
sudo apt install pavucontrol  
pavucontrol &  
# Onglet Lecture : volume individuel par application
# Onglet Périphériques d'entrée : gain du microphone
```

## Accessibilité motrice

### Touches rémanentes (Sticky Keys)

Les touches rémanentes permettent d'effectuer des combinaisons de touches (Ctrl+C, Alt+Tab) en appuyant séquentiellement sur chaque touche au lieu de les maintenir simultanément. C'est essentiel pour les personnes qui ne peuvent utiliser qu'un seul doigt ou un dispositif de pointage :

```bash
# GNOME
gsettings set org.gnome.desktop.a11y.keyboard stickykeys-enable true
# Comportement : appuyer sur Ctrl, relâcher, puis appuyer sur C = Ctrl+C

# Activer avec un raccourci physique : presser Shift 5 fois de suite
gsettings set org.gnome.desktop.a11y.keyboard stickykeys-two-key-off true
# Désactivation automatique si deux touches sont pressées simultanément

# Via Paramètres → Accessibilité → Frappe → Touches rémanentes

# KDE
# Paramètres système → Accessibilité → Touche modificatrice rémanente

# XFCE
# Paramètres → Accessibilité → Touches rémanentes
```

### Touches lentes (Slow Keys)

Les touches lentes exigent qu'une touche soit maintenue pendant un certain temps avant d'être prise en compte. Cela empêche les frappes accidentelles pour les personnes ayant des tremblements ou un contrôle moteur réduit :

```bash
# GNOME
gsettings set org.gnome.desktop.a11y.keyboard slowkeys-enable true  
gsettings set org.gnome.desktop.a11y.keyboard slowkeys-delay 300  
# Délai en millisecondes (300 ms = la touche doit être maintenue 0,3 seconde)

# Via Paramètres → Accessibilité → Frappe → Touches lentes

# KDE : Paramètres système → Accessibilité → Filtre clavier
```

### Touches filtres (Bounce Keys)

Les touches filtres ignorent les pressions rapides et répétées involontaires sur la même touche :

```bash
# GNOME
gsettings set org.gnome.desktop.a11y.keyboard bouncekeys-enable true  
gsettings set org.gnome.desktop.a11y.keyboard bouncekeys-delay 300  
# Délai minimum entre deux pressions de la même touche

# Via Paramètres → Accessibilité → Frappe → Touches filtres
```

### Clavier virtuel (clavier à l'écran)

Le clavier virtuel permet la saisie de texte en cliquant sur des touches affichées à l'écran, utile pour les personnes utilisant un dispositif de pointage alternatif (trackball, joystick, eye-tracker, contacteur) :

```bash
# GNOME : clavier à l'écran intégré
# Il apparaît automatiquement lorsqu'un champ de texte reçoit le focus
# sur un appareil tactile ou lorsque l'accessibilité est activée

# Activer le clavier à l'écran
gsettings set org.gnome.desktop.a11y.applications screen-keyboard-enabled true

# Via Paramètres → Accessibilité → Frappe → Clavier à l'écran

# Alternatives
# Onboard (clavier à l'écran indépendant et configurable)
sudo apt install onboard  
onboard &  
# Onboard offre plus d'options de personnalisation :
# - Disposition configurable
# - Prédiction de mots
# - Thèmes
# - Balayage automatique (scanning) pour les contacteurs
```

### Contrôle du pointeur au clavier (MouseKeys)

MouseKeys permet de contrôler le curseur de la souris avec le pavé numérique du clavier :

```bash
# GNOME
gsettings set org.gnome.desktop.a11y.keyboard mousekeys-enable true

# Via Paramètres → Accessibilité → Pointage et clic → Touches de la souris

# Fonctionnement avec le pavé numérique :
# 4, 6, 8, 2         → Déplacer le curseur (gauche, droite, haut, bas)
# 7, 9, 1, 3         → Déplacer en diagonale
# 5                   → Clic gauche
# + (plus)            → Double-clic
# 0                   → Maintenir le bouton (début de glisser)
# . (point)           → Relâcher le bouton (fin de glisser)
# / (diviser)         → Sélectionner bouton gauche
# * (multiplier)      → Sélectionner bouton du milieu
# - (soustraire)      → Sélectionner bouton droit

# KDE : Paramètres système → Accessibilité → Navigation de la souris
```

### Clic automatique (Dwell Click)

Le clic automatique déclenche un clic lorsque le curseur reste immobile pendant un certain temps. C'est utile pour les personnes qui peuvent déplacer le pointeur mais pas cliquer :

```bash
# GNOME
gsettings set org.gnome.desktop.a11y.mouse dwell-click-enabled true  
gsettings set org.gnome.desktop.a11y.mouse dwell-time 1.2  
# Durée d'immobilité en secondes avant le clic automatique

# Choix du type de clic (affiché dans un panneau en haut de l'écran)
gsettings set org.gnome.desktop.a11y.mouse dwell-threshold 10
# Seuil de mouvement en pixels (en dessous, le curseur est considéré immobile)

# Via Paramètres → Accessibilité → Pointage et clic → Délai du clic
```

### Technologies d'assistance spécialisées

Pour les personnes à mobilité très réduite, des dispositifs d'entrée alternatifs existent et sont supportés sous Linux :

```bash
# Contrôle oculaire (eye tracking)
# Certains eye-trackers (Tobii) ont un support expérimental sous Linux
# Le projet OpenGazer et les pilotes communautaires offrent un support partiel

# Contacteurs (switch access)
# Le clavier virtuel Onboard supporte le mode balayage (scanning)
# Un contacteur unique permet de naviguer dans les options du clavier

# Reconnaissance vocale
# Le projet KDE Simon n'est plus packagé dans Debian Trixie.
# Alternatives actuelles :
#  - Numen : contrôle vocal pour Linux (commandes vocales pour la souris/clavier)
#    https://git.sr.ht/~geb/numen
#  - Talon Voice : contrôle vocal avancé tiers, multiplateforme (gratuit)
#    https://talonvoice.com/
#  - whisper.cpp / faster-whisper : moteur Whisper (OpenAI) en local
#    (dictée et transcription, voir extensions GNOME Shell Blurt/Speech2Text)
#  - pocketsphinx : moteur historique de CMU Sphinx
sudo apt install pocketsphinx

# Contrôle par mouvement de tête
# eViacam : contrôle du curseur par webcam et mouvements de tête
sudo apt install eviacam  
eviacam &  
# Utilise la webcam pour suivre les mouvements de la tête
# et les convertir en mouvements du curseur
```

## Accessibilité cognitive

### Simplification de l'interface

Pour les personnes ayant des troubles cognitifs ou attentionnels, un environnement simplifié réduit la surcharge d'informations :

```bash
# Réduire les animations (moins de distractions visuelles)
gsettings set org.gnome.desktop.interface enable-animations false

# Désactiver les notifications non essentielles
# GNOME : Paramètres → Notifications → désactiver par application

# Mode « Ne pas déranger »
gsettings set org.gnome.desktop.notifications show-banners false

# Utiliser des espaces de travail pour séparer les contextes
# (un espace par tâche, réduire le multitâche visuel)

# Réduire le nombre d'applications dans les menus
# Masquer les applications rarement utilisées via les fichiers .desktop
# (voir section 2.2 pour la technique NoDisplay=true)
```

### Polices adaptées à la dyslexie

Certaines polices de caractères sont conçues pour améliorer la lisibilité pour les personnes dyslexiques :

```bash
# OpenDyslexic : police conçue pour la dyslexie
# Les lettres ont des bases plus lourdes pour réduire la confusion
sudo apt install fonts-opendyslexic

# Appliquer comme police système
# GNOME
gsettings set org.gnome.desktop.interface font-name 'OpenDyslexic 12'

# D'autres polices sans empattement (Noto Sans, Verdana, Arial)
# sont également recommandées pour la lisibilité
```

## Afficheur braille

Pour les personnes aveugles utilisant un afficheur braille (plage tactile braille), le système **BRLTTY** assure l'interface entre le matériel et les applications :

```bash
# Installation
sudo apt install brltty

# BRLTTY détecte automatiquement de nombreux afficheurs braille USB
# La configuration se fait dans /etc/brltty.conf

# Vérifier la détection
brltty -V       # Version  
brltty -v       # Mode verbose pour le diagnostic  

# BRLTTY s'intègre avec Orca pour fournir une sortie braille
# synchronisée avec la synthèse vocale
# Configuration dans Orca → Préférences → Braille

# Afficheurs supportés : Baum, Freedom Scientific, Handy Tech,
# HIMS, Humanware, Papenmeier, et de nombreux autres
# Liste complète : http://brltty.app/doc/Drivers.html
```

## Tester l'accessibilité

### Accessi (inspecteur d'accessibilité)

Pour les développeurs ou les administrateurs qui souhaitent vérifier l'accessibilité d'une application :

```bash
# Accerciser : inspecteur d'accessibilité AT-SPI
sudo apt install accerciser  
accerciser &  

# Accerciser permet de :
# - Visualiser l'arbre d'accessibilité de chaque application
# - Vérifier que les éléments ont un nom, un rôle et un état corrects
# - Simuler les interactions d'un lecteur d'écran
# - Identifier les éléments non accessibles
```

### Tester avec un lecteur d'écran

La méthode la plus fiable pour évaluer l'accessibilité d'un poste de travail est de l'utiliser avec un lecteur d'écran pendant quelques minutes :

```bash
# Activer Orca
# Super + Alt + S

# Essayer les actions courantes :
# 1. Naviguer dans les menus de l'environnement de bureau
# 2. Ouvrir et naviguer dans le gestionnaire de fichiers
# 3. Naviguer sur une page web dans Firefox
# 4. Rédiger un texte dans un éditeur
# 5. Utiliser les paramètres système

# Chaque élément doit être annoncé de manière compréhensible :
# nom de l'élément, type (bouton, lien, champ, etc.), état (actif, coché, etc.)
```

## Résumé

L'accessibilité sous Debian est un domaine mature, particulièrement dans l'environnement GNOME qui offre la couverture la plus complète : lecteur d'écran Orca, zoom d'écran, contraste élevé, clavier à l'écran, touches rémanentes, clic automatique, support braille et accessibilité dès l'écran de connexion. KDE Plasma et XFCE offrent un support solide pour les fonctionnalités les plus courantes (touches rémanentes, zoom, contraste), mais ne rivalisent pas avec GNOME pour les besoins avancés (lecteur d'écran, braille, clic automatique).

Pour un poste de travail destiné à une personne en situation de handicap, GNOME est le choix recommandé en raison de son investissement historique et continu dans l'accessibilité. L'installation des outils complémentaires (Onboard, eViacam, BRLTTY, polices OpenDyslexic) permet de couvrir des besoins spécifiques que les paramètres intégrés ne couvrent pas.
