# 2.5.1 Suite LibreOffice

## Introduction

LibreOffice est la suite bureautique par défaut de Debian et l'une des suites bureautiques libres les plus complètes et les plus matures. Issue d'un fork d'OpenOffice.org en 2010, elle est développée par The Document Foundation et bénéficie d'une communauté active de contributeurs et d'entreprises sponsors. LibreOffice est installée par défaut avec toutes les tâches desktop Debian et constitue l'outil de productivité bureautique central du système.

Cette section couvre l'installation et la configuration de LibreOffice sous Debian, la présentation de chaque composant, la compatibilité avec Microsoft Office, les fonctionnalités avancées (macros, modèles, publipostage) et les astuces pour une utilisation quotidienne efficace.

## Installation

### Paquets et composants

LibreOffice est découpée en plusieurs paquets correspondant à chaque composant, ce qui permet une installation modulaire :

```bash
# Installation complète (normalement déjà présente via task-*-desktop)
sudo apt install libreoffice

# Installation par composant
sudo apt install libreoffice-writer    # Traitement de texte  
sudo apt install libreoffice-calc      # Tableur  
sudo apt install libreoffice-impress   # Présentations  
sudo apt install libreoffice-draw      # Dessin vectoriel et édition PDF  
sudo apt install libreoffice-base      # Base de données relationnelle  
sudo apt install libreoffice-math      # Éditeur de formules mathématiques  

# Vérifier la version installée
libreoffice --version
```

### Packs de langue et correcteurs

```bash
# Interface en français
sudo apt install libreoffice-l10n-fr

# Aide intégrée en français
sudo apt install libreoffice-help-fr

# Dictionnaire orthographique français
sudo apt install hunspell-fr-comprehensive
# ou la version standard
sudo apt install hunspell-fr

# Thésaurus (synonymes)
sudo apt install mythes-fr

# Coupure de mots (césure)
sudo apt install hyphen-fr

# Correcteur grammatical (Grammalecte)
# Grammalecte est une extension LibreOffice, non disponible dans les dépôts
# Installation depuis le site : https://grammalecte.net
# Outils → Gestionnaire des extensions → Ajouter → grammalecte.oxt

# Vérifier les dictionnaires installés
# Outils → Options → Paramètres linguistiques → Langues
# La colonne de droite liste les dictionnaires disponibles par langue

# Installer des langues supplémentaires
sudo apt install libreoffice-l10n-en-gb hunspell-en-gb    # Anglais britannique  
sudo apt install libreoffice-l10n-de hunspell-de-de       # Allemand  
sudo apt install libreoffice-l10n-es hunspell-es           # Espagnol  
```

### Intégration au bureau

LibreOffice s'intègre visuellement à l'environnement de bureau via des paquets d'intégration spécifiques :

```bash
# Intégration GNOME (dialogues d'ouverture/sauvegarde natifs, thème GTK)
sudo apt install libreoffice-gnome

# Intégration KDE Plasma 6 (Debian 13)
sudo apt install libreoffice-kf6      # Dialogues natifs Plasma (KDE Frameworks 6)  
sudo apt install libreoffice-qt6      # Backend de rendu VCL en Qt 6  

# Activer explicitement le backend Qt 6 si la détection automatique ne fonctionne pas
echo 'export SAL_USE_VCLPLUGIN=qt6' >> ~/.profile
# Valeurs possibles : qt6 (Plasma 6), gtk3 (GNOME/XFCE), gen (X11 minimal)
# Note : sous Bookworm/Plasma 5, on utilisait `libreoffice-kf5` + `libreoffice-qt5`
# avec `SAL_USE_VCLPLUGIN=qt5`.

# Intégration pour les icônes de thème
# LibreOffice propose plusieurs jeux d'icônes
# Outils → Options → LibreOffice → Affichage → Taille et style des icônes
# Styles disponibles : Breeze, Colibre, Elementary, Karasa Jaga, Sifr, Sukapura
# Le style est normalement détecté automatiquement selon l'environnement
```

### Obtenir une version plus récente

La version de LibreOffice dans Debian Stable peut avoir un ou deux cycles de retard par rapport à la dernière version upstream. Pour obtenir une version plus récente :

```bash
# Option 1 : Backports Debian (si disponible)
sudo apt install -t trixie-backports libreoffice

# Option 2 : Flatpak (dernière version stable)
flatpak install flathub org.libreoffice.LibreOffice

# Option 3 : PPA/dépôt tiers (non recommandé pour Debian)
# LibreOffice fournit ses propres paquets .deb sur son site
# mais ils ne sont pas intégrés au système de dépendances Debian

# La version Flatpak cohabite avec la version système
# Les deux utilisent des profils de configuration séparés
```

## Présentation des composants

### Writer : traitement de texte

Writer est l'équivalent de Microsoft Word. Il gère la rédaction de documents structurés, de la simple lettre au rapport technique de plusieurs centaines de pages.

**Fonctionnalités clés** :

- **Styles** : Writer repose sur un système de styles (paragraphe, caractère, page, cadre, liste) qui est la clé d'un usage productif. Les styles permettent de séparer le contenu de la mise en forme, d'assurer la cohérence visuelle et de modifier l'apparence d'un document entier en changeant un seul style.
- **Table des matières automatique** : générée à partir des styles de titre (Titre 1, Titre 2, etc.), mise à jour en un clic.
- **Suivi des modifications** : enregistrement et visualisation des modifications, avec acceptation ou rejet individuel ou global. Compatible avec le suivi des modifications de Microsoft Word.
- **Commentaires et annotations** : insertion de commentaires dans la marge, avec support des réponses et des fils de discussion.
- **Publipostage** : fusion de documents avec une source de données (base de données, tableur, carnet d'adresses) pour générer des courriers personnalisés en série.
- **Modèles** : documents pré-formatés réutilisables (lettres, rapports, factures, CV).
- **Export PDF** : export natif en PDF avec contrôle sur la qualité, la sécurité (mot de passe, restrictions), les signets et la structure du document.
- **Bibliographie** : gestion des références bibliographiques avec insertion de citations et génération de la bibliographie.
- **Formulaires** : création de formulaires remplissables (champs de texte, cases à cocher, listes déroulantes).

### Calc : tableur

Calc est l'équivalent de Microsoft Excel. Il gère les feuilles de calcul, de la simple liste au tableau de bord analytique complexe.

**Fonctionnalités clés** :

- **Formules et fonctions** : plus de 500 fonctions intégrées (mathématiques, statistiques, financières, logiques, texte, date, recherche). La syntaxe est compatible avec Excel pour la majorité des formules.
- **Tableaux croisés dynamiques** : appelés « DataPilot » dans LibreOffice, ils permettent de résumer et d'analyser de grands ensembles de données interactivement.
- **Graphiques** : création de graphiques intégrés (barres, lignes, secteurs, nuages de points, etc.) avec personnalisation complète.
- **Formatage conditionnel** : mise en forme automatique des cellules selon leur valeur (barres de données, échelles de couleurs, jeux d'icônes, règles personnalisées).
- **Validation des données** : restriction des valeurs saisissables dans une cellule (liste déroulante, plage de valeurs, expression régulière).
- **Macro et automatisation** : langage de macros LibreOffice Basic (et support partiel de VBA) pour automatiser les tâches répétitives.
- **Connexion à des sources de données** : importation depuis des bases de données, des fichiers CSV, des flux web.
- **Capacité** : jusqu'à 1 048 576 lignes × 16 384 colonnes par feuille (identique à Excel).

### Impress : présentations

Impress est l'équivalent de Microsoft PowerPoint. Il permet de créer des présentations avec des diapositives contenant du texte, des images, des graphiques, des vidéos et des animations.

**Fonctionnalités clés** :

- **Modèles de présentation** : thèmes visuels prédéfinis et personnalisables.
- **Animations et transitions** : effets d'entrée, de sortie et d'accentuation sur les objets, transitions entre diapositives.
- **Mode présentateur** : affichage des notes et du minuteur sur l'écran du présentateur, avec la diapositive projetée sur l'écran externe.
- **Export PDF et vidéo** : export des diapositives en PDF ou en fichier vidéo (nécessite un encodeur vidéo).
- **Télécommande** : l'application Impress Remote (Android/iOS) permet de contrôler la présentation depuis un téléphone.

### Draw : dessin vectoriel

Draw est un outil de dessin vectoriel polyvalent qui couvre un spectre large d'usages : schémas, organigrammes, plans, posters, et surtout édition de fichiers PDF.

**Fonctionnalités clés** :

- **Édition de PDF** : Draw peut ouvrir un fichier PDF et permet de modifier le texte, les images et les objets vectoriels. C'est l'un des rares outils libres offrant cette fonctionnalité.
- **Formes et connecteurs** : bibliothèque de formes géométriques, flèches, connecteurs pour les organigrammes et les diagrammes.
- **Calques** : gestion par calques pour les dessins complexes.
- **Export multi-format** : SVG, PNG, PDF, EPS, etc.

### Base : base de données

Base est l'équivalent de Microsoft Access. Il offre une interface graphique pour créer et gérer des bases de données relationnelles, avec des formulaires, des requêtes et des rapports.

```bash
# Base nécessite un moteur de base de données Java
sudo apt install libreoffice-base default-jre

# Moteurs supportés : HSQLDB (embarqué, par défaut), PostgreSQL,
# MySQL/MariaDB, SQLite, ODBC, JDBC
```

### Math : éditeur de formules

Math est un éditeur de formules mathématiques qui s'intègre dans les autres composants LibreOffice (Writer, Calc, Impress). Il utilise un langage de balisage textuel pour décrire les formules :

```
# Exemples de formules Math
# Fraction : a over b
# Racine carrée : sqrt{x}
# Intégrale : int from{a} to{b} f(x) dx
# Somme : sum from{i=1} to{n} x_i
# Matrice : matrix{a # b ## c # d}
```

## Configuration globale

### Paramètres essentiels

La configuration de LibreOffice se fait via **Outils → Options** (ou `Alt+F12`). Les paramètres les plus importants :

#### Enregistrement et formats par défaut

```
Outils → Options → Charger/Enregistrer → Général

# Format d'enregistrement par défaut
# Pour maximiser la compatibilité Microsoft Office :
# Document texte : Format de document Microsoft Word 2007-365 (.docx)
# Feuille de calcul : Format Microsoft Excel 2007-365 (.xlsx)
# Présentation : Format Microsoft PowerPoint 2007-365 (.pptx)

# Pour rester en format libre (recommandé si l'écosystème le permet) :
# Document texte : Format de document ODF (.odt)
# Feuille de calcul : Classeur ODF (.ods)
# Présentation : Présentation ODF (.odp)

# Sauvegarde automatique
# Outils → Options → Charger/Enregistrer → Général
# Activer « Enregistrer les informations de récupération automatique toutes les X minutes »
# Recommandation : 5 à 10 minutes
# Activer « Toujours créer une copie de sauvegarde »
```

#### Mémoire et performances

```
Outils → Options → LibreOffice → Mémoire

# Utilisation de la mémoire pour LibreOffice
# Nombre d'étapes d'annulation : 100 (défaut, réduire à 30-50 si la RAM est limitée)
# Cache pour les images : 128 Mo (augmenter à 256 Mo si beaucoup d'images sont utilisées)

# Activer l'utilisation du GPU pour le rendu
Outils → Options → LibreOffice → Affichage
# Cocher « Utiliser le rendu matériel » (activé par défaut)
# Cocher « Utiliser l'anticrénelage » pour un texte plus lisse

# Activer le démarrage rapide (quickstarter)
# Outils → Options → LibreOffice → Mémoire
# Cocher « Activer systray Quickstarter »
# LibreOffice reste partiellement chargé en mémoire pour un démarrage plus rapide
```

#### Chemins et modèles

```bash
# Les modèles utilisateur sont stockés dans
~/.config/libreoffice/4/user/template/

# Les extensions utilisateur sont stockées dans
~/.config/libreoffice/4/user/extensions/

# Le profil complet de LibreOffice
~/.config/libreoffice/4/user/

# Réinitialiser le profil en cas de problème
# (renommer le répertoire, LibreOffice en créera un nouveau au démarrage)
mv ~/.config/libreoffice/4/user ~/.config/libreoffice/4/user.backup
```

## Compatibilité Microsoft Office

### Polices de compatibilité

L'installation des polices de compatibilité Microsoft est le facteur le plus important pour une bonne fidélité de rendu (voir section 2.2.3 pour les détails) :

```bash
# Polices Microsoft de base
sudo apt install ttf-mscorefonts-installer

# Polices métriquement compatibles (Calibri → Carlito, Cambria → Caladea)
sudo apt install fonts-crosextra-carlito fonts-crosextra-caladea

# Polices Liberation (compatibles avec Arial, Times, Courier)
sudo apt install fonts-liberation2

# Rafraîchir le cache
fc-cache -fv
```

### Configuration pour la compatibilité maximale

```
# Utiliser les formats Microsoft par défaut
Outils → Options → Charger/Enregistrer → Général
# Définir les formats Microsoft comme défaut pour chaque type de document

# Compatibilité VBA
Outils → Options → Charger/Enregistrer → Propriétés VBA
# Cocher « Charger le code Basic pour édition »
# Cocher « Enregistrer le code Basic d'origine lors de l'enregistrement du document »
# Cela préserve les macros VBA lors de l'ouverture et de la sauvegarde

# Compatibilité de mise en page avec Word
Outils → Options → LibreOffice Writer → Compatibilité
# Cocher « Utiliser les métriques d'impression pour le formatage du document »
# Cela rapproche le rendu de celui de Word
```

### Test de compatibilité

Lorsqu'un document doit être envoyé à un destinataire sous Microsoft Office et que la fidélité visuelle est critique :

```bash
# Méthode 1 : exporter en PDF (fidélité visuelle garantie)
# Fichier → Exporter en PDF

# Méthode 2 : convertir et vérifier en ligne de commande
libreoffice --headless --convert-to docx document.odt
# Puis vérifier le résultat dans LibreOffice ou sur un poste Windows

# Méthode 3 : utiliser Microsoft 365 en ligne pour vérifier le rendu
# Téléverser le document sur OneDrive et l'ouvrir dans Word Online
```

## Modèles et styles

### Utilisation des styles

Les styles sont la fonctionnalité la plus puissante et la plus sous-utilisée de LibreOffice. Un document bien structuré avec des styles permet de modifier l'apparence de l'ensemble du document en quelques clics, de générer automatiquement la table des matières, et d'assurer une cohérence visuelle parfaite.

```
# Panneau des styles
# F11 ou Affichage → Styles
# Catégories : Styles de paragraphe, Styles de caractère, Styles de cadre,
#              Styles de page, Styles de liste

# Appliquer un style de paragraphe
# Sélectionner le texte, double-cliquer sur le style dans le panneau F11
# ou utiliser la liste déroulante des styles dans la barre d'outils

# Créer un style personnalisé
# Formater un paragraphe comme souhaité, puis dans le panneau F11 :
# Clic droit → Nouveau style → Nommer le style

# Styles essentiels pour un document structuré :
# - Titre 1, Titre 2, Titre 3 (hiérarchie du document)
# - Corps de texte (texte courant)
# - Citation (bloc de texte en retrait)
# - Code (texte en police à chasse fixe)
# - Légende (sous les figures et tableaux)
```

### Modèles de documents

Un modèle (template) est un document pré-formaté qui sert de base pour de nouveaux documents. Il contient les styles, la mise en page, les éléments graphiques récurrents (en-têtes, logos) et éventuellement du texte de remplacement.

```bash
# Emplacement des modèles utilisateur
~/.config/libreoffice/4/user/template/

# Créer un modèle
# 1. Créer un document avec la mise en page souhaitée
# 2. Fichier → Modèles → Enregistrer comme modèle
# 3. Nommer le modèle et choisir une catégorie

# Utiliser un modèle
# Fichier → Nouveau → Modèles
# Sélectionner le modèle dans la galerie

# Installer des modèles depuis le web
# Le site LibreOffice Extensions & Templates propose des modèles :
# https://extensions.libreoffice.org/en/templates

# Définir un modèle par défaut
# Fichier → Modèles → Gérer les modèles
# Clic droit sur un modèle → Définir comme modèle par défaut
```

## Fonctionnalités avancées

### Publipostage (mailing)

Le publipostage permet de générer des documents personnalisés en masse (lettres, étiquettes, enveloppes) à partir d'un document modèle et d'une source de données :

```
# Source de données
# Calc (fichier .ods ou .xlsx), Base, CSV, ou carnet d'adresses

# Procédure
# 1. Préparer la source de données (un tableur avec les colonnes Nom, Prénom, Adresse, etc.)
# 2. Dans Writer : Outils → Assistant Publipostage
# 3. Suivre l'assistant :
#    - Sélectionner le document de base
#    - Sélectionner la source de données
#    - Insérer les champs de fusion dans le document
#    - Prévisualiser le résultat
#    - Générer les documents (impression, e-mail ou fichiers individuels)

# Insertion manuelle de champs
# Insertion → Champ → Plus de champs → Base de données
# Sélectionner le champ dans la source de données enregistrée
```

### Macros et automatisation

LibreOffice dispose de plusieurs langages de macro pour l'automatisation :

```
# LibreOffice Basic (langage par défaut, similaire à VBA)
# Outils → Macros → Organiser les macros → LibreOffice Basic

# Exemple : macro qui met en gras tout le texte sélectionné
Sub MetEnGras()
    Dim oDoc As Object
    Dim oSel As Object
    oDoc = ThisComponent
    oSel = oDoc.getCurrentSelection()
    oSel.setPropertyValue("CharWeight", com.sun.star.awt.FontWeight.BOLD)
End Sub

# Python (langage alternatif, plus puissant)
# Les macros Python sont stockées dans
# ~/.config/libreoffice/4/user/Scripts/python/

# Exemple de macro Python
def hello_world():
    desktop = XSCRIPTCONTEXT.getDesktop()
    doc = desktop.getCurrentComponent()
    text = doc.getText()
    cursor = text.createTextCursor()
    text.insertString(cursor, "Hello World!", False)

# Enregistrement de macros
# Outils → Macros → Enregistrer une macro
# Effectuer les actions souhaitées, arrêter l'enregistrement
# La macro est enregistrée en LibreOffice Basic
```

### Formulaires remplissables

Writer permet de créer des formulaires PDF remplissables :

```
# 1. Activer la barre d'outils Contrôles de formulaire
# Affichage → Barres d'outils → Contrôles de formulaire

# 2. Insérer des champs de formulaire
# Champ de texte, case à cocher, bouton radio, liste déroulante, etc.

# 3. Protéger le document
# Format → Sections → protéger la section pour empêcher la modification
# du texte tout en permettant la saisie dans les champs de formulaire

# 4. Exporter en PDF
# Fichier → Exporter en PDF
# Cocher « Créer un formulaire PDF »
# Format de soumission : FDF ou PDF (selon le cas d'usage)
```

## Utilisation en ligne de commande

LibreOffice dispose d'un mode « headless » (sans interface graphique) qui permet la conversion de fichiers et le traitement par lots depuis le terminal ou dans des scripts :

```bash
# Conversion de format
libreoffice --headless --convert-to pdf document.docx  
libreoffice --headless --convert-to docx document.odt  
libreoffice --headless --convert-to xlsx tableau.ods  
libreoffice --headless --convert-to png --outdir images/ dessin.odg  

# Conversion par lot
libreoffice --headless --convert-to pdf *.docx  
libreoffice --headless --convert-to pdf --outdir /sortie/ /dossier/*.odt  

# « Impression » fichier (génère un .ps utilisant les paramètres d'un pilote)
libreoffice --headless --print-to-file --printer-name mon_imprimante \
  --outdir /tmp document.odt
# Note : `--print-to-file` ne lance PAS un travail d'impression réel — il
# convertit le document en PostScript brut en utilisant la configuration du
# pilote `mon_imprimante`. Pour une impression réelle vers une imprimante CUPS :
libreoffice --headless --pt mon_imprimante document.odt
# ou via la chaîne classique : convertir en PDF puis envoyer à `lp`
libreoffice --headless --convert-to pdf document.odt  
lp -d mon_imprimante document.pdf  

# Exécuter une macro depuis la ligne de commande
libreoffice --headless --invisible \
  "macro:///Standard.Module1.MaMacro" document.ods

# Ouvrir un document en lecture seule
libreoffice --view document.odt

# Ouvrir un composant spécifique
libreoffice --writer       # Ouvrir Writer vide  
libreoffice --calc         # Ouvrir Calc vide  
libreoffice --impress      # Ouvrir Impress vide  
```

## Extensions

Le système d'extensions de LibreOffice permet d'ajouter des fonctionnalités. Les extensions sont des fichiers `.oxt` installables via **Outils → Gestionnaire des extensions**.

### Extensions recommandées

```bash
# Grammalecte : correcteur grammatical français avancé
# Télécharger depuis https://grammalecte.net
# Installation : Outils → Gestionnaire des extensions → Ajouter

# LanguageTool : correcteur grammatical multilingue
# Télécharger depuis https://extensions.libreoffice.org
# Supporte le français, l'anglais, l'allemand et de nombreuses autres langues

# TexMaths : insertion d'équations LaTeX dans Writer/Impress
# Pour les utilisateurs familiers avec la syntaxe LaTeX

# MultiFormatSave : sauvegarder simultanément en ODF et en format Microsoft
# Utile pour maintenir les deux formats en parallèle

# Zotero Connector : intégration du gestionnaire de références bibliographiques
# Installé automatiquement par Zotero. Note : `zotero` n'est PAS packagé
# dans Debian Trixie (`apt install zotero` retourne « Unable to locate
# package »). L'installation se fait via Flatpak ou via le tarball
# officiel téléchargeable depuis zotero.org :
flatpak install flathub org.zotero.Zotero
```

### Gestion des extensions en ligne de commande

```bash
# Lister les extensions installées
libreoffice --headless --list-extensions

# Installer une extension
unopkg add extension.oxt

# Désinstaller une extension
unopkg remove identifiant.extension
```

## Dépannage

### LibreOffice est lent au démarrage

```bash
# 1. Désactiver le démarrage rapide de Java si non nécessaire
# Outils → Options → LibreOffice → Avancé
# Décocher « Utiliser un environnement d'exécution Java »
# (Base nécessite Java, les autres composants non)

# 2. Réduire le nombre d'extensions
# Outils → Gestionnaire des extensions
# Désactiver les extensions non utilisées

# 3. Réinitialiser le profil utilisateur (corrige la plupart des lenteurs)
mv ~/.config/libreoffice/4/user ~/.config/libreoffice/4/user.old
# LibreOffice créera un nouveau profil au prochain démarrage

# 4. Désactiver l'assistant de démarrage
# Outils → Options → LibreOffice → Général
# Décocher « Afficher le dialogue de démarrage »
```

### Problèmes de mise en page avec des documents Microsoft Office

```bash
# 1. Vérifier les polices
fc-match "Calibri"
# Si la police n'est pas trouvée, installer les polices de compatibilité

# 2. Forcer les métriques d'impression
# Outils → Options → LibreOffice Writer → Compatibilité
# Cocher « Utiliser les métriques d'impression pour le formatage du document »

# 3. Pour les documents critiques, comparer avec Microsoft 365 en ligne
```

## Résumé

LibreOffice est une suite bureautique complète et mature qui couvre la totalité des besoins de bureautique quotidiens sous Debian. Sa compatibilité avec les formats Microsoft Office est bonne pour les documents courants, et les polices de compatibilité et les paramètres de configuration permettent de l'améliorer significativement. L'utilisation des styles, des modèles et des macros transforme LibreOffice d'un simple outil de saisie en un véritable instrument de productivité. Le mode headless en fait également un outil de conversion et de traitement par lots puissant dans les scripts d'automatisation.

La section suivante (2.5.2) aborde les navigateurs web et leurs extensions, l'application la plus utilisée au quotidien sur un poste de travail moderne.
