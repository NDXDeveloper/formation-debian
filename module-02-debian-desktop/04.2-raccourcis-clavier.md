# 2.4.2 Raccourcis clavier

## Introduction

Les raccourcis clavier sont le levier de productivité le plus sous-estimé et le plus rentable d'un poste de travail. Un raccourci maîtrisé élimine un trajet souris, un menu contextuel, un glisser-déposer — autant de micro-interruptions qui, cumulées sur une journée de travail, représentent un temps significatif. Au-delà du gain de temps, les raccourcis réduisent la charge cognitive en rendant les actions fréquentes automatiques et instinctives.

Sous Linux, les raccourcis clavier sont gérés à plusieurs niveaux : le serveur d'affichage (Xorg ou Wayland), le gestionnaire de fenêtres (Mutter, KWin, Xfwm4), l'environnement de bureau (GNOME Shell, Plasma Shell) et chaque application individuellement. Cette architecture en couches offre une grande flexibilité mais nécessite de comprendre quel composant gère quel raccourci pour éviter les conflits.

Cette section présente les raccourcis essentiels de chaque environnement, les méthodes de personnalisation, la création de raccourcis personnalisés pour lancer des commandes et des scripts, et les bonnes pratiques pour construire un système de raccourcis efficace.

## Architecture des raccourcis clavier sous Linux

### Les couches de gestion des raccourcis

Lorsqu'une touche est pressée, l'événement traverse plusieurs couches. Chaque couche peut capturer l'événement et empêcher sa propagation aux couches suivantes :

```
Frappe clavier
      │
      ▼
┌─────────────────────────────────────┐
│  1. Raccourcis globaux du DE        │  ← Super, Alt+Tab, Alt+F2, etc.
│     (gestionnaire de fenêtres +     │     Captés avant tout le reste
│      shell graphique)               │
├─────────────────────────────────────┤
│  2. Raccourcis système              │  ← Ctrl+Alt+Suppr, Impr. écran
│     (systemd-logind, acpid)         │     Actions système (verrouillage,
│                                     │     extinction, capture d'écran)
├─────────────────────────────────────┤
│  3. Raccourcis d'application        │  ← Ctrl+S, Ctrl+C, Ctrl+V, etc.
│     (l'application au premier plan) │     Gérés par chaque application
└─────────────────────────────────────┘
```

Un raccourci global (couche 1) est toujours prioritaire sur un raccourci d'application (couche 3). C'est pourquoi un raccourci comme `Super` ouvre toujours la vue d'ensemble de GNOME, même si une application utilise aussi cette touche.

### Touches modificatrices

| Touche | Nom XKB | Symbole courant | Rôle habituel |
|--------|---------|-----------------|---------------|
| **Ctrl** (Control) | Control_L / Control_R | Ctrl | Raccourcis d'application (Ctrl+S, Ctrl+C, etc.) |
| **Alt** | Alt_L | Alt | Raccourcis d'application et de fenêtre (Alt+Tab, Alt+F4) |
| **Super** (touche Windows/Commande) | Super_L | ⊞ / ◆ | Raccourcis du bureau (lanceur, espaces de travail) |
| **Shift** (Majuscule) | Shift_L / Shift_R | ⇧ | Modificateur secondaire (extension des raccourcis) |
| **Alt Gr** | ISO_Level3_Shift | Alt Gr | Caractères spéciaux du clavier (€, @, #, etc.) |
| **Caps Lock** | Caps_Lock | ⇪ | Verrouillage majuscules (souvent réaffecté) |

### Conventions de notation

Dans cette section et dans la documentation Linux en général, les raccourcis sont notés avec les modificateurs séparés par `+` :

- `Ctrl+C` : maintenir Ctrl, appuyer sur C
- `Super+L` : maintenir Super, appuyer sur L
- `Ctrl+Shift+T` : maintenir Ctrl et Shift, appuyer sur T
- `Alt+F4` : maintenir Alt, appuyer sur F4

## Raccourcis essentiels communs à tous les environnements

Certains raccourcis sont partagés par GNOME, KDE Plasma et XFCE car ils proviennent de conventions héritées de X11 ou de standards freedesktop.

### Navigation entre applications

| Raccourci | Action |
|-----------|--------|
| `Alt+Tab` | Basculer entre les applications (classeur d'applications) |
| `Alt+F4` | Fermer la fenêtre active |
| `Alt+F2` | Boîte de dialogue de commande / lanceur rapide |

### Presse-papiers

| Raccourci | Action |
|-----------|--------|
| `Ctrl+C` | Copier |
| `Ctrl+X` | Couper |
| `Ctrl+V` | Coller |
| `Ctrl+A` | Sélectionner tout |
| `Ctrl+Z` | Annuler |
| `Ctrl+Shift+Z` ou `Ctrl+Y` | Rétablir |

### Terminal

| Raccourci | Action |
|-----------|--------|
| `Ctrl+Shift+C` | Copier (dans le terminal, `Ctrl+C` envoie SIGINT) |
| `Ctrl+Shift+V` | Coller dans le terminal |
| `Ctrl+Shift+T` | Nouvel onglet de terminal |
| `Ctrl+Shift+W` | Fermer l'onglet de terminal |
| `Ctrl+D` | Fermer le terminal (EOF) |
| `Ctrl+L` | Effacer l'écran |
| `Ctrl+R` | Recherche dans l'historique des commandes |
| `Tab` | Autocomplétion des commandes et chemins |

### Consoles virtuelles (TTY)

| Raccourci | Action |
|-----------|--------|
| `Ctrl+Alt+F1` | TTY 1 — souvent occupé par le gestionnaire de connexion (GDM, SDDM) |
| `Ctrl+Alt+F2` | TTY 2 — souvent occupé par la session graphique active sur Debian moderne |
| `Ctrl+Alt+F3` à `F6` | TTY 3 à 6 — consoles texte additionnelles |
| `Ctrl+Alt+F7` | TTY 7 — emplacement historique de Xorg (avant logind) |

> 💡 **Trouver le TTY de la session graphique** : `loginctl` puis `loginctl show-session <ID> -p VTNr`. Sur Debian Trixie + GDM, le gestionnaire de connexion est sur TTY 1 et la session utilisateur s'ouvre généralement sur TTY 2. Sur Bookworm avec un gestionnaire historique, on peut encore trouver Xorg sur TTY 7.

### Personnalisation des raccourcis GNOME

```bash
# Interface graphique
# Paramètres → Clavier → Raccourcis clavier
# Les raccourcis sont organisés par catégorie : navigation, fenêtres,
# système, accessibilité, raccourcis personnalisés

# En ligne de commande (gsettings)
# Lister les raccourcis du gestionnaire de fenêtres
gsettings list-keys org.gnome.desktop.wm.keybindings

# Voir un raccourci spécifique
gsettings get org.gnome.desktop.wm.keybindings maximize
# Résultat : ['<Super>Up']

# Modifier un raccourci
gsettings set org.gnome.desktop.wm.keybindings maximize "['<Super>Up', '<Super>k']"
# Ajoute Super+K comme raccourci alternatif pour maximiser

# Désactiver un raccourci
gsettings set org.gnome.desktop.wm.keybindings minimize "['disabled']"

# Les raccourcis du shell GNOME
gsettings list-keys org.gnome.shell.keybindings

# Les raccourcis media
gsettings list-keys org.gnome.settings-daemon.plugins.media-keys

# Exporter tous les raccourcis pour sauvegarde
dconf dump /org/gnome/desktop/wm/keybindings/ > keybindings-wm.txt  
dconf dump /org/gnome/shell/keybindings/ > keybindings-shell.txt  
dconf dump /org/gnome/settings-daemon/plugins/media-keys/ > keybindings-media.txt  

# Restaurer
dconf load /org/gnome/desktop/wm/keybindings/ < keybindings-wm.txt
```

### Raccourcis personnalisés GNOME (lancement de commandes)

GNOME permet de créer des raccourcis qui exécutent une commande arbitraire :

```bash
# Via l'interface : Paramètres → Clavier → Raccourcis clavier
# → Section « Raccourcis personnalisés » → « + » en bas

# Via gsettings (pour les scripts d'automatisation)
# Les raccourcis personnalisés sont stockés dans un schéma indexé

# Exemple : ajouter Ctrl+Alt+T pour ouvrir un terminal
# Étape 1 : définir le chemin du raccourci
KEYS_PATH="/org/gnome/settings-daemon/plugins/media-keys/custom-keybindings"  
KEY="/org/gnome/settings-daemon/plugins/media-keys/custom-keybindings/custom0/"  

# Étape 2 : enregistrer le raccourci dans la liste
gsettings set org.gnome.settings-daemon.plugins.media-keys custom-keybindings \
  "['${KEY}']"

# Étape 3 : configurer le raccourci
dconf write "${KEY}name" "'Ouvrir Terminal'"  
dconf write "${KEY}command" "'gnome-terminal'"  
dconf write "${KEY}binding" "'<Ctrl><Alt>t'"  

# Exemple : raccourci pour capturer une zone et la copier au presse-papiers
# Note : depuis GNOME 42, l'outil intégré au shell remplace `gnome-screenshot`
# qui n'est plus installé par défaut. Le paquet reste disponible
# (`sudo apt install gnome-screenshot`, version 41.0-3 dans Trixie).
KEY2="/org/gnome/settings-daemon/plugins/media-keys/custom-keybindings/custom1/"  
gsettings set org.gnome.settings-daemon.plugins.media-keys custom-keybindings \  
  "['${KEY}', '${KEY2}']"
dconf write "${KEY2}name" "'Screenshot selection to clipboard'"  
dconf write "${KEY2}command" "'sh -c \"gnome-screenshot -a -c\"'"  
dconf write "${KEY2}binding" "'<Ctrl><Shift>s'"  
```

# 1. Raccourcis du gestionnaire de fenêtres (Xfwm4)

  -p "/xfwm4/custom/<Super>Up" -n -t string -s "maximize_window_key"

## Raccourcis personnalisés avancés

### Lancer des scripts avec des raccourcis

Associer un raccourci clavier à un script shell permet d'automatiser des tâches récurrentes :

```bash
# Exemple 1 : basculer entre les modes audio (haut-parleurs / casque)
# Créer le script
mkdir -p ~/.local/bin  
cat > ~/.local/bin/toggle-audio.sh << 'EOF'  
#!/bin/bash
CURRENT=$(pactl get-default-sink)  
if [[ "$CURRENT" == *"hdmi"* ]]; then  
    pactl set-default-sink alsa_output.pci-0000_00_1f.3.analog-stereo
    notify-send "Audio" "Sortie : Haut-parleurs"
else
    pactl set-default-sink alsa_output.pci-0000_00_1f.3.hdmi-stereo
    notify-send "Audio" "Sortie : HDMI"
fi  
EOF  
chmod +x ~/.local/bin/toggle-audio.sh  

# Associer à un raccourci (ex: Super+F12) via les Paramètres du DE

# Exemple 2 : verrouiller l'écran et éteindre l'affichage immédiatement
cat > ~/.local/bin/lock-screen.sh << 'EOF'
#!/bin/bash
# Verrouiller la session
loginctl lock-session
# Attendre que l'écran de verrouillage soit affiché
sleep 1
# Éteindre l'écran
xset dpms force off 2>/dev/null || true  
EOF  
chmod +x ~/.local/bin/lock-screen.sh  

# Exemple 3 : noter rapidement une idée dans un fichier
cat > ~/.local/bin/quick-note.sh << 'EOF'
#!/bin/bash
NOTE=$(zenity --entry --title="Note rapide" --text="Votre note :")  
if [ -n "$NOTE" ]; then  
    echo "$(date '+%Y-%m-%d %H:%M') — $NOTE" >> ~/Documents/notes.md
    notify-send "Note ajoutée" "$NOTE"
fi  
EOF  
chmod +x ~/.local/bin/quick-note.sh  

# zenity fournit des dialogues graphiques depuis les scripts
sudo apt install zenity
```

### xdotool : simulation de frappes et automatisation (Xorg)

Sous Xorg, `xdotool` permet de simuler des frappes clavier et des clics de souris, ce qui ouvre la porte à des automatisations avancées :

```bash
sudo apt install xdotool

# Simuler une frappe clavier
xdotool key ctrl+s                    # Sauvegarder dans l'application active  
xdotool type "Hello World"            # Taper du texte  

# Cliquer
xdotool click 1                       # Clic gauche  
xdotool click 3                       # Clic droit  

# Manipuler des fenêtres
xdotool getactivewindow windowminimize       # Minimiser la fenêtre active  
xdotool search --name "Firefox" windowactivate  # Activer Firefox  

# Exemple : raccourci pour coller du texte non formaté
# (copie depuis le presse-papiers, supprime le formatage, colle en texte brut)
cat > ~/.local/bin/paste-plain.sh << 'EOF'
#!/bin/bash
xclip -selection clipboard -o | xclip -selection clipboard -i  
xdotool key ctrl+v  
EOF  
chmod +x ~/.local/bin/paste-plain.sh  
# Associer à Ctrl+Shift+V en raccourci global
```

### Équivalents Wayland (wtype, ydotool)

Sous Wayland, `xdotool` ne fonctionne pas (par conception de sécurité — voir section 2.1.5).

**`wtype`** est le seul outil packagé dans Debian Trixie. Il simule des frappes clavier au niveau du protocole Wayland (extension `virtual-keyboard`). Il ne gère pas les clics souris :

```bash
sudo apt install wtype

# Simuler une frappe
wtype "Hello World"

# Simuler un raccourci avec modificateur
wtype -M ctrl s -m ctrl
```

**`ydotool`** est plus complet (clavier ET souris) mais n'est PAS packagé dans Debian Trixie. Il faut le compiler depuis [https://github.com/ReimuNotMoe/ydotool](https://github.com/ReimuNotMoe/ydotool) ou utiliser un dépôt tiers. Une fois installé, il fonctionne au niveau du noyau (uinput) :

```bash
# Procédure une fois ydotool installé (compilation manuelle ou dépôt tiers)
sudo systemctl enable --now ydotoold.service

# Simuler une frappe
ydotool type "Hello World"  
ydotool key ctrl+s  

# ydotool nécessite des permissions sur /dev/uinput
# L'utilisateur doit appartenir au groupe input
sudo usermod -aG input $USER
# Se déconnecter/reconnecter pour appliquer
```

## Réaffectation de touches

### Caps Lock : la touche la plus réaffectée

La touche Caps Lock occupe une position ergonomiquement avantageuse (rangée du milieu, accessible sans déplacer la main) mais sa fonction par défaut (verrouillage des majuscules) est rarement utilisée. La réaffecter est l'une des personnalisations les plus populaires :

```bash
# Option 1 : Caps Lock → Escape (populaire chez les utilisateurs de Vim)
# GNOME
gsettings set org.gnome.desktop.input-sources xkb-options "['caps:escape']"

# KDE : Paramètres système → Périphériques d'entrée → Clavier
#        → Avancé → Comportement de la touche Caps Lock

# XFCE / tous les environnements (Xorg)
setxkbmap -option caps:escape

# Option 2 : Caps Lock → Ctrl supplémentaire
gsettings set org.gnome.desktop.input-sources xkb-options "['ctrl:nocaps']"
# ou
setxkbmap -option ctrl:nocaps

# Option 3 : Caps Lock → Hyper (touche modificatrice supplémentaire)
# Utile pour créer un espace de raccourcis entièrement séparé
gsettings set org.gnome.desktop.input-sources xkb-options "['caps:hyper']"

# Option 4 : Caps Lock → Escape quand pressé seul, Ctrl quand maintenu
# Nécessite l'outil interception-tools ou keyd
# (voir section suivante)

# Rendre permanent via localectl (affecte la console et Xorg)
sudo localectl set-x11-keymap fr "" "" "caps:escape"

# Réinitialiser les options
gsettings reset org.gnome.desktop.input-sources xkb-options
# ou
setxkbmap -option ""
```

### Options XKB utiles

XKB (X Keyboard Extension) propose de nombreuses options de réaffectation. Voici les plus courantes :

```bash
# Lister toutes les options XKB disponibles
grep -E "^\s+\w+:" /usr/share/X11/xkb/rules/base.lst | head -60

# Options populaires
setxkbmap -option compose:ralt           # Alt Gr = touche Compose  
setxkbmap -option terminate:ctrl_alt_bksp # Ctrl+Alt+Backspace tue le serveur X  
setxkbmap -option lv3:ralt_switch        # Alt Gr pour le niveau 3  
setxkbmap -option eurosign:e             # Alt Gr+E = €  
setxkbmap -option ctrl:swap_lalt_lctl    # Échanger Alt gauche et Ctrl gauche  
setxkbmap -option altwin:swap_lalt_lwin  # Échanger Alt gauche et Super  

# Combiner plusieurs options
setxkbmap -option "caps:escape,compose:ralt,terminate:ctrl_alt_bksp"

# GNOME : combiner plusieurs options
gsettings set org.gnome.desktop.input-sources xkb-options \
  "['caps:escape', 'compose:ralt']"
```

### keyd : réaffectation avancée des touches

Pour les réaffectations complexes qui dépassent les capacités de XKB (touches à double fonction, couches personnalisées, macros), **keyd** est un outil puissant qui fonctionne au niveau du noyau (lecture directe d'`/dev/input/event*` + injection via `uinput`) et supporte à la fois Xorg et Wayland :

```bash
# keyd est disponible dans Debian Trixie depuis 2025
sudo apt install keyd  
sudo apt install keyd-application-mapper   # Mappings par application  

# Activer le service système
sudo systemctl enable --now keyd.service

# Configuration globale (/etc/keyd/default.conf)
# Caps Lock = Escape quand pressé seul, Ctrl quand maintenu avec une autre touche
sudo tee /etc/keyd/default.conf > /dev/null << 'EOF'
[ids]
*

[main]
capslock = overload(control, esc)  
EOF  

# Recharger la configuration sans redémarrer
sudo keyd reload

# Tester en direct (logs)
sudo keyd monitor    # Affiche les frappes telles qu'elles sont vues par keyd
```

## Touche Compose : caractères spéciaux

La touche **Compose** est un mécanisme élégant pour saisir des caractères spéciaux qui ne figurent pas directement sur le clavier. On presse la touche Compose, puis une séquence de deux ou trois caractères, et le caractère composé est inséré :

```bash
# Activer la touche Compose (en général sur Alt Gr droit)
setxkbmap -option compose:ralt

# Séquences Compose courantes :
# Compose + ' + e    → é
# Compose + ` + a    → à
# Compose + ^ + o    → ô
# Compose + " + u    → ü
# Compose + c + ,    → ç
# Compose + - + >    → →
# Compose + < + =    → ≤
# Compose + = + =    → ≡
# Compose + / + =    → ≠
# Compose + o + c    → ©
# Compose + o + r    → ®
# Compose + + + -    → ±
# Compose + s + s    → ß
# Compose + . + .    → …

# La liste complète des séquences est dans
less /usr/share/X11/locale/en_US.UTF-8/Compose

# Des séquences personnalisées peuvent être ajoutées dans
~/.XCompose
# Format : <Multi_key> <a> <e> : "æ"  # ae ligature
```

## Bonnes pratiques

### Construire un système de raccourcis cohérent

Un bon système de raccourcis suit une logique interne qui facilite la mémorisation :

**Organiser par modificateur** : réserver chaque touche modificatrice à un domaine :
- `Super+...` : actions du bureau (lancement d'applications, espaces de travail, fenêtres)
- `Ctrl+...` : actions d'application (copier, coller, sauvegarder, chercher)
- `Alt+...` : navigation et commutation (Alt+Tab, Alt+F4)
- `Ctrl+Alt+...` : actions système (terminal, verrouillage, TTY)

**Utiliser des associations mnémotiques** :
- `Super+E` pour l'explorateur de fichiers (E = Explorer)
- `Super+T` pour le terminal (T = Terminal)
- `Super+B` pour le navigateur (B = Browser)

**Documenter ses raccourcis personnalisés** : maintenir un fichier de référence dans `~/Documents/raccourcis.md` avec la liste des raccourcis personnalisés ajoutés, leur justification et leur commande associée.

### Éviter les conflits

```bash
# Identifier les conflits potentiels
# GNOME : les raccourcis en conflit sont signalés dans les Paramètres
# KDE : les conflits sont détectés et signalés lors de la configuration

# Tester un raccourci avant de l'affecter
# Appuyer sur la combinaison souhaitée et vérifier qu'elle ne fait rien
# d'inattendu dans les applications courantes

# Raccourcis à éviter (souvent utilisés par les applications) :
# Ctrl+W          (fermer onglet dans les navigateurs)
# Ctrl+T          (nouvel onglet)
# Ctrl+N          (nouvelle fenêtre)
# Ctrl+Q          (quitter l'application)
# F11             (plein écran)
# Ces raccourcis sont mieux réservés aux applications
```

### Apprendre progressivement

Adopter trop de raccourcis en même temps est contre-productif. Une approche efficace consiste à en apprendre deux ou trois par semaine, en commençant par ceux qui ont le plus grand impact sur le flux de travail quotidien :

1. `Super` (vue d'ensemble / recherche) — remplace la navigation dans les menus
2. `Alt+Tab` / `Alt+²` — navigation entre applications et fenêtres
3. `Super+←` / `→` — ancrage de fenêtres (split screen)
4. `Super+Page Haut` / `Bas` — navigation entre espaces de travail
5. `Ctrl+Alt+T` — ouvrir un terminal (si configuré)

Une fois ces raccourcis devenus instinctifs, ajouter les suivants selon ses besoins.

## Résumé

Les raccourcis clavier sont un investissement dont le rendement s'accumule quotidiennement. Chaque environnement de bureau Debian offre un système complet de raccourcis par défaut, extensible avec des raccourcis personnalisés qui peuvent lancer n'importe quelle commande ou script. La personnalisation des raccourcis passe par les Paramètres de l'environnement de bureau pour les ajustements simples, par gsettings (GNOME), les fichiers de configuration KDE ou xfconf-query (XFCE) pour l'automatisation, et par des outils spécialisés (xdotool, ydotool, keyd) pour les besoins avancés.

La réaffectation de touches sous-utilisées (Caps Lock en Escape ou Ctrl) et l'activation de la touche Compose sont deux optimisations à fort impact qui améliorent à la fois la productivité et le confort de frappe.
