# 2.5.4 Outils de développement (IDE, éditeurs, VS Code)

## Introduction

Linux est historiquement le système d'exploitation des développeurs. Les outils fondamentaux du développement logiciel — compilateurs, interpréteurs, systèmes de contrôle de version, shells, outils de build — y sont natifs et parfaitement intégrés. Un poste Debian constitue un environnement de développement naturel, capable de couvrir tous les langages et tous les paradigmes, du développement web au développement embarqué, de l'administration système au machine learning.

Cette section présente l'outillage de développement disponible sous Debian : les éditeurs de texte, les IDE complets, les environnements d'exécution par langage, les outils de versionnement, et les utilitaires qui transforment un bureau Debian en station de développement productive.

## Éditeurs de texte

### Le spectre des éditeurs

Les éditeurs de texte pour le développement se répartissent sur un spectre allant du minimalisme pur (édition de fichiers, coloration syntaxique) aux environnements quasi-IDE (autocomplétion, débogage, intégration Git, terminal intégré). Le choix de l'éditeur est personnel et souvent passionné, mais il est guidé par des critères objectifs : les langages utilisés, la puissance de la machine, le flux de travail préféré et le temps qu'on est prêt à investir dans la configuration.

### Vim et Neovim

**Vim** est l'éditeur modal légendaire du monde Unix. Son modèle d'édition (modes normal, insertion, visuel, commande) est déroutant au premier abord mais d'une efficacité redoutable une fois maîtrisé. **Neovim** est un fork modernisé de Vim qui conserve la compatibilité avec la configuration Vim tout en apportant une architecture de plugins moderne (Lua), un terminal intégré natif et un support LSP (Language Server Protocol) intégré.

```bash
# Vim
sudo apt install vim

# Neovim (recommandé pour les nouvelles configurations)
sudo apt install neovim
# Ou via Flatpak pour une version plus récente
flatpak install flathub io.neovim.nvim

# Alias optionnel
echo "alias vi='nvim'" >> ~/.bashrc  
echo "alias vim='nvim'" >> ~/.bashrc  
```

Neovim avec une distribution de configuration préconfigurée est la manière la plus rapide d'obtenir un environnement Vim moderne et productif :

```bash
# Distributions Neovim populaires (configurations prêtes à l'emploi) :

# LazyVim — distribution moderne, modulaire, basée sur lazy.nvim
# Installation :
# 1. Sauvegarder la configuration existante
mv ~/.config/nvim ~/.config/nvim.bak  
mv ~/.local/share/nvim ~/.local/share/nvim.bak  
# 2. Cloner LazyVim
git clone https://github.com/LazyVim/starter ~/.config/nvim
# 3. Lancer Neovim (les plugins s'installent automatiquement)
nvim

# Autres distributions : AstroNvim, NvChad, LunarVim
# Chacune offre un niveau différent de pré-configuration

# Configuration personnelle : ~/.config/nvim/init.lua (Neovim)
#                             ~/.vimrc (Vim classique)
```

Les fonctionnalités de Neovim avec une distribution moderne incluent l'autocomplétion intelligente via LSP, la navigation dans le code (aller à la définition, références, renommer), le débogage intégré (DAP), l'intégration Git (signes dans la gouttière, diff, blame), le fuzzy finder (recherche de fichiers, de symboles, de texte), le terminal intégré, la gestion de sessions et la coloration syntaxique avancée via Tree-sitter.

### Emacs

**Emacs** est l'autre éditeur historique du monde Unix, souvent décrit comme un « système d'exploitation déguisé en éditeur de texte ». Son extensibilité via Emacs Lisp en fait un environnement capable de tout : édition de code, client e-mail (mu4e), gestionnaire de tâches (Org Mode), navigateur web, client IRC, terminal, et bien plus.

```bash
# Emacs avec interface graphique
sudo apt install emacs

# Emacs en mode terminal uniquement
sudo apt install emacs-nox

# Distribution préconfigurée populaire
# Doom Emacs : configuration moderne, réactive, inspirée de Vim
git clone --depth 1 https://github.com/doomemacs/doomemacs ~/.config/emacs
~/.config/emacs/bin/doom install

# Spacemacs : configuration hybride Vim/Emacs
git clone https://github.com/syl20bnr/spacemacs ~/.emacs.d

# Configuration personnelle : ~/.emacs.d/init.el ou ~/.config/emacs/init.el
```

La fonctionnalité la plus distinctive d'Emacs est **Org Mode**, un système de gestion de documents structurés, de prise de notes, de gestion de tâches, de suivi du temps et de literate programming. Org Mode est considéré par beaucoup comme la raison suffisante d'apprendre Emacs.

### Kate

**Kate** (KDE Advanced Text Editor) est l'éditeur de texte avancé de KDE. Il offre une expérience riche sans la complexité d'un IDE complet, avec une interface graphique familière :

```bash
sudo apt install kate
```

Kate propose la coloration syntaxique pour plus de 300 langages, un support LSP intégré (autocomplétion, diagnostic, navigation), un terminal intégré, la gestion de projets (dossiers de travail), la recherche et remplacement avec expressions régulières dans plusieurs fichiers, la gestion de sessions, le mode vi (émulation des raccourcis Vim) et le split d'éditeur horizontal et vertical.

Kate est un excellent choix pour les développeurs sous KDE Plasma qui souhaitent un éditeur puissant sans la lourdeur d'un IDE et sans l'investissement d'apprentissage de Vim/Emacs.

### Geany

**Geany** est un éditeur de texte léger qui se rapproche d'un petit IDE. Il est idéal pour les environnements à ressources limitées (XFCE, LXQt) ou pour les développeurs qui préfèrent un outil simple et rapide :

```bash
sudo apt install geany geany-plugins
```

Geany offre la coloration syntaxique, l'autocomplétion basique (basée sur les symboles du fichier), un terminal intégré, la compilation et l'exécution directement depuis l'interface, un navigateur de symboles (fonctions, classes, variables), et la gestion de projets. Il est compatible avec tous les environnements de bureau et consomme très peu de ressources.

### GNOME Text Editor et Mousepad

Pour l'édition occasionnelle de fichiers de configuration ou de scripts simples :

```bash
# GNOME Text Editor (successeur de gedit, intégré à GNOME)
sudo apt install gnome-text-editor
# Coloration syntaxique, recherche, préférences minimales

# Mousepad (éditeur XFCE, très léger)
sudo apt install mousepad
# Coloration syntaxique, onglets, recherche/remplacement
```

## Visual Studio Code

### Présentation

**Visual Studio Code** (VS Code) est l'éditeur de code le plus utilisé au monde. Développé par Microsoft, c'est une application Electron (Chromium + Node.js) qui offre un équilibre entre la légèreté d'un éditeur et la puissance d'un IDE grâce à son écosystème d'extensions massif. VS Code est devenu le standard de facto dans de nombreuses équipes de développement.

### Installation

```bash
# Méthode 1 : dépôt Microsoft (version officielle, propriétaire)
wget -qO- https://packages.microsoft.com/keys/microsoft.asc | \
  gpg --dearmor | sudo tee /usr/share/keyrings/microsoft.gpg > /dev/null
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/microsoft.gpg] \
  https://packages.microsoft.com/repos/code stable main" | \
  sudo tee /etc/apt/sources.list.d/vscode.list
sudo apt update && sudo apt install code

# Méthode 2 : VSCodium (version libre, sans télémétrie Microsoft)
flatpak install flathub com.vscodium.codium
# VSCodium est un build de VS Code sans les composants propriétaires
# (télémétrie, branding Microsoft, certaines extensions du Marketplace)
# Le Marketplace d'extensions est remplacé par Open VSX Registry

# Méthode 3 : Flatpak (version officielle Microsoft)
flatpak install flathub com.visualstudio.code
```

### Configuration Wayland

```bash
# ⚠️ Le mécanisme `~/.config/code-flags.conf` documenté sur le wiki
# d'ArchLinux N'EST PAS supporté par le paquet officiel Microsoft (.deb)
# ni par VSCodium : c'est un wrapper spécifique au paquet `code` d'Arch.

# Méthode portable : modifier le lanceur utilisateur (.desktop)
cp /usr/share/applications/code.desktop ~/.local/share/applications/  
sed -i 's|Exec=/usr/share/code/code|Exec=/usr/share/code/code --ozone-platform-hint=auto --enable-features=WaylandWindowDecorations|g' \  
  ~/.local/share/applications/code.desktop
update-desktop-database ~/.local/share/applications/

# Pour VSCodium en Flatpak, utiliser flatpak override :
flatpak override --user --env=ELECTRON_OZONE_PLATFORM_HINT=auto com.vscodium.codium
```

## IDE complets

### JetBrains (IntelliJ IDEA, PyCharm, WebStorm, GoLand, etc.)

Les IDE JetBrains sont considérés parmi les meilleurs pour le développement professionnel. Chaque IDE est spécialisé pour un langage ou un écosystème, avec une intelligence de code parmi les plus avancées du marché.

```bash
# Méthode recommandée : JetBrains Toolbox App
# Télécharger depuis https://www.jetbrains.com/toolbox-app/
# La Toolbox gère l'installation, les mises à jour et les licences
# de tous les IDE JetBrains

# Via Flatpak (éditions Community, gratuites)
flatpak install flathub com.jetbrains.IntelliJ-IDEA-Community   # Java/Kotlin  
flatpak install flathub com.jetbrains.PyCharm-Community          # Python  
flatpak install flathub com.jetbrains.CLion                      # C/C++ (payant)  

# Les éditions Community sont gratuites et open source
# Les éditions Professional/Ultimate sont payantes (licence annuelle)
# Licence gratuite disponible pour les étudiants et les projets open source
```

Les forces des IDE JetBrains sont l'autocomplétion contextuelle profonde, le refactoring automatisé fiable, le débogueur intégré avec évaluation d'expressions, l'intégration native des outils de build (Maven, Gradle, npm, pip), les outils de base de données intégrés et l'analyse de code statique en temps réel.

### Eclipse

**Eclipse** est un IDE libre et extensible, historiquement dominant dans le monde Java mais utilisable pour de nombreux langages via les plugins :

```bash
# ⚠️ Le paquet `eclipse` n'est plus distribué dans Debian Trixie
# (retiré faute de mainteneur en amont pour le packaging Debian).
# Méthodes d'installation actuelles :

# Option 1 : Flatpak (édition Java de la Eclipse Foundation)
flatpak install flathub org.eclipse.Java

# Option 2 : Téléchargement officiel depuis eclipse.org
# https://www.eclipse.org/downloads/
# Décompresser le tarball et lancer `./eclipse`

# Option 3 : Eclipse Installer
# Outil officiel multi-paquets (Eclipse IDE for Java, RCP, C/C++, etc.)
```

Eclipse est le choix établi pour le développement Java d'entreprise, le développement Eclipse RCP et les environnements où les outils JetBrains ne sont pas disponibles.

### GNOME Builder

**GNOME Builder** est l'IDE intégré à GNOME, conçu spécifiquement pour le développement d'applications GNOME (C, Python, Vala, JavaScript) et les projets utilisant le système de build Meson :

```bash
sudo apt install gnome-builder
```

Builder offre une intégration native avec Flatpak (construction et test d'applications dans un environnement sandbox), le support LSP, un profileur intégré et un éditeur d'interface utilisateur GTK.

## Outils de versionnement

### Git

Git est le système de contrôle de version standard. Il est indispensable pour tout développement logiciel moderne :

```bash
# Installation
sudo apt install git

# Configuration initiale
git config --global user.name "Prénom Nom"  
git config --global user.email "email@example.com"  
git config --global init.defaultBranch main  

# Éditeur par défaut pour les messages de commit
git config --global core.editor "nvim"
# ou "code --wait" pour VS Code

# Configuration recommandée
git config --global pull.rebase true          # Rebase au lieu de merge lors du pull  
git config --global fetch.prune true          # Nettoyer les branches distantes supprimées  
git config --global diff.colorMoved zebra     # Coloriser les lignes déplacées dans les diffs  
git config --global rerere.enabled true       # Mémoriser les résolutions de conflits  

# Alias utiles
git config --global alias.st "status -sb"  
git config --global alias.lg "log --oneline --graph --all --decorate"  
git config --global alias.co checkout  
git config --global alias.br branch  
git config --global alias.ci commit  
git config --global alias.unstage "reset HEAD --"  
git config --global alias.last "log -1 HEAD"  
```

### Interfaces graphiques pour Git

```bash
# gitk : visualiseur d'historique intégré à Git
sudo apt install gitk  
gitk --all &  

# git-gui : interface graphique basique pour Git
sudo apt install git-gui

# gitg : client Git graphique pour GNOME
sudo apt install gitg

# Outils avancés
# lazygit : interface Git en terminal, rapide et intuitive
sudo apt install lazygit  
lazygit    # Lancer dans un dépôt Git  

# tig : navigateur d'historique Git en mode texte
sudo apt install tig  
tig  
```

### Clés SSH pour Git

```bash
# Générer une clé SSH pour l'authentification Git
ssh-keygen -t ed25519 -C "email@example.com"

# Démarrer l'agent SSH
eval "$(ssh-agent -s)"  
ssh-add ~/.ssh/id_ed25519  

# Afficher la clé publique (à ajouter sur GitHub/GitLab)
cat ~/.ssh/id_ed25519.pub

# Tester la connexion
ssh -T git@github.com  
ssh -T git@gitlab.com  

# Configuration SSH pour utiliser des clés différentes par service
# Éditer ~/.ssh/config
# Host github.com
#     HostName github.com
#     User git
#     IdentityFile ~/.ssh/id_ed25519_github
#
# Host gitlab.com
#     HostName gitlab.com
#     User git
#     IdentityFile ~/.ssh/id_ed25519_gitlab
```

## Environnements d'exécution par langage

### Python

```bash
# Python est pré-installé sur Debian
python3 --version

# Pip (gestionnaire de paquets Python)
sudo apt install python3-pip python3-venv

# Créer un environnement virtuel (bonne pratique pour chaque projet)
python3 -m venv ~/projets/mon-projet/.venv  
source ~/projets/mon-projet/.venv/bin/activate  
pip install flask requests numpy    # Installer des dépendances dans le venv  
deactivate                          # Quitter l'environnement virtuel  

# pyenv : gestion de plusieurs versions de Python
# Installation depuis https://github.com/pyenv/pyenv
# Permet d'installer et de basculer entre Python 3.10, 3.11, 3.12, etc.

# Outils de qualité de code
# ⚠️ PEP 668 : Debian Trixie marque l'environnement Python système comme
# « externally-managed-environment ». `pip install` global est refusé.
# Trois solutions :

# 1. Outils CLI globaux : utiliser pipx (un venv dédié par outil, recommandé)
sudo apt install pipx  
pipx ensurepath              # Ajoute ~/.local/bin au PATH (rouvrir le shell)  
pipx install ruff            # Linter et formateur ultra-rapide (remplace flake8 + black)  
pipx install mypy            # Vérificateur de types statiques  
pipx install pytest          # Framework de tests (souvent géré par projet : voir option 2)  

# 2. Outils par projet : utiliser un venv (préférable pour les libs/projets)
python3 -m venv .venv  
source .venv/bin/activate  
pip install ruff mypy pytest  

# 3. Paquets Debian (versions parfois en retard mais bien intégrées)
sudo apt install python3-pytest python3-mypy
# (ruff n'est pas encore packagé dans Trixie — pipx ou cargo install ruff)
```

## Outils de productivité du développeur

### Terminal et shell

```bash
# Émulateurs de terminal avancés
sudo apt install kitty         # GPU-accelerated, configurable, images inline

sudo apt install fd-find       # Alternative à find (fd)
```

```bash
# Docker : voir le Module 10 pour l'installation et la configuration détaillées
```

## Résumé

Debian offre un environnement de développement complet et flexible. Le choix de l'éditeur ou de l'IDE dépend des préférences personnelles, des langages utilisés et du contexte de travail. VS Code avec ses extensions couvre la majorité des besoins avec un investissement minimal en configuration. Neovim avec une distribution comme LazyVim offre une alternative légère et puissante pour les développeurs qui investissent dans l'apprentissage de l'édition modale. Les IDE JetBrains sont imbattables pour le développement professionnel dans un langage spécifique.

Au-delà de l'éditeur, l'écosystème Linux offre les meilleurs outils de développement en ligne de commande : Git, les gestionnaires de versions de langages (nvm, rustup, pyenv), les outils de conteneurisation (Docker, Dev Containers) et une collection d'utilitaires modernes (ripgrep, fzf, jq, bat) qui accélèrent le flux de travail quotidien.

Ce chapitre conclut le Module 2 sur Debian Desktop. Le Module 3 aborde l'administration système de base : gestion du système de fichiers, des utilisateurs, des processus, de systemd et des logs.
