# 2.1.4 Installation et configuration des DE

## Introduction

Les sections précédentes ont présenté les principaux environnements de bureau disponibles sous Debian : GNOME, KDE Plasma, XFCE, LXDE et LXQt. Cette section aborde les aspects pratiques : comment installer un environnement de bureau sur un système Debian, comment le configurer, comment gérer la cohabitation de plusieurs environnements, et comment résoudre les problèmes courants liés à l'installation et au changement d'environnement.

Que l'on parte d'une installation Debian minimale (serveur, sans interface graphique) ou que l'on souhaite ajouter un second environnement à un système existant, les mécanismes sont les mêmes. Debian utilise un système de méta-paquets et de tâches (tasks) qui simplifie considérablement le processus, tout en laissant la possibilité d'une installation granulaire pour les utilisateurs avancés.

## Prérequis

### Composants graphiques fondamentaux

Avant d'installer un environnement de bureau, le système doit disposer des composants graphiques de base. Sur une installation Debian avec interface graphique, ces composants sont déjà présents. Sur une installation minimale (serveur), ils doivent être installés.

La pile graphique Linux repose sur plusieurs couches :

```
┌───────────────────────────────────────────────┐
│          Environnement de bureau              │
│  (GNOME, KDE Plasma, XFCE, LXQt, etc.)        │
├───────────────────────────────────────────────┤
│         Gestionnaire de connexion             │
│  (GDM, SDDM, LightDM)                         │
├───────────────────────────────────────────────┤
│         Serveur d'affichage                   │
│  (Xorg et/ou Wayland)                         │
├───────────────────────────────────────────────┤
│     Pilotes graphiques (Mesa, NVIDIA, etc.)   │
├───────────────────────────────────────────────┤
│        DRM/KMS (noyau Linux)                  │
└───────────────────────────────────────────────┘
```

Sur une installation minimale, la vérification suivante permet de s'assurer que le système est prêt à recevoir un environnement graphique :

```bash
# Vérifier que le serveur d'affichage est installé
dpkg -l | grep -E "xorg|xserver-xorg"

# Vérifier les pilotes graphiques disponibles
lspci | grep -i vga  
apt list --installed 2>/dev/null | grep -iE "mesa|nvidia|amdgpu"  

# Vérifier la résolution de la console framebuffer (indique que DRM/KMS fonctionne)
cat /sys/class/drm/card0/device/driver/module/drivers
```

### Identification du matériel graphique

Le choix des pilotes graphiques dépend du matériel présent dans la machine. Cette identification est un prérequis avant toute installation :

```bash
# Identifier la carte graphique
lspci -nn | grep -iE "vga|3d|display"

# Résultat typique :
# 00:02.0 VGA compatible controller [0300]: Intel Corporation UHD Graphics 620 [8086:3ea0]
# 01:00.0 3D controller [0302]: NVIDIA Corporation GP107M [GeForce GTX 1050] [10de:1c8d]
```

Les pilotes à installer selon le fabricant :

| Fabricant | Pilote libre (recommandé) | Pilote propriétaire | Paquets Debian |
|-----------|--------------------------|---------------------|----------------|
| **Intel** | i915 (intégré au noyau) | — | `xserver-xorg-video-intel` (optionnel, le mode modesetting du noyau suffit généralement) |
| **AMD/ATI** | amdgpu (intégré au noyau) | — | `xserver-xorg-video-amdgpu`, `firmware-amd-graphics` |
| **NVIDIA** | nouveau (intégré au noyau) | nvidia-driver | `xserver-xorg-video-nouveau` (par défaut) ou `nvidia-driver` (non-free) |

```bash
# Installer les firmwares nécessaires (si les dépôts non-free-firmware sont activés)
sudo apt install firmware-linux firmware-linux-nonfree

# Pour les cartes AMD
sudo apt install firmware-amd-graphics

# Pour les cartes Intel récentes nécessitant du firmware
sudo apt install firmware-misc-nonfree
```

> **Note** : l'installation des pilotes NVIDIA propriétaires est couverte en détail dans la section 2.3.1. Pour l'installation initiale d'un environnement de bureau, le pilote libre `nouveau` est suffisant dans la majorité des cas.

## Installation via tasksel

### Présentation de tasksel

**tasksel** est l'outil Debian dédié à l'installation de groupes cohérents de paquets appelés « tâches » (tasks). Chaque tâche correspond à un usage spécifique : environnement de bureau, serveur web, serveur de fichiers, etc. C'est l'outil utilisé par l'installateur Debian lors de la phase de sélection des logiciels.

```bash
# Lancer tasksel en mode interactif (interface ncurses)
sudo tasksel

# Lister toutes les tâches disponibles avec leur état
tasksel --list-tasks

# Résultat typique :
# u desktop        Debian desktop environment
# u gnome-desktop  GNOME
# u xfce-desktop   XFCE
# u kde-desktop    KDE Plasma
# u cinnamon-desktop  Cinnamon
# u mate-desktop   MATE
# u lxde-desktop   LXDE
# u lxqt-desktop   LXQt
# u web-server     Web server
# u ssh-server     SSH server
# i laptop         Laptop
# (i = installée, u = non installée)

# Installer une tâche spécifique
sudo tasksel install kde-desktop

# Désinstaller une tâche
sudo tasksel remove gnome-desktop
```

### Structure d'une tâche desktop

Chaque tâche desktop installe en réalité un méta-paquet `task-*-desktop` qui dépend de plusieurs couches de paquets :

```
task-gnome-desktop
├── task-desktop           (composants communs à tous les DE)
│   ├── xorg               (serveur d'affichage X11)
│   ├── xserver-xorg       (serveur X et pilotes)
│   ├── desktop-base       (fonds d'écran et thèmes Debian)
│   ├── fonts              (polices de base)
│   └── cups               (impression)
├── gnome-core             (composants essentiels de GNOME)
│   ├── gnome-shell        (interface utilisateur)
│   ├── mutter             (gestionnaire de fenêtres)
│   ├── nautilus            (gestionnaire de fichiers)
│   ├── gdm3               (gestionnaire de connexion)
│   └── gnome-settings-daemon
├── gnome                  (applications GNOME supplémentaires)
│   ├── gnome-console / ptyxis  (terminal — Ptyxis défaut depuis GNOME 48.4)
│   ├── gnome-text-editor
│   ├── evince             (lecteur PDF)
│   ├── loupe              (visionneuse d'images Rust, remplace eog depuis GNOME 45)
│   └── ...
├── libreoffice            (suite bureautique)
├── firefox-esr            (navigateur web)
└── system-config-printer  (configuration d'imprimantes)
```

### La tâche « desktop » commune

La tâche `desktop` (méta-paquet `task-desktop`) est un socle commun installé par toutes les tâches d'environnement de bureau. Elle comprend les composants partagés indispensables : le serveur Xorg, les polices de base, le support de l'impression, les thèmes Debian par défaut et les utilitaires communs. Lors de l'installation d'un second environnement, cette couche n'est pas réinstallée puisqu'elle est déjà présente.

## Installation via apt

### Installation complète (équivalent tasksel)

L'installation via `apt` avec les méta-paquets `task-*-desktop` est strictement équivalente à l'utilisation de tasksel :

```bash
# GNOME (environnement complet avec applications et bureautique)
sudo apt install task-gnome-desktop

# KDE Plasma (environnement complet)
sudo apt install task-kde-desktop

# XFCE (environnement complet)
sudo apt install task-xfce-desktop

# LXDE (environnement complet)
sudo apt install task-lxde-desktop

# LXQt (environnement complet)
sudo apt install task-lxqt-desktop

# MATE (environnement complet)
sudo apt install task-mate-desktop

# Cinnamon (environnement complet)
sudo apt install task-cinnamon-desktop
```

### Installation intermédiaire (environnement sans bureautique)

Pour installer un environnement de bureau sans les applications tierces volumineuses (LibreOffice, etc.) :

```bash
# GNOME : le méta-paquet gnome installe l'environnement + les applications GNOME
sudo apt install gnome

# KDE Plasma : idem avec le méta-paquet kde-standard
sudo apt install kde-standard

# XFCE avec les goodies
sudo apt install xfce4 xfce4-goodies
```

### Installation minimale (cœur de l'environnement uniquement)

Pour les installations les plus légères possibles, seul le cœur de l'environnement est installé, sans les applications complémentaires :

```bash
# GNOME minimal
sudo apt install gnome-core

# KDE Plasma minimal
sudo apt install kde-plasma-desktop

# XFCE minimal
sudo apt install xfce4

# LXQt minimal
sudo apt install lxqt

# LXDE minimal
sudo apt install lxde-core
```

Ces installations minimales fournissent un bureau fonctionnel mais dépouillé. Il faudra ensuite installer manuellement les applications souhaitées (terminal, gestionnaire de fichiers avancé, éditeur, navigateur, etc.).

### Installation depuis une base serveur (sans interface graphique)

Lorsque le point de départ est une installation Debian minimale (serveur, sans aucun composant graphique), il est nécessaire d'installer d'abord les composants graphiques de base :

```bash
# Méthode 1 : via tasksel (installe tout d'un coup)
sudo apt install tasksel  
sudo tasksel install gnome-desktop  

# Méthode 2 : via apt, étape par étape

# 1. Installer le serveur d'affichage et les pilotes
sudo apt install xorg

# 2. Installer l'environnement de bureau choisi
sudo apt install gnome-core
# ou
sudo apt install kde-plasma-desktop

# 3. Le gestionnaire de connexion est normalement installé comme dépendance
# Vérifier qu'il est bien présent et actif
systemctl status gdm3     # pour GNOME  
systemctl status sddm     # pour KDE  
systemctl status lightdm   # pour XFCE/LXQt  

# 4. Activer le démarrage en mode graphique
sudo systemctl set-default graphical.target

# 5. Démarrer l'interface graphique immédiatement (sans redémarrer)
sudo systemctl start gdm3
# ou redémarrer le système
sudo systemctl reboot
```

## Gestionnaires de connexion (Display Managers)

### Rôle et fonctionnement

Le gestionnaire de connexion est le service qui affiche l'écran de connexion au démarrage du système. Il remplit trois fonctions essentielles :

1. **Authentification** : il vérifie les identifiants de l'utilisateur (nom d'utilisateur et mot de passe).
2. **Sélection de session** : il propose le choix de l'environnement de bureau si plusieurs sont installés.
3. **Lancement de la session** : il démarre l'environnement de bureau sélectionné avec les variables d'environnement appropriées.

Le gestionnaire de connexion est un service systemd qui s'exécute en tant que root et contrôle le processus de connexion graphique :

```bash
# Le gestionnaire de connexion actif est celui pointé par display-manager.service
ls -la /etc/systemd/system/display-manager.service
# Résultat : lien symbolique vers le .service du DM actif

# Vérifier l'état du gestionnaire de connexion
systemctl status display-manager.service
```

### Les principaux gestionnaires de connexion

#### GDM3 (GNOME Display Manager)

GDM3 est le gestionnaire de connexion de GNOME. C'est historiquement le seul gestionnaire qui exécute son **greeter** (l'écran de connexion lui-même) directement sous Wayland sur Debian. **SDDM** a ajouté un support Wayland expérimental à partir de la version 0.20, mais il faut l'activer manuellement (`DisplayServer=wayland` dans `/etc/sddm.conf.d/override.conf`) — par défaut, le greeter SDDM reste sous X11 même quand la session lancée est Wayland. **LightDM** reste exclusivement sous X11 pour son greeter. Quel que soit le greeter, tous lancent correctement une session Wayland si l'utilisateur la sélectionne.

```bash
# Installation
sudo apt install gdm3

# Configuration principale
/etc/gdm3/daemon.conf

# Fichier de configuration important : activer/désactiver Wayland
# Éditer /etc/gdm3/daemon.conf
# [daemon]
# WaylandEnable=true    (activer Wayland, par défaut)
# WaylandEnable=false   (forcer X11 uniquement)
# AutomaticLoginEnable=true    (connexion automatique)
# AutomaticLogin=nom_utilisateur

# Appliquer les changements
sudo systemctl restart gdm3
```

GDM3 offre un écran de connexion sobre et intégré à GNOME. Il affiche la liste des utilisateurs avec leurs avatars, un champ de mot de passe et un sélecteur de session (icône d'engrenage). L'accessibilité (lecteur d'écran, contraste élevé, zoom) est disponible directement depuis l'écran de connexion.

#### SDDM (Simple Desktop Display Manager)

SDDM est le gestionnaire de connexion recommandé pour KDE Plasma. Il est écrit en C++/QML et supporte les thèmes QML, ce qui le rend visuellement flexible et cohérent avec l'esthétique Plasma.

```bash
# Installation
sudo apt install sddm

# Répertoire de configuration
/etc/sddm.conf.d/
# ou fichier unique
/etc/sddm.conf

# Générer un fichier de configuration par défaut
sddm --example-config | sudo tee /etc/sddm.conf

# Options de configuration courantes
# [Autologin]
# User=nom_utilisateur
# Session=plasma.desktop
#
# [Theme]
# Current=breeze
#
# [General]
# InputMethod=          (méthode de saisie)

# Installer des thèmes SDDM supplémentaires
sudo apt install sddm-theme-breeze
# Les thèmes sont situés dans /usr/share/sddm/themes/

# Tester un thème sans redémarrer
sddm-greeter --test-mode --theme /usr/share/sddm/themes/breeze
```

#### LightDM

LightDM est un gestionnaire de connexion léger et indépendant de tout environnement de bureau. Il est le choix recommandé pour XFCE, LXDE, LXQt et les gestionnaires de fenêtres standalone. Son architecture repose sur un démon principal et des « greeters » (interfaces d'accueil) interchangeables.

```bash
# Installation avec le greeter GTK (adapté à XFCE, LXDE, MATE)
sudo apt install lightdm lightdm-gtk-greeter

# Pour LXQt, on installe également LightDM avec le greeter GTK : il n'existe pas
# de paquet `lightdm-qt-greeter` dans Trixie (`apt-cache search lightdm-qt-greeter`
# ne retourne rien). Si l'on souhaite un greeter à l'esthétique différente, les
# alternatives packagées sont `slick-greeter` ou `arctica-greeter`.
sudo apt install lightdm lightdm-gtk-greeter

# Configuration principale
/etc/lightdm/lightdm.conf

# Configuration du greeter GTK
/etc/lightdm/lightdm-gtk-greeter.conf

# Options de configuration du greeter GTK
# [greeter]
# background=/usr/share/backgrounds/debian.jpg   (fond d'écran)
# theme-name=Greybird                            (thème GTK)
# icon-theme-name=elementary-xfce                (icônes)
# font-name=Noto Sans 11                         (police)
# indicators=~host;~spacer;~clock;~spacer;~session;~a11y;~power
#                                                 (indicateurs dans la barre)

# Personnaliser davantage le greeter
sudo apt install lightdm-gtk-greeter-settings  
lightdm-gtk-greeter-settings   # Interface graphique de configuration  
```

### Changement de gestionnaire de connexion

Lorsque plusieurs gestionnaires de connexion sont installés simultanément, Debian propose un dialogue de sélection via `debconf`. Ce choix peut être modifié à tout moment :

```bash
# Méthode 1 : via dpkg-reconfigure (interface interactive)
sudo dpkg-reconfigure gdm3
# ou
sudo dpkg-reconfigure sddm
# ou
sudo dpkg-reconfigure lightdm
# (l'un des trois affichera un dialogue permettant de choisir le DM par défaut)

# Méthode 2 : manipulation directe du lien symbolique systemd
sudo systemctl disable gdm3  
sudo systemctl enable sddm  
# Puis redémarrer

# Vérifier quel gestionnaire est actif
cat /etc/X11/default-display-manager
```

### Fichiers de session (.desktop)

Le sélecteur de session du gestionnaire de connexion affiche les environnements disponibles en scannant les fichiers `.desktop` dans deux répertoires :

```bash
# Sessions X11 (Xorg)
ls /usr/share/xsessions/
# gnome.desktop
# gnome-xorg.desktop
# plasma.desktop
# xfce.desktop
# lxqt.desktop
# LXDE.desktop
# ...

# Sessions Wayland
ls /usr/share/wayland-sessions/
# gnome.desktop
# gnome-wayland.desktop
# plasma-wayland.desktop     (Plasma 5)
# plasma.desktop             (Plasma 6, Wayland par défaut)
# ...
```

Chaque fichier `.desktop` de session contient les informations nécessaires au lancement de l'environnement :

```ini
# Exemple : /usr/share/xsessions/xfce.desktop
[Desktop Entry]
Name=Xfce Session  
Comment=Use this session to run Xfce as your desktop environment  
Exec=startxfce4  
Type=Application  
DesktopNames=XFCE  
```

Si un environnement installé n'apparaît pas dans le sélecteur de session, il faut vérifier la présence de son fichier `.desktop` dans ces répertoires.

## Gestion de la cohabitation de plusieurs DE

### Installation d'un second environnement

L'installation d'un second environnement de bureau sur un système existant est techniquement simple. Le processus ajoute les composants du nouvel environnement sans supprimer l'existant :

```bash
# Exemple : ajouter KDE Plasma sur un système GNOME
sudo apt install kde-plasma-desktop

# Exemple : ajouter XFCE sur un système GNOME
sudo apt install xfce4 xfce4-goodies

# Après l'installation, le nouvel environnement apparaît
# dans le sélecteur de session du gestionnaire de connexion
```

Lors de l'installation d'un nouvel environnement, le système peut proposer de changer de gestionnaire de connexion si le nouvel environnement en installe un différent. Ce choix est libre, mais il est recommandé d'utiliser le gestionnaire de connexion associé à l'environnement principal.

### Problèmes courants de la cohabitation

La cohabitation de plusieurs environnements de bureau sur un même système fonctionne, mais elle entraîne plusieurs effets secondaires qu'il est important de connaître.

#### Doublons d'applications dans les menus

Chaque environnement installe ses propres applications (gestionnaire de fichiers, terminal, éditeur, visionneuse d'images, etc.). Les menus de chaque environnement affichent toutes les applications installées, y compris celles des autres environnements. On retrouve ainsi Nautilus, Dolphin et Thunar dans les menus, trois terminaux différents, plusieurs éditeurs de texte, etc.

Solution partielle : masquer les applications indésirables dans un environnement spécifique en créant des fichiers `.desktop` locaux avec la directive `NoDisplay=true` :

```bash
# Masquer une application dans les menus
# Copier le fichier .desktop de l'application dans le répertoire local
cp /usr/share/applications/org.kde.dolphin.desktop ~/.local/share/applications/

# Éditer la copie et ajouter ou modifier la ligne
# NoDisplay=true

# Pour masquer une application uniquement dans certains environnements,
# utiliser la directive OnlyShowIn ou NotShowIn
# OnlyShowIn=KDE;           → n'apparaît que dans KDE
# NotShowIn=GNOME;XFCE;     → n'apparaît pas dans GNOME et XFCE
```

#### Conflits de thèmes et d'apparence

Les applications GTK dans un environnement Qt (et inversement) peuvent présenter une apparence incohérente. Les thèmes de l'un ne s'appliquent pas automatiquement aux applications de l'autre :

```bash
# Sous KDE Plasma : configurer l'apparence des applications GTK
# Paramètres système → Apparence → Style de l'application → Style des applications GNOME/GTK
# Sélectionner le thème Breeze GTK pour la cohérence

# Installer le thème GTK correspondant au thème Qt
sudo apt install breeze-gtk-theme

# Sous GNOME : configurer l'apparence des applications Qt
# Installer qt6ct (ou qt5ct pour Qt5) pour contrôler l'apparence Qt
sudo apt install qt6ct

# Définir la variable d'environnement pour que les applications Qt
# utilisent qt6ct comme plateforme de style
# Ajouter dans ~/.profile ou ~/.bashrc :
export QT_QPA_PLATFORMTHEME=qt6ct
```

#### Conflits de services et de démons

Certains services peuvent entrer en conflit lorsque deux environnements essaient de gérer la même fonctionnalité. Les conflits les plus fréquents concernent :

- **La gestion de l'alimentation** : `xfce4-power-manager`, `powerdevil` (KDE) et `gnome-settings-daemon` (composant power) peuvent interférer.
- **La gestion du réseau** : NetworkManager est partagé, mais les applets de chaque environnement peuvent créer des incohérences dans la zone de notification.
- **Le trousseau de clés** : `gnome-keyring` et `kwallet` peuvent coexister mais l'utilisateur doit potentiellement s'authentifier dans les deux.
- **Les portails XDG** : `xdg-desktop-portal-gnome` et `xdg-desktop-portal-kde` peuvent entrer en conflit pour les applications Flatpak.

Pour les portails XDG, la configuration de priorité se fait dans :

```bash
# Forcer l'utilisation d'un portail spécifique par environnement
# Créer ou éditer ~/.config/xdg-desktop-portal/portals.conf

# Exemple pour GNOME
[preferred]
default=gnome  
org.freedesktop.impl.portal.FileChooser=gnome  
org.freedesktop.impl.portal.Screenshot=gnome  

# Exemple pour KDE
[preferred]
default=kde
```

#### Impact sur l'espace disque

L'installation de plusieurs environnements complets consomme un espace disque significatif. À titre indicatif :

| Installation | Espace disque approximatif |
|--------------|---------------------------|
| GNOME complet (`task-gnome-desktop`) | ~3 – 4 Go |
| KDE Plasma complet (`task-kde-desktop`) | ~3 – 4 Go |
| XFCE complet (`task-xfce-desktop`) | ~2 – 3 Go |
| LXQt complet (`task-lxqt-desktop`) | ~2 Go |
| GNOME + KDE Plasma (cohabitation) | ~5 – 6 Go (composants partagés) |

### Recommandations pour la cohabitation

La cohabitation est utile pour tester différents environnements avant de faire un choix définitif. Pour un usage pérenne, les recommandations sont :

- **Choisir un environnement principal** et n'installer que des applications individuelles de l'autre environnement si nécessaire (par exemple, installer Dolphin sous GNOME ou Nautilus sous KDE, plutôt que l'environnement complet).
- **Utiliser le gestionnaire de connexion de l'environnement principal** pour une intégration optimale.
- **Éviter plus de deux environnements complets simultanés** : les conflits de services et la pollution des menus augmentent rapidement.
- **Tester les environnements dans une machine virtuelle** avant de les installer sur le système principal.

## Suppression d'un environnement de bureau

### Suppression propre

La suppression d'un environnement de bureau nécessite de retirer le méta-paquet et ses dépendances, puis de nettoyer les paquets devenus orphelins :

```bash
# Exemple : supprimer KDE Plasma
# 1. Supprimer le méta-paquet et ses dépendances
sudo apt remove --purge task-kde-desktop kde-plasma-desktop

# 2. Supprimer les paquets qui ne sont plus nécessaires
sudo apt autoremove --purge

# 3. Vérifier les paquets KDE restants
dpkg -l | grep kde  
dpkg -l | grep plasma  

# 4. Si des paquets KDE subsistent et ne sont pas souhaités
sudo apt remove --purge $(dpkg -l | grep kde | awk '{print $2}')

# 5. Nettoyer le cache APT
sudo apt clean
```

> **Attention** : avant de supprimer un environnement de bureau, s'assurer qu'un autre environnement est installé et fonctionnel, et que le gestionnaire de connexion actif ne dépend pas de l'environnement supprimé. Supprimer GNOME alors que GDM3 est le gestionnaire de connexion actif peut empêcher le démarrage graphique.

```bash
# Changer de gestionnaire de connexion AVANT de supprimer l'environnement
sudo dpkg-reconfigure lightdm
# Sélectionner LightDM ou SDDM selon l'environnement restant
# Puis procéder à la suppression
```

### Nettoyage des fichiers de configuration utilisateur

La suppression des paquets ne supprime pas les fichiers de configuration situés dans le répertoire personnel de l'utilisateur. Pour un nettoyage complet :

```bash
# Fichiers de configuration GNOME
rm -rf ~/.config/gnome-*  
rm -rf ~/.config/dconf/  
rm -rf ~/.local/share/gnome-shell/  

# Fichiers de configuration KDE/Plasma
rm -rf ~/.config/kde*  
rm -rf ~/.config/plasma*  
rm -rf ~/.config/kwin*  
rm -rf ~/.local/share/plasma/  
rm -rf ~/.local/share/kscreen/  

# Fichiers de configuration XFCE
rm -rf ~/.config/xfce4/  
rm -rf ~/.cache/xfce4/  

# Fichiers de configuration LXQt
rm -rf ~/.config/lxqt/

# Fichiers de configuration LXDE
rm -rf ~/.config/lxsession/  
rm -rf ~/.config/lxpanel/  
rm -rf ~/.config/openbox/lxde-rc.xml  
```

> **Note** : ces opérations sont irréversibles. Il est recommandé de sauvegarder ces répertoires avant de les supprimer, au cas où un retour en arrière serait nécessaire.

## Configuration du démarrage graphique

### Targets systemd

Le démarrage en mode graphique ou en mode texte est contrôlé par les targets systemd :

```bash
# Vérifier le mode de démarrage actuel
systemctl get-default

# Résultats possibles :
# graphical.target   → démarrage en mode graphique (gestionnaire de connexion)
# multi-user.target  → démarrage en mode texte (console)

# Passer en mode graphique au prochain démarrage
sudo systemctl set-default graphical.target

# Passer en mode texte au prochain démarrage
sudo systemctl set-default multi-user.target

# Basculer immédiatement (sans redémarrer)
# Vers le mode graphique
sudo systemctl isolate graphical.target

# Vers le mode texte (ferme toutes les sessions graphiques !)
sudo systemctl isolate multi-user.target
```

### Démarrage manuel d'une session graphique

Sur un système configuré en mode `multi-user.target` (sans gestionnaire de connexion), il est possible de démarrer manuellement une session graphique via `startx` ou en lançant le gestionnaire de connexion à la demande :

```bash
# Méthode 1 : lancer le gestionnaire de connexion manuellement
sudo systemctl start gdm3
# ou
sudo systemctl start sddm
# ou
sudo systemctl start lightdm

# Méthode 2 : démarrer une session X avec startx
# Nécessite le paquet xinit
sudo apt install xinit

# Créer ou éditer ~/.xinitrc pour spécifier l'environnement à lancer
echo "exec startxfce4" > ~/.xinitrc
# ou
echo "exec startplasma-x11" > ~/.xinitrc

# Lancer la session
startx

# Note : cette méthode ne lance pas de gestionnaire de connexion
# L'authentification se fait dans le terminal avant startx
```

### Configuration de la connexion automatique

Pour les postes de travail mono-utilisateur (machines personnelles, bornes), la connexion automatique évite la saisie du mot de passe au démarrage :

```bash
# GDM3 (GNOME)
# Éditer /etc/gdm3/daemon.conf
[daemon]
AutomaticLoginEnable=true  
AutomaticLogin=nom_utilisateur  

# SDDM (KDE Plasma)
# Éditer /etc/sddm.conf.d/autologin.conf (créer le fichier si nécessaire)
[Autologin]
User=nom_utilisateur  
Session=plasma.desktop  

# LightDM
# Éditer /etc/lightdm/lightdm.conf
[Seat:*]
autologin-user=nom_utilisateur  
autologin-user-timeout=0  
# Note : l'utilisateur doit appartenir au groupe autologin
sudo groupadd -r autologin  
sudo gpasswd -a nom_utilisateur autologin  
```

> **Sécurité** : la connexion automatique est déconseillée sur les machines accessibles physiquement par d'autres personnes, les ordinateurs portables susceptibles d'être volés et les serveurs.

## Dépannage

### L'interface graphique ne démarre pas

Si le système démarre en mode texte alors qu'un environnement graphique est installé :

```bash
# 1. Vérifier que le target graphique est configuré
systemctl get-default

# 2. Vérifier l'état du gestionnaire de connexion
systemctl status display-manager.service

# 3. Consulter les logs du gestionnaire de connexion
journalctl -u gdm3 -b
# ou
journalctl -u sddm -b
# ou
journalctl -u lightdm -b

# 4. Vérifier les logs du serveur d'affichage
# Pour Xorg :
cat /var/log/Xorg.0.log | grep -E "(EE|WW)"
# (EE) = erreur, (WW) = avertissement

# Pour Wayland :
journalctl --user -b | grep -iE "mutter|kwin|wayland"

# 5. Vérifier que les pilotes graphiques sont chargés
lsmod | grep -iE "i915|amdgpu|nouveau|nvidia"

# 6. Tester le lancement manuel du serveur X
# (depuis un TTY, Ctrl+Alt+F2)
startx 2>&1 | tee /tmp/startx.log
```

### Écran noir après connexion

Un écran noir après la saisie du mot de passe indique généralement un problème dans le lancement de la session :

```bash
# 1. Basculer vers un TTY (Ctrl+Alt+F2) et se connecter en mode texte

# 2. Vérifier les fichiers de log de la session
cat ~/.local/share/xorg/Xorg.0.log 2>/dev/null | grep "(EE)"  
journalctl --user -b --no-pager | tail -50  

# 3. Vérifier les fichiers d'initialisation de session
# Un fichier .xinitrc ou .xsession corrompu peut empêcher le démarrage
ls -la ~/.xinitrc ~/.xsession ~/.xprofile 2>/dev/null
# En cas de doute, renommer ces fichiers pour tester
mv ~/.xinitrc ~/.xinitrc.bak 2>/dev/null  
mv ~/.xsession ~/.xsession.bak 2>/dev/null  

# 4. Tester avec un utilisateur différent (élimine les problèmes de config utilisateur)
sudo -u autre_utilisateur startx

# 5. Réinitialiser la configuration de l'environnement de bureau
# GNOME : supprimer le cache dconf
mv ~/.config/dconf ~/.config/dconf.bak
# KDE : renommer les fichiers de configuration
mv ~/.config/plasma-org.kde.plasma.desktop-appletsrc \
   ~/.config/plasma-org.kde.plasma.desktop-appletsrc.bak
# XFCE : renommer le répertoire de configuration
mv ~/.config/xfce4 ~/.config/xfce4.bak
```

### Résolution ou affichage incorrect

```bash
# Vérifier les écrans détectés et les résolutions supportées
# Sous X11 :
xrandr

# Sous Wayland (GNOME) :
gnome-randr  # si installé, sinon utiliser les Paramètres
# Sous Wayland (KDE) :
kscreen-doctor --outputs

# Forcer une résolution sous X11
xrandr --output HDMI-1 --mode 1920x1080 --rate 60

# Rendre le changement permanent sous X11
# Créer /etc/X11/xorg.conf.d/10-resolution.conf
# Section "Monitor"
#     Identifier "HDMI-1"
#     Option "PreferredMode" "1920x1080"
# EndSection
```

### Le gestionnaire de connexion ne propose pas tous les environnements

```bash
# Vérifier que les fichiers de session existent
ls /usr/share/xsessions/  
ls /usr/share/wayland-sessions/  

# Si un fichier de session est manquant, vérifier que le paquet est installé
dpkg -S /usr/share/xsessions/xfce.desktop
# Résultat : xfce4-session: /usr/share/xsessions/xfce.desktop

# Vérifier que le paquet de session est installé
dpkg -l | grep xfce4-session

# Si le fichier existe mais l'environnement n'apparaît pas,
# vérifier que le fichier n'est pas masqué
grep -i "hidden\|nodisplay" /usr/share/xsessions/xfce.desktop
```

### Revenir à un état fonctionnel depuis un TTY

En cas de problème graphique rendant le bureau inutilisable, les consoles virtuelles (TTY) permettent de reprendre le contrôle :

```bash
# Accéder à un TTY
# Ctrl+Alt+F2 (ou F3, F4, F5, F6)
# Le TTY 1 (Ctrl+Alt+F1) est généralement occupé par le gestionnaire de connexion
# Les TTY 2 à 6 sont des consoles texte

# Se connecter avec son nom d'utilisateur et mot de passe

# Arrêter le gestionnaire de connexion
sudo systemctl stop gdm3

# Redémarrer le gestionnaire de connexion
sudo systemctl restart gdm3

# Si nécessaire, passer temporairement en mode texte
sudo systemctl isolate multi-user.target

# Effectuer les réparations nécessaires, puis revenir en mode graphique
sudo systemctl isolate graphical.target
```

## Résumé

L'installation et la configuration d'un environnement de bureau sous Debian sont des opérations bien documentées et relativement simples grâce au système de méta-paquets et de tâches. L'approche recommandée dépend du contexte : `tasksel` ou les méta-paquets `task-*-desktop` pour une installation complète, les méta-paquets intermédiaires pour un environnement sans bureautique, et les paquets `*-core` ou `*-desktop` pour une installation minimale.

La cohabitation de plusieurs environnements est possible mais entraîne des effets secondaires (doublons d'applications, conflits de thèmes, consommation d'espace disque) qu'il convient d'anticiper. Pour un usage quotidien, le choix d'un environnement principal complété par quelques applications individuelles d'autres environnements est l'approche la plus propre.

La section suivante (2.1.5) aborde en profondeur la distinction entre Wayland et Xorg, un sujet transversal qui influence le choix et la configuration de l'environnement de bureau.

⏭️ [Wayland vs Xorg](/module-02-debian-desktop/01.5-wayland-vs-xorg.md)
