# 2.2.2 Installation de logiciels courants

## Introduction

L'un des premiers réflexes après l'installation d'un bureau Debian est d'installer les applications nécessaires au quotidien. Debian propose un catalogue de près de **70 000 paquets binaires** (69 830 dans Trixie à sa sortie) dans ses dépôts officiels, couvrant la quasi-totalité des besoins. Cependant, certains logiciels populaires (Google Chrome, Spotify, Visual Studio Code, Steam) ne figurent pas dans les dépôts officiels pour des raisons de licence, et doivent être installés depuis des sources externes.

Cette section présente les logiciels courants par catégorie d'usage, avec pour chacun la méthode d'installation recommandée sous Debian : paquet officiel, Flatpak ou dépôt tiers. L'objectif est de fournir un guide pratique pour constituer un poste de travail complet, que ce soit pour un usage personnel, professionnel ou de développement.

## Navigateurs web

### Firefox ESR (installé par défaut)

Firefox ESR (Extended Support Release) est le navigateur par défaut de Debian. La version ESR reçoit les correctifs de sécurité pendant environ un an sans changement majeur de fonctionnalités, ce qui correspond à la philosophie de stabilité de Debian.

```bash
# Normalement déjà installé
sudo apt install firefox-esr

# Firefox ESR utilise un profil séparé de Firefox standard
# Les extensions et les paramètres sont stockés dans
~/.mozilla/firefox/
```

Firefox ESR est en retard de quelques versions par rapport à la dernière version stable de Firefox. Si la version la plus récente est nécessaire (pour des raisons de compatibilité web ou de fonctionnalités), Firefox standard est disponible via Flatpak.

### Firefox (dernière version stable)

```bash
# Via Flatpak (méthode recommandée pour la dernière version)
flatpak install flathub org.mozilla.firefox

# Cohabitation avec Firefox ESR
# Les deux versions peuvent coexister. Elles utilisent des profils séparés.
# Firefox Flatpak stocke ses données dans
~/.var/app/org.mozilla.firefox/
```

### Chromium (version libre de Chrome)

Chromium est la base open source de Google Chrome. Il est disponible dans les dépôts Debian et offre une expérience très proche de Chrome sans les composants propriétaires Google (synchronisation du compte Google, certains codecs intégrés, mises à jour automatiques).

```bash
# Via les dépôts Debian
sudo apt install chromium

# Ou via Flatpak pour une version plus récente
flatpak install flathub org.chromium.Chromium
```

### Google Chrome (propriétaire)

Google Chrome n'est pas dans les dépôts Debian en raison de sa licence propriétaire. Google fournit son propre dépôt APT.

```bash
# Télécharger et installer le paquet .deb depuis le site Google
# Le paquet .deb ajoute automatiquement le dépôt APT de Google
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb  
sudo apt install ./google-chrome-stable_current_amd64.deb  

# Vérifier que le dépôt a bien été ajouté
cat /etc/apt/sources.list.d/google-chrome.list

# Les mises à jour seront ensuite gérées par apt update/upgrade
```

### Autres navigateurs

```bash
# Brave (basé sur Chromium, axé vie privée)
# Via Flatpak
flatpak install flathub com.brave.Browser

# Vivaldi (basé sur Chromium, très personnalisable)
# Via le dépôt officiel Vivaldi
wget -qO- https://repo.vivaldi.com/archive/linux_signing_key.pub | \
  gpg --dearmor | sudo tee /usr/share/keyrings/vivaldi-browser.gpg > /dev/null
echo "deb [signed-by=/usr/share/keyrings/vivaldi-browser.gpg] \
  https://repo.vivaldi.com/archive/deb/ stable main" | \
  sudo tee /etc/apt/sources.list.d/vivaldi.list
sudo apt update && sudo apt install vivaldi-stable

# GNOME Web (Epiphany) — navigateur minimaliste intégré à GNOME
sudo apt install epiphany-browser

# Tor Browser — navigation anonyme via le réseau Tor
# Via Flatpak (méthode la plus simple)
flatpak install flathub com.github.nickvergessen.TorBrowser
# Ou via le paquet Debian
sudo apt install torbrowser-launcher
```

## Suite bureautique

### LibreOffice (installé par défaut)

LibreOffice est la suite bureautique par défaut de Debian. Elle comprend un traitement de texte (Writer), un tableur (Calc), un logiciel de présentation (Impress), un outil de dessin vectoriel (Draw), un gestionnaire de bases de données (Base) et un éditeur de formules mathématiques (Math).

```bash
# Normalement déjà installé avec les tâches desktop
sudo apt install libreoffice

# Installation de composants individuels si nécessaire
sudo apt install libreoffice-writer    # Traitement de texte  
sudo apt install libreoffice-calc      # Tableur  
sudo apt install libreoffice-impress   # Présentations  
sudo apt install libreoffice-draw      # Dessin vectoriel  
sudo apt install libreoffice-base      # Base de données  
sudo apt install libreoffice-math      # Formules mathématiques  

# Packs de langue français
sudo apt install libreoffice-l10n-fr   # Interface en français  
sudo apt install libreoffice-help-fr   # Aide en français  

# Dictionnaires et correcteur orthographique
sudo apt install hunspell-fr           # Dictionnaire français  
sudo apt install mythes-fr             # Thésaurus français  
sudo apt install hyphen-fr             # Coupure de mots  

# Intégration avec l'environnement de bureau
sudo apt install libreoffice-gnome     # Intégration GNOME (dialogues natifs)  
sudo apt install libreoffice-qt6       # Intégration Qt 6 (Plasma 6)  
# Note Trixie : le paquet `libreoffice-kde5` (pour Plasma 5) n'est plus
# distribué — Trixie embarquant Plasma 6, c'est `libreoffice-qt6` qui sert
# d'intégration KDE.
```

LibreOffice dans Debian Stable peut être en retard de une à deux versions majeures par rapport à la dernière release upstream. Pour obtenir la version la plus récente :

```bash
# Via Flatpak
flatpak install flathub org.libreoffice.LibreOffice

# Ou via les backports Debian (si disponible)
sudo apt install -t trixie-backports libreoffice
```

### OnlyOffice

OnlyOffice est une suite bureautique alternative qui offre une meilleure compatibilité de rendu avec les formats Microsoft Office (DOCX, XLSX, PPTX). Elle est particulièrement utile dans les environnements où les échanges de documents avec des utilisateurs Microsoft Office sont fréquents.

```bash
# Via Flatpak
flatpak install flathub org.onlyoffice.desktopeditors
```

### Microsoft 365 (Office en ligne)

Microsoft Office n'est pas disponible nativement sous Linux. Les utilisateurs qui dépendent de Microsoft Office peuvent accéder à **Microsoft 365** (anciennement Office 365) via le navigateur web. Les applications en ligne (Word, Excel, PowerPoint, Outlook, Teams) sont entièrement fonctionnelles dans Firefox et Chromium.

## Bonnes pratiques pour l'installation de logiciels

### Privilégier les sources officielles

L'ordre de préférence recommandé (présenté dans la section 2.2) reste la référence : dépôts Debian d'abord, Flatpak ensuite, dépôts tiers en troisième option. Chaque source ajoutée au système augmente la surface d'attaque et la complexité de maintenance.

### Gérer les dépôts tiers avec précaution

Chaque dépôt tiers ajouté est une source de confiance supplémentaire. Quelques règles de prudence :

```bash
# Toujours utiliser le format moderne de stockage des clés GPG
# (keyrings dans /usr/share/keyrings/ avec signed-by dans sources.list)

# Vérifier les dépôts tiers configurés sur le système
ls /etc/apt/sources.list.d/

# Vérifier les clés GPG importées
ls /usr/share/keyrings/  
ls /etc/apt/trusted.gpg.d/  

# Désactiver temporairement un dépôt sans le supprimer
# Renommer le fichier avec une extension différente
sudo mv /etc/apt/sources.list.d/google-chrome.list \
        /etc/apt/sources.list.d/google-chrome.list.disabled
```

### Éviter les installations manuelles non suivies

Les logiciels installés manuellement (compilation depuis les sources, binaires copiés dans `/usr/local/bin/`, AppImages éparpillées) ne sont pas gérés par le système de paquets. Ils ne reçoivent pas de mises à jour automatiques et peuvent créer des conflits silencieux avec les paquets Debian. Si un logiciel n'est disponible que sous cette forme, il est préférable de le documenter et de le centraliser dans un emplacement dédié :

```bash
# Emplacement recommandé pour les AppImages
mkdir -p ~/Applications

# Emplacement recommandé pour les binaires compilés manuellement
/usr/local/bin/    # Accessible à tous les utilisateurs
~/bin/             # Accessible uniquement à l'utilisateur courant
~/.local/bin/      # Standard freedesktop, ajouté au PATH par défaut dans Debian
```

### Vérifier les mises à jour de toutes les sources

Un système Debian desktop avec Flatpak et des dépôts tiers nécessite plusieurs commandes pour être entièrement à jour :

```bash
# Mettre à jour les paquets Debian (dépôts officiels + tiers)
sudo apt update && sudo apt upgrade

# Mettre à jour les applications Flatpak
flatpak update

# Vérifier les firmwares disponibles (si fwupd est installé)
fwupdmgr refresh && fwupdmgr get-updates
```

Les logithèques graphiques (GNOME Logiciels, KDE Discover) centralisent ces mises à jour dans une interface unique, ce qui simplifie la maintenance pour les utilisateurs non techniques.

## Résumé

Constituer un poste de travail Debian complet est aujourd'hui une opération accessible. Les dépôts officiels couvrent la majorité des besoins en logiciels libres, Flatpak via Flathub comble les manques pour les applications récentes ou propriétaires, et les dépôts tiers des éditeurs permettent d'installer les quelques logiciels restants. La combinaison de ces trois sources offre un catalogue d'applications comparable à celui de n'importe quel autre système d'exploitation desktop.
