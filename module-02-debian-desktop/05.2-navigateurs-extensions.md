# 2.5.2 Navigateurs web et extensions

## Introduction

Le navigateur web est l'application la plus utilisée sur un poste de travail moderne. Il sert de point d'accès à la messagerie, aux outils collaboratifs, aux applications SaaS (Google Workspace, Microsoft 365, Figma, Notion), au streaming, à la documentation technique, aux réseaux sociaux et à la quasi-totalité des services en ligne. Sur de nombreux postes, le navigateur est ouvert en permanence avec des dizaines d'onglets actifs, ce qui en fait le premier consommateur de ressources système.

Debian propose plusieurs navigateurs dans ses dépôts et via des sources externes. Chacun a ses forces, sa philosophie et son écosystème d'extensions. Cette section présente les navigateurs disponibles, leur installation et configuration, les extensions essentielles pour la productivité et la vie privée, et les bonnes pratiques pour une navigation efficace et sécurisée.

## Navigateurs disponibles sous Debian

### Vue d'ensemble

| Navigateur | Moteur | Licence | Source sous Debian | Wayland natif |
|------------|--------|---------|-------------------|---------------|
| **Firefox ESR** | Gecko | Libre (MPL 2.0) | Dépôts officiels | Oui (par défaut) |
| **Firefox** (dernière stable) | Gecko | Libre (MPL 2.0) | Flatpak | Oui |
| **Chromium** | Blink | Libre (BSD) | Dépôts officiels | Via flag `--ozone-platform` |
| **Google Chrome** | Blink | Propriétaire | Dépôt tiers Google | Via flag `--ozone-platform` |
| **GNOME Web (Epiphany)** | WebKitGTK | Libre (GPL) | Dépôts officiels | Oui (natif GTK) |
| **Brave** | Blink | Libre (MPL 2.0) | Flatpak | Via flag |
| **Vivaldi** | Blink | Propriétaire (UI), libre (moteur) | Dépôt tiers Vivaldi | Via flag |
| **Tor Browser** | Gecko (modifié) | Libre | Dépôts / Flatpak | Variable |

### Deux familles de moteurs

Le web contemporain est dominé par deux moteurs de rendu :

**Gecko** (Mozilla) : moteur de Firefox. C'est le seul moteur de rendu majeur indépendant, développé par une organisation à but non lucratif (Mozilla Foundation). Son existence est considérée comme essentielle pour la diversité du web et la prévention d'un monopole de rendu.

**Blink** (Google) : dérivé de WebKit, utilisé par Chromium, Google Chrome, Brave, Vivaldi, Edge, Opera et la quasi-totalité des navigateurs basés sur Electron. La domination de Blink est un sujet de préoccupation pour les défenseurs du web ouvert, car un moteur unique peut dicter l'évolution des standards web.

Le choix entre Firefox (Gecko) et un navigateur basé sur Chromium (Blink) est souvent le premier choix technique à faire. Les deux offrent une expérience de navigation comparable, mais ils diffèrent sur des aspects comme la gestion de la mémoire, l'écosystème d'extensions, la philosophie de vie privée et la compatibilité avec certains sites web.

## Firefox ESR

### Présentation

Firefox ESR (Extended Support Release) est le navigateur par défaut de Debian. La version ESR est une branche de Firefox qui reçoit les correctifs de sécurité pendant environ un an sans changements majeurs de fonctionnalités. Cette approche correspond à la philosophie de stabilité de Debian : les utilisateurs bénéficient des correctifs de sécurité sans les surprises de changements d'interface ou de fonctionnalités.

```bash
# Normalement déjà installé
sudo apt install firefox-esr

# Packs de langue
sudo apt install firefox-esr-l10n-fr

# Vérifier la version
firefox-esr --version
```

### Profils Firefox

Firefox stocke toute sa configuration, ses extensions, ses favoris, son historique et ses cookies dans un **profil**. Chaque profil est un répertoire indépendant, ce qui permet de maintenir plusieurs configurations distinctes :

```bash
# Emplacement des profils
~/.mozilla/firefox/

# Le fichier profiles.ini liste les profils disponibles
cat ~/.mozilla/firefox/profiles.ini

# Lancer le gestionnaire de profils
firefox-esr --ProfileManager

# Créer un profil dédié (utile pour séparer usage personnel/professionnel)
firefox-esr --CreateProfile "Professionnel"

# Lancer Firefox avec un profil spécifique
firefox-esr -P "Professionnel"

# Lancer une instance séparée avec un profil temporaire
firefox-esr --no-remote -P "Autre"
```

### Configuration avancée (about:config)

Firefox dispose d'un système de configuration avancée accessible via `about:config` dans la barre d'adresse. Ce système expose des centaines de paramètres internes modifiables :

```
# Paramètres de performance
browser.cache.disk.capacity          # Taille du cache disque (Ko)  
browser.sessionstore.interval        # Intervalle de sauvegarde de session (ms)  
                                     # Défaut : 15000 (15s). Augmenter à 60000
                                     # réduit les écritures disque

# Paramètres de vie privée
privacy.resistFingerprinting         # Résistance au fingerprinting (false par défaut)  
privacy.trackingprotection.enabled   # Protection contre le pistage (true)  
network.cookie.cookieBehavior        # Comportement des cookies  
                                     # 5 = protection totale (isolement par site)

# Paramètres d'interface
browser.tabs.closeWindowWithLastTab  # Fermer la fenêtre avec le dernier onglet  
browser.urlbar.suggest.searches      # Suggestions de recherche dans la barre d'URL  
browser.newtabpage.activity-stream.feeds.topsites  # Sites fréquents sur les nouveaux onglets  

# Paramètres Wayland
widget.use-xdg-desktop-portal.file-picker  # Utiliser le sélecteur de fichiers natif
                                            # 1 = toujours utiliser le portail XDG
```

### Configuration de Wayland

À partir de Firefox **121** (stable) — et donc Firefox ESR **128 et 140** présents dans Trixie — Mozilla active Wayland par défaut sur les sessions Wayland. Pour vérifier et, le cas échéant, forcer le mode :

```bash
# Vérifier le protocole utilisé
# Dans Firefox : about:support → Window Protocol
# "wayland" = natif, "xwayland" = via XWayland, "x11" = session X11

# Forcer Wayland (si ce n'est pas automatique)
# Ajouter dans ~/.profile ou /etc/environment
MOZ_ENABLE_WAYLAND=1

# Activer le sélecteur de fichiers natif du bureau (portail XDG)
# about:config → widget.use-xdg-desktop-portal.file-picker → 1

# Activer l'accélération matérielle vidéo (VA-API)
# about:config → media.ffmpeg.vaapi.enabled → true
# about:config → gfx.webrender.all → true
```

### Synchronisation Firefox

Firefox Sync permet de synchroniser les favoris, l'historique, les mots de passe, les onglets ouverts et les extensions entre plusieurs appareils :

```
# Créer un compte Firefox
# Menu → Se connecter à Firefox (ou Paramètres → Sync)

# Les données synchronisées sont chiffrées de bout en bout
# Seul l'utilisateur peut déchiffrer ses données

# Données synchronisables :
# - Favoris
# - Historique de navigation
# - Onglets ouverts (accessible sur les autres appareils)
# - Identifiants et mots de passe
# - Extensions installées
# - Paramètres et préférences
# - Adresses et moyens de paiement
```

## Chromium

### Présentation

Chromium est la base open source de Google Chrome. Il offre une expérience très proche de Chrome mais sans les composants propriétaires Google : pas de synchronisation avec un compte Google intégrée, pas de certains codecs propriétaires (Widevine DRM n'est pas inclus par défaut), pas de rapports de crash envoyés à Google.

```bash
# Installation depuis les dépôts Debian
sudo apt install chromium

# Ou via Flatpak pour une version plus récente
flatpak install flathub org.chromium.Chromium

# Pack de langue (Chromium utilise la locale système par défaut)
sudo apt install chromium-l10n
```

### Wayland et Chromium

Chromium ne démarre pas en mode Wayland natif par défaut. Le mode doit être activé explicitement :

```bash
# Lancer Chromium avec Wayland natif (test ponctuel)
chromium --ozone-platform-hint=auto

# Rendre permanent — méthode 1 : flags système (Debian)
# Le paquet Debian fournit un wrapper qui sourcerait /etc/chromium.d/*
sudo tee /etc/chromium.d/wayland > /dev/null << 'EOF'  
CHROMIUM_FLAGS="$CHROMIUM_FLAGS --ozone-platform-hint=auto"  
EOF  
# ⚠️ Attention : le mécanisme `~/.config/chromium-flags.conf`
# documenté dans le wiki ArchLinux N'EST PAS supporté nativement par
# le paquet `chromium` de Debian (différence de packaging).

# Rendre permanent — méthode 2 : modifier le lanceur utilisateur
cp /usr/share/applications/chromium.desktop ~/.local/share/applications/  
sed -i 's|Exec=/usr/bin/chromium|Exec=/usr/bin/chromium --ozone-platform-hint=auto|g' \  
  ~/.local/share/applications/chromium.desktop
update-desktop-database ~/.local/share/applications/
```

### Différences avec Google Chrome

| Aspect | Chromium | Google Chrome |
|--------|----------|---------------|
| **Licence** | Libre (BSD) | Propriétaire |
| **Synchronisation** | Pas de sync Google intégré | Sync Google complète |
| **Codecs** | Codecs libres uniquement | Tous codecs (AAC, H.264, Widevine DRM) |
| **Mises à jour** | Via les dépôts Debian (cycle Debian) | Mises à jour automatiques via le dépôt Google |
| **Télémétrie** | Minimale | Rapports d'utilisation et de crash vers Google |
| **Widevine DRM** | Non inclus (Netflix, Spotify web en SD) | Inclus (streaming HD/4K) |
| **PDF viewer** | Inclus | Inclus |
| **Traduction** | Non inclus | Google Translate intégré |

### Installer Widevine DRM dans Chromium

Pour la lecture de contenus protégés par DRM (Netflix, Disney+, Amazon Prime Video, Spotify web) en qualité HD :

```bash
# Sous Debian, Widevine n'est pas installé par défaut avec Chromium
# Le paquet chromium-sandbox peut être nécessaire

# Option 1 : installer Google Chrome (inclut Widevine)
# Voir section 2.2.2 pour l'installation

# Option 2 : utiliser Firefox ESR (Widevine est téléchargé automatiquement)
# Firefox gère Widevine de manière transparente

# Option 3 : utiliser un navigateur Chromium-like qui embarque Widevine
# (Brave, Vivaldi, Microsoft Edge) — Widevine est inclus dans le paquet officiel
# Note : redistribuer ou extraire Widevine depuis Chrome viole la licence Google.
```

## Google Chrome

```bash
# Installation via le dépôt officiel Google
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb  
sudo apt install ./google-chrome-stable_current_amd64.deb  

# Le paquet .deb configure automatiquement le dépôt APT Google
# Les mises à jour sont ensuite gérées par apt update/upgrade

# Vérifier le dépôt
cat /etc/apt/sources.list.d/google-chrome.list

# Activer Wayland
# ⚠️ Le paquet google-chrome-stable de Google ne lit pas `chrome-flags.conf`
# (c'est un mécanisme spécifique à AUR/ArchLinux, pas au paquet officiel).
# Sur Debian, modifier le lanceur :
cp /usr/share/applications/google-chrome.desktop ~/.local/share/applications/  
sed -i 's|Exec=/usr/bin/google-chrome-stable|Exec=/usr/bin/google-chrome-stable --ozone-platform-hint=auto|g' \  
  ~/.local/share/applications/google-chrome.desktop
update-desktop-database ~/.local/share/applications/
```

Chrome offre la synchronisation complète avec un compte Google (favoris, historique, mots de passe, extensions, onglets), le support DRM Widevine natif, la traduction automatique intégrée et les mises à jour les plus rapides. La contrepartie est la collecte de données d'utilisation par Google.

## GNOME Web (Epiphany)

GNOME Web est le navigateur minimaliste intégré à GNOME. Il utilise le moteur WebKitGTK et offre une interface épurée, parfaitement intégrée à l'esthétique GNOME :

```bash
sudo apt install epiphany-browser
```

GNOME Web est adapté à la navigation occasionnelle et légère. Ses avantages sont la parfaite intégration Wayland et GNOME (portails XDG, thème libadwaita, synchronisation GNOME Online Accounts), la légèreté de la consommation mémoire et le support natif des PWA (Progressive Web Apps) qui peuvent être installées comme des applications de bureau.

Ses limitations sont l'absence d'écosystème d'extensions, un support web parfois incomplet par rapport à Firefox et Chrome (certains sites complexes peuvent ne pas fonctionner correctement), et un nombre réduit de fonctionnalités avancées.

---

## Brave

Brave est un navigateur basé sur Chromium qui se positionne sur la protection de la vie privée. Il bloque nativement les traqueurs, les publicités et les cookies tiers sans nécessiter d'extensions :

```bash
# Via Flatpak
flatpak install flathub com.brave.Browser
```

Brave intègre un bloqueur de publicités et de traqueurs activé par défaut, le support Tor pour la navigation privée (onglets Tor intégrés), une protection contre le fingerprinting et un modèle économique basé sur un système de récompenses optionnel (Brave Rewards) plutôt que sur la collecte de données.

## Extensions essentielles

Les extensions (ou add-ons) ajoutent des fonctionnalités au navigateur. L'écosystème d'extensions est l'un des critères de choix les plus importants entre les navigateurs. Firefox et les navigateurs basés sur Chromium disposent tous deux de catalogues riches, mais avec des architectures différentes.

### Extensions de productivité

| Extension | Disponibilité | Fonction |
|-----------|--------------|----------|
| **uBlock Origin** | Firefox, Chromium | Bloqueur de publicités et de traqueurs. L'extension de protection la plus efficace et la plus légère. Indispensable. |
| **Bitwarden** / **KeePassXC-Browser** | Firefox, Chromium | Gestionnaire de mots de passe. Remplissage automatique des identifiants. |
| **Dark Reader** | Firefox, Chromium | Mode sombre forcé sur tous les sites web. Génère dynamiquement un thème sombre pour les sites qui n'en proposent pas. |
| **Tridactyl** (Firefox) / **Vimium** (Chromium) | Respectifs | Navigation au clavier style Vim. Permet de naviguer, cliquer sur les liens et faire défiler les pages sans toucher la souris. |
| **Tree Style Tab** | Firefox uniquement | Affiche les onglets dans un panneau latéral sous forme d'arborescence. Indispensable pour les utilisateurs qui gèrent des dizaines d'onglets. |
| **SingleFile** | Firefox, Chromium | Sauvegarde une page web complète (HTML, CSS, images) dans un seul fichier HTML autonome. |
| **Zotero Connector** | Firefox, Chromium | Intégration avec le gestionnaire de références bibliographiques Zotero. Capture les métadonnées des articles académiques. |

### Extensions de vie privée

| Extension | Disponibilité | Fonction |
|-----------|--------------|----------|
| **uBlock Origin** | Firefox, Chromium | Bloque également les traqueurs et les scripts de pistage (en plus des publicités). |
| **Privacy Badger** | Firefox, Chromium | Bloque automatiquement les traqueurs tiers en apprenant leur comportement. Développé par l'EFF. |
| **ClearURLs** | Firefox, Chromium | Supprime les paramètres de pistage des URL (utm_source, fbclid, etc.). |
| **Cookie AutoDelete** | Firefox, Chromium | Supprime automatiquement les cookies des onglets fermés, en préservant ceux des sites de confiance. |
| **HTTPS Everywhere** | Firefox, Chromium | Force les connexions HTTPS quand disponible. Moins nécessaire aujourd'hui car la plupart des navigateurs intègrent ce comportement. |
| **NoScript** | Firefox | Bloque l'exécution de JavaScript par défaut, avec autorisation par site. Niveau de protection maximal mais configuration exigeante. |

### Extensions pour le développement web

| Extension | Disponibilité | Fonction |
|-----------|--------------|----------|
| **React Developer Tools** | Firefox, Chromium | Inspection des composants React |
| **Vue.js devtools** | Firefox, Chromium | Inspection des composants Vue.js |
| **JSON Formatter** | Firefox, Chromium | Affichage formaté des réponses JSON |
| **Wappalyzer** | Firefox, Chromium | Identification des technologies utilisées par un site web |
| **Lighthouse** | Chromium (intégré aux DevTools) | Audit de performance, accessibilité, SEO |

### Installation et gestion des extensions

```bash
# Firefox : les extensions sont installées depuis
# https://addons.mozilla.org
# Menu → Extensions et thèmes → Rechercher

# Chromium/Chrome : les extensions sont installées depuis
# https://chrome.google.com/webstore
# (fonctionne aussi pour Brave et Vivaldi)

# Gestion en ligne de commande des extensions Firefox
# Les extensions sont stockées dans le profil
ls ~/.mozilla/firefox/*.default-esr/extensions/

# Les extensions peuvent aussi être déployées par politique système
# (voir la section Déploiement en entreprise)
```

## Gestion de la mémoire et des performances

Les navigateurs modernes sont les applications les plus gourmandes en mémoire sur un poste de travail. Chaque onglet, chaque extension, chaque iframe est un processus ou un thread séparé qui consomme de la RAM.

### Surveiller la consommation

```bash
# Firefox : gestionnaire de tâches interne
# about:processes (ou Shift+Esc dans les versions récentes)
# Affiche la consommation CPU et mémoire par onglet et par extension

# Chromium/Chrome : gestionnaire de tâches interne
# Menu → Plus d'outils → Gestionnaire de tâches
# ou Shift+Esc
# Affiche la mémoire, le CPU et le réseau par onglet, extension et processus

# Depuis le système
ps aux --sort=-%mem | grep -E "firefox|chromium|chrome" | head -10  
smem -t -k -P "firefox|chromium" | tail -5  
```

### Réduire la consommation mémoire

```bash
# Firefox : limiter le nombre de processus de contenu
# about:config → dom.ipc.processCount
# Défaut : 8. Réduire à 4 sur les machines avec peu de RAM
# Chaque processus de contenu gère un ou plusieurs onglets

# Firefox : activer le déchargement automatique des onglets
# about:config → browser.tabs.unloadOnLowMemory → true
# Les onglets inactifs sont déchargés quand la mémoire est faible

# Chromium : limiter les processus
# Lancer avec --renderer-process-limit=4
# Ajouter dans ~/.config/chromium-flags.conf

# Général : réduire le nombre d'onglets ouverts
# Utiliser des extensions de gestion d'onglets :
# - Tab Session Manager (sauvegarde/restauration de sessions d'onglets)
# - Auto Tab Discard (décharge automatiquement les onglets inactifs)
# - OneTab (convertit tous les onglets en une liste de liens)
```

### Accélération matérielle

L'accélération matérielle permet au GPU de gérer le rendu des pages web, les vidéos et les animations CSS/WebGL, réduisant la charge CPU :

```bash
# Firefox : vérifier l'accélération matérielle
# about:support → Section "Compositing"
# "WebRender" ou "WebRender (Software)" indique l'état du rendu

# about:config
# gfx.webrender.all → true (forcer WebRender)
# media.ffmpeg.vaapi.enabled → true (décodage vidéo VA-API)
# layers.acceleration.force-enabled → true (forcer l'accélération)

# Chromium : vérifier l'accélération
# chrome://gpu
# Affiche l'état de chaque fonctionnalité GPU

# Forcer l'accélération si désactivée
# chrome://flags → Override software rendering list → Enabled
# ou via flag : --ignore-gpu-blocklist
```

## Vie privée et sécurité

### Configuration de base pour la vie privée

```bash
# Firefox : la protection renforcée contre le pistage
# Paramètres → Vie privée et sécurité → Protection renforcée contre le pistage
# Trois niveaux :
# Standard : bloque les traqueurs sociaux, cookies tiers, cryptomineurs, fingerprinters
# Stricte : idem + bloque tous les cookies tiers (peut casser certains sites)
# Personnalisée : contrôle fin de chaque catégorie

# DNS over HTTPS (DoH)
# Paramètres → Vie privée et sécurité → DNS via HTTPS
# Fournisseurs : Cloudflare, NextDNS, ou personnalisé
# DoH chiffre les requêtes DNS, empêchant le FAI de voir les sites visités

# Chromium : paramètres de vie privée
# Paramètres → Confidentialité et sécurité
# - Cookies tiers : Bloquer les cookies tiers
# - DNS sécurisé : activer avec le fournisseur souhaité
# - Envoyer une requête "Do Not Track" : activer
```

### Conteneurs Firefox (Multi-Account Containers)

Les conteneurs Firefox sont une fonctionnalité unique qui isole les cookies et les sessions par contexte. Chaque conteneur est un environnement de navigation séparé :

```
# Installer l'extension Firefox Multi-Account Containers
# https://addons.mozilla.org/fr/firefox/addon/multi-account-containers/

# Cas d'usage :
# - Conteneur "Personnel" : comptes personnels (Google, réseaux sociaux)
# - Conteneur "Travail" : comptes professionnels (Microsoft 365, Slack)
# - Conteneur "Achats" : sites e-commerce (isole le pistage publicitaire)
# - Conteneur "Banque" : sites bancaires (isolation maximale)

# Chaque conteneur a sa propre session :
# Être connecté à un compte Google dans le conteneur Personnel
# et à un autre compte dans le conteneur Travail, simultanément

# L'extension Facebook Container (par Mozilla) isole automatiquement
# Facebook/Instagram/WhatsApp dans un conteneur dédié
```

### Moteurs de recherche respectueux de la vie privée

```
# Moteurs alternatifs à Google :
# DuckDuckGo : pas de pistage, résultats provenant de Bing + sources propres
# Startpage : résultats Google sans le pistage
# SearXNG : méta-moteur open source auto-hébergeable
# Qwant : moteur européen (serveurs en France)
# Brave Search : index propre, pas de pistage

# Changer le moteur de recherche par défaut
# Firefox : Paramètres → Recherche → Moteur de recherche par défaut
# Chromium : Paramètres → Moteur de recherche
```

## Déploiement et configuration en entreprise

### Politiques Firefox (policies.json)

Firefox supporte un système de politiques qui permet aux administrateurs de préconfigurer et de verrouiller les paramètres du navigateur pour un déploiement en entreprise :

```bash
# Créer le fichier de politiques
sudo mkdir -p /etc/firefox-esr/policies/  
sudo nano /etc/firefox-esr/policies/policies.json  
```

```json
{
  "policies": {
    "Homepage": {
      "URL": "https://intranet.entreprise.fr",
      "Locked": true
    },
    "SearchEngines": {
      "Default": "DuckDuckGo"
    },
    "ExtensionSettings": {
      "uBlock0@raymondhill.net": {
        "installation_mode": "force_installed",
        "install_url": "https://addons.mozilla.org/firefox/downloads/latest/ublock-origin/latest.xpi"
      }
    },
    "DisableTelemetry": true,
    "DisableFirefoxStudies": true,
    "DisablePocket": true,
    "DNSOverHTTPS": {
      "Enabled": true,
      "ProviderURL": "https://dns.entreprise.fr/dns-query",
      "Locked": true
    },
    "Certificates": {
      "ImportEnterpriseRoots": true
    },
    "PasswordManagerEnabled": false,
    "NoDefaultBookmarks": true
  }
}
```

```bash
# Vérifier que les politiques sont appliquées
# about:policies dans Firefox
# Les paramètres verrouillés apparaissent grisés dans les Paramètres
```

### Politiques Chromium

```bash
# Les politiques Chromium sont définies dans des fichiers JSON
sudo mkdir -p /etc/chromium/policies/managed/  
sudo nano /etc/chromium/policies/managed/enterprise.json  
```

```json
{
  "HomepageLocation": "https://intranet.entreprise.fr",
  "DefaultSearchProviderEnabled": true,
  "DefaultSearchProviderName": "DuckDuckGo",
  "DefaultSearchProviderSearchURL": "https://duckduckgo.com/?q={searchTerms}",
  "BlockExternalExtensions": true,
  "PasswordManagerEnabled": false
}
```

## Résumé

Le choix du navigateur web sous Debian dépend des priorités de l'utilisateur. Firefox ESR est le choix par défaut, recommandé pour sa stabilité, son respect de la vie privée, ses fonctionnalités uniques (conteneurs, Tree Style Tab) et son rôle dans la diversité du web. Chromium et Google Chrome offrent une compatibilité web maximale et une synchronisation avec l'écosystème Google. Brave combine la compatibilité Chromium avec un focus sur la vie privée. GNOME Web convient pour une navigation légère et intégrée.

Les extensions uBlock Origin et un gestionnaire de mots de passe (Bitwarden, KeePassXC) sont considérées comme essentielles quel que soit le navigateur. La configuration de la vie privée (protection contre le pistage, DNS chiffré, moteur de recherche respectueux) mérite une attention particulière dès la première utilisation.

La section suivante (2.5.3) aborde les clients de messagerie électronique, un outil de productivité fondamental pour l'usage professionnel.
