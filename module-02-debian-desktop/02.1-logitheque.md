# 2.2.1 Logithèque et centres d'applications

## Introduction

Les logithèques (ou centres d'applications) sont des interfaces graphiques qui permettent de découvrir, installer, mettre à jour et désinstaller des logiciels sans passer par la ligne de commande. Elles jouent un rôle essentiel dans l'accessibilité de Debian en tant que système desktop, en offrant une expérience comparable aux boutiques d'applications des autres systèmes d'exploitation.

Debian et ses environnements de bureau proposent plusieurs logithèques, chacune avec ses forces, ses sources de paquets supportées et son niveau d'intégration. Cette section présente les principales logithèques disponibles, leur configuration et leur utilisation, ainsi que la technologie sous-jacente qui les alimente : **PackageKit** et **AppStream**.

---

## Technologies sous-jacentes

Avant d'aborder les logithèques elles-mêmes, il est important de comprendre les couches techniques sur lesquelles elles reposent.

### AppStream : les métadonnées applicatives

**AppStream** est un standard freedesktop.org qui définit un format de métadonnées pour les applications. Ces métadonnées enrichissent les informations brutes des paquets (nom, version, dépendances) avec des données orientées utilisateur : description longue, captures d'écran, catégorie, licence, site web, historique des versions, notes de publication et évaluations.

Sans AppStream, une logithèque ne pourrait afficher que les informations techniques du paquet. Avec AppStream, elle peut présenter une fiche d'application complète avec des visuels et une description humainement lisible.

```bash
# Les métadonnées AppStream sont fournies par le paquet
sudo apt install appstream

# Emplacement des métadonnées système
/usr/share/metainfo/              # Métadonnées fournies par les paquets individuels
/usr/share/app-info/              # Catalogue agrégé par la distribution

# Interroger les métadonnées AppStream en ligne de commande
appstreamcli search "text editor"  
appstreamcli get org.gnome.TextEditor  
appstreamcli what-provides mediatype "application/pdf"  

# Rafraîchir le cache AppStream
sudo appstreamcli refresh --force
```

Les métadonnées AppStream sont intégrées dans les dépôts Debian eux-mêmes. Lors d'un `apt update`, les fichiers AppStream sont téléchargés en même temps que les listes de paquets, ce qui permet aux logithèques d'afficher leur catalogue sans téléchargement supplémentaire.

### PackageKit : l'abstraction de la gestion de paquets

**PackageKit** est un service D-Bus qui fournit une interface unifiée pour la gestion de paquets, indépendamment du gestionnaire de paquets sous-jacent (APT pour Debian, DNF pour Fedora, etc.). Les logithèques communiquent avec PackageKit plutôt que directement avec APT, ce qui leur permet de fonctionner de manière identique sur différentes distributions.

```bash
# Installation de PackageKit
sudo apt install packagekit

# Vérifier l'état du service
systemctl status packagekit.service

# PackageKit fournit également un outil en ligne de commande
pkcon search name firefox  
pkcon get-details firefox-esr  
pkcon install vlc  
pkcon get-updates  
```

PackageKit gère les autorisations via Polkit : lorsqu'une installation est demandée depuis la logithèque, un dialogue d'authentification apparaît pour autoriser l'opération avec les droits administrateur.

## GNOME Logiciels (gnome-software)

### Présentation

**GNOME Logiciels** (gnome-software) est la logithèque intégrée à l'environnement GNOME et installée par défaut avec Debian GNOME. Son interface est épurée et orientée vers la découverte d'applications, avec un design qui s'inspire des boutiques d'applications mobiles. Elle est conçue pour les utilisateurs qui souhaitent une expérience simple et visuelle pour gérer leurs logiciels.

```bash
# Installation (normalement déjà présent avec GNOME)
sudo apt install gnome-software

# Lancement
gnome-software &
```

### Interface et navigation

L'interface de GNOME Logiciels se compose de trois sections principales, accessibles via la barre de navigation supérieure :

**Explorer** : la page d'accueil présente des applications mises en avant (choix éditoriaux), des catégories thématiques (Audio & Vidéo, Développement, Graphisme, Jeux, Productivité, etc.) et des sélections curatées. C'est la porte d'entrée pour découvrir de nouveaux logiciels.

**Installées** : cette vue liste toutes les applications graphiques installées sur le système, permettant de les lancer, de consulter leurs détails ou de les désinstaller. Seules les applications disposant de métadonnées AppStream et d'un fichier `.desktop` apparaissent dans cette vue — les paquets système, les bibliothèques et les outils en ligne de commande n'y figurent pas.

**Mises à jour** : cette section centralise les mises à jour disponibles, qu'elles proviennent des dépôts Debian, de Flatpak ou d'autres sources. Les mises à jour peuvent être appliquées individuellement ou globalement. GNOME Logiciels peut également gérer les mises à jour hors-ligne (offline updates), qui s'appliquent au redémarrage pour éviter les incohérences.

### Fiche d'application

Chaque application dispose d'une fiche détaillée qui comprend :

- Le nom, l'icône et une description longue
- Des captures d'écran de l'application
- La version installée et la version disponible
- La source du paquet (Debian, Flathub, etc.)
- La taille de téléchargement et l'espace disque requis
- La licence du logiciel
- Le site web du projet
- Les permissions demandées (pour les applications Flatpak)
- Les avis et évaluations (si disponibles via ODRS — Open Desktop Ratings Service)
- L'historique des versions avec les notes de changement

### Sources de paquets supportées

GNOME Logiciels peut gérer des applications provenant de plusieurs sources simultanément grâce à un système de plugins :

| Source | Plugin | Activé par défaut |
|--------|--------|-------------------|
| **Dépôts Debian (APT)** | packagekit | Oui |
| **Flatpak** | flatpak | Oui, si le plugin est installé |
| **Snap** | snap | Non (plugin disponible séparément) |
| **Firmware (fwupd)** | fwupd | Oui (mises à jour de firmware matériel) |

Lorsqu'une application est disponible dans plusieurs sources (par exemple Firefox dans les dépôts Debian et sur Flathub), GNOME Logiciels affiche les deux versions et permet à l'utilisateur de choisir celle qu'il préfère. Un indicateur de source identifie clairement l'origine de chaque paquet.

### Activation du support Flatpak

Le support Flatpak n'est pas toujours activé par défaut dans GNOME Logiciels sous Debian. Pour l'activer :

```bash
# Installer le plugin Flatpak pour GNOME Logiciels
sudo apt install gnome-software-plugin-flatpak

# Ajouter le dépôt Flathub (source principale d'applications Flatpak)
flatpak remote-add --if-not-exists flathub https://dl.flathub.org/repo/flathub.flatpakrepo

# Redémarrer GNOME Logiciels pour qu'il détecte le nouveau plugin
# Fermer l'application et la relancer, ou se déconnecter/reconnecter
```

Une fois Flathub ajouté, le catalogue de GNOME Logiciels s'enrichit considérablement avec des milliers d'applications supplémentaires, souvent dans leurs versions les plus récentes.

### Mises à jour et notifications

GNOME Logiciels vérifie périodiquement les mises à jour disponibles et affiche une notification lorsque des mises à jour sont prêtes. Le comportement peut être configuré :

```bash
# Paramétrer la fréquence de vérification et le téléchargement automatique
# Via dconf/gsettings
gsettings get org.gnome.software download-updates  
gsettings set org.gnome.software download-updates true  

# Activer/désactiver les notifications de mises à jour
gsettings get org.gnome.software download-updates-notify  
gsettings set org.gnome.software download-updates-notify true  
```

### Limitations

GNOME Logiciels présente certaines limitations qu'il est utile de connaître :

- **Catalogue incomplet** : seules les applications disposant de métadonnées AppStream apparaissent. De nombreux paquets Debian (outils système, bibliothèques, applications sans métadonnées) ne sont pas visibles. Pour ces paquets, il faut utiliser APT en ligne de commande ou Synaptic.
- **Performances** : le chargement initial du catalogue peut être lent, particulièrement après un `apt update` ou lorsque Flathub est activé. Le cache met du temps à se construire.
- **Résolution des conflits** : GNOME Logiciels n'offre pas d'interface pour résoudre les conflits de dépendances complexes. En cas de problème, APT reste l'outil de référence.

## KDE Discover

### Présentation

**KDE Discover** est la logithèque de l'environnement KDE Plasma. Elle propose une interface riche et bien intégrée à l'esthétique Plasma, avec un catalogue d'applications mais également un accès aux extensions Plasma (widgets, thèmes, effets KWin), ce qui en fait un centre de gestion de contenu plus large qu'une simple logithèque.

```bash
# Installation (normalement présent avec KDE Plasma)
sudo apt install plasma-discover

# Lancement
plasma-discover &
```

### Interface et navigation

Discover s'organise autour de plusieurs catégories accessibles depuis le panneau latéral gauche :

**Applications** : le catalogue principal, organisé par catégories (Éducation, Graphisme, Internet, Jeux, Multimédia, Bureautique, Développement, Système, Utilitaires). Une barre de recherche permet de trouver rapidement une application par son nom.

**Extensions Plasma** : c'est une spécificité de Discover. Cette section permet d'installer directement des widgets Plasma (plasmoids), des thèmes globaux, des thèmes de fenêtres, des schémas de couleurs, des thèmes d'icônes et des effets KWin depuis le KDE Store, sans quitter l'application. C'est la méthode la plus pratique pour personnaliser Plasma.

**Mises à jour** : centralise les mises à jour système (paquets Debian), les mises à jour Flatpak et les mises à jour d'extensions Plasma. Discover propose des mises à jour hors-ligne et peut être configuré pour notifier l'utilisateur automatiquement.

### Sources de paquets supportées

| Source | Backend | Activé par défaut |
|--------|---------|-------------------|
| **Dépôts Debian (APT)** | PackageKit | Oui |
| **Flatpak** | plasma-discover-backend-flatpak | Oui, si le backend est installé |
| **Snap** | discover-backend-snap | Non par défaut sous Debian |
| **Firmware (fwupd)** | fwupd | Oui |
| **KDE Store** | knewstuff | Oui (pour les extensions Plasma) |

### Activation du support Flatpak

```bash
# Installer le backend Flatpak pour Discover
sudo apt install plasma-discover-backend-flatpak

# Ajouter Flathub
flatpak remote-add --if-not-exists flathub https://dl.flathub.org/repo/flathub.flatpakrepo

# Redémarrer Discover
```

### Spécificités de Discover

**Choix de la source** : lorsqu'une application est disponible depuis plusieurs sources, Discover affiche clairement les différentes versions et leurs origines. L'utilisateur peut choisir entre le paquet Debian et le Flatpak, avec des informations sur la version et la taille.

**Gestion des extensions Plasma** : Discover est le seul moyen graphique intégré pour installer des widgets, des thèmes et des effets depuis le KDE Store. Les extensions sont catégorisées, notées par la communauté et présentées avec des captures d'écran.

**Mises à jour système** : Discover peut gérer les mises à jour complètes du système (équivalent de `apt upgrade`), pas uniquement les applications graphiques. Il propose des redémarrages lorsque des mises à jour du noyau ou de bibliothèques critiques ont été installées.

## Synaptic : le gestionnaire de paquets avancé

### Présentation

**Synaptic** n'est pas une logithèque au sens moderne du terme, mais un gestionnaire de paquets graphique complet qui donne accès à l'intégralité des dépôts Debian. Là où GNOME Logiciels et Discover ne montrent que les applications disposant de métadonnées AppStream, Synaptic affiche tous les paquets : applications, bibliothèques, outils, fichiers de documentation, paquets de développement — soit près de **70 000 paquets binaires** dans les dépôts Debian Trixie (69 830 à la sortie).

Synaptic est l'interface graphique la plus proche d'APT en ligne de commande. Il est particulièrement utile pour les opérations que les logithèques modernes ne prennent pas en charge : recherche de paquets spécifiques, inspection des dépendances, vérification des fichiers installés, pinning de versions, application de filtres complexes.

```bash
# Installation
sudo apt install synaptic

# Lancement (nécessite les droits administrateur)
sudo synaptic &
# ou via le menu d'applications (un dialogue Polkit demandera le mot de passe)
```

### Interface

L'interface de Synaptic est divisée en plusieurs zones :

**Le panneau de filtres** (à gauche) permet de filtrer les paquets par catégorie (sections Debian), par état (installé, non installé, mis à jour, cassé, obsolète), par origine (dépôt), par filtre personnalisé ou par recherche.

**La liste des paquets** (au centre) affiche les paquets correspondant au filtre sélectionné, avec leur nom, la version installée, la dernière version disponible, et un indicateur d'état visuel (installé, à installer, à supprimer, à mettre à jour).

**Le panneau de détails** (en bas) affiche les informations détaillées du paquet sélectionné : description, dépendances, paquets suggérés, fichiers installés, version du mainteneur, taille.

### Opérations courantes

**Recherche de paquets** : le bouton « Rechercher » ou le raccourci `Ctrl+F` ouvre un dialogue de recherche qui peut porter sur le nom, la description ou le mainteneur du paquet.

**Installation** : clic droit sur un paquet → « Sélectionner pour installation ». Les dépendances sont automatiquement sélectionnées. Cliquer sur « Appliquer » pour lancer l'installation effective.

**Suppression** : clic droit → « Sélectionner pour suppression » (conserve les fichiers de configuration) ou « Sélectionner pour suppression complète » (équivalent de `apt purge`).

**Verrouillage de version** : clic droit → « Verrouiller la version ». Empêche la mise à jour d'un paquet spécifique, ce qui est utile pour conserver une version particulière d'un logiciel.

**Inspection des dépendances** : dans le panneau de détails, l'onglet « Dépendances » affiche l'arbre complet des dépendances d'un paquet, avec un indicateur visuel pour chaque dépendance (satisfaite, manquante, conflit).

### Filtres utiles

Synaptic propose des filtres prédéfinis particulièrement utiles :

| Filtre | Fonction |
|--------|----------|
| **Installés (local ou obsolète)** | Paquets installés qui ne sont plus dans aucun dépôt — utile pour détecter les résidus de dépôts supprimés |
| **Non installés (résidus de configuration)** | Paquets supprimés mais dont les fichiers de configuration persistent — équivalent de ce que `apt purge` nettoierait |
| **Mis à jour** | Paquets pour lesquels une nouvelle version est disponible |
| **Cassés** | Paquets dont les dépendances ne sont pas satisfaites — essentiel pour diagnostiquer un système en état incohérent |

### Synaptic vs les logithèques modernes

| Critère | Synaptic | GNOME Logiciels / Discover |
|---------|----------|---------------------------|
| **Couverture** | Tous les paquets Debian (60 000+) | Applications avec métadonnées AppStream uniquement |
| **Captures d'écran** | Non | Oui |
| **Support Flatpak** | Non | Oui |
| **Inspection des dépendances** | Oui, détaillée | Non |
| **Verrouillage de version** | Oui | Non |
| **Résolution de conflits** | Oui, interactive | Limitée |
| **Interface** | Utilitaire, technique | Moderne, visuelle |
| **Public cible** | Administrateurs, utilisateurs avancés | Utilisateurs de bureau |
| **Mises à jour firmware** | Non | Oui (via fwupd) |

Synaptic et les logithèques modernes sont complémentaires. Synaptic est l'outil de choix lorsqu'on sait précisément quel paquet on cherche ou lorsqu'on doit effectuer des opérations avancées sur les paquets. Les logithèques modernes sont préférables pour la découverte d'applications et la gestion quotidienne des mises à jour.

## Autres outils graphiques

### GDebi : installateur de paquets .deb individuels

**GDebi** est un outil spécialisé dans l'installation de fichiers `.deb` téléchargés manuellement (hors dépôts). Lorsqu'un utilisateur télécharge un fichier `.deb` depuis un site web (par exemple Google Chrome, Slack, ou un logiciel propriétaire), un double-clic ouvre GDebi qui affiche les informations du paquet, vérifie les dépendances et permet l'installation.

```bash
# Installation
sudo apt install gdebi

# Utilisation en ligne de commande
sudo gdebi paquet-telecharge.deb

# Utilisation graphique
gdebi-gtk paquet-telecharge.deb
# ou simplement double-cliquer sur le fichier .deb dans le gestionnaire de fichiers
```

GDebi est supérieur à `dpkg -i` pour l'installation de fichiers `.deb` isolés car il résout automatiquement les dépendances en les téléchargeant depuis les dépôts APT. `dpkg` seul ne gère pas les dépendances et échoue si elles ne sont pas déjà satisfaites.

Sans GDebi, GNOME Logiciels tente de prendre en charge l'ouverture des fichiers `.deb`, mais cette fonctionnalité est souvent moins fiable et moins informative que GDebi dédié.

### Apper : alternative PackageKit pour KDE (retiré de Trixie)

**Apper** était un gestionnaire de paquets graphique basé sur PackageKit, initialement conçu pour KDE. Il a été **retiré de Debian Trixie** : `apt install apper` retourne *Unable to locate package*. Sur la branche stable actuelle, il n'a plus de remplaçant graphique direct dans cette niche (entre Discover et Synaptic). Pour un accès KDE plus direct aux paquets Debian que ce que Discover offre, les options pratiques sont :

- **Synaptic** (interface GTK universelle) ;
- la ligne de commande `apt` / `aptitude` lancée depuis Konsole ;
- l'utilitaire `pkcon` du paquet `packagekit-tools` pour scripter les opérations PackageKit.

```bash
# Recherche / installation via PackageKit en CLI
pkcon search name firefox  
pkcon install firefox-esr  
```

## Configuration de Flatpak pour les logithèques

Flatpak étant devenu un élément central de l'écosystème desktop Linux, sa configuration mérite une attention particulière dans le contexte des logithèques.

### Installation et configuration initiale

```bash
# Installer Flatpak
sudo apt install flatpak

# Ajouter le dépôt Flathub (principal dépôt Flatpak)
flatpak remote-add --if-not-exists flathub https://dl.flathub.org/repo/flathub.flatpakrepo

# Installer le plugin pour la logithèque de votre environnement
# GNOME
sudo apt install gnome-software-plugin-flatpak
# KDE
sudo apt install plasma-discover-backend-flatpak

# Un redémarrage de session est recommandé pour que les portails XDG
# et les intégrations desktop fonctionnent correctement
```

### Cohabitation Flatpak et paquets Debian

Lorsque Flathub est activé dans la logithèque, une même application peut apparaître deux fois : une fois comme paquet Debian et une fois comme Flatpak. Les logithèques permettent de choisir la source, mais il est utile de comprendre les implications de chaque choix.

| Aspect | Paquet Debian (.deb) | Flatpak (Flathub) |
|--------|---------------------|-------------------|
| **Version** | Celle de Debian Stable (souvent plus ancienne) | Dernière version upstream (généralement la plus récente) |
| **Intégration système** | Complète (thèmes, polices, raccourcis, services) | Bonne mais parfois imparfaite (thèmes GTK/Qt, accès fichiers) |
| **Sécurité** | Mises à jour de sécurité Debian | Mises à jour par le mainteneur Flathub |
| **Sandboxing** | Aucun (accès complet au système) | Oui (permissions contrôlées via portails XDG) |
| **Espace disque** | Optimisé (dépendances partagées avec le système) | Plus important (runtimes embarqués, partagés entre Flatpaks) |
| **Performances** | Natives | Généralement identiques, parfois légère latence au démarrage |
| **Mises à jour** | Via `apt upgrade` | Via `flatpak update` (ou via la logithèque) |

En règle générale, les applications système (terminal, gestionnaire de fichiers, éditeur de texte) sont mieux servies par les paquets Debian pour leur intégration optimale. Les applications tierces (Spotify, Discord, Slack, Steam, navigateurs alternatifs, logiciels créatifs) sont souvent plus récentes et mieux maintenues en Flatpak.

### Gestion des permissions Flatpak

Les applications Flatpak s'exécutent dans un sandbox. Leurs permissions peuvent être consultées et modifiées :

```bash
# Lister les permissions d'une application Flatpak
flatpak info --show-permissions org.mozilla.firefox

# L'outil Flatseal offre une interface graphique pour gérer les permissions
flatpak install flathub com.github.tchx84.Flatseal

# Flatseal permet de :
# - Accorder ou retirer l'accès au système de fichiers
# - Contrôler l'accès réseau
# - Gérer l'accès aux périphériques (webcam, microphone, GPU)
# - Configurer les variables d'environnement
# - Visualiser les sockets D-Bus et Wayland autorisés
```

GNOME Logiciels et Discover affichent également les permissions des applications Flatpak sur leur fiche de détails, avec la possibilité de les modifier dans certains cas.

## Recommandations par profil d'utilisation

**Utilisateur débutant sur GNOME** : utiliser GNOME Logiciels comme outil principal, avec Flathub activé pour un catalogue étendu. Installer GDebi pour les fichiers `.deb` téléchargés ponctuellement.

**Utilisateur débutant sur KDE** : utiliser Discover comme outil principal, avec le backend Flatpak activé. Discover couvre également la personnalisation de l'environnement (thèmes, widgets).

**Utilisateur intermédiaire** : combiner la logithèque de son environnement pour la découverte et les mises à jour quotidiennes, avec APT en ligne de commande pour les opérations précises et les paquets système.

**Administrateur ou utilisateur avancé** : APT en ligne de commande comme outil principal, complété par Synaptic pour la visualisation des dépendances et le diagnostic. La logithèque de l'environnement reste utile pour les mises à jour Flatpak et firmware.

**Poste d'entreprise** : Synaptic ou APT uniquement, avec les sources Flatpak désactivées si la politique de l'entreprise impose un contrôle strict des logiciels installés. Les dépôts tiers et Flathub peuvent être restreints via la configuration APT et Flatpak.

## Résumé

Les logithèques sous Debian offrent des expériences complémentaires. GNOME Logiciels et KDE Discover proposent une interface moderne orientée vers la découverte et l'installation simple d'applications, avec le support de Flatpak pour accéder aux versions les plus récentes. Synaptic reste l'outil de référence pour un accès complet aux dépôts Debian et les opérations avancées de gestion de paquets. La combinaison d'une logithèque moderne avec APT en ligne de commande couvre l'intégralité des besoins, du débutant à l'administrateur système.
