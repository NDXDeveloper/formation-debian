# 2.3.4 Bluetooth et périphériques

## Introduction

Un poste de travail moderne interagit avec une multitude de périphériques : claviers et souris Bluetooth, casques audio sans fil, manettes de jeu, imprimantes réseau ou USB, scanners, tablettes graphiques, webcams, lecteurs de cartes, clés de sécurité, téléphones connectés, etc. Sous Debian, la prise en charge de ces périphériques repose sur des composants noyau (pilotes, firmware), des services système (BlueZ pour le Bluetooth, CUPS pour l'impression, libinput pour les entrées) et des outils d'intégration propres à chaque environnement de bureau.

Cette section couvre la configuration du Bluetooth et la gestion des principaux types de périphériques rencontrés au quotidien sur un bureau Debian.

## Bluetooth

### Architecture du Bluetooth sous Linux

La pile Bluetooth Linux repose sur plusieurs couches :

```
┌──────────────────────────────────────────────────────┐
│              Environnement de bureau                 │
│  (applet Bluetooth GNOME, KDE, Blueman)              │
├──────────────────────────────────────────────────────┤
│                     BlueZ                            │
│  (démon bluetoothd, pile protocolaire Bluetooth)     │
│  - Gestion des appairages                            │
│  - Profils (A2DP, HFP, HID, OBEX, etc.)              │
│  - Communication D-Bus avec les applications         │
├──────────────────────────────────────────────────────┤
│              Pilote Bluetooth noyau                  │
│  (btusb, btintel, btrtl, btbcm, hci_uart, etc.)      │
├──────────────────────────────────────────────────────┤
│            Firmware Bluetooth                        │
│  (chargé depuis /lib/firmware/)                      │
├──────────────────────────────────────────────────────┤
│          Contrôleur Bluetooth matériel               │
│  (intégré, USB, PCIe)                                │
└──────────────────────────────────────────────────────┘
```

**BlueZ** est la pile Bluetooth officielle de Linux. C'est le composant central qui gère tous les aspects du Bluetooth : découverte des appareils, appairage, authentification, gestion des profils et communication avec les applications via D-Bus. BlueZ est installé par défaut dans la plupart des installations desktop Debian.

### Installation et vérification

```bash
# Installer les composants Bluetooth (normalement déjà présents)
sudo apt install bluetooth bluez bluez-tools

# Vérifier que le service Bluetooth est actif
systemctl status bluetooth.service

# Activer le service au démarrage s'il ne l'est pas
sudo systemctl enable --now bluetooth.service

# Vérifier que le contrôleur Bluetooth est détecté
hciconfig
# ou avec l'outil moderne :
bluetoothctl show
# Affiche l'adresse MAC, le nom, l'état (powered, discoverable, pairable)

# Vérifier les messages noyau liés au Bluetooth
dmesg | grep -i bluetooth  
dmesg | grep -i "btusb\|btintel\|btrtl\|btbcm\|hci"  

# Si le contrôleur n'est pas détecté, un firmware est probablement manquant
# Voir la section 2.3.1 pour l'installation des firmwares Bluetooth
```

### bluetoothctl : l'outil en ligne de commande

`bluetoothctl` est l'outil interactif de BlueZ pour gérer le Bluetooth depuis le terminal. Il offre un contrôle complet sur tous les aspects du Bluetooth :

```bash
# Lancer bluetoothctl (mode interactif)
bluetoothctl

# Commandes de base dans le shell bluetoothctl :
[bluetooth]# power on              # Activer le Bluetooth
[bluetooth]# power off             # Désactiver le Bluetooth
[bluetooth]# show                  # Afficher les informations du contrôleur

# Découverte et appairage
[bluetooth]# scan on               # Lancer la recherche d'appareils
# Les appareils détectés s'affichent au fur et à mesure :
# [NEW] Device AA:BB:CC:DD:EE:FF Mon Casque Bluetooth

[bluetooth]# scan off              # Arrêter la recherche
[bluetooth]# devices               # Lister les appareils découverts

# Appairer un appareil
[bluetooth]# pair AA:BB:CC:DD:EE:FF
# Un code PIN peut être demandé (afficher et confirmer ou saisir)

# Faire confiance à un appareil (reconnexion automatique sans confirmation)
[bluetooth]# trust AA:BB:CC:DD:EE:FF

# Se connecter à un appareil appairé
[bluetooth]# connect AA:BB:CC:DD:EE:FF

# Se déconnecter
[bluetooth]# disconnect AA:BB:CC:DD:EE:FF

# Supprimer un appairage
[bluetooth]# remove AA:BB:CC:DD:EE:FF

# Lister les appareils appairés
[bluetooth]# paired-devices

# Rendre le contrôleur détectable (pour qu'un téléphone puisse le trouver)
[bluetooth]# discoverable on
[bluetooth]# discoverable off

# Quitter
[bluetooth]# exit
```

Utilisation non interactive (dans des scripts) :

```bash
# Activer le Bluetooth
bluetoothctl power on

# Lister les appareils appairés
bluetoothctl paired-devices

# Se connecter à un appareil connu
bluetoothctl connect AA:BB:CC:DD:EE:FF

# Script de connexion automatique
bluetoothctl << EOF  
power on  
connect AA:BB:CC:DD:EE:FF  
EOF  
```

### Outils graphiques Bluetooth

Chaque environnement de bureau fournit son propre outil de gestion Bluetooth :

#### GNOME

GNOME intègre la gestion Bluetooth dans **Paramètres → Bluetooth** et dans le menu système de la barre supérieure. L'interface permet d'activer/désactiver le Bluetooth, de scanner les appareils, de les appairer et de les connecter. Le package `gnome-bluetooth` est installé avec l'environnement GNOME.

```bash
# Normalement déjà installé avec GNOME
sudo apt install gnome-bluetooth-3-common
```

#### KDE Plasma

KDE Plasma propose la gestion Bluetooth via **Paramètres système → Bluetooth** et via l'applet Bluetooth dans la zone de notification. Le module `bluedevil` gère l'intégration.

```bash
# Normalement déjà installé avec KDE Plasma
sudo apt install bluedevil
```

#### Blueman (XFCE, LXQt, gestionnaires de fenêtres)

**Blueman** est un gestionnaire Bluetooth graphique indépendant de l'environnement de bureau. C'est le choix recommandé pour XFCE, LXQt et les gestionnaires de fenêtres qui n'intègrent pas nativement la gestion Bluetooth :

```bash
# Installation
sudo apt install blueman

# Lancer le gestionnaire
blueman-manager &

# L'applet Blueman s'affiche dans la zone de notification
# et propose les fonctions courantes : activer/désactiver, appairer,
# envoyer des fichiers, configurer les profils audio, etc.

# Blueman est aussi utile sous GNOME ou KDE lorsque les outils natifs
# ne fonctionnent pas correctement avec un périphérique spécifique
```

### Profils Bluetooth courants

| Profil | Sigle | Usage | Exemples |
|--------|-------|-------|----------|
| **Advanced Audio Distribution** | A2DP | Écoute audio stéréo haute qualité | Casques, enceintes |
| **Hands-Free** | HFP | Appels téléphoniques (audio + microphone) | Casques, kits mains-libres |
| **Human Interface Device** | HID | Périphériques d'entrée | Claviers, souris, manettes |
| **Object Push** | OPP | Transfert de fichiers simple | Échange entre téléphone et PC |
| **File Transfer** | FTP | Transfert de fichiers avancé | Navigateur de fichiers distant |
| **Personal Area Network** | PAN | Réseau via Bluetooth (tethering) | Partage de connexion |
| **Serial Port** | SPP | Port série virtuel | Modules Arduino, GPS |
| **Battery Service** | BAS | Niveau de batterie | Casques, claviers, souris |

La commutation entre les profils audio (A2DP pour la musique, HFP pour les appels avec microphone) est gérée automatiquement par PipeWire ou peut être forcée manuellement via `pavucontrol` (onglet Configuration).

### Dépannage Bluetooth

```bash
# 1. Le Bluetooth est bloqué (soft block ou hard block)
rfkill list bluetooth
# Si "Soft blocked: yes" :
sudo rfkill unblock bluetooth
# Si "Hard blocked: yes" :
# Vérifier le commutateur matériel / touche Fn du portable

# 2. Le contrôleur n'est pas détecté
# Vérifier le chargement du module noyau
lsmod | grep btusb
# Charger manuellement si absent
sudo modprobe btusb
# Vérifier les logs pour les erreurs de firmware
dmesg | grep -i "bluetooth\|firmware"
# Installer le firmware si manquant (voir section 2.3.1)

# 3. Un appareil ne s'appaire pas
# Supprimer l'ancien appairage et recommencer
bluetoothctl remove AA:BB:CC:DD:EE:FF  
bluetoothctl scan on  
bluetoothctl pair AA:BB:CC:DD:EE:FF  
bluetoothctl trust AA:BB:CC:DD:EE:FF  
bluetoothctl connect AA:BB:CC:DD:EE:FF  

# 4. Le casque Bluetooth est connecté mais pas de son
# Vérifier le profil audio dans pavucontrol
pavucontrol &
# Onglet Configuration : le casque doit être en profil A2DP
# Onglet Lecture : l'application doit utiliser le casque comme sortie

# Vérifier que le plugin Bluetooth de PipeWire est installé
dpkg -l | grep libspa-0.2-bluetooth  
sudo apt install libspa-0.2-bluetooth  
systemctl --user restart pipewire.service  

# 5. Reconnexion automatique qui ne fonctionne pas
# Vérifier que l'appareil est marqué comme "trusted"
bluetoothctl info AA:BB:CC:DD:EE:FF | grep Trusted
# Si "Trusted: no" :
bluetoothctl trust AA:BB:CC:DD:EE:FF

# 6. Redémarrer la pile Bluetooth complète
sudo systemctl restart bluetooth.service  
systemctl --user restart pipewire.service  

# 7. Réinitialisation complète (dernier recours)
sudo systemctl stop bluetooth.service  
sudo rm -rf /var/lib/bluetooth/*    # Supprime TOUS les appairages  
sudo systemctl start bluetooth.service  
```

## Périphériques d'entrée

### libinput : le gestionnaire d'entrée unifié

**libinput** est la bibliothèque qui gère tous les périphériques d'entrée sous Linux moderne : claviers, souris, pavés tactiles, écrans tactiles, tablettes graphiques. Elle est utilisée à la fois par Xorg (via le pilote `xf86-input-libinput`) et par les compositeurs Wayland.

```bash
# Lister les périphériques d'entrée détectés
sudo libinput list-devices

# Informations détaillées sur chaque périphérique :
# - Nom, chemin sysfs, ID vendeur/produit
# - Capacités (pointer, keyboard, touch, tablet, etc.)
# - Boutons, axes, résolution
# - Taille physique (touchpad)

# Surveiller les événements d'entrée en temps réel
sudo libinput debug-events
# Affiche chaque mouvement de souris, frappe clavier, toucher
# Utile pour diagnostiquer si un périphérique est détecté et fonctionne

# Tester un périphérique spécifique
sudo libinput debug-events --device /dev/input/event5
```

### Souris et pavé tactile

#### Configuration du pavé tactile

Le pavé tactile est un périphérique d'entrée fondamental sur les portables. Sa configuration permet d'ajuster de nombreux comportements :

```bash
# GNOME : Paramètres → Souris et pavé tactile
# Options : vitesse du pointeur, défilement naturel, clic par tapotement,
#           clic secondaire (deux doigts ou coin), désactivation pendant la frappe

gsettings set org.gnome.desktop.peripherals.touchpad tap-to-click true  
gsettings set org.gnome.desktop.peripherals.touchpad natural-scroll true  
gsettings set org.gnome.desktop.peripherals.touchpad two-finger-scrolling-enabled true  
gsettings set org.gnome.desktop.peripherals.touchpad speed 0.3  
# speed : -1.0 (très lent) à 1.0 (très rapide)

# Désactiver le pavé tactile lorsqu'une souris externe est connectée
gsettings set org.gnome.desktop.peripherals.touchpad send-events 'disabled-on-external-mouse'

# KDE : Paramètres système → Périphériques d'entrée → Pavé tactile
# Fichier de configuration : ~/.config/touchpadxlibinputrc

# XFCE : Paramètres → Souris et pavé tactile
```

#### Configuration de la souris

```bash
# Profil d'accélération
# "flat" = accélération linéaire (préféré pour le gaming, correspondance 1:1)
# "adaptive" = accélération progressive (défaut, confortable pour le bureau)

# GNOME
gsettings set org.gnome.desktop.peripherals.mouse accel-profile 'flat'  
gsettings set org.gnome.desktop.peripherals.mouse speed 0.0  

# Sous Xorg, configuration via un fichier xorg.conf.d
sudo nano /etc/X11/xorg.conf.d/50-mouse.conf
```

```
Section "InputClass"
    Identifier "My Mouse"
    MatchIsPointer "yes"
    Option "AccelProfile" "flat"
    Option "AccelSpeed" "0"
EndSection
```

#### Souris et claviers Logitech (récepteur Unifying)

Les périphériques Logitech utilisant un récepteur USB Unifying nécessitent un outil spécifique pour l'appairage et la configuration :

```bash
# Solaar : outil de gestion des périphériques Logitech
sudo apt install solaar

# Lancer Solaar (se place dans la zone de notification)
solaar &

# Solaar permet de :
# - Voir les périphériques connectés au récepteur Unifying
# - Appairer de nouveaux périphériques
# - Configurer les paramètres spécifiques (DPI souris, rétroéclairage clavier)
# - Surveiller le niveau de batterie

# Pour les souris gaming Logitech haute résolution
# Piper offre une interface de configuration avancée
sudo apt install piper  
piper &  
# Configure les boutons, le DPI, les macros, le rétroéclairage
# (nécessite le démon libratbagd)
sudo apt install ratbagd  
sudo systemctl enable --now ratbagd.service  
```

### Claviers

#### Disposition du clavier

```bash
# Vérifier la disposition actuelle
setxkbmap -query    # Xorg
# ou
localectl status

# Changer la disposition du clavier
# Via les Paramètres de l'environnement de bureau :
# GNOME : Paramètres → Clavier → Sources d'entrée
# KDE : Paramètres système → Périphériques d'entrée → Clavier → Dispositions
# XFCE : Paramètres → Clavier → Disposition

# En ligne de commande (temporaire)
setxkbmap fr                    # Français AZERTY  
setxkbmap us                    # US QWERTY  
setxkbmap fr -variant bepo      # BÉPO (disposition française ergonomique)  
setxkbmap de                    # Allemand QWERTZ  

# Permanent via localectl
sudo localectl set-x11-keymap fr "" "" ""
# Format : layout variant model options

# Options de clavier utiles
setxkbmap -option caps:escape          # Caps Lock → Escape (Vim users)  
setxkbmap -option ctrl:nocaps          # Caps Lock → Ctrl supplémentaire  
setxkbmap -option compose:ralt         # Alt Gr → touche Compose  
setxkbmap -option terminate:ctrl_alt_bksp  # Ctrl+Alt+Backspace tue le serveur X  

# Réinitialiser les options
setxkbmap -option ""

# Configuration permanente via Xorg
sudo nano /etc/X11/xorg.conf.d/00-keyboard.conf
```

```
Section "InputClass"
    Identifier "system-keyboard"
    MatchIsKeyboard "on"
    Option "XkbLayout" "fr"
    Option "XkbOptions" "caps:escape,compose:ralt"
EndSection
```

#### Touches spéciales et multimédia

Les touches multimédia (volume, luminosité, lecture/pause, etc.) sont gérées par l'environnement de bureau via le système d'événements ACPI et XKB. Si certaines touches ne fonctionnent pas :

```bash
# Identifier le code de la touche
# Sous Xorg :
xev
# Appuyer sur la touche problématique et noter le keycode et le keysym

# Sous Wayland :
sudo libinput debug-events
# ou
sudo evtest
# Sélectionner le périphérique clavier et appuyer sur la touche

# Si la touche est détectée mais sans effet,
# configurer un raccourci dans l'environnement de bureau :
# GNOME : Paramètres → Clavier → Raccourcis clavier
# KDE : Paramètres système → Raccourcis
# XFCE : Paramètres → Clavier → Raccourcis d'applications
```

### Tablettes graphiques

Les tablettes graphiques Wacom sont les mieux supportées sous Linux grâce au pilote noyau `wacom` et aux outils de l'espace utilisateur :

```bash
# Installation des outils Wacom
sudo apt install xserver-xorg-input-wacom
# Note : ce paquet fournit le pilote Xorg ainsi que l'utilitaire `xsetwacom`
# (pas de paquet séparé). Sous Wayland, le support tablette est natif.

# Vérifier la détection de la tablette
xsetwacom list devices
# Résultat : liste les dispositifs (stylet, gomme, pad tactile, etc.)

# Configuration en ligne de commande
# Définir la zone active de la tablette pour correspondre à un écran
xsetwacom set "Wacom Intuos Pro M Pen stylus" MapToOutput HDMI-1

# Ajuster la sensibilité à la pression
xsetwacom set "Wacom Intuos Pro M Pen stylus" PressureCurve 0 0 100 100

# Configurer les boutons de la tablette
xsetwacom set "Wacom Intuos Pro M Pad pad" Button 1 "key ctrl z"    # Annuler

# KDE : Paramètres système → Périphériques d'entrée → Tablette graphique
# Interface graphique complète pour la configuration

# GNOME : Paramètres → Tablette Wacom
# Interface basique (mappage d'écran, calibration, boutons)

# Sous Wayland, le support des tablettes est natif via le protocole tablet
# La configuration se fait via les outils de l'environnement de bureau
```

Pour les tablettes d'autres fabricants (Huion, XP-Pen, Gaomon), le support est assuré par le pilote noyau `hid-uclogic`. La qualité du support varie selon le modèle :

```bash
# Vérifier si la tablette est détectée
lsusb | grep -iE "huion|xp-pen|gaomon|ugee"  
sudo libinput list-devices | grep -i tablet  

# Le pilote hid-uclogic est intégré au noyau depuis la version 5.x
# Pour les modèles récents, un noyau backports peut être nécessaire
```

# Les manettes sont détectées automatiquement via le sous-système input du noyau

## Impression

### CUPS : le système d'impression

**CUPS** (Common Unix Printing System) est le système d'impression standard sous Linux. Il gère la découverte des imprimantes (réseau et USB), la file d'attente des travaux d'impression, la conversion des formats et la communication avec les pilotes d'impression.

```bash
# Installation (normalement déjà présent)
sudo apt install cups cups-client

# Vérifier que le service est actif
systemctl status cups.service  
sudo systemctl enable --now cups.service  

# L'interface web de CUPS (administration complète)
# Ouvrir dans le navigateur :
# http://localhost:631
```

### Ajout d'une imprimante

```bash
# Méthode graphique (recommandée)
# GNOME : Paramètres → Imprimantes → Ajouter une imprimante
# KDE : Paramètres système → Imprimantes → Ajouter une imprimante
# Tous : system-config-printer (outil graphique universel)
sudo apt install system-config-printer  
system-config-printer &  

# Les imprimantes réseau (IPP/AirPrint) et USB sont généralement
# détectées automatiquement grâce au service avahi-daemon (mDNS/DNS-SD)
sudo apt install avahi-daemon  
sudo systemctl enable --now avahi-daemon.service  

# Méthode en ligne de commande
# Lister les imprimantes détectées sur le réseau
lpinfo -v
# Résultat :
# network ipp://192.168.1.50/ipp/print
# direct usb://HP/LaserJet?serial=ABC123
# network dnssd://HP%20LaserJet._ipp._tcp.local

# Ajouter une imprimante
sudo lpadmin -p mon_imprimante -v ipp://192.168.1.50/ipp/print \
  -m everywhere -E
# -p : nom de l'imprimante
# -v : URI de l'imprimante
# -m everywhere : pilote IPP Everywhere (driverless, recommandé)
# -E : activer l'imprimante

# Définir comme imprimante par défaut
sudo lpadmin -d mon_imprimante

# Imprimer un fichier
lp document.pdf  
lp -d mon_imprimante -n 2 document.pdf    # 2 copies sur une imprimante spécifique  
```

### Impression driverless (sans pilote)

Les imprimantes modernes supportent le protocole **IPP Everywhere** (ou AirPrint pour Apple), ce qui permet l'impression sans installer de pilote spécifique. CUPS 2.x supporte nativement ce mode :

```bash
# Le paquet cups-filters fournit le support driverless
sudo apt install cups-filters cups-browsed

# cups-browsed découvre automatiquement les imprimantes réseau
# via mDNS/DNS-SD et les rend disponibles dans CUPS

# Vérifier les imprimantes découvertes
lpstat -v
```

### Pilotes d'imprimantes spécifiques

Pour les imprimantes qui ne supportent pas IPP Everywhere, des pilotes spécifiques peuvent être nécessaires :

```bash
# Pilotes HP (la majorité des imprimantes HP)
sudo apt install hplip
# HPLIP fournit également un outil graphique de gestion
hp-setup    # Assistant d'installation d'imprimante HP  
hp-toolbox  # Outil de maintenance (niveaux d'encre, nettoyage)  

# Pilotes Gutenprint (pilotes génériques haute qualité)
sudo apt install printer-driver-gutenprint

# Pilotes Epson
sudo apt install printer-driver-escpr
# Note Trixie : il n'existe pas (encore) de paquet `printer-driver-escpr2` dans
# les dépôts officiels. Le pilote « ESC/P-R version 2 » d'Epson doit être
# téléchargé directement depuis https://download.ebz.epson.net/dsc/search/01/search/
# si une imprimante Epson récente n'est pas couverte par `printer-driver-escpr`.

# Pilotes Brother (téléchargement depuis le site Brother)
# Brother fournit des paquets .deb sur son site de support
# https://support.brother.com/

# Pilotes Canon (selon le modèle)
# Canon fournit parfois des pilotes Linux sur son site
# Certains modèles sont supportés par Gutenprint

# PPD PostScript pour imprimantes HP (filtre hpps + descriptions PPD)
# Ce paquet est issu du source HPLIP et complète le support des imprimantes
# HP en PostScript (utile pour les modèles non gérés par les pilotes hpcups).
sudo apt install printer-driver-postscript-hp

# Méta-paquet regroupant la majorité des pilotes d'impression Debian
sudo apt install printer-driver-all
```

### Scanner

```bash
# SANE (Scanner Access Now Easy) — framework de numérisation
sudo apt install sane-utils libsane1
# `sane-utils` fournit `scanimage`, le démon `saned` et `sane-find-scanner`.
# `libsane1` est l'API tirée automatiquement comme dépendance.

# Détecter les scanners connectés
scanimage -L
# Résultat : device 'epson2:libusb:001:003' is a Epson GT-1500 flatbed scanner

# Numériser depuis la ligne de commande
scanimage --format=png --resolution=300 > scan.png

# Interfaces graphiques pour la numérisation
sudo apt install simple-scan    # GNOME (simple et efficace)  
sudo apt install skanlite       # KDE  
sudo apt install xsane          # Interface avancée (indépendante du DE)  

# Pour les imprimantes multifonctions HP
hp-scan    # Outil HP dédié

# Scanners réseau : vérifier que saned est configuré
# /etc/sane.d/net.conf → ajouter l'adresse IP du scanner réseau
```

## Webcams

```bash
# Les webcams USB sont généralement reconnues automatiquement via le pilote UVC
# (USB Video Class), intégré au noyau Linux

# Vérifier la détection
v4l2-ctl --list-devices
# ou
ls /dev/video*

# Tester la webcam
# GNOME
sudo apt install cheese  
cheese &  

# KDE
sudo apt install kamoso  
kamoso &  

# En ligne de commande
mpv av://v4l2:/dev/video0
# ou
ffplay /dev/video0

# Régler les paramètres de la webcam (luminosité, contraste, etc.)
v4l2-ctl -d /dev/video0 --list-ctrls  
v4l2-ctl -d /dev/video0 --set-ctrl brightness=140  
v4l2-ctl -d /dev/video0 --set-ctrl white_balance_automatic=1  

# Webcam virtuelle (utile pour les visioconférences avec filtres)
sudo apt install v4l2loopback-dkms  
sudo modprobe v4l2loopback  
# Crée un périphérique /dev/videoX virtuel que OBS peut alimenter
```

### Webcam et vie privée

Sur certains portables, la webcam peut être désactivée au niveau logiciel :

```bash
# Désactiver le module webcam
sudo modprobe -r uvcvideo

# Empêcher le chargement automatique au démarrage
echo "blacklist uvcvideo" | sudo tee /etc/modprobe.d/disable-webcam.conf

# Réactiver
sudo rm /etc/modprobe.d/disable-webcam.conf  
sudo modprobe uvcvideo  
```

## Stockage amovible

### Montage automatique des clés USB et disques externes

Les environnements de bureau montent automatiquement les périphériques de stockage USB grâce à **udisks2** :

```bash
# udisks2 est normalement déjà installé
sudo apt install udisks2

# Monter un périphérique en ligne de commande (sans root)
udisksctl mount -b /dev/sdb1
# Le périphérique est monté dans /media/$USER/label_du_volume

# Démonter proprement
udisksctl unmount -b /dev/sdb1

# Éjecter (pour les disques USB avec éjection sécurisée)
udisksctl power-off -b /dev/sdb

# Informations sur un périphérique
udisksctl info -b /dev/sdb1

# Sous XFCE, le montage automatique est géré par thunar-volman
sudo apt install thunar-volman
```

### Systèmes de fichiers supportés

Pour accéder à tous les types de volumes couramment rencontrés :

```bash
# NTFS (disques Windows)
sudo apt install ntfs-3g
# Le montage en lecture/écriture est automatique

# exFAT (cartes SD, clés USB grandes capacités)
sudo apt install exfatprogs
# Supporté nativement par le noyau 5.4+

# HFS+ (disques macOS anciens)
sudo apt install hfsprogs

# APFS (disques macOS récents) — support expérimental
sudo apt install apfsprogs    # Outils userspace : mkfs.apfs, fsck.apfs  
sudo apt install apfs-dkms    # Module noyau APFS (lecture/écriture expérimentale)  
# Le support APFS sous Linux reste expérimental : préférer le mode lecture seule
# pour les volumes en production. Les fonctionnalités avancées (snapshots, FileVault)
# ne sont pas toutes prises en charge.

# Systèmes de fichiers chiffrés (LUKS)
sudo apt install cryptsetup
# Les environnements de bureau proposent automatiquement un dialogue
# de mot de passe à l'insertion d'un volume LUKS
```

## Téléphones et appareils mobiles

### Android : MTP (Media Transfer Protocol)

```bash
# Le protocole MTP est utilisé par les appareils Android pour le transfert de fichiers
# La prise en charge est intégrée aux gestionnaires de fichiers modernes

# GNOME : Nautilus accède aux appareils MTP via GVfs.
# Sur Trixie, `gvfs-mtp` n'existe plus en paquet séparé : le backend MTP
# (basé sur `libmtp`) est désormais inclus directement dans `gvfs-backends`.
sudo apt install gvfs-backends
# Connecter le téléphone en USB, sélectionner "Transfert de fichiers" sur le téléphone
# Le téléphone apparaît dans le panneau latéral de Nautilus

# KDE : Dolphin accède aux appareils MTP via KIO
# Sous Trixie (Plasma 6), le support MTP est intégré dans `kio-extras`
# (le paquet `kio-mtp` séparé n'existe plus).
sudo apt install kio-extras
# Le téléphone apparaît dans les "Emplacements" de Dolphin
# (saisir `mtp:/` dans la barre d'adresse pour lister tous les appareils MTP)

# XFCE / LXQt : utiliser gvfs-mtp ou jmtpfs
sudo apt install jmtpfs  
mkdir ~/telephone  
jmtpfs ~/telephone                # Monter  
fusermount -u ~/telephone          # Démonter  

# KDE Connect (intégration avancée, voir section 2.1.2)
sudo apt install kdeconnect
# Transfert de fichiers, notifications, presse-papiers, etc.
# Fonctionne via le réseau WiFi, pas via USB
```

### iPhone : AFC (Apple File Conduit)

```bash
# Support iOS (accès limité aux photos et fichiers)
sudo apt install ifuse libimobiledevice-utils usbmuxd

# Vérifier la détection
ideviceinfo

# Appairer l'appareil (accepter la confiance sur l'iPhone)
idevicepair pair

# Monter le système de fichiers
mkdir ~/iphone

fusermount -u ~/iphone
```

## Clés de sécurité (FIDO2, U2F)

Les clés de sécurité matérielles (YubiKey, SoloKey, Google Titan) sont utilisées pour l'authentification à deux facteurs (2FA) et l'authentification sans mot de passe (FIDO2/WebAuthn) :

```bash
# Support de base (bibliothèque + règles udev intégrées)
sudo apt install libfido2-1 fido2-tools
# Depuis udev 244+ (et `libfido2-1` ≥ 1.x), les règles udev pour les clés
# FIDO/U2F sont fournies directement : aucun paquet supplémentaire requis
# pour l'accès non-root.

# Paquet historique conservé pour compatibilité (transitionnel)
# Ne plus installer sur un système moderne — `libfido2-1` ship déjà les règles
# # sudo apt install libu2f-udev

# Tester la clé
fido2-token -L    # Lister les clés connectées  
fido2-token -I /dev/hidraw0    # Informations sur une clé  

# Les navigateurs (Firefox, Chromium) supportent nativement les clés FIDO2/U2F
# pour l'authentification web (WebAuthn)

# YubiKey : outil de gestion spécifique
sudo apt install yubikey-manager  
ykman info    # Informations sur la YubiKey connectée  

# Interface graphique pour YubiKey
sudo apt install yubikey-manager-qt
```

## Résumé

La gestion des périphériques sous Debian est largement automatisée grâce à la détection matérielle du noyau, aux services système (BlueZ, CUPS, udisks2, libinput) et aux outils d'intégration des environnements de bureau. Le Bluetooth fonctionne de manière transparente avec la pile BlueZ et PipeWire pour l'audio. L'impression est prise en charge par CUPS, avec une tendance à l'impression sans pilote via IPP Everywhere. Les périphériques d'entrée (souris, claviers, pavés tactiles, tablettes graphiques) sont gérés uniformément par libinput.

Les cas nécessitant une intervention manuelle sont principalement les firmwares Bluetooth manquants (traités dans la section 2.3.1), les imprimantes anciennes nécessitant des pilotes spécifiques, et certains périphériques USB spécialisés. Pour la majorité des périphériques courants, le fonctionnement est immédiat (plug-and-play) dès lors que les firmwares appropriés sont installés.

Ce chapitre conclut la section 2.3 sur le matériel et les pilotes, et le Module 2 sur Debian Desktop dans son ensemble. Le Module 3 aborde l'administration système de base : gestion des fichiers, des utilisateurs, des processus, de systemd et des logs.
