# 1.2.5 Sélection des paquets de base (tasksel)

## Introduction

La sélection des paquets est la dernière étape majeure avant la finalisation de l'installation. C'est à ce moment que l'administrateur définit le **profil logiciel initial** de la machine : un serveur minimaliste en ligne de commande, un poste de travail avec environnement graphique complet, un serveur web, ou toute combinaison de ces profils. Le debian-installer s'appuie pour cela sur un outil appelé **tasksel**, qui présente des ensembles cohérents de logiciels sous forme de « tâches » (*tasks*) sélectionnables d'un simple coche.

Le choix effectué à cette étape n'est pas irréversible — tout peut être installé ou désinstallé ultérieurement via APT — mais il détermine l'état initial du système au premier démarrage et influence significativement l'expérience de configuration post-installation.

## Tasksel : mécanisme et fonctionnement

### Qu'est-ce qu'une tâche ?

Une **tâche** (*task*) dans le vocabulaire Debian est un ensemble logique de paquets qui, installés ensemble, fournissent une fonctionnalité complète. Une tâche n'est pas un métapaquet au sens strict (bien qu'elle puisse en contenir) : c'est une description déclarative qui liste les paquets à installer, certains de manière obligatoire et d'autres de manière recommandée ou optionnelle.

Les tâches sont définies dans des fichiers de description inclus dans le paquet `tasksel` et dans les métadonnées de l'archive Debian. Chaque paquet `.deb` de l'archive peut déclarer dans son champ de contrôle `Task:` les tâches auxquelles il appartient. Tasksel agrège ces informations pour construire les ensembles présentés à l'utilisateur.

### Fonctionnement interne

Lorsque l'écran de sélection des logiciels apparaît dans le d-i, tasksel consulte la liste des tâches disponibles, affiche celles qui sont pertinentes pour l'architecture et le type d'installation, et pré-sélectionne certaines tâches selon le contexte (par exemple, l'environnement de bureau GNOME est pré-sélectionné sur une installation desktop standard).

Pour chaque tâche sélectionnée, tasksel génère la liste complète des paquets à installer en incluant les paquets marqués comme `key` (essentiels à la tâche, toujours installés), les paquets recommandés par ces paquets clés (installés par défaut via le comportement standard d'APT), et les dépendances transitives de l'ensemble.

Tasksel délègue ensuite l'installation effective à APT, qui se charge du téléchargement (depuis le miroir réseau ou le DVD), de la résolution des dépendances et de l'appel à dpkg pour l'installation de chaque paquet.

### L'écran de sélection dans le d-i

L'écran de sélection des logiciels du d-i présente une liste de tâches avec des cases à cocher. L'interface est volontairement simple : chaque ligne correspond à une tâche, une brève description est affichée, et l'administrateur coche ou décoche selon ses besoins. Plusieurs tâches peuvent être sélectionnées simultanément.

L'ordre d'affichage et les tâches proposées varient légèrement selon le mode d'installation (standard ou expert) et l'architecture matérielle.

## Les tâches disponibles

### Environnements de bureau

Les tâches d'environnement de bureau installent un système graphique complet : serveur d'affichage (Wayland ou Xorg), gestionnaire de connexion, environnement de bureau, et un ensemble d'applications desktop (navigateur, gestionnaire de fichiers, éditeur de texte, terminal, etc.).

**Debian desktop environment** est la tâche chapeau qui installe les composants communs à tous les environnements de bureau : le serveur d'affichage, les bibliothèques graphiques partagées, les polices de caractères, les pilotes d'entrée (clavier, souris, touchpad), et les outils d'accessibilité. Cette tâche est automatiquement incluse lorsqu'un environnement de bureau spécifique est sélectionné.

**GNOME** est l'environnement de bureau par défaut de Debian. Basé sur la bibliothèque GTK et le shell GNOME, il offre une interface moderne et épurée, orientée vers la productivité. GNOME sur Debian utilise **Wayland** par défaut comme protocole d'affichage (avec Xorg disponible en repli). Debian 13 (Trixie) embarque **GNOME 48**. L'installation complète de GNOME via tasksel représente environ **2 à 3 Go** de paquets téléchargés et inclut des applications intégrées telles que Nautilus (gestionnaire de fichiers), **Ptyxis** (terminal devenu défaut depuis GNOME 48.4 ; **GNOME Console** reste l'application terminal officielle de l'écosystème GNOME), **Firefox ESR** (navigateur par défaut sur Debian — Epiphany est aussi packagé), Evince (lecteur PDF), **Loupe** (visualiseur d'images en Rust devenu défaut depuis GNOME 45 — il remplace l'historique Eye of GNOME / `eog` qui n'est plus installé par défaut), et d'autres utilitaires cohérents avec l'écosystème GNOME. GNOME est le choix recommandé pour les utilisateurs qui privilégient une interface standardisée, bien intégrée et largement documentée.

**KDE Plasma** est l'alternative majeure à GNOME. Basé sur le framework Qt, Plasma offre une interface hautement personnalisable avec une philosophie de configuration exposant davantage d'options à l'utilisateur. L'empreinte mémoire et disque est comparable à celle de GNOME. KDE Plasma inclut son propre écosystème d'applications : Dolphin (gestionnaire de fichiers), Konsole (terminal), Kate (éditeur de texte), Okular (lecteur PDF), et l'ensemble des KDE Applications. KDE Plasma est le choix privilégié pour les utilisateurs qui apprécient un degré élevé de personnalisation.

**XFCE** est un environnement de bureau léger qui conserve une interface traditionnelle (barre des tâches, menu d'applications, bureau avec icônes) tout en consommant significativement moins de ressources que GNOME ou KDE. Il représente environ **1 à 1,5 Go** de paquets et fonctionne confortablement sur des machines disposant de 1 Go de RAM. XFCE est recommandé pour le matériel ancien, les machines virtuelles aux ressources limitées, et les utilisateurs préférant une interface classique et sobre.

**LXQt** est l'environnement de bureau le plus léger parmi ceux proposés par tasksel. Basé sur Qt (comme KDE) mais avec une empreinte minimale, il convient aux machines très modestes ou aux utilisateurs souhaitant un environnement graphique fonctionnel sans surcharge.

**LXDE** est le prédécesseur de LXQt, basé sur GTK 2. Il est encore disponible mais son développement est essentiellement en maintenance. LXQt est préférable pour les nouvelles installations.

**MATE** est un fork de GNOME 2, maintenant l'interface classique de bureau à panneaux que GNOME a abandonnée lors du passage à GNOME 3 (shell). Il offre un compromis entre légèreté et fonctionnalités.

**Cinnamon** est l'environnement de bureau développé par le projet Linux Mint, également disponible dans Debian. Il propose une interface moderne mais traditionnelle, avec un menu d'applications complet, une barre des tâches et un système de bureau classique.

### Tâches serveur

**SSH server** installe le serveur **OpenSSH** (`openssh-server`), qui permet l'accès distant sécurisé à la machine via le protocole SSH. C'est la tâche la plus fréquemment sélectionnée pour les installations serveur, et souvent la seule en dehors des utilitaires standard. L'installation via tasksel configure le service pour un démarrage automatique et génère les clés d'hôte SSH.

**Web server** installe le serveur web **Apache** (`apache2`) avec une configuration par défaut fonctionnelle. Cette tâche est un point de départ ; en production, le choix entre Apache, Nginx ou Caddy et leur configuration avancée seront traités dans le module 7.

### Tâches utilitaires

**Standard system utilities** est une tâche particulière qui installe l'ensemble des paquets ayant la priorité APT `standard` (cf. `apt-cache show <paquet> | grep ^Priority`). Cette tâche est **pré-sélectionnée par défaut** et il est fortement recommandé de la conserver. Elle inclut notamment `file` (identification de types de fichiers), `less` (pager avancé), `man-db` (pages de manuel), `reportbug` (outil de signalement de bugs Debian), `bzip2` et `xz-utils` (outils de compression), `bind9-dnsutils` (commandes DNS `dig`, `nslookup` — l'ancien paquet `dnsutils` n'est plus qu'un méta-paquet de transition), `traceroute`, `nano`, `vim-tiny`, `openssh-client`, `nftables`, `cron`, `logrotate` et d'autres utilitaires courants. À noter que des outils souvent attendus comme `aptitude`, `net-tools` (`ifconfig`, `netstat`) ou `whois` sont en priorité `optional` et **ne** font **pas** partie de cette tâche : ils s'installent à la demande via APT.

Sans cette tâche, le système installé sera fonctionnel mais dépouillé — il manquera de nombreuses commandes que l'administrateur s'attend à trouver sur un système Linux.

> **Note (Trixie) :** Une tâche **Print server** existait dans les versions précédentes pour installer **CUPS** (*Common Unix Printing System*) et les pilotes courants. Elle a été retirée de tasksel sur Debian 13 (`tasksel --list-tasks` ne propose plus que les environnements de bureau, `standard`, `ssh-server`, `web-server`, `laptop` et `blendsel`). Pour mettre en place un serveur d'impression sur Trixie, on installe directement les paquets adéquats via APT : `apt install cups cups-bsd printer-driver-all` pour un serveur d'impression complet, ou `apt install cups cups-client` pour un poste client.

### Tâche de localisation

Le d-i propose automatiquement une tâche de **localisation** correspondant à la langue sélectionnée en début d'installation. Pour une installation en français, la tâche installe les paquets de traduction française pour les applications, les dictionnaires orthographiques, les polices adaptées, et les méthodes d'entrée si nécessaire (pertinent pour les langues CJK — chinois, japonais, coréen).

## Estimation des volumes par profil

Le volume de données téléchargées et l'espace disque occupé varient considérablement selon les tâches sélectionnées. Les estimations suivantes donnent un ordre de grandeur pour l'architecture amd64.

| Profil | Tâches sélectionnées | Téléchargement | Espace disque |
|--------|---------------------|----------------|---------------|
| Serveur minimal | SSH server + standard utilities | ~300 Mo | ~1,5 Go |
| Serveur web | SSH + web server + standard | ~350 Mo | ~1,8 Go |
| Desktop XFCE | XFCE + standard | ~1,5 Go | ~4 Go |
| Desktop GNOME | GNOME + standard | ~2,5 Go | ~6 Go |
| Desktop KDE | KDE Plasma + standard | ~2,5 Go | ~6 Go |
| Desktop complet | GNOME + standard + CUPS via APT | ~3 Go | ~7 Go |

Ces chiffres sont indicatifs et varient selon la version de Debian, les paquets recommandés installés, et les mises à jour appliquées.

## La notion de système de base (base system)

### Ce qui est installé avant tasksel

Avant l'étape de sélection des logiciels, le d-i installe le **système de base** (*base system*), un ensemble minimal de paquets indispensables au fonctionnement du système. Ce socle est installé quelle que soit la sélection tasksel et ne peut pas être désélectionné.

Le système de base comprend le **noyau Linux** et ses modules, les **outils essentiels** du système (le paquet `base-files` qui définit l'arborescence FHS, `coreutils`, `bash`, `login`, `passwd`, `adduser`, `grep`, `sed`, `gzip`, `tar`), le **système d'initialisation** (`systemd`, `systemd-sysv`), le **gestionnaire de paquets** (`dpkg`, `apt`), les **bibliothèques fondamentales** (`libc6`, `libssl`, `zlib1g`), et les **outils de démarrage** (`grub-efi-amd64` ou `grub-pc` selon le mode de démarrage, `initramfs-tools`).

Ce système de base est fonctionnel mais spartiate : il permet de démarrer la machine, de se connecter en console locale, de gérer les paquets, et rien de plus. C'est le point de départ à partir duquel tasksel — ou l'administrateur manuellement — construit le système adapté au rôle de la machine.

### Les paquets prioritaires

Debian classe chaque paquet selon un niveau de **priorité** qui reflète son importance pour le fonctionnement du système. Les niveaux `required` (indispensable au fonctionnement de dpkg et du système de base) et `important` (présent sur tout système Unix raisonnable) constituent le socle incompressible. Le niveau `standard` correspond aux paquets installés par la tâche « Standard system utilities ». Les niveaux `optional` et `extra` couvrent tous les autres paquets de l'archive.

Le d-i installe systématiquement les paquets `required` et `important`, puis propose la tâche « Standard system utilities » pour les paquets `standard`.

## Utilisation de tasksel après l'installation

### Invocation en ligne de commande

Tasksel n'est pas limité au contexte du d-i : il peut être invoqué à tout moment après l'installation pour ajouter ou supprimer des tâches complètes. La commande `tasksel` lancée en tant que root ouvre l'interface de sélection interactive :

```bash
sudo tasksel
```

L'interface affiche la même liste de tâches que pendant l'installation, avec les tâches actuellement installées déjà cochées. L'administrateur peut cocher de nouvelles tâches (qui seront installées) ou décocher des tâches existantes (qui seront supprimées).

### Commandes non interactives

Pour une utilisation scriptée ou en ligne de commande directe, tasksel accepte des commandes spécifiques :

```bash
# Lister toutes les tâches disponibles et leur état
tasksel --list-tasks

# Installer une tâche spécifique
sudo tasksel install kde-desktop

# Supprimer une tâche
sudo tasksel remove gnome-desktop

# Afficher les paquets associés à une tâche
tasksel --task-packages kde-desktop
```

La commande `--task-packages` est particulièrement utile pour prévisualiser l'impact d'une installation de tâche avant de l'exécuter.

### Identifiants des tâches

Chaque tâche possède un identifiant interne utilisé en ligne de commande. Les principaux identifiants sont les suivants :

| Identifiant | Description |
|-------------|-------------|
| `standard` | Utilitaires système standard |
| `desktop` | Composants communs bureau |
| `gnome-desktop` | Environnement GNOME |
| `kde-desktop` | Environnement KDE Plasma |
| `xfce-desktop` | Environnement XFCE |
| `lxqt-desktop` | Environnement LXQt |
| `lxde-desktop` | Environnement LXDE |
| `mate-desktop` | Environnement MATE |
| `cinnamon-desktop` | Environnement Cinnamon |
| `web-server` | Serveur web Apache |
| `ssh-server` | Serveur SSH |
| `laptop` | Outils pour portables |
| `blendsel` | Sélection d'un *Debian Pure Blend* (Edu, Med, Junior…) |

### Tasksel vs APT : quand utiliser quoi

Tasksel est un outil de **haut niveau** qui installe des profils cohérents de logiciels. APT est l'outil de gestion de paquets **individuel**. Les deux sont complémentaires.

Tasksel est approprié pour installer ou changer d'environnement de bureau complet, pour configurer rapidement un profil serveur standard, et pour reproduire une sélection de logiciels identique sur plusieurs machines.

APT est approprié pour installer des paquets individuels (`apt install nginx`), pour un contrôle fin sur les paquets installés (choisir Nginx plutôt qu'Apache, par exemple, alors que la tâche web-server installe Apache), et pour toute opération de gestion de paquets au quotidien.

En pratique, après l'installation initiale, la grande majorité des opérations de gestion de paquets se font via APT. Tasksel est surtout utilisé lors de l'installation et ponctuellement pour les changements de profil majeurs.

## Installation minimale : le choix du serveur

### Tout décocher

Un choix courant pour les serveurs de production et les environnements automatisés consiste à **ne sélectionner aucune tâche** dans l'écran tasksel (en décochant même « Standard system utilities » et « SSH server ») pour obtenir un système absolument minimal.

Le système résultant ne contient que les paquets du système de base. Il est fonctionnel mais limité : pas de `man`, pas de `less`, pas de `dig`, pas de serveur SSH. L'administrateur construit ensuite le système pièce par pièce, en n'installant que les paquets strictement nécessaires au rôle de la machine.

Cette approche est cohérente avec le principe de **surface d'attaque minimale** en sécurité : chaque paquet installé est un paquet supplémentaire à maintenir et à surveiller. Elle est également privilégiée pour les **images de conteneurs** et les **images cloud** où le minimalisme est une vertu.

En contrepartie, un système aussi dépouillé nécessite une connaissance précise des paquets à installer manuellement et rend le diagnostic plus difficile en l'absence des outils habituels. Un compromis raisonnable pour la plupart des serveurs est de conserver la tâche « Standard system utilities » et « SSH server », puis d'ajouter uniquement les paquets applicatifs nécessaires via APT.

### La tâche laptop

Pour les installations sur ordinateur portable, la tâche **laptop** (parfois sélectionnée automatiquement par le d-i lorsqu'il détecte un châssis de type portable) installe des paquets spécifiques à la gestion de l'alimentation et à l'autonomie : `acpi-support` (gestion ACPI), `powertop` (analyse de la consommation), `wireless-tools` et `wpasupplicant` (outils WiFi), et d'autres utilitaires liés à la mobilité. Cette tâche est complémentaire à la tâche d'environnement de bureau et n'inclut pas elle-même d'interface graphique.

## Preseed et automatisation de la sélection

Dans un contexte de déploiement automatisé (preseed, abordé dans le module 7), la sélection des tâches est spécifiée dans le fichier preseed sans aucune interaction :

```
# Sélection des tâches dans un fichier preseed
tasksel tasksel/first multiselect standard, ssh-server

# Paquets individuels supplémentaires à installer
d-i pkgsel/include string vim curl htop tmux
```

La directive `tasksel/first` définit les tâches sélectionnées (séparées par des virgules). La directive `pkgsel/include` permet d'ajouter des paquets individuels qui ne font partie d'aucune tâche mais que l'on souhaite voir présents dès le premier démarrage. Ce mécanisme est fréquemment utilisé pour inclure les outils d'administration préférés de l'équipe (éditeurs, outils de monitoring, agents de configuration management) sans créer une tâche personnalisée.

## Après la sélection : finalisation de l'installation

Une fois les tâches sélectionnées et les paquets installés, le d-i effectue les dernières opérations de configuration avant le premier démarrage.

**L'installation du bootloader** : GRUB est installé sur le disque de démarrage (dans l'ESP en mode UEFI, dans le MBR ou la partition biosgrub en mode BIOS). Le d-i demande confirmation du disque cible pour éviter les erreurs.

**La configuration de popularity-contest** : le d-i propose d'activer le programme `popcon` (*popularity-contest*), qui envoie anonymement et de manière hebdomadaire la liste des paquets installés sur la machine aux serveurs Debian. Ces statistiques sont utilisées pour classer les paquets par popularité — c'est notamment ce classement qui détermine l'ordre des paquets sur les images DVD. La participation est volontaire et les données sont anonymisées.

**L'horloge système** : le d-i configure l'horloge matérielle (UTC recommandé pour les systèmes exclusivement Linux, heure locale si un dual-boot avec Windows est prévu).

**Le retrait du support d'installation** : le d-i demande de retirer la clé USB ou le DVD avant de redémarrer, pour éviter que la machine ne redémarre sur l'installateur.

Le système redémarre alors sur le Debian nouvellement installé, prêt pour la configuration post-installation qui sera traitée dans la section 1.3.

## Points clés à retenir

Tasksel est l'outil de sélection de profils logiciels utilisé par le debian-installer et disponible après l'installation. Il propose des ensembles cohérents de paquets organisés en tâches — environnements de bureau, services serveur, utilitaires système — qui définissent le profil initial de la machine. La tâche « Standard system utilities » devrait presque toujours être conservée. Pour les serveurs, la combinaison « standard + SSH server » constitue un point de départ solide et minimal. Pour les postes de travail, le choix de l'environnement de bureau (GNOME, KDE, XFCE, LXQt) dépend des ressources matérielles et des préférences de l'utilisateur. Tous les choix effectués via tasksel sont réversibles après l'installation via APT, ce qui permet une approche itérative de la construction du système.
