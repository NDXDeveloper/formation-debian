## Métadonnées

- **ID** : RB-002
- **Catégorie** : Routine
- **Sévérité** : Info
- **Temps estimé** : 15 minutes
- **Fréquence** : Quotidienne (vérification), automatique (sauvegarde)
- **Compétences** : Kubernetes, etcd

## Contexte

Le CronJob de sauvegarde etcd (déployé en section 19.2.2) s'exécute toutes  
les 6 heures. Cette procédure vérifie que les sauvegardes sont  
fonctionnelles et que le dernier snapshot est restaurable.  

## Procédure

### Étape 1 : Vérifier l'exécution du CronJob

```bash
# Lister les dernières exécutions
kubectl -n kube-system get jobs --sort-by=.metadata.creationTimestamp \
    | grep etcd-backup | tail -5

# Vérifier les logs du dernier job
kubectl -n kube-system logs "job/$(kubectl -n kube-system get jobs \
    --sort-by=.metadata.creationTimestamp \
    -o jsonpath='{.items[-1].metadata.name}' \
    -l app=etcd-backup)"
```

### Étape 2 : Vérifier l'intégrité du snapshot

```bash
ssh admin@cp-1.k8s.internal.example.com

# Note : etcdctl snapshot status est déprécié depuis etcd 3.5 au profit de
# etcdutl (utilitaire qui opère directement sur le fichier, sans contacter
# l'API etcd).
LATEST=$(ls -t /var/backups/etcd/etcd-*.db | head -1)  
sudo etcdutl snapshot status "${LATEST}" --write-out=table  
# Doit afficher : hash, revision, total-keys, total-size
```

### Étape 3 : Vérifier la copie distante

```bash
ssh admin@backup.infra.internal.example.com \
    "ls -lh /srv/backups/etcd/ | tail -5"
# Le fichier le plus récent doit dater de moins de 6 heures

ssh admin@backup.infra.internal.example.com \
    "etcdutl snapshot status \$(ls -t /srv/backups/etcd/etcd-*.db | head -1) --write-out=table"
```

## Validation

- ✅ CronJob exécuté avec succès dans les 6 dernières heures
- ✅ Snapshot local vérifié (commande `etcdutl snapshot status` sans erreur)
- ✅ Copie distante présente et vérifiée

## Escalade

- CronJob en échec depuis plus de 12h : → alerte Slack #infra-alerts
- Snapshot corrompu : → déclencher une sauvegarde manuelle immédiate
- Copie distante absente : → vérifier la connectivité vers le serveur de backup
