# 5.3.2 Scripts d'administration système en Python

## Introduction

La section 5.2 a présenté des scripts d'administration complets en Bash — rotation de logs, nettoyage, provisionnement, interaction avec des APIs. Cette sous-section aborde les mêmes catégories de tâches en Python, en mettant en évidence les patterns fondamentaux du langage qui s'appliquent à l'administration système : parsing d'arguments, manipulation du système de fichiers, exécution de commandes, journalisation et gestion des fichiers de configuration.

L'objectif n'est pas de réécrire chaque script Bash en Python — ce serait souvent contre-productif pour les tâches simples — mais de montrer comment Python structure le code d'administration de manière plus robuste, plus testable et plus maintenable à mesure que la complexité augmente.

## Structure d'un script d'administration Python

### Le squelette standard

Un script Python d'administration suit une structure conventionnelle qui garantit la lisibilité, la testabilité et la robustesse :

```python
#!/usr/bin/env python3
"""
check_disk_space.py — Vérification de l'espace disque avec alertes.

Vérifie l'utilisation des partitions et alerte si un seuil est dépassé.  
Conçu pour être exécuté via un timer systemd ou cron.  

Codes de retour :
    0 : toutes les partitions sous le seuil
    1 : au moins une partition dépasse le seuil warning
    2 : au moins une partition dépasse le seuil critique
"""

import argparse  
import logging  
import shutil  
import sys  
from pathlib import Path  

# ── Constantes ──────────────────────────────────────────────
DEFAULT_WARNING = 80  
DEFAULT_CRITICAL = 90  
EXCLUDED_FS = {"tmpfs", "devtmpfs", "squashfs", "overlay"}  

# ── Logging ─────────────────────────────────────────────────
logger = logging.getLogger(__name__)


def setup_logging(verbose: bool = False) -> None:
    """Configure la journalisation."""
    level = logging.DEBUG if verbose else logging.INFO
    logging.basicConfig(
        level=level,
        format="%(asctime)s [%(levelname)-5s] %(message)s",
        datefmt="%Y-%m-%d %H:%M:%S",
        stream=sys.stderr,
    )


# ── Logique métier ──────────────────────────────────────────
def check_partition(path: str, warning: int, critical: int) -> int:
    """Vérifie l'utilisation d'une partition.
    
    Retourne 0 (ok), 1 (warning) ou 2 (critical).
    """
    try:
        usage = shutil.disk_usage(path)
    except PermissionError:
        logger.warning("Permission refusée pour %s", path)
        return 0

    percent = int(usage.used / usage.total * 100)

    if percent >= critical:
        logger.error(
            "CRITIQUE : %s à %d%% (%s libre)",
            path, percent, format_size(usage.free),
        )
        return 2
    elif percent >= warning:
        logger.warning(
            "WARNING : %s à %d%% (%s libre)",
            path, percent, format_size(usage.free),
        )
        return 1
    else:
        logger.info("%s : %d%% (%s libre)", path, percent, format_size(usage.free))
        return 0


def format_size(size_bytes: int) -> str:
    """Formate une taille en octets vers un format lisible."""
    for unite in ("o", "Ko", "Mo", "Go", "To"):
        if abs(size_bytes) < 1024:
            return f"{size_bytes:.1f} {unite}"
        size_bytes /= 1024
    return f"{size_bytes:.1f} Po"


def get_mount_points() -> list[str]:
    """Récupère les points de montage en excluant les FS virtuels."""
    points = []
    with open("/proc/mounts") as f:
        for line in f:
            parts = line.split()
            if len(parts) >= 3 and parts[2] not in EXCLUDED_FS:
                mount_point = parts[1]
                if Path(mount_point).exists():
                    points.append(mount_point)
    # Dédupliquer (certains montages bind)
    return sorted(set(points))


# ── Arguments CLI ───────────────────────────────────────────
def parse_args(argv: list[str] | None = None) -> argparse.Namespace:
    """Parse les arguments de la ligne de commande."""
    parser = argparse.ArgumentParser(
        description="Vérification de l'espace disque avec alertes.",
        formatter_class=argparse.RawDescriptionHelpFormatter,
    )
    parser.add_argument(
        "-w", "--warning",
        type=int, default=DEFAULT_WARNING,
        help=f"Seuil warning en %% (défaut : {DEFAULT_WARNING})",
    )
    parser.add_argument(
        "-c", "--critical",
        type=int, default=DEFAULT_CRITICAL,
        help=f"Seuil critique en %% (défaut : {DEFAULT_CRITICAL})",
    )
    parser.add_argument(
        "-v", "--verbose",
        action="store_true",
        help="Mode verbeux (affiche les partitions OK)",
    )
    parser.add_argument(
        "paths",
        nargs="*",
        default=None,
        help="Points de montage à vérifier (défaut : tous)",
    )
    return parser.parse_args(argv)


# ── Point d'entrée ──────────────────────────────────────────
def main(argv: list[str] | None = None) -> int:
    """Point d'entrée principal. Retourne le code de sortie."""
    args = parse_args(argv)
    setup_logging(verbose=args.verbose)

    mount_points = args.paths if args.paths else get_mount_points()
    logger.debug("Partitions à vérifier : %s", mount_points)

    worst = 0
    for mp in mount_points:
        result = check_partition(mp, args.warning, args.critical)
        worst = max(worst, result)

    return worst


if __name__ == "__main__":
    sys.exit(main())
```

### Anatomie du squelette

Chaque élément de cette structure remplit un rôle précis :

**La docstring de module** (triple guillemets en tête de fichier) documente l'objectif du script, son usage et ses codes de retour. Elle est accessible via `--help` si `argparse` est configuré pour l'utiliser, et elle sert de documentation de premier recours pour l'administrateur qui ouvre le fichier.

**Les imports** sont regroupés en trois blocs selon la convention PEP 8 : bibliothèque standard, bibliothèques tierces, modules locaux. L'ordre est alphabétique au sein de chaque bloc.

**Le module `logging`** remplace les fonctions `log_info`/`log_error` manuelles de Bash. Il offre nativement les niveaux de sévérité, le formatage, la rotation et la redirection vers des fichiers ou syslog.

**La fonction `parse_args`** utilise `argparse`, le module standard de parsing d'arguments. Il génère automatiquement le `--help`, valide les types et les valeurs, et gère les arguments optionnels et positionnels.

**La fonction `main`** centralise la logique d'orchestration et retourne un code de sortie entier (et non pas `sys.exit()` directement), ce qui la rend testable :

```python
# Dans un test unitaire
def test_main_toutes_partitions_ok():
    result = main(["--warning", "99", "--critical", "100", "/tmp"])
    assert result == 0
```

**Le bloc `if __name__ == "__main__"`** est l'idiome Python standard pour distinguer l'exécution directe du script de son importation en tant que module. Quand le script est exécuté directement, `main()` est appelée et son code de retour est passé à `sys.exit()`. Quand le script est importé, seules les définitions de fonctions sont chargées, ce qui permet les tests et la réutilisation.

## Parsing d'arguments avec `argparse`

### Au-delà du basique

`argparse` est considérablement plus puissant que le parsing manuel d'arguments en Bash (`getopts`, boucle `case`). Il gère les types, les valeurs par défaut, la validation, les sous-commandes, les groupes mutuellement exclusifs et la génération automatique de l'aide :

```python
def parse_args():
    parser = argparse.ArgumentParser(
        description="Gestion des sauvegardes PostgreSQL.",
        epilog="Exemples :\n"
               "  %(prog)s backup --databases production staging\n"
               "  %(prog)s cleanup --retention 30\n"
               "  %(prog)s verify --latest",
        formatter_class=argparse.RawDescriptionHelpFormatter,
    )

    # Arguments globaux
    parser.add_argument(
        "-v", "--verbose", action="store_true",
        help="Mode verbeux",
    )
    parser.add_argument(
        "-n", "--dry-run", action="store_true",
        help="Simulation (aucune modification)",
    )
    parser.add_argument(
        "--config", type=Path, default=Path("/etc/backup/config.yaml"),
        help="Fichier de configuration (défaut : %(default)s)",
    )

    # Sous-commandes
    subparsers = parser.add_subparsers(dest="command", required=True)

    # Sous-commande : backup
    backup_parser = subparsers.add_parser("backup", help="Effectuer une sauvegarde")
    backup_parser.add_argument(
        "--databases", nargs="+", default=["all"],
        help="Bases à sauvegarder (défaut : toutes)",
    )
    backup_parser.add_argument(
        "--compress", choices=["gzip", "zstd", "none"], default="gzip",
        help="Algorithme de compression (défaut : %(default)s)",
    )

    # Sous-commande : cleanup
    cleanup_parser = subparsers.add_parser("cleanup", help="Nettoyer les anciennes sauvegardes")
    cleanup_parser.add_argument(
        "--retention", type=int, default=30,
        help="Rétention en jours (défaut : %(default)s)",
    )

    # Sous-commande : verify
    verify_parser = subparsers.add_parser("verify", help="Vérifier l'intégrité")
    verify_parser.add_argument(
        "--latest", action="store_true",
        help="Vérifier uniquement la dernière sauvegarde",
    )

    return parser.parse_args()
```

Utilisation résultante :

```bash
$ ./backup_pg.py --help
$ ./backup_pg.py backup --databases production --compress zstd
$ ./backup_pg.py cleanup --retention 14 --dry-run
$ ./backup_pg.py verify --latest
```

## Manipulation du système de fichiers avec `pathlib`

### Remplacement de `os.path`

Le module `pathlib` (Python 3.4+) remplace avantageusement les fonctions dispersées dans `os.path` par une API orientée objet cohérente :

```python
from pathlib import Path

# Construire des chemins (l'opérateur / concatène)
config_dir = Path("/etc") / "nginx"  
access_log = config_dir / "logs" / "access.log"  

# Informations sur un chemin
print(access_log.name)        # access.log  
print(access_log.stem)        # access  
print(access_log.suffix)      # .log  
print(access_log.parent)      # /etc/nginx/logs  
print(access_log.parts)       # ('/', 'etc', 'nginx', 'logs', 'access.log')  

# Tests d'existence et de type
access_log.exists()            # True/False  
access_log.is_file()           # True/False  
access_log.is_dir()            # False  
access_log.is_symlink()        # True/False  

# Métadonnées
stat = access_log.stat()  
print(stat.st_size)            # Taille en octets  
print(stat.st_mtime)           # Timestamp de dernière modification  
```

### Opérations courantes


### Exemple concret : nettoyage de fichiers anciens

```python
def cleanup_old_files(
    directory: Path,
    pattern: str,
    max_age_days: int,
    dry_run: bool = False,
) -> dict[str, int]:
    """Supprime les fichiers correspondant au pattern et plus anciens que max_age_days.

    Retourne un dictionnaire avec les compteurs de résultats.
    """
    cutoff = datetime.now().timestamp() - (max_age_days * 86400)
    stats = {"scanned": 0, "deleted": 0, "freed_bytes": 0, "errors": 0}

    if not directory.is_dir():
        logger.error("Répertoire inexistant : %s", directory)
        return stats

    for filepath in directory.glob(pattern):
        if not filepath.is_file():
            continue

        stats["scanned"] += 1

        try:
            file_mtime = filepath.stat().st_mtime
            if file_mtime < cutoff:
                size = filepath.stat().st_size
                if dry_run:
                    logger.info("[DRY-RUN] Suppression : %s (%s)", filepath.name, format_size(size))
                else:
                    filepath.unlink()
                    logger.info("Supprimé : %s (%s)", filepath.name, format_size(size))
                stats["deleted"] += 1
                stats["freed_bytes"] += size
        except OSError as e:
            logger.error("Erreur sur %s : %s", filepath, e)
            stats["errors"] += 1

    logger.info(
        "Nettoyage terminé : %d/%d fichier(s) supprimé(s), %s libéré(s)",
        stats["deleted"], stats["scanned"], format_size(stats["freed_bytes"]),
    )
    return stats
```

## Exécution de commandes avec `subprocess`

### Le module `subprocess`

`subprocess` est l'interface Python pour exécuter des commandes système. Il remplace les anciennes fonctions `os.system()` et `os.popen()` et offre un contrôle complet sur l'entrée, la sortie, les codes de retour et les erreurs :

```python
import subprocess

# Exécution simple avec vérification du code de retour
result = subprocess.run(
    ["systemctl", "is-active", "nginx"],
    capture_output=True,
    text=True,
)
print(f"Code de retour : {result.returncode}")  
print(f"Sortie : {result.stdout.strip()}")  

# check=True : lève une exception si le code de retour est non-zéro
try:
    subprocess.run(
        ["systemctl", "reload", "nginx"],
        check=True,
        capture_output=True,
        text=True,
    )
    logger.info("Nginx rechargé avec succès")
except subprocess.CalledProcessError as e:
    logger.error("Échec du rechargement : %s", e.stderr)
```

### Paramètres importants de `subprocess.run`

| Paramètre | Effet |
|-----------|-------|
| `capture_output=True` | Capture stdout et stderr (équiv. `stdout=PIPE, stderr=PIPE`) |
| `text=True` | Décode la sortie en chaîne (sinon `bytes`) |
| `check=True` | Lève `CalledProcessError` si code retour ≠ 0 |
| `timeout=30` | Timeout en secondes (`TimeoutExpired` si dépassé) |
| `cwd="/tmp"` | Répertoire de travail |
| `env={...}` | Variables d'environnement (remplace l'environnement) |
| `input="data"` | Données envoyées sur stdin |
| `shell=True` | Exécuter via `/bin/sh -c` (à éviter, voir ci-dessous) |

### Sécurité : éviter `shell=True`

L'option `shell=True` passe la commande à `/bin/sh -c` comme une seule chaîne, ce qui est vulnérable à l'injection de commandes si des données utilisateur sont incluses :

```python
# DANGEREUX : injection possible si nom_fichier contient "; rm -rf /"
nom_fichier = input("Nom du fichier : ")  
subprocess.run(f"cat {nom_fichier}", shell=True)  

# CORRECT : passer la commande comme liste (pas d'interprétation shell)
subprocess.run(["cat", nom_fichier])
```

> **Règle** : utilisez toujours la forme liste (`["commande", "arg1", "arg2"]`) sauf quand les fonctionnalités du shell sont réellement nécessaires (pipes, redirections, globbing). Dans ces cas, construisez la commande de manière sûre ou utilisez les équivalents Python.

### Remplacer les pipelines Bash

Au lieu de `commande1 | commande2` via `shell=True`, chaînez les processus avec `subprocess.PIPE` :

```python
# Équivalent de : grep "Failed password" /var/log/auth.log | wc -l
p1 = subprocess.Popen(
    ["grep", "Failed password", "/var/log/auth.log"],
    stdout=subprocess.PIPE,
    stderr=subprocess.DEVNULL,
)
p2 = subprocess.Popen(
    ["wc", "-l"],
    stdin=p1.stdout,
    capture_output=True,
    text=True,
)
p1.stdout.close()  # Permettre à p1 de recevoir SIGPIPE si p2 termine  
output = p2.communicate()[0]  
count = int(output.strip())  
```

Dans la plupart des cas, il est cependant plus lisible et plus Pythonique de traiter les données directement en Python :

```python
# Approche Pythonique : pas de pipeline, traitement natif
from pathlib import Path

count = sum(
    1 for line in Path("/var/log/auth.log").open()
    if "Failed password" in line
)
```

### Exemple concret : wrapper de commande système

```python
def run_command(
    cmd: list[str],
    description: str = "",
    dry_run: bool = False,
    timeout: int = 300,
) -> subprocess.CompletedProcess:
    """Exécute une commande système avec logging et gestion d'erreurs."""
    desc = description or " ".join(cmd)

    if dry_run:
        logger.info("[DRY-RUN] %s", desc)
        return subprocess.CompletedProcess(cmd, 0, "", "")

    logger.debug("Exécution : %s", " ".join(cmd))

    try:
        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=timeout,
            check=True,
        )
        logger.info("[OK] %s", desc)
        return result

    except subprocess.CalledProcessError as e:
        logger.error("[ÉCHEC] %s (code %d)", desc, e.returncode)
        if e.stderr:
            logger.error("stderr : %s", e.stderr.strip())
        raise

    except subprocess.TimeoutExpired:
        logger.error("[TIMEOUT] %s (> %ds)", desc, timeout)
        raise
```

## Journalisation avec `logging`

### Configuration pour les scripts d'administration

Le module `logging` de la bibliothèque standard est bien plus riche que les fonctions `log_info`/`log_error` écrites manuellement en Bash :

```python
import logging  
import logging.handlers  
import sys  

def setup_logging(
    verbose: bool = False,
    log_file: str | None = None,
    syslog: bool = False,
) -> None:
    """Configure la journalisation avec plusieurs destinations."""
    root_logger = logging.getLogger()
    root_logger.setLevel(logging.DEBUG)

    # Format commun
    formatter = logging.Formatter(
        fmt="%(asctime)s [%(levelname)-5s] [%(name)s] %(message)s",
        datefmt="%Y-%m-%d %H:%M:%S",
    )

    # Handler : stderr (toujours actif)
    console_handler = logging.StreamHandler(sys.stderr)
    console_handler.setLevel(logging.DEBUG if verbose else logging.INFO)
    console_handler.setFormatter(formatter)
    root_logger.addHandler(console_handler)

    # Handler : fichier (optionnel, avec rotation automatique)
    if log_file:
        file_handler = logging.handlers.RotatingFileHandler(
            log_file,
            maxBytes=10 * 1024 * 1024,  # 10 Mo
            backupCount=5,
            encoding="utf-8",
        )
        file_handler.setLevel(logging.DEBUG)
        file_handler.setFormatter(formatter)
        root_logger.addHandler(file_handler)

    # Handler : syslog/journald (optionnel)
    if syslog:
        syslog_handler = logging.handlers.SysLogHandler(
            address="/dev/log",
            facility=logging.handlers.SysLogHandler.LOG_USER,
        )
        syslog_formatter = logging.Formatter(
            fmt="%(name)s[%(process)d]: [%(levelname)s] %(message)s"
        )
        syslog_handler.setFormatter(syslog_formatter)
        syslog_handler.setLevel(logging.WARNING)
        root_logger.addHandler(syslog_handler)
```

Avec cette configuration, le logger écrit sur stderr (toujours), dans un fichier avec rotation (si demandé) et dans syslog/journald (si demandé). Chaque handler a son propre niveau de sévérité, ce qui permet par exemple de ne journaliser que les warnings et erreurs dans syslog tout en gardant le détail complet dans le fichier de log.

### Utilisation dans le code

```python
# Créer un logger par module (bonne pratique)
logger = logging.getLogger(__name__)

def sauvegarder_base(database: str, backup_dir: Path) -> Path:
    logger.info("Début de la sauvegarde de %s", database)

    output_file = backup_dir / f"{database}_{datetime.now():%Y%m%d_%H%M%S}.sql.gz"
    logger.debug("Fichier de sortie : %s", output_file)

    try:
        result = subprocess.run(
            ["pg_dump", "-Fc", database],
            capture_output=True,
            check=True,
            timeout=3600,
        )
        output_file.write_bytes(result.stdout)
        size = output_file.stat().st_size
        logger.info("Sauvegarde terminée : %s (%s)", output_file.name, format_size(size))
        return output_file

    except subprocess.CalledProcessError as e:
        logger.error("Échec de pg_dump pour %s : %s", database, e.stderr)
        raise
    except subprocess.TimeoutExpired:
        logger.critical("Timeout de pg_dump pour %s (> 1h)", database)
        raise
```

### INI avec `configparser`

```python
from configparser import ConfigParser  
from pathlib import Path  

def charger_config_ini(chemin: Path) -> ConfigParser:
    """Charge un fichier de configuration INI avec valeurs par défaut."""
    config = ConfigParser()

    # Valeurs par défaut
    config["DEFAULT"] = {
        "log_level": "info",
        "timeout": "30",
    }
    config["backup"] = {
        "directory": "/var/backups",
        "retention_days": "30",
        "compress": "gzip",
    }
    config["notification"] = {
        "enabled": "false",
        "email": "",
    }

    # Charger le fichier (écrase les défauts)
    if chemin.exists():
        config.read(chemin)
    else:
        logger.warning("Fichier de configuration absent, utilisation des défauts : %s", chemin)

    return config

# Utilisation
config = charger_config_ini(Path("/etc/monapp/backup.ini"))

backup_dir = Path(config.get("backup", "directory"))  
retention = config.getint("backup", "retention_days")  
compress = config.get("backup", "compress")  
notify_enabled = config.getboolean("notification", "enabled")  
```

### YAML avec `pyyaml`

```python
import yaml  
from pathlib import Path  

def charger_config_yaml(chemin: Path) -> dict:
    """Charge un fichier YAML avec validation basique."""
    try:
        contenu = chemin.read_text(encoding="utf-8")
        config = yaml.safe_load(contenu)
    except FileNotFoundError:
        logger.error("Fichier introuvable : %s", chemin)
        raise
    except yaml.YAMLError as e:
        logger.error("Erreur de syntaxe YAML dans %s : %s", chemin, e)
        raise

    if not isinstance(config, dict):
        raise ValueError(f"Le fichier YAML doit contenir un dictionnaire, pas {type(config)}")

    return config

# Utilisation
config = charger_config_yaml(Path("/etc/monapp/config.yaml"))

# Accès avec valeurs par défaut
backup_dir = Path(config.get("backup", {}).get("directory", "/var/backups"))  
retention = config.get("backup", {}).get("retention_days", 30)  
```

> **Sécurité** : utilisez toujours `yaml.safe_load()` et jamais `yaml.load()`. Ce dernier peut exécuter du code Python arbitraire contenu dans le fichier YAML, ce qui constitue une vulnérabilité critique.

### Variables d'environnement et valeurs par défaut

Un pattern courant combine la configuration par fichier et les variables d'environnement, ces dernières ayant la priorité :

```python
import os  
from dataclasses import dataclass, field  
from pathlib import Path  

@dataclass
class AppConfig:
    """Configuration de l'application avec résolution multi-sources."""
    backup_dir: Path = field(default_factory=lambda: Path("/var/backups"))
    retention_days: int = 30
    db_host: str = "localhost"
    db_port: int = 5432
    db_name: str = "production"
    log_level: str = "info"
    dry_run: bool = False

    @classmethod
    def from_env_and_file(cls, config_path: Path | None = None) -> "AppConfig":
        """Charge la config depuis un fichier YAML puis applique les variables d'env."""
        # Charger depuis le fichier si présent
        file_config = {}
        if config_path and config_path.exists():
            file_config = yaml.safe_load(config_path.read_text()) or {}

        # Les variables d'environnement priment sur le fichier
        return cls(
            backup_dir=Path(os.getenv("BACKUP_DIR", file_config.get("backup_dir", "/var/backups"))),
            retention_days=int(os.getenv("RETENTION_DAYS", file_config.get("retention_days", 30))),
            db_host=os.getenv("DB_HOST", file_config.get("db_host", "localhost")),
            db_port=int(os.getenv("DB_PORT", file_config.get("db_port", 5432))),
            db_name=os.getenv("DB_NAME", file_config.get("db_name", "production")),
            log_level=os.getenv("LOG_LEVEL", file_config.get("log_level", "info")),
            dry_run=os.getenv("DRY_RUN", "").lower() in ("1", "true", "yes"),
        )
```

## Script complet : rapport d'inventaire système

Voici un script complet qui combine tous les patterns abordés — arguments, logging, système de fichiers, subprocess, configuration — dans un cas d'usage concret :

```python
#!/usr/bin/env python3
"""
system_inventory.py — Génération d'un inventaire système au format JSON.

Collecte les informations système (OS, CPU, mémoire, disques, services,  
paquets) et produit un rapport JSON sur stdout.  
"""

import argparse  
import json  
import logging  
import platform  
import subprocess  
import sys  
from datetime import datetime, timezone  
from pathlib import Path  

logger = logging.getLogger("inventory")


def setup_logging(verbose: bool = False) -> None:
    logging.basicConfig(
        level=logging.DEBUG if verbose else logging.INFO,
        format="%(asctime)s [%(levelname)-5s] %(message)s",
        datefmt="%Y-%m-%d %H:%M:%S",
        stream=sys.stderr,
    )


def get_os_info() -> dict:
    """Informations sur le système d'exploitation."""
    info = {
        "hostname": platform.node(),
        "system": platform.system(),
        "kernel": platform.release(),
        "architecture": platform.machine(),
    }

    debian_version = Path("/etc/debian_version")
    if debian_version.exists():
        info["debian_version"] = debian_version.read_text().strip()

    return info


def get_cpu_info() -> dict:
    """Informations sur le processeur."""
    info = {"cores": 0, "model": "unknown"}

    cpuinfo = Path("/proc/cpuinfo")
    if cpuinfo.exists():
        lines = cpuinfo.read_text().splitlines()
        models = [l.split(":")[1].strip() for l in lines if l.startswith("model name")]
        info["cores"] = len(models)
        if models:
            info["model"] = models[0]

    load = Path("/proc/loadavg")
    if load.exists():
        parts = load.read_text().split()
        info["load_average"] = {
            "1min": float(parts[0]),
            "5min": float(parts[1]),
            "15min": float(parts[2]),
        }

    return info


def get_memory_info() -> dict:
    """Informations sur la mémoire."""
    meminfo = {}

    for line in Path("/proc/meminfo").read_text().splitlines():
        if ":" in line:
            key, val = line.split(":", 1)
            # Convertir en octets (les valeurs sont en kB)
            parts = val.strip().split()
            meminfo[key.strip()] = int(parts[0]) * 1024 if parts else 0

    total = meminfo.get("MemTotal", 0)
    available = meminfo.get("MemAvailable", 0)
    used = total - available

    return {
        "total_bytes": total,
        "available_bytes": available,
        "used_bytes": used,
        "usage_pct": round(used / total * 100, 1) if total else 0,
    }


def get_disk_info() -> list[dict]:
    """Informations sur les partitions montées."""
    disks = []
    seen = set()

    for line in Path("/proc/mounts").read_text().splitlines():
        parts = line.split()
        if len(parts) < 3:
            continue

        device, mount_point, fs_type = parts[0], parts[1], parts[2]
        if fs_type in ("tmpfs", "devtmpfs", "sysfs", "proc", "cgroup2",
                        "squashfs", "overlay", "securityfs", "efivarfs"):
            continue
        if mount_point in seen:
            continue
        seen.add(mount_point)

        try:
            import shutil
            usage = shutil.disk_usage(mount_point)
            disks.append({
                "mount": mount_point,
                "device": device,
                "fs_type": fs_type,
                "total_bytes": usage.total,
                "used_bytes": usage.used,
                "free_bytes": usage.free,
                "usage_pct": round(usage.used / usage.total * 100, 1) if usage.total else 0,
            })
        except (PermissionError, OSError) as e:
            logger.debug("Impossible de lire %s : %s", mount_point, e)

    return sorted(disks, key=lambda d: d["mount"])


def get_services_status(services: list[str]) -> list[dict]:
    """État des services systemd."""
    results = []

    for svc in services:
        try:
            result = subprocess.run(
                ["systemctl", "show", svc, "--property=ActiveState,MainPID,StateChangeTimestamp"],
                capture_output=True, text=True, timeout=10,
            )
            props = {}
            for line in result.stdout.strip().splitlines():
                if "=" in line:
                    k, v = line.split("=", 1)
                    props[k] = v

            results.append({
                "name": svc,
                "active": props.get("ActiveState", "unknown"),
                "pid": int(props.get("MainPID", 0)),
                "since": props.get("StateChangeTimestamp", ""),
            })
        except (subprocess.TimeoutExpired, OSError):
            results.append({"name": svc, "active": "unknown", "pid": 0, "since": ""})

    return results


def get_package_counts() -> dict:
    """Nombre de paquets installés et mises à jour disponibles."""
    info = {"installed": 0, "upgradable": 0, "security": 0}

    try:
        result = subprocess.run(
            ["dpkg-query", "-f", "${Status}\n", "-W"],
            capture_output=True, text=True, timeout=30,
        )
        info["installed"] = sum(
            1 for line in result.stdout.splitlines()
            if "install ok installed" in line
        )
    except (subprocess.SubprocessError, OSError):
        logger.warning("Impossible de compter les paquets installés")

    return info


def build_inventory(services: list[str]) -> dict:
    """Construit l'inventaire complet."""
    logger.info("Collecte des informations système...")

    inventory = {
        "meta": {
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "generator": "system_inventory.py",
        },
        "os": get_os_info(),
        "cpu": get_cpu_info(),
        "memory": get_memory_info(),
        "disks": get_disk_info(),
        "services": get_services_status(services),
        "packages": get_package_counts(),
    }

    logger.info("Inventaire collecté pour %s", inventory["os"]["hostname"])
    return inventory


def parse_args(argv: list[str] | None = None) -> argparse.Namespace:
    parser = argparse.ArgumentParser(description="Inventaire système au format JSON.")
    parser.add_argument("-v", "--verbose", action="store_true")
    parser.add_argument(
        "-s", "--services", nargs="+",
        default=["ssh", "cron", "nginx", "postgresql"],
        help="Services à vérifier (défaut : ssh cron nginx postgresql)",
    )
    parser.add_argument(
        "-o", "--output", type=Path, default=None,
        help="Fichier de sortie (défaut : stdout)",
    )
    return parser.parse_args(argv)


def main(argv: list[str] | None = None) -> int:
    args = parse_args(argv)
    setup_logging(verbose=args.verbose)

    inventory = build_inventory(args.services)
    json_output = json.dumps(inventory, indent=2, ensure_ascii=False, default=str)

    if args.output:
        args.output.write_text(json_output + "\n", encoding="utf-8")
        logger.info("Inventaire écrit dans %s", args.output)
    else:
        print(json_output)

    return 0


if __name__ == "__main__":
    sys.exit(main())
```

Ce script illustre comment les patterns Python — `pathlib` pour les fichiers, `subprocess` pour les commandes, `logging` pour le diagnostic, `argparse` pour les options, `json` pour la sortie structurée — se combinent pour produire un outil d'administration propre, testable et extensible.

## Résumé

Les points clés abordés dans cette sous-section sont :

- **Le squelette standard** d'un script Python d'administration comprend une docstring de module, des imports organisés, une fonction `parse_args` avec `argparse`, une fonction `main` retournant un code de sortie, et le bloc `if __name__ == "__main__"` pour la testabilité.
- **`argparse`** remplace avantageusement `getopts` et le parsing manuel en Bash : types validés, sous-commandes, aide auto-générée, groupes mutuellement exclusifs, validateurs personnalisés.
- **`pathlib.Path`** offre une API orientée objet pour la manipulation du système de fichiers : construction de chemins avec `/`, glob, lecture/écriture, métadonnées. Combiné avec `shutil` et `tempfile`, il couvre tous les besoins.
- **`subprocess.run`** exécute les commandes système avec `capture_output=True` et `text=True` pour capturer les sorties, `check=True` pour lever une exception en cas d'erreur, et `timeout` pour éviter les blocages. La forme liste (`["cmd", "arg"]`) est toujours préférée à `shell=True`.
- **`logging`** fournit une journalisation structurée avec niveaux de sévérité, handlers multiples (stderr, fichier avec rotation, syslog), et loggers par module. La configuration est centralisée dans une fonction `setup_logging`.
- **Les fichiers de configuration** sont gérés par `configparser` (INI), `json` (JSON), `pyyaml` (YAML) ou `tomllib` (TOML). Le pattern de priorité variable d'environnement > fichier > valeur par défaut est recommandé pour les scripts de production.
