# 5.3.3 Bibliothèques utiles (os, subprocess, paramiko, requests)

## Introduction

La sous-section précédente a posé les fondations du scripting Python pour l'administration : structure de script, `argparse`, `pathlib`, `subprocess`, `logging` et fichiers de configuration. Cette sous-section élargit la boîte à outils en explorant les bibliothèques — standard et tierces — que tout administrateur système Python devrait connaître.

L'objectif est de couvrir les cas d'usage les plus fréquents en administration Debian : interaction avec le système d'exploitation, exécution à distance via SSH, appels HTTP vers des APIs, collecte de métriques système, génération de configurations à partir de templates et manipulation de bases de données. Pour chaque bibliothèque, l'accent est mis sur les patterns directement applicables en production, plutôt que sur une couverture exhaustive de l'API.

## Bibliothèques standard

### `os` — Interface avec le système d'exploitation

Le module `os` fournit l'accès aux fonctionnalités du système d'exploitation. Si `pathlib` couvre la manipulation de chemins et de fichiers (voir 5.3.2), `os` reste indispensable pour les interactions système plus larges :

```python
import os

# ── Informations système ──────────────────────────────────
os.getuid()                     # UID de l'utilisateur courant  
os.geteuid()                    # UID effectif (après setuid)  
os.getpid()                     # PID du processus courant  
os.getppid()                    # PID du processus parent  
os.uname()                      # (sysname, nodename, release, version, machine)  
os.cpu_count()                  # Nombre de CPUs logiques  
os.getloadavg()                 # Load average (1, 5, 15 minutes)  

# ── Variables d'environnement ─────────────────────────────
os.environ["PATH"]              # Lire (lève KeyError si absente)  
os.environ.get("EDITOR", "vi")  # Lire avec valeur par défaut  
os.environ["MA_VAR"] = "valeur" # Définir (visible par les processus enfants)  

# ── Utilisateurs et groupes ───────────────────────────────
os.getlogin()                   # Nom de l'utilisateur connecté

import pwd, grp  
pwd.getpwnam("www-data")        # Infos utilisateur par nom  
pwd.getpwuid(33)                # Infos utilisateur par UID  
grp.getgrnam("sudo")            # Infos groupe par nom  
# pwd.struct_passwd(pw_name='www-data', pw_uid=33, pw_gid=33, pw_dir='/var/www', ...)

# ── Permissions et propriété ──────────────────────────────
os.chmod("/opt/scripts/backup.sh", 0o750)  
os.chown("/var/backups/dump.sql", uid=0, gid=34)  # root:backup  

# ── Processus ─────────────────────────────────────────────
os.kill(pid, signal.SIGTERM)    # Envoyer un signal  
os.waitpid(pid, 0)              # Attendre la fin d'un processus enfant  
```

### `os` pour la détection de l'environnement

Un pattern fréquent en administration est l'adaptation du comportement selon l'environnement d'exécution :

```python
import os  
import sys  

def verifier_environnement() -> None:
    """Vérifie les prérequis d'exécution."""
    # Vérifier que le script est exécuté en tant que root
    if os.geteuid() != 0:
        sys.exit("Ce script doit être exécuté en tant que root")

    # Vérifier le système d'exploitation
    if not os.path.isfile("/etc/debian_version"):
        sys.exit("Ce script est conçu pour Debian")

    # Vérifier qu'on n'est pas dans un conteneur (heuristique)
    in_container = (
        os.path.isfile("/.dockerenv")
        or os.environ.get("container") == "docker"
        or "docker" in open("/proc/1/cgroup", "r").read()
    )
    if in_container:
        logger.info("Exécution détectée dans un conteneur")
```

### `shutil` — Opérations de fichiers de haut niveau

`shutil` complète `pathlib` et `os` pour les opérations qui portent sur des arborescences entières :

```python
import shutil  
from pathlib import Path  

# Copie de fichiers
shutil.copy2(src, dst)              # Copie avec métadonnées (timestamps, permissions)  
shutil.copyfile(src, dst)           # Copie du contenu uniquement  

# Copie d'arborescence
shutil.copytree(
    "/etc/nginx",
    "/var/backups/nginx_20250414",
    dirs_exist_ok=True,              # Ne pas échouer si la destination existe (3.8+)
)

# Suppression d'arborescence
shutil.rmtree("/tmp/work_dir", ignore_errors=True)

# Archivage (équivalent de tar czf)
shutil.make_archive(
    base_name="/var/backups/etc_nginx",   # Sans extension
    format="gztar",                       # tar.gz
    root_dir="/etc",                      # Référence pour les chemins relatifs
    base_dir="nginx",                     # Répertoire à archiver
)
# Produit /var/backups/etc_nginx.tar.gz

# Espace disque (vu en 5.3.2)
usage = shutil.disk_usage("/var")  
print(f"Total: {usage.total}, Utilisé: {usage.used}, Libre: {usage.free}")  

# Trouver un exécutable dans le PATH (équivalent de 'which')
shutil.which("pg_dump")               # '/usr/bin/pg_dump' ou None
```

### `socket` — Réseau bas niveau

Le module `socket` est utile pour les vérifications de connectivité et la découverte réseau :

```python
import socket

# Résolution DNS
socket.gethostname()                            # Nom de la machine locale  
socket.getfqdn()                                # FQDN complet  
socket.gethostbyname("db.example.com")          # Résolution A  
socket.getaddrinfo("db.example.com", 5432)      # Résolution complète (IPv4+IPv6)  

# Vérifier la connectivité TCP vers un service
def check_port(host: str, port: int, timeout: float = 5.0) -> bool:
    """Teste si un port TCP est accessible."""
    try:
        with socket.create_connection((host, port), timeout=timeout):
            return True
    except (socket.timeout, ConnectionRefusedError, OSError):
        return False

# Vérification de services
services = {
    "postgresql": ("db01.example.com", 5432),
    "redis": ("cache01.example.com", 6379),
    "nginx": ("web01.example.com", 443),
}

for name, (host, port) in services.items():
    status = "OK" if check_port(host, port) else "ÉCHEC"
    logger.info("%s (%s:%d) : %s", name, host, port, status)
```

### `hashlib` — Vérification d'intégrité

```python
import hashlib  
from pathlib import Path  

def checksum_file(filepath: Path, algorithm: str = "sha256") -> str:
    """Calcule le hash d'un fichier (lecture par blocs pour les gros fichiers)."""
    h = hashlib.new(algorithm)
    with open(filepath, "rb") as f:
        for chunk in iter(lambda: f.read(8192), b""):
            h.update(chunk)
    return h.hexdigest()

# Vérifier l'intégrité d'une sauvegarde
backup = Path("/var/backups/dump_20250414.sql.gz")  
sha = checksum_file(backup)  
logger.info("SHA-256 de %s : %s", backup.name, sha)  

# Comparer avec un checksum de référence
reference = Path("/var/backups/dump_20250414.sha256").read_text().split()[0]  
if sha != reference:  
    logger.error("Intégrité compromise pour %s", backup.name)
```

### `tarfile` et `zipfile` — Archivage programmatique

```python
import tarfile  
from pathlib import Path  
from datetime import datetime  

def creer_archive_config(
    repertoire: Path,
    destination: Path,
) -> Path:
    """Crée une archive tar.gz d'un répertoire de configuration."""
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    archive_name = destination / f"config_{repertoire.name}_{timestamp}.tar.gz"

    with tarfile.open(archive_name, "w:gz") as tar:
        tar.add(repertoire, arcname=repertoire.name)

    logger.info("Archive créée : %s (%s)",
                archive_name.name,
                format_size(archive_name.stat().st_size))
    return archive_name

def extraire_archive(archive: Path, destination: Path) -> None:
    """Extrait une archive tar.gz de manière sécurisée."""
    with tarfile.open(archive, "r:gz") as tar:
        # Protection contre les attaques par traversée de chemin
        for member in tar.getmembers():
            member_path = Path(destination / member.name).resolve()
            if not str(member_path).startswith(str(destination.resolve())):
                raise ValueError(f"Chemin suspect dans l'archive : {member.name}")
        tar.extractall(path=destination)

    logger.info("Archive extraite dans %s", destination)
```

> **Sécurité** : l'appel à `tar.extractall()` sans vérification préalable est vulnérable aux attaques par traversée de chemin (un fichier nommé `../../etc/passwd` dans l'archive). Python 3.12+ offre le paramètre `filter='data'` pour se protéger automatiquement. Sur les versions antérieures, la vérification manuelle montrée ci-dessus est nécessaire.

### `concurrent.futures` — Parallélisme simplifié

Pour les tâches d'administration qui doivent s'exécuter en parallèle sur plusieurs serveurs ou ressources :

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

def verifier_serveur(host: str) -> dict:
    """Vérifie la connectivité et collecte les infos d'un serveur."""
    result = {"host": host, "ssh": False, "http": False, "latency_ms": None}

    start = time.time()
    result["ssh"] = check_port(host, 22, timeout=5)
    result["http"] = check_port(host, 80, timeout=5)
    result["latency_ms"] = round((time.time() - start) * 1000, 1)

    return result

def verifier_parc(serveurs: list[str], max_workers: int = 10) -> list[dict]:
    """Vérifie un parc de serveurs en parallèle."""
    resultats = []

    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        futures = {
            executor.submit(verifier_serveur, host): host
            for host in serveurs
        }

        for future in as_completed(futures):
            host = futures[future]
            try:
                result = future.result(timeout=30)
                resultats.append(result)
                status = "OK" if result["ssh"] else "ÉCHEC"
                logger.info("%s : %s (%s ms)", host, status, result["latency_ms"])
            except Exception as e:
                logger.error("%s : exception — %s", host, e)
                resultats.append({"host": host, "error": str(e)})

    return resultats

# Vérifier 50 serveurs en parallèle (10 threads simultanés)
serveurs = [f"web{i:02d}.example.com" for i in range(1, 51)]  
resultats = verifier_parc(serveurs)  
```

`ThreadPoolExecutor` est idéal pour les opérations liées à l'I/O (réseau, disque, SSH). Pour les opérations liées au CPU (parsing intensif, compression), utilisez `ProcessPoolExecutor` qui contourne le GIL de Python.

### Sessions `requests` pour les appels multiples

Une session HTTP réutilise les connexions TCP, persiste les en-têtes et gère les cookies :

```python
class APIClient:
    """Client API réutilisable avec session HTTP."""

    def __init__(self, base_url: str, token: str, timeout: int = 30):
        self.base_url = base_url.rstrip("/")
        self.timeout = timeout
        self.session = requests.Session()
        self.session.headers.update({
            "Authorization": f"Bearer {token}",
            "Accept": "application/json",
            "User-Agent": "admin-scripts/1.0",
        })

    def get(self, endpoint: str, **kwargs) -> dict:
        return self._request("GET", endpoint, **kwargs)

    def post(self, endpoint: str, data: dict = None, **kwargs) -> dict:
        return self._request("POST", endpoint, json=data, **kwargs)

    def _request(self, method: str, endpoint: str, **kwargs) -> dict:
        url = f"{self.base_url}{endpoint}"
        kwargs.setdefault("timeout", self.timeout)

        try:
            response = self.session.request(method, url, **kwargs)
            response.raise_for_status()
            return response.json() if response.content else {}

        except requests.exceptions.ConnectionError as e:
            logger.error("Connexion impossible à %s : %s", url, e)
            raise
        except requests.exceptions.Timeout:
            logger.error("Timeout sur %s %s (> %ds)", method, url, self.timeout)
            raise
        except requests.exceptions.HTTPError as e:
            logger.error(
                "Erreur HTTP %d sur %s %s : %s",
                e.response.status_code, method, url,
                e.response.text[:200],
            )
            raise

    def close(self):
        self.session.close()

    def __enter__(self):
        return self

    def __exit__(self, *args):
        self.close()
```

Utilisation :

```python
with APIClient("https://api.example.com/v1", token=API_TOKEN) as api:

        details = api.get(f"/servers/{srv['id']}/metrics")
        logger.info("%s : CPU %.1f%%, RAM %.1f%%",
                     srv["name"], details["cpu"], details["memory"])

    # Créer un incident

        "title": "Rapport automatisé",

    })
```

### Retry automatique avec `requests`

```python
from requests.adapters import HTTPAdapter  
from urllib3.util.retry import Retry  

def creer_session_avec_retry(
    retries: int = 3,
    backoff_factor: float = 1.0,
    status_forcelist: tuple = (429, 500, 502, 503, 504),
) -> requests.Session:
    """Crée une session HTTP avec retry automatique sur les erreurs transitoires."""
    session = requests.Session()

    retry_strategy = Retry(
        total=retries,
        backoff_factor=backoff_factor,        # 1s, 2s, 4s...
        status_forcelist=status_forcelist,
        allowed_methods=["GET", "POST", "PUT", "DELETE"],
    )

    adapter = HTTPAdapter(max_retries=retry_strategy)
    session.mount("https://", adapter)
    session.mount("http://", adapter)

    return session

# Utilisation — les retries sont automatiques et transparents
session = creer_session_avec_retry()  
session.headers["Authorization"] = f"Bearer {token}"  
response = session.get("https://api.example.com/v1/data", timeout=30)  
```

### `paramiko` — SSH programmatique

`paramiko` permet d'exécuter des commandes à distance via SSH et de transférer des fichiers via SFTP, sans dépendre de la commande `ssh` système :

```python
import paramiko  
from pathlib import Path  

class SSHClient:
    """Client SSH simplifié pour l'administration."""

    def __init__(
        self,
        hostname: str,
        username: str = "root",
        key_filename: str | None = None,
        timeout: int = 10,
        known_hosts: str | None = None,
    ):
        self.hostname = hostname
        self.client = paramiko.SSHClient()

        # Sécurité : charger les clés d'hôte connues (~/.ssh/known_hosts par défaut)
        # et REJETER toute connexion vers un hôte non vérifié.
        # AutoAddPolicy() est volontairement évité car il accepte silencieusement
        # n'importe quelle clé, ce qui est vulnérable au MITM en production.
        self.client.load_system_host_keys()
        if known_hosts:
            self.client.load_host_keys(known_hosts)
        self.client.set_missing_host_key_policy(paramiko.RejectPolicy())

        connect_kwargs = {
            "hostname": hostname,
            "username": username,
            "timeout": timeout,
        }
        if key_filename:
            connect_kwargs["key_filename"] = key_filename

        self.client.connect(**connect_kwargs)
        logger.debug("Connecté à %s en tant que %s", hostname, username)

    def run(self, command: str, timeout: int = 300) -> tuple[int, str, str]:
        """Exécute une commande et retourne (code_retour, stdout, stderr)."""
        stdin, stdout, stderr = self.client.exec_command(command, timeout=timeout)
        exit_code = stdout.channel.recv_exit_status()

        out = stdout.read().decode("utf-8", errors="replace")
        err = stderr.read().decode("utf-8", errors="replace")

        logger.debug("[%s] %s → code=%d", self.hostname, command, exit_code)
        return exit_code, out, err

    def run_checked(self, command: str, timeout: int = 300) -> str:
        """Exécute une commande et lève une exception en cas d'erreur."""
        code, out, err = self.run(command, timeout)
        if code != 0:
            raise RuntimeError(
                f"Commande échouée sur {self.hostname} (code {code}): {err.strip()}"
            )
        return out

    def upload(self, local_path: Path, remote_path: str) -> None:
        """Upload un fichier via SFTP."""
        sftp = self.client.open_sftp()
        try:
            sftp.put(str(local_path), remote_path)
            logger.info("Upload %s → %s:%s", local_path.name, self.hostname, remote_path)
        finally:
            sftp.close()

    def download(self, remote_path: str, local_path: Path) -> None:
        """Télécharge un fichier via SFTP."""
        sftp = self.client.open_sftp()
        try:
            sftp.get(remote_path, str(local_path))
            logger.info("Download %s:%s → %s", self.hostname, remote_path, local_path.name)
        finally:
            sftp.close()

    def close(self):
        self.client.close()

    def __enter__(self):
        return self

    def __exit__(self, *args):
        self.close()
```

Utilisation :

```python
# Exécuter une commande sur un serveur distant
with SSHClient("web01.example.com", username="admin") as ssh:
    code, stdout, stderr = ssh.run("df -h /var")
    print(stdout)

    # Commande avec vérification automatique
    uptime = ssh.run_checked("uptime -p")
    print(f"Uptime : {uptime.strip()}")

    # Transférer un fichier de configuration
    ssh.upload(Path("./nginx.conf"), "/etc/nginx/nginx.conf")
    ssh.run_checked("nginx -t")
    ssh.run_checked("systemctl reload nginx")
```

> **Pré-chargement des clés d'hôte** : pour un parc nouveau, les clés des serveurs cibles doivent être présentes dans `~/.ssh/known_hosts` avant la première connexion. Sinon, `RejectPolicy()` interrompt la connexion. Procédure standard avec `ssh-keyscan` :  
>  
> ```bash  
> # Récolter les clés de tous les serveurs et les ajouter au known_hosts  
> for host in web01 web02 db01 db02; do  
>     ssh-keyscan -t ed25519,rsa "$host" >> ~/.ssh/known_hosts  
> done  
> ```  
>  
> Idéalement, les clés sont distribuées via un canal vérifié (gestion de configuration, base d'inventaire signée), pas via `ssh-keyscan` qui ne valide pas la première rencontre. Sur un parc Ansible-géré, l'option `ssh_args=-o StrictHostKeyChecking=accept-new` lors du premier provisionnement crée le fichier known_hosts de manière contrôlée.

### Exécution sur un parc de serveurs

```python
def executer_sur_parc(
    serveurs: list[str],
    commande: str,
    username: str = "admin",
    max_workers: int = 10,
) -> dict[str, dict]:
    """Exécute une commande sur plusieurs serveurs en parallèle."""
    resultats = {}

    def executer_un(host: str) -> dict:
        try:
            with SSHClient(host, username=username, timeout=10) as ssh:
                code, stdout, stderr = ssh.run(commande, timeout=60)
                return {"status": "ok", "code": code, "stdout": stdout, "stderr": stderr}
        except Exception as e:
            return {"status": "error", "error": str(e)}

    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        futures = {executor.submit(executer_un, host): host for host in serveurs}
        for future in as_completed(futures):
            host = futures[future]
            resultats[host] = future.result()

    # Résumé
    ok = sum(1 for r in resultats.values() if r["status"] == "ok" and r["code"] == 0)
    ko = len(resultats) - ok
    logger.info("Résultat : %d/%d succès, %d échecs", ok, len(resultats), ko)

    return resultats

# Vérifier l'espace disque sur tout le parc
resultats = executer_sur_parc(
    ["web01", "web02", "db01", "db02", "cache01"],
    "df -h --output=target,pcent / | tail -1",
)

for host, r in sorted(resultats.items()):
    if r["status"] == "ok":
        print(f"{host}: {r['stdout'].strip()}")
    else:
        print(f"{host}: ERREUR — {r['error']}")
```

> **Note** : pour la gestion de configuration sur un grand nombre de serveurs, Ansible (module 13) est plus adapté que des scripts `paramiko` manuels. `paramiko` est pertinent pour les opérations ponctuelles, les scripts de diagnostic et les cas où Ansible n'est pas disponible.

### `psutil` — Métriques système

`psutil` (process and system utilities) fournit un accès portable et riche aux métriques système, remplaçant avantageusement le parsing de `/proc` et les appels à des commandes comme `df`, `free`, `ps` :

```python
import psutil

# ── CPU ───────────────────────────────────────────────────
psutil.cpu_count(logical=True)           # Nombre de CPUs logiques  
psutil.cpu_count(logical=False)          # Nombre de CPUs physiques  
psutil.cpu_percent(interval=1)           # Utilisation CPU globale (%)  
psutil.cpu_percent(interval=1, percpu=True)  # Par CPU  
psutil.getloadavg()                      # Load average (1, 5, 15 min)  

# ── Mémoire ───────────────────────────────────────────────
mem = psutil.virtual_memory()  
print(f"RAM : {mem.total / 1024**3:.1f} Go total, {mem.percent}% utilisé")  
print(f"Disponible : {mem.available / 1024**3:.1f} Go")  

swap = psutil.swap_memory()  
print(f"Swap : {swap.total / 1024**3:.1f} Go, {swap.percent}% utilisé")  

# ── Disques ───────────────────────────────────────────────
for partition in psutil.disk_partitions():
    if "loop" in partition.device:
        continue
    usage = psutil.disk_usage(partition.mountpoint)
    print(f"{partition.mountpoint}: {usage.percent}% "
          f"({usage.free / 1024**3:.1f} Go libre)")

# I/O disque
io = psutil.disk_io_counters()  
print(f"Lectures : {io.read_bytes / 1024**3:.1f} Go, "  
      f"Écritures : {io.write_bytes / 1024**3:.1f} Go")

# ── Réseau ────────────────────────────────────────────────
for iface, addrs in psutil.net_if_addrs().items():
    for addr in addrs:
        if addr.family.name == "AF_INET":
            print(f"{iface}: {addr.address}")

net_io = psutil.net_io_counters()  
print(f"Envoyé : {net_io.bytes_sent / 1024**3:.1f} Go, "  
      f"Reçu : {net_io.bytes_recv / 1024**3:.1f} Go")

# Connexions réseau actives
for conn in psutil.net_connections(kind="tcp"):
    if conn.status == "ESTABLISHED":
        print(f"{conn.laddr} ↔ {conn.raddr}")

# ── Processus ─────────────────────────────────────────────
# Top 5 processus par utilisation mémoire
processes = sorted(
    psutil.process_iter(["pid", "name", "memory_percent", "cpu_percent"]),
    key=lambda p: p.info["memory_percent"] or 0,
    reverse=True,
)[:5]

for p in processes:
    print(f"PID {p.info['pid']:>6} {p.info['name']:<20} "
          f"RAM: {p.info['memory_percent']:.1f}% CPU: {p.info['cpu_percent']:.1f}%")

# ── Uptime / boot ────────────────────────────────────────
from datetime import datetime  
boot_time = datetime.fromtimestamp(psutil.boot_time())  
uptime = datetime.now() - boot_time  
print(f"Démarré le {boot_time:%Y-%m-%d %H:%M}, uptime : {uptime.days}j {uptime.seconds//3600}h")  
```

### Collecte d'inventaire avec `psutil`

```python
def collecter_metriques() -> dict:
    """Collecte un snapshot complet des métriques système."""
    mem = psutil.virtual_memory()
    swap = psutil.swap_memory()

    return {
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "hostname": socket.getfqdn(),
        "cpu": {
            "count": psutil.cpu_count(),
            "percent": psutil.cpu_percent(interval=1),
            "load_avg": list(psutil.getloadavg()),
        },
        "memory": {
            "total_gb": round(mem.total / 1024**3, 2),
            "used_pct": mem.percent,
            "available_gb": round(mem.available / 1024**3, 2),
        },
        "swap": {
            "total_gb": round(swap.total / 1024**3, 2),
            "used_pct": swap.percent,
        },
        "disks": [
            {
                "mount": p.mountpoint,
                "device": p.device,
                "fs": p.fstype,
                "used_pct": psutil.disk_usage(p.mountpoint).percent,
                "free_gb": round(psutil.disk_usage(p.mountpoint).free / 1024**3, 2),
            }
            for p in psutil.disk_partitions()
            if "loop" not in p.device and p.fstype not in ("squashfs", "tmpfs")
        ],
        "uptime_days": (datetime.now() - datetime.fromtimestamp(psutil.boot_time())).days,
    }
```

### `jinja2` — Génération de configurations par templates

`jinja2` est le moteur de templates utilisé par Ansible, Flask et de nombreux outils DevOps. Il permet de générer des fichiers de configuration à partir de templates et de données structurées :

```python
from jinja2 import Environment, FileSystemLoader, StrictUndefined

def generer_config(
    template_dir: Path,
    template_name: str,
    variables: dict,
    output_path: Path,
) -> None:
    """Génère un fichier de configuration à partir d'un template Jinja2."""
    env = Environment(
        loader=FileSystemLoader(str(template_dir)),
        undefined=StrictUndefined,     # Erreur si une variable est manquante
        trim_blocks=True,
        lstrip_blocks=True,
        keep_trailing_newline=True,
    )

    template = env.get_template(template_name)
    contenu = template.render(**variables)

    output_path.write_text(contenu, encoding="utf-8")
    logger.info("Configuration générée : %s", output_path)
```

Exemple de template Nginx (`templates/nginx_vhost.conf.j2`) :

```jinja
# Généré automatiquement — ne pas modifier manuellement
# Template : {{ template_name }}
# Date : {{ generated_at }}

server {
    listen 80;
    server_name {{ server_name }};

    {% if ssl_enabled %}
    listen 443 ssl;
    ssl_certificate /etc/letsencrypt/live/{{ server_name }}/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/{{ server_name }}/privkey.pem;
    {% endif %}

    root {{ document_root }};
    index index.html;

    {% for location in locations %}
    location {{ location.path }} {
        {% if location.proxy_pass is defined %}
        proxy_pass {{ location.proxy_pass }};
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        {% else %}
        try_files $uri $uri/ =404;
        {% endif %}
    }
    {% endfor %}

    access_log /var/log/nginx/{{ server_name }}_access.log;
    error_log /var/log/nginx/{{ server_name }}_error.log;
}
```

Utilisation :

```python
from datetime import datetime

generer_config(
    template_dir=Path("./templates"),
    template_name="nginx_vhost.conf.j2",
    variables={
        "template_name": "nginx_vhost.conf.j2",
        "generated_at": datetime.now().isoformat(),
        "server_name": "app.example.com",
        "ssl_enabled": True,
        "document_root": "/var/www/app",
        "locations": [
            {"path": "/", "proxy_pass": "http://127.0.0.1:8080"},
            {"path": "/static"},
            {"path": "/api", "proxy_pass": "http://127.0.0.1:3000"},
        ],
    },
    output_path=Path("/etc/nginx/sites-available/app.example.com"),
)
```

Ce pattern — template + données → configuration — est exactement celui qu'Ansible utilise à grande échelle. Maîtriser Jinja2 dans un contexte Python facilite la compréhension et l'écriture des templates Ansible (module 13).

## Résumé

Les points clés abordés dans cette sous-section sont :

- **`os`** fournit l'accès aux informations système (UID, PID, load), aux variables d'environnement, aux permissions et aux utilisateurs/groupes. **`shutil`** complète avec les opérations sur les arborescences (copie récursive, archivage, `which`).
- **`socket`** permet les vérifications de connectivité TCP et la résolution DNS. Le pattern `check_port` est un outil de diagnostic réseau fondamental.
- **`hashlib`** calcule les checksums pour la vérification d'intégrité des sauvegardes et des fichiers transférés.
- **`concurrent.futures`** simplifie le parallélisme pour les opérations sur un parc de serveurs. `ThreadPoolExecutor` est adapté aux tâches I/O-bound (réseau, SSH).
- **`requests`** remplace `curl` + `jq` pour les interactions HTTP/API : sessions réutilisables, retry automatique via `urllib3.Retry`, parsing JSON natif, gestion des erreurs par exceptions.
- **`paramiko`** fournit un client SSH programmatique complet (exécution de commandes, SFTP upload/download) sans dépendance à la commande `ssh` système. Il est adapté aux opérations ponctuelles ; Ansible est préférable pour la gestion de configuration à grande échelle.
- **`psutil`** offre un accès portable et riche aux métriques système (CPU, mémoire, disques, réseau, processus), remplaçant le parsing manuel de `/proc` et les appels à des commandes système.
- **`jinja2`** génère des fichiers de configuration à partir de templates et de données structurées, suivant le même pattern que les templates Ansible.
