# 2.4 Personnalisation et optimisation

## 2.4.1 Thèmes et apparence

### Comprendre les thèmes sous Linux

#### Composants d'un thème

Un **thème complet** sous Linux se compose de plusieurs éléments indépendants :

**Éléments visuels principaux :**
- **Theme GTK** : Style des applications GNOME/GTK
- **Theme Qt** : Style des applications KDE/Qt
- **Icons** : Pack d'icônes système et applications
- **Cursors** : Pointeurs de souris personnalisés
- **Wallpaper** : Fond d'écran
- **Fonts** : Polices de caractères

**Hiérarchie des thèmes :**
```
Thème Global
├── GTK Theme (Applications GNOME)
├── Qt Theme (Applications KDE)
├── Icon Theme (Icônes système)
├── Cursor Theme (Pointeurs souris)
├── Shell Theme (Interface DE)
└── Sound Theme (Sons système)
```

#### Où sont stockés les thèmes

**Emplacements des thèmes :**
```bash
# Thèmes système (tous utilisateurs)
/usr/share/themes/          # Thèmes GTK/Qt
/usr/share/icons/           # Packs d'icônes
/usr/share/pixmaps/         # Icônes diverses
/usr/share/sounds/          # Thèmes sonores

# Thèmes utilisateur (personnel)
~/.themes/                  # Thèmes GTK personnels
~/.icons/                   # Icônes personnelles
~/.local/share/themes/      # Alternative thèmes
~/.local/share/icons/       # Alternative icônes
```


### Optimisation mémoire

#### Gestion swap

**Configuration swappiness :**
```bash
# Voir valeur actuelle (défaut 60)
cat /proc/sys/vm/swappiness

# Réduire utilisation swap (plus réactif)
echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf

# Appliquer immédiatement
sudo sysctl vm.swappiness=10
```

**Optimisation fichier swap :**
```bash
# Vérifier swap actuel
swapon --show

# Si besoin d'ajuster taille swap (exemple 4GB)
sudo swapoff -a
sudo rm /swapfile  # Si fichier swap existant
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

#### Nettoyage mémoire

**Cache système :**
```bash
# Vider caches (attention, peut ralentir temporairement)
sudo sync
echo 3 | sudo tee /proc/sys/vm/drop_caches

# Nettoyage automatique quotidien
echo '0 2 * * * root sync && echo 3 > /proc/sys/vm/drop_caches' | sudo tee -a /etc/crontab
```

**Applications gourmandes :**
```bash
# Identifier applications utilisant le plus de RAM
ps aux --sort=-%mem | head -20

# Tuer processus gourmand (attention !)
kill -9 PID-processus
```

### Optimisation stockage

#### Nettoyage disque dur

**APT cache et paquets :**
```bash
# Nettoyer cache APT
sudo apt autoclean    # Anciens paquets
sudo apt clean        # Tout le cache

# Supprimer paquets orphelins
sudo apt autoremove

# Voir espace cache
du -sh /var/cache/apt/archives/
```

**Logs système :**
```bash
# Taille logs actuels
sudo journalctl --disk-usage

# Nettoyer logs anciens (garder 1 mois)
sudo journalctl --vacuum-time=1month

# Ou limiter taille totale (500MB)
sudo journalctl --vacuum-size=500M

# Configuration permanente
sudo nano /etc/systemd/journald.conf
# Ajouter : SystemMaxUse=500M
```

**Fichiers temporaires :**
```bash
# Nettoyer /tmp (automatique au redémarrage)
sudo rm -rf /tmp/*

# Vider corbeille
rm -rf ~/.local/share/Trash/files/*
rm -rf ~/.local/share/Trash/info/*

# Miniatures images
rm -rf ~/.cache/thumbnails/*
```

```bash
while true; do
```

## 2.4.4 Accessibilité

### Introduction à l'accessibilité

#### Qu'est-ce que l'accessibilité ?

L'**accessibilité informatique** consiste à rendre les ordinateurs et logiciels **utilisables par tous**, y compris les personnes ayant des **déficiences visuelles, auditives, motrices ou cognitives**.

**Principes fondamentaux :**
- **Perceptible** : Information présentée de façon perceptible
- **Utilisable** : Interface utilisable par tous
- **Compréhensible** : Information et interface compréhensibles
- **Robuste** : Contenu interprétable par diverses technologies

**Types de déficiences couvertes :**
- **Visuelles** : Cécité, malvoyance, daltonisme
- **Auditives** : Surdité, malentendance
- **Motrices** : Mobilité réduite, paralysie
- **Cognitives** : Dyslexie, troubles attention
