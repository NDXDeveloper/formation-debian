# 1.1 Introduction à Debian

## 1.1.1 Histoire et philosophie de Debian

### Les origines de Debian

Debian a été créé en **1993** par Ian Murdock, alors étudiant à l'université Purdue. Le nom "Debian" vient de la combinaison des prénoms d'Ian Murdock et de sa compagne de l'époque, Debra Lynn.

**Pourquoi Debian a-t-il été créé ?**
À l'époque, les distributions Linux existantes étaient souvent maintenues par une seule personne ou une petite équipe, ce qui posait des problèmes de continuité et de qualité. Ian Murdock voulait créer une distribution :
- Maintenue par une **communauté de développeurs**
- Basée sur des **principes éthiques clairs**
- Focalisée sur la **qualité et la stabilité**

### La philosophie Debian : Le Contrat Social

Debian n'est pas qu'une simple distribution Linux, c'est un **projet communautaire** guidé par des valeurs fortes. Le Contrat Social Debian, créé en 1997, définit les engagements de la communauté :

1. **Debian restera 100% libre** - Tous les composants principaux seront libres
2. **Nous rendrons à la communauté du logiciel libre** - Nos améliorations bénéficieront à tous
3. **Nous ne cacherons pas les problèmes** - Transparency totale sur les bugs et développements
4. **Nos priorités sont nos utilisateurs et les logiciels libres** - Les besoins de la communauté passent avant tout
5. **Travaux non conformes à nos standards** - Nous fournirons des outils pour les logiciels non-libres sans les promouvoir

### Les Debian Free Software Guidelines (DFSG)

Ces principes définissent ce qu'est un **logiciel libre** selon Debian :
- Redistribution libre
- Code source disponible
- Travaux dérivés autorisés
- Intégrité du code source de l'auteur
- Pas de discrimination contre des personnes ou groupes
- Pas de discrimination contre des domaines d'application
- Distribution de la licence
- La licence ne doit pas être spécifique à Debian
- La licence ne doit pas contaminer d'autres logiciels
- Exemples de licences libres conformes : GPL, MIT, BSD

### Le modèle de développement communautaire

**Comment Debian fonctionne-t-il ?**

Debian est développé par plus de **1000 développeurs bénévoles** dans le monde entier, coordonnés par :
- **Le Projet Debian** : L'organisation globale
- **Le Leader du Projet** : Élu chaque année par les développeurs
- **Le Comité Technique** : Résout les conflits techniques
- **Les équipes spécialisées** : Maintenance, sécurité, publication, etc.

**Qui peut contribuer ?**
- **Utilisateurs** : Rapports de bugs, traductions, documentation
- **Contributeurs** : Paquets, corrections, tests
- **Développeurs Debian** : Statut officiel après un processus de candidature
- **Mainteneurs de paquets** : Responsables de logiciels spécifiques

## 1.1.2 Les versions et cycles de release (Stable, Testing, Unstable)

### Le système à trois branches

Debian utilise un **système unique** de développement avec trois branches principales qui coexistent en permanence :

#### 1. Unstable (Sid)

**Qu'est-ce que c'est ?**
- La branche de **développement actif**
- Nom de code permanent : **"Sid"** (du méchant garçon de Toy Story)
- Reçoit les **nouveaux paquets** et mises à jour en premier

**Caractéristiques :**
- **Avantages** : Logiciels les plus récents, nouvelles fonctionnalités
- **Inconvénients** : Peut être cassée, instable, bugs fréquents
- **Public cible** : Développeurs, testeurs expérimentés
- **Fréquence de mise à jour** : Continue (plusieurs fois par jour)

**Pour les débutants :** ❌ **Évitez Unstable** - réservé aux experts !

#### 2. Testing

**Qu'est-ce que c'est ?**
- La **future version Stable** en préparation
- Reçoit les paquets d'Unstable après une **période de test** (2-10 jours)
- Change de nom à chaque cycle (actuellement "Trixie")

**Caractéristiques :**
- **Avantages** : Plus stable qu'Unstable, logiciels assez récents
- **Inconvénients** : Peut avoir des bugs, mises à jour de sécurité retardées
- **Public cible** : Utilisateurs avancés, développeurs
- **Fréquence de mise à jour** : Quotidienne

**Pour les débutants :** ⚠️ **Utilisable mais déconseillé** - nécessite de l'expérience

#### 3. Stable

**Qu'est-ce que c'est ?**
- La version **officiellement supportée** et recommandée
- **Très stable** et testée intensivement
- Change de nom à chaque release (actuellement "Bookworm")

**Caractéristiques :**
- **Avantages** : Très stable, support de sécurité, documentation complète
- **Inconvénients** : Logiciels parfois anciens
- **Public cible** : Tous les utilisateurs, production, serveurs
- **Fréquence de mise à jour** : Sécurité uniquement (sauf exceptions)

**Pour les débutants :** ✅ **Version recommandée** - parfaite pour débuter !

### Le cycle de développement Debian

**Comment une nouvelle version Stable naît-elle ?**

1. **Développement continu** dans Unstable (Sid)
2. **Migration automatique** vers Testing après tests
3. **Période de gel** (freeze) : plus de nouvelles fonctionnalités
4. **Correction intensive** des bugs critiques
5. **Publication** de la nouvelle Stable
6. **L'ancienne Testing devient Stable**, nouvelle Testing créée

**Durée typique d'un cycle :**
- Environ **2-3 ans** entre chaque version Stable
- **6-12 mois** de gel avant publication
- **Support de sécurité** pendant 3 ans minimum (+ 2 ans LTS)

### Noms de code des versions

Debian utilise des **noms de personnages de Toy Story** :

| Version | Nom de code | Année | Statut actuel |
|---------|-------------|-------|---------------|
| 12 | Bookworm | 2023 | **Stable actuelle** |
| 13 | Trixie | ~2025 | Testing |
| 14 | Forky | ~2027 | Future |
| - | Sid | - | Unstable permanent |

## 1.1.3 Différences avec les autres distributions Linux

### Debian vs Ubuntu

**Ubuntu** est basé sur Debian, mais avec des différences importantes :

| Aspect | Debian | Ubuntu |
|--------|--------|--------|
| **Philosophie** | 100% logiciel libre par défaut | Pragmatique, inclut du propriétaire |
| **Cycles** | ~2-3 ans | 6 mois (LTS : 2 ans) |
| **Stabilité** | Très conservateur | Plus agressif sur les nouveautés |
| **Support** | Communautaire | Canonical Ltd (commercial) |
| **Installation** | Plus technique | Plus guidée et automatisée |
| **Public** | Avancé/serveurs | Grand public/desktop |

**Pourquoi choisir Debian plutôt qu'Ubuntu ?**
- **Contrôle total** sur le système
- **Philosophie du libre** respectée
- **Stabilité** éprouvée pour les serveurs
- **Indépendance** d'une entreprise
- **Apprentissage** plus approfondi de Linux

### Debian vs Red Hat/CentOS/Fedora

**Famille Red Hat** (RPM) vs **Famille Debian** (DEB) :

| Aspect | Debian | Red Hat/CentOS |
|--------|--------|----------------|
| **Paquets** | .deb / APT | .rpm / YUM/DNF |
| **Philosophie** | Communautaire | Entreprise |
| **Cible** | Universelle | Professionnelle |
| **Certification** | Informelle | Officielle (RHCE) |
| **Documentation** | Wiki communautaire | Documentation officielle |

### Debian vs Arch Linux

| Aspect | Debian | Arch Linux |
|--------|--------|------------|
| **Philosophie** | Stabilité avant tout | Simplicité et actualité |
| **Installation** | Guidée | Manuelle complète |
| **Paquets** | Très testés | Très récents |
| **Rolling release** | Non (versions fixes) | Oui (mise à jour continue) |
| **Difficulté** | Débutant à expert | Expert |

### Pourquoi choisir Debian ?

**Avantages uniques de Debian :**

1. **Stabilité légendaire** - "Ça marche et ça continue de marcher"
2. **Respect du libre** - Pas de compromis sur l'éthique
3. **Flexibilité totale** - S'adapte à tous les usages
4. **Communauté mature** - 30+ ans d'expérience
5. **Documentation excellent** - Guides détaillés et précis
6. **Support long** - 5+ ans par version
7. **Architecture multiple** - Fonctionne partout
8. **Base solide** - Fondation de nombreuses autres distributions

## 1.1.4 Architecture du système Debian

### Vue d'ensemble de l'architecture

Le système Debian s'organise en **couches** bien définies :

```
┌─────────────────────────────────────┐
│     Applications utilisateur        │
├─────────────────────────────────────┤
│    Environnement de bureau          │
├─────────────────────────────────────┤
│         Bibliothèques               │
├─────────────────────────────────────┤
│      Système de paquets             │
├─────────────────────────────────────┤
│     Utilitaires système             │
├─────────────────────────────────────┤
│        Noyau Linux                  │
├─────────────────────────────────────┤
│        Matériel                     │
└─────────────────────────────────────┘
```

### Le système de paquets - Cœur de Debian

**Qu'est-ce qu'un paquet ?**
Un paquet Debian (.deb) contient :
- Les **fichiers** du logiciel
- Les **métadonnées** (version, description, dépendances)
- Les **scripts** d'installation/suppression
- Les **informations de contrôle**

**Hiérarchie des outils de paquets :**

1. **dpkg** (niveau bas)
   - Installe/supprime les paquets .deb
   - Ne gère **pas** les dépendances
   - Outil fondamental mais rarement utilisé directement

2. **APT** (niveau haut)
   - **apt-get**, **apt-cache**, **apt** (moderne)
   - Résout automatiquement les **dépendances**
   - Télécharge depuis les **dépôts**
   - Interface recommandée pour les utilisateurs

3. **Aptitude** (interface avancée)
   - Interface textuelle interactive
   - Résolution intelligente des conflits
   - Plus puissant qu'APT pour les cas complexes

### Structure des dépôts Debian

**Organisation logique :**

```
Dépôt Debian
├── main (logiciels libres officiels)
├── contrib (libres mais dépendent du non-libre)
└── non-free (logiciels propriétaires)
```

**Composants par version :**
- **main** : ✅ Recommandé - logiciels 100% libres
- **contrib** : ⚠️ Optionnel - libres mais avec dépendances propriétaires
- **non-free** : ❌ Si nécessaire - pilotes, firmware propriétaires

**Structure géographique :**
- **deb.debian.org** : Serveur principal
- **Miroirs locaux** : Serveurs régionaux plus rapides
- **Security** : Mises à jour de sécurité prioritaires
- **Backports** : Versions récentes pour Stable

### Architecture matérielle supportée

Debian supporte **officiellement** de nombreuses architectures :

| Architecture | Description | Usage typique |
|--------------|-------------|---------------|
| **amd64** | 64-bit x86 | PC, serveurs modernes |
| **i386** | 32-bit x86 | Anciens PC |
| **arm64** | ARM 64-bit | Raspberry Pi 4, serveurs ARM |
| **armhf** | ARM hard-float | Raspberry Pi 2/3 |
| **armel** | ARM soft-float | Anciens ARM |
| **mips64el** | MIPS 64-bit | Routeurs, systèmes embarqués |
| **ppc64el** | PowerPC 64-bit | Serveurs IBM |
| **s390x** | IBM z/Architecture | Mainframes IBM |

**Pour les débutants :** Utilisez **amd64** pour les PC modernes (depuis ~2005)

### Processus de démarrage (boot)

**Séquence de démarrage Debian :**

1. **BIOS/UEFI** - Initialisation matérielle
2. **Bootloader** (GRUB2) - Sélection du système
3. **Noyau Linux** - Chargement du kernel
4. **initramfs** - Système de fichiers temporaire
5. **systemd** - Gestionnaire de services (init moderne)
6. **Services système** - Réseau, logs, etc.
7. **Login manager** - Écran de connexion
8. **Session utilisateur** - Bureau ou shell

### Gestion des services avec systemd

**systemd** est le **système d'init** moderne de Debian :

**Concepts clés :**
- **Unit** : Unité de service (service, timer, mount, etc.)
- **Target** : Groupes d'unités (équivalent des runlevels)
- **Journal** : Logs centralisés
- **Socket activation** : Démarrage à la demande

**Avantages de systemd :**
- Démarrage **parallèle** et plus rapide
- Gestion **unifiée** des services
- **Logs structurés** avec journalctl
- **Dépendances** entre services
- **Isolation** des processus

### Systèmes de fichiers

**Systèmes de fichiers supportés :**

| Type | Usage | Caractéristiques |
|------|-------|------------------|
| **ext4** | Standard Linux | Journalisé, fiable, performant |
| **btrfs** | Moderne | Snapshots, compression, RAID |
| **XFS** | Gros volumes | Haute performance, scaling |
| **ZFS** | Avancé | Intégrité, snapshots, déduplication |
| **FAT32** | Compatibilité | Windows, clés USB |
| **NTFS** | Windows | Lecture/écriture possible |

**Pour les débutants :** **ext4** est le choix sûr et universel

### Structure des répertoires système

**Arborescence standard Linux (FHS - Filesystem Hierarchy Standard) :**

```
/
├── bin/     → Commandes essentielles (ls, cp, mv)
├── boot/    → Fichiers de démarrage (kernel, GRUB)
├── dev/     → Fichiers de périphériques
├── etc/     → Fichiers de configuration système
├── home/    → Répertoires des utilisateurs
├── lib/     → Bibliothèques partagées essentielles
├── media/   → Points de montage amovibles (CD, USB)
├── mnt/     → Points de montage temporaires
├── opt/     → Logiciels optionnels tiers
├── proc/    → Système de fichiers virtuel (processus)
├── root/    → Répertoire de l'utilisateur root
├── run/     → Données de runtime
├── sbin/    → Commandes système (mount, fsck)
├── srv/     → Données des services (web, ftp)
├── sys/     → Interface avec le kernel
├── tmp/     → Fichiers temporaires
├── usr/     → Programmes et données utilisateur
│   ├── bin/     → Commandes utilisateur
│   ├── lib/     → Bibliothèques
│   ├── local/   → Logiciels locaux
│   └── share/   → Données partagées
└── var/     → Données variables (logs, caches)
    ├── log/     → Fichiers de logs
    ├── cache/   → Caches des applications
    └── lib/     → Données d'état variables
```

**Répertoires importants pour les débutants :**
- **/home/utilisateur/** : Vos fichiers personnels
- **/etc/** : Configuration à modifier avec précaution
- **/var/log/** : Logs pour diagnostiquer les problèmes
- **/usr/bin/** : Commandes que vous utilisez
- **/tmp/** : Stockage temporaire (nettoyé au redémarrage)

Cette architecture solide et cohérente fait de Debian une base excellente pour apprendre Linux et construire des systèmes fiables.
