# 7.4.1 Samba (partage Windows)

## Introduction

Samba est l'implémentation libre du protocole SMB/CIFS qui permet à un serveur Debian de fournir des partages de fichiers accessibles nativement depuis les postes Windows, macOS et Linux. Le projet Samba existe depuis 1992 et offre aujourd'hui une compatibilité étendue avec l'écosystème Microsoft : partage de fichiers et d'imprimantes, authentification, et même la capacité de fonctionner comme contrôleur de domaine Active Directory.

Cette section couvre l'installation de Samba sur Debian, la configuration de partages autonomes (serveur standalone avec sa propre base d'utilisateurs), l'intégration avec Active Directory en tant que membre de domaine, la gestion fine des permissions et les fonctionnalités avancées utiles en production.

---

## Installation

```bash
$ sudo apt update
$ sudo apt install samba samba-common-bin
```

L'installation crée le fichier de configuration principal `/etc/samba/smb.conf`, les répertoires de données internes de Samba et deux services systemd : `smbd` (serveur de fichiers et d'impression) et `nmbd` (résolution de noms NetBIOS, nécessaire pour la découverte réseau dans les environnements legacy).

```bash
# Vérifier les services
$ sudo systemctl status smbd
$ sudo systemctl status nmbd

# Vérifier les ports d'écoute
$ sudo ss -tlnp | grep -E 'smbd|nmbd'
LISTEN  0  50  *:445  *:*  users:(("smbd",pid=...))  
LISTEN  0  50  *:139  *:*  users:(("smbd",pid=...))  

# Vérifier la version (Trixie embarque Samba 4.22.x)
$ smbd --version
Version 4.22.x
```

Les ports utilisés par Samba :

| Port | Protocole | Service | Nécessaire |
|------|-----------|---------|------------|
| 445 | TCP | SMB direct (SMB2/SMB3) | Oui (indispensable) |
| 139 | TCP | SMB sur NetBIOS | Optionnel (clients anciens) |
| 137 | UDP | NetBIOS Name Service | Optionnel (découverte réseau) |
| 138 | UDP | NetBIOS Datagram | Optionnel (parcours réseau) |

Pour un déploiement moderne avec des clients SMB2/SMB3 uniquement, seul le port 445 est nécessaire. Les ports 137-139 (NetBIOS) peuvent être désactivés si la découverte réseau legacy n'est pas requise.

## Structure de configuration

### Le fichier smb.conf

Toute la configuration de Samba réside dans `/etc/samba/smb.conf`. Le fichier est organisé en sections délimitées par des crochets :

- `[global]` — Paramètres globaux du serveur.
- `[nom_partage]` — Définition d'un partage de fichiers. Le nom entre crochets est celui qui apparaît dans l'Explorateur Windows.

La syntaxe est de type INI : `paramètre = valeur`, un paramètre par ligne. Les commentaires commencent par `#` ou `;`. Les paramètres booléens acceptent `yes`/`no`, `true`/`false` ou `1`/`0`.

### Vérification de la syntaxe

L'outil `testparm` vérifie la syntaxe du fichier de configuration et affiche la configuration effective (paramètres par défaut inclus) :

```bash
# Vérifier la syntaxe
$ testparm
Load smb config files from /etc/samba/smb.conf  
Loaded services file OK.  

# Afficher la configuration effective complète
$ testparm -s

# Afficher un paramètre spécifique
$ testparm -s --parameter-name="server min protocol"
```

`testparm` doit être exécuté systématiquement après chaque modification de `smb.conf`, avant le rechargement du service. Une erreur de syntaxe dans `smb.conf` peut empêcher Samba de démarrer.

## Configuration du serveur standalone

### Section [global]

La section `[global]` définit l'identité du serveur, les protocoles autorisés, la méthode d'authentification et les paramètres par défaut appliqués à tous les partages.

```ini
# /etc/samba/smb.conf

[global]
    # --- Identité du serveur ---
    workgroup = EXAMPLE
    server string = Serveur de fichiers Debian
    netbios name = SRV-FILES

    # --- Réseau ---
    # Interfaces sur lesquelles Samba écoute
    interfaces = enp0s3
    bind interfaces only = yes

    # --- Protocoles ---
    # SMB2 minimum (désactive SMB1, obsolète et vulnérable)
    server min protocol = SMB2_10
    server max protocol = SMB3

    # --- Authentification ---
    # Mode standalone (pas de domaine)
    security = user
    
    # Backend de mots de passe
    passdb backend = tdbsam

    # Correspondance des comptes avec les utilisateurs système
    # (les comptes Samba doivent exister en tant qu'utilisateurs système)
    unix password sync = yes
    passwd program = /usr/bin/passwd %u
    passwd chat = *Enter\snew\s*\spassword:* %n\n *Retype\snew\s*\spassword:* %n\n *password\supdated\ssuccessfully* .

    # --- Permissions par défaut ---
    # Masques de création pour les nouveaux fichiers et répertoires
    create mask = 0664
    directory mask = 0775

    # Forcer le groupe propriétaire des fichiers créés
    # force group = partage

    # --- Journalisation ---
    log file = /var/log/samba/log.%m
    max log size = 10000
    log level = 1

    # --- Performance ---
    # Désactiver NetBIOS si non nécessaire (environnement SMB2/SMB3 pur)
    # disable netbios = yes

    # Cache de répertoire (améliore les performances de listage)
    directory name cache size = 100

    # --- Impression ---
    # Désactiver le partage d'imprimantes si non nécessaire
    load printers = no
    printing = bsd
    printcap name = /dev/null
    disable spoolss = yes
```

### Gestion des utilisateurs Samba

Samba maintient sa propre base de mots de passe, distincte de celle du système (`/etc/shadow`). Les utilisateurs Samba doivent d'abord exister en tant qu'utilisateurs système, puis être ajoutés à la base Samba avec un mot de passe spécifique.

```bash
# Créer l'utilisateur système (sans accès shell)
$ sudo adduser --no-create-home --shell /usr/sbin/nologin --disabled-password alice

# Définir le mot de passe Samba pour cet utilisateur
$ sudo smbpasswd -a alice
New SMB password:  
Retype new SMB password:  
Added user alice.  

# Activer un utilisateur Samba existant
$ sudo smbpasswd -e alice

# Désactiver un utilisateur Samba (sans le supprimer)
$ sudo smbpasswd -d alice

# Supprimer un utilisateur de la base Samba
$ sudo smbpasswd -x alice

# Lister les utilisateurs Samba
$ sudo pdbedit -L
alice:1001:  
bob:1002:  

# Afficher les détails d'un utilisateur
$ sudo pdbedit -Lv alice
```

Le mot de passe Samba et le mot de passe système sont indépendants. Le paramètre `unix password sync = yes` peut synchroniser le mot de passe système quand l'utilisateur change son mot de passe Samba, mais les deux bases restent distinctes.

### Groupes et organisation

Les groupes système Linux servent de base pour les permissions de partage :

```bash
# Créer un groupe pour le partage
$ sudo groupadd partage-docs

# Ajouter des utilisateurs au groupe
$ sudo usermod -aG partage-docs alice
$ sudo usermod -aG partage-docs bob
```

## Création de partages

### Partage basique

```ini
# /etc/samba/smb.conf (suite)

[Documents]
    comment = Documents partages
    path = /srv/samba/documents
    browseable = yes
    read only = no
    valid users = @partage-docs
    create mask = 0660
    directory mask = 0770
    force group = partage-docs
```

Création du répertoire et attribution des permissions :

```bash
$ sudo mkdir -p /srv/samba/documents
$ sudo chown root:partage-docs /srv/samba/documents
$ sudo chmod 2770 /srv/samba/documents
```

Le bit SGID (`2770`) garantit que tous les fichiers et sous-répertoires créés dans ce répertoire héritent du groupe `partage-docs`, quel que soit le groupe principal de l'utilisateur qui les crée. C'est un mécanisme essentiel pour que le partage fonctionne correctement en multi-utilisateurs.

Les paramètres du partage :

- `browseable = yes` — Le partage apparaît dans la liste des partages réseau (Explorateur Windows).
- `read only = no` — Les utilisateurs autorisés peuvent écrire.
- `valid users = @partage-docs` — Seuls les membres du groupe `partage-docs` peuvent accéder au partage. Le `@` indique un groupe.
- `create mask` et `directory mask` — Permissions POSIX maximales pour les fichiers et répertoires créés via Samba.
- `force group` — Force le groupe propriétaire de tous les fichiers créés.

### Partage en lecture seule

```ini
[Public]
    comment = Documents publics en lecture seule
    path = /srv/samba/public
    browseable = yes
    read only = yes
    guest ok = yes
    # write list = admin
```

Le paramètre `guest ok = yes` autorise l'accès sans authentification. C'est adapté aux partages de documentation interne en lecture seule sur un réseau de confiance. En production, limiter l'accès à des utilisateurs authentifiés est préférable.

### Partage avec accès différenciés

```ini
[Projets]
    comment = Repertoires de projets
    path = /srv/samba/projets
    browseable = yes
    read only = yes

    # Utilisateurs avec accès en écriture
    write list = @chefs-projet, admin

    # Utilisateurs avec accès en lecture uniquement (en plus des membres en écriture)
    valid users = @chefs-projet, @developpeurs, @stagiaires

    # Masques de permissions
    create mask = 0660
    directory mask = 0770
    force group = projets
```

L'option `read only = yes` combinée à `write list` crée un partage en lecture par défaut, avec des droits d'écriture accordés uniquement aux utilisateurs ou groupes listés dans `write list`.

### Répertoires personnels

Samba peut exposer automatiquement le répertoire home de chaque utilisateur :

```ini
[homes]
    comment = Repertoire personnel de %U
    browseable = no
    read only = no
    valid users = %S
    create mask = 0600
    directory mask = 0700
```

La section spéciale `[homes]` crée automatiquement un partage nommé par le login de l'utilisateur connecté, pointant vers son répertoire home. La variable `%S` représente le nom du partage (identique au nom d'utilisateur dans ce cas). `browseable = no` empêche l'affichage du partage `homes` générique dans la liste — seul le partage personnalisé (portant le nom de l'utilisateur) apparaît après connexion.

### Activation des partages

```bash
# Vérifier la syntaxe
$ testparm

# Recharger la configuration (sans interrompre les connexions actives)
$ sudo smbcontrol all reload-config

# Ou redémarrer les services
$ sudo systemctl restart smbd
```

La commande `smbcontrol all reload-config` est préférée au redémarrage car elle recharge la configuration sans déconnecter les utilisateurs actifs. Un redémarrage (`restart`) ferme toutes les connexions en cours.

## Intégration Active Directory

### Jonction au domaine

Dans un environnement d'entreprise, Samba fonctionne typiquement comme **membre de domaine** Active Directory. Les utilisateurs s'authentifient avec leurs identifiants AD et les permissions sont gérées via les groupes AD.

```bash
# Installer les paquets nécessaires
$ sudo apt install samba winbind libpam-winbind libnss-winbind krb5-config krb5-user

# Lors de l'installation de krb5-config, renseigner :
# - Realm Kerberos : EXAMPLE.COM (en majuscules)
# - Serveurs KDC : dc1.example.com
# - Serveur admin : dc1.example.com
```

Configuration de Kerberos :

```ini
# /etc/krb5.conf

[libdefaults]
    default_realm = EXAMPLE.COM
    dns_lookup_realm = false
    dns_lookup_kdc = true

[realms]
    EXAMPLE.COM = {
        kdc = dc1.example.com
        admin_server = dc1.example.com
    }

[domain_realm]
    .example.com = EXAMPLE.COM
    example.com = EXAMPLE.COM
```

Configuration de Samba pour le domaine :

```ini
# /etc/samba/smb.conf

[global]
    workgroup = EXAMPLE
    realm = EXAMPLE.COM
    server string = Serveur de fichiers Debian

    security = ADS
    encrypt passwords = yes

    # Backend Winbind pour la correspondance utilisateurs AD → Unix
    idmap config * : backend = tdb
    idmap config * : range = 3000-7999

    idmap config EXAMPLE : backend = rid
    idmap config EXAMPLE : range = 10000-999999

    # Correspondance des noms
    winbind use default domain = yes
    winbind enum users = yes
    winbind enum groups = yes

    # Template pour les comptes Unix créés par Winbind
    template shell = /bin/bash
    template homedir = /home/%U

    # Protocoles
    server min protocol = SMB2_10

    # Permissions
    vfs objects = acl_xattr
    map acl inherit = yes
    store dos attributes = yes
```

Configurer NSS pour utiliser Winbind :

```bash
# /etc/nsswitch.conf (modifier les lignes passwd et group)
passwd:     files winbind  
group:      files winbind  
```

Joindre le domaine :

```bash
# Tester l'authentification Kerberos
$ kinit administrateur@EXAMPLE.COM
Password for administrateur@EXAMPLE.COM:
$ klist

# Joindre le domaine
$ sudo net ads join -U administrateur
Enter administrateur's password:  
Using short domain name -- EXAMPLE  
Joined 'SRV-FILES' to dns domain 'example.com'  

# Redémarrer les services
$ sudo systemctl restart smbd nmbd winbind

# Vérifier la jonction
$ sudo net ads testjoin
Join is OK

# Vérifier que les utilisateurs AD sont visibles
$ wbinfo -u    # Liste les utilisateurs du domaine
$ wbinfo -g    # Liste les groupes du domaine
$ getent passwd alice.dupont   # Résoudre un utilisateur AD
$ id alice.dupont              # Voir les UID/GID
```

### Partage utilisant les groupes AD

Une fois le serveur joint au domaine, les partages peuvent référencer directement les groupes Active Directory :

```ini
[Finance]
    comment = Documents du service financier
    path = /srv/samba/finance
    browseable = yes
    read only = no
    valid users = "@EXAMPLE\Finance", "@EXAMPLE\Direction"
    write list = "@EXAMPLE\Finance"
    read list = "@EXAMPLE\Direction"
    create mask = 0660
    directory mask = 0770
    force group = "EXAMPLE\Finance"

    # ACL Windows complètes
    vfs objects = acl_xattr
    map acl inherit = yes
    store dos attributes = yes
```

Le préfixe `EXAMPLE\` identifie les groupes AD. Si `winbind use default domain = yes` est configuré, le préfixe de domaine peut être omis pour le domaine par défaut.

## Corbeille réseau

Samba peut intercepter les suppressions de fichiers et les déplacer vers un répertoire corbeille au lieu de les supprimer définitivement. C'est une fonctionnalité précieuse en production pour récupérer des fichiers supprimés par erreur sans recourir aux sauvegardes.

```ini
[Documents]
    path = /srv/samba/documents
    # ... autres paramètres ...

    # Activer la corbeille
    vfs objects = recycle
    recycle:repository = .corbeille/%U
    recycle:keeptree = yes
    recycle:versions = yes
    recycle:touch = yes
    recycle:maxsize = 104857600
    recycle:exclude = *.tmp, *.temp, ~$*
    recycle:exclude_dir = .corbeille, tmp
```

Les paramètres :

- `recycle:repository = .corbeille/%U` — Répertoire corbeille par utilisateur dans le partage. `%U` est remplacé par le nom de l'utilisateur.
- `recycle:keeptree = yes` — Préserve l'arborescence des fichiers supprimés.
- `recycle:versions = yes` — Conserve les versions multiples (ajoute un suffixe numérique si un fichier du même nom existe déjà dans la corbeille).
- `recycle:touch = yes` — Met à jour la date de modification du fichier lors de la mise en corbeille (utile pour les scripts de purge basés sur l'âge).
- `recycle:maxsize` — Taille maximale d'un fichier recyclé (en octets, ici 100 Mo). Les fichiers plus gros sont supprimés définitivement.
- `recycle:exclude` — Motifs de fichiers à ne pas recycler (fichiers temporaires).

Un script de purge automatique nettoie la corbeille :

```bash
# Supprimer les fichiers de la corbeille de plus de 30 jours
$ find /srv/samba/documents/.corbeille -type f -mtime +30 -delete
$ find /srv/samba/documents/.corbeille -type d -empty -delete
```

Ce script est planifié via un timer systemd quotidien.

## Audit et journalisation

### Audit des accès aux fichiers

Le module VFS `full_audit` journalise les opérations sur les fichiers : ouvertures, lectures, écritures, suppressions, renommages. C'est un outil essentiel pour la conformité (traçabilité des accès) et le diagnostic.

```ini
[Documents]
    path = /srv/samba/documents
    # ... autres paramètres ...

    # Audit des opérations
    vfs objects = full_audit recycle
    full_audit:prefix = %u|%I|%S
    full_audit:success = mkdir rmdir open rename unlink write
    full_audit:failure = none
    full_audit:facility = local5
    full_audit:priority = notice
```

Les variables dans le préfixe :

- `%u` — Nom de l'utilisateur Samba.
- `%I` — Adresse IP du client.
- `%S` — Nom du partage.

Les logs d'audit sont envoyés à syslog (facility `local5`). Sur Trixie, `rsyslog`  
n'est plus installé par défaut (depuis Debian 12) — le journal système est  
géré par `journald`. Deux options pour exploiter les messages `local5` :  

**Option A — Installer rsyslog et le configurer**

```bash
$ sudo apt install rsyslog

# /etc/rsyslog.d/samba-audit.conf
local5.notice    /var/log/samba/audit.log
```

```bash
$ sudo systemctl restart rsyslog
$ sudo systemctl restart smbd
```

**Option B — Filtrer dans journald** (sans rsyslog)

```bash
# Voir les événements d'audit Samba
$ journalctl SYSLOG_FACILITY=21 -t smbd_audit -f
# (facility 21 = local5)

# Pour exporter vers un fichier dédié, créer un service systemd
# qui pipe journalctl vers /var/log/samba/audit.log :
$ sudo tee /etc/systemd/system/samba-audit-export.service <<'EOF'
[Unit]
Description=Export Samba audit logs to file  
After=smbd.service  

[Service]
Type=simple  
ExecStart=/bin/sh -c 'journalctl -f SYSLOG_FACILITY=21 -t smbd_audit \  
                       --output=short-iso > /var/log/samba/audit.log'
Restart=on-failure

[Install]
WantedBy=multi-user.target  
EOF  

$ sudo systemctl enable --now samba-audit-export.service
```

Le fichier `audit.log` contiendra des lignes comme :

```
Apr 15 10:30:45 srv-files smbd_audit: alice|192.0.2.50|Documents|open|ok|r|rapport-Q1.xlsx  
Apr 15 10:31:12 srv-files smbd_audit: bob|192.0.2.51|Documents|unlink|ok|budget-2025.xlsx  
```

## Performances

### Paramètres d'optimisation

```ini
[global]
    # --- Optimisation réseau ---
    # Utiliser sendfile (transfert direct noyau → réseau)
    use sendfile = yes

    # Taille minimale de lecture/écriture pour utiliser sendfile
    min receivefile size = 16384

    # Signalement asynchrone des écritures (améliore le débit)
    aio read size = 16384
    aio write size = 16384

    # Socket options
    socket options = TCP_NODELAY IPTOS_LOWDELAY

    # --- Cache ---
    # Activer le cache de permission
    directory name cache size = 100

    # Taille maximale du cache de stat
    max stat cache size = 512
```

### Multichannel (SMB3)

Le multichannel SMB3 permet d'utiliser plusieurs connexions réseau simultanées pour un même partage, augmentant le débit sur les serveurs équipés de plusieurs interfaces réseau ou de cartes réseau à bonding :

```ini
[global]
    server multi channel support = yes
    aio read size = 1
    aio write size = 1
```

Le multichannel est négocié automatiquement par les clients Windows 8+ et les clients Linux avec `cifs-utils` récent. Le serveur doit disposer de plusieurs interfaces réseau ou d'interfaces bondées pour que le multichannel apporte un bénéfice réel.

## Test et accès client

### Depuis un poste Windows

```
\\SRV-FILES\Documents
```

Ou via l'Explorateur de fichiers : menu Réseau, ou mapper un lecteur réseau (`Z:` → `\\SRV-FILES\Documents`).

### Depuis un poste Linux

```bash
# Installer le client SMB
$ sudo apt install cifs-utils

# Montage ponctuel
# Note : vers=3.0 force SMB 3.0 ; on peut préférer vers=3.1.1 (SMB 3.1.1,
# le plus moderne avec chiffrement AES-128-GCM). vers=default laisse la
# négociation choisir la meilleure version commune entre client et serveur.
$ sudo mount -t cifs //192.0.2.10/Documents /mnt/documents \
    -o username=alice,password=mot_de_passe,vers=3.1.1

# Montage permanent via /etc/fstab
# Créer un fichier de credentials (permissions restrictives)
$ sudo tee /etc/samba/credentials-alice << 'EOF'
username=alice  
password=mot_de_passe  
domain=EXAMPLE  
EOF  
$ sudo chmod 600 /etc/samba/credentials-alice

# Ligne fstab
//192.0.2.10/Documents /mnt/documents cifs credentials=/etc/samba/credentials-alice,vers=3.1.1,uid=1000,gid=1000,file_mode=0644,dir_mode=0755 0 0
```

`smbclient` est fourni par un paquet séparé sur Debian (le serveur `samba`
ne l'installe pas) :

```bash

$ sudo apt install smbclient

# Lister les partages disponibles
$ smbclient -L //192.0.2.10 -U alice

# Se connecter à un partage (interface FTP-like)
$ smbclient //192.0.2.10/Documents -U alice
```

## Bonnes pratiques

**Désactiver SMB1** — Le protocole SMB1 est vulnérable (WannaCry, EternalBlue) et obsolète. La directive `server min protocol = SMB2_10` garantit que seuls les protocoles SMB2 et SMB3 sont acceptés. Les clients Windows XP et les appareils très anciens ne pourront plus se connecter — c'est le comportement souhaité.

**Restreindre les interfaces d'écoute** — La directive `interfaces` combinée à `bind interfaces only = yes` empêche Samba d'écouter sur des interfaces non prévues (interface de management, VPN). Sur un serveur multi-interfaces, ne lister que l'interface du réseau bureautique.

**Utiliser le bit SGID sur les répertoires partagés** — Le chmod `2770` (ou `2775`) sur le répertoire racine d'un partage garantit que tous les fichiers et sous-répertoires héritent du groupe du répertoire parent. Sans ce bit, les fichiers créés appartiennent au groupe principal de l'utilisateur, ce qui casse les permissions de groupe partagé.

**Activer la corbeille** — La corbeille réseau via le module `recycle` est un filet de sécurité peu coûteux qui évite de nombreuses restaurations depuis les sauvegardes. Coupler la corbeille avec une purge automatique pour ne pas consommer d'espace disque indéfiniment.

**Séparer les mots de passe Samba de l'authentification système** — En mode standalone, les comptes Samba sont créés avec `nologin` comme shell. Ils n'ont pas besoin de pouvoir ouvrir une session sur le serveur. En environnement AD, l'authentification est déléguée au contrôleur de domaine et aucun mot de passe n'est stocké localement.

**Auditer les accès en production** — Le module `full_audit` fournit une traçabilité complète des opérations sur les fichiers. Dans les environnements soumis à des exigences de conformité (RGPD, ISO 27001), cet audit est souvent obligatoire. Centraliser les logs d'audit dans un système de collecte (ELK, Loki) pour faciliter l'analyse et la rétention.

**Tester avec `testparm` avant chaque rechargement** — Une erreur de syntaxe dans `smb.conf` peut empêcher Samba de démarrer ou désactiver silencieusement un partage. `testparm` détecte les erreurs de syntaxe et affiche la configuration effective, y compris les valeurs par défaut implicites.

**Surveiller l'espace disque et les quotas** — Les partages de fichiers sans quotas se remplissent inévitablement. Mettre en place des quotas de disque (par utilisateur ou par groupe) avec le système de quotas Linux et alerter quand l'espace disque descend sous un seuil de sécurité. La gestion des quotas est traitée en section 7.4.4.
